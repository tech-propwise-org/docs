This catalog serves as the comprehensive reference for all notification types in the PropWise CRM system. Each notification includes details about when it's triggered, who receives it, priority levels, and available actions.

<Note>
This documentation is sourced from `notification-type.enum.ts`, `notification-type.registry.ts`, and domain listeners under `src/modules/notification/listeners/`. External emitters include AI agent, escalation, distribution cron, workflow engine, and reminder scheduler.
</Note>

## Notification Categories

The system organizes notifications into five main categories:
- **Properties** (formerly `real_estate`)
- **CRM** 
- **Access** (formerly `team`)
- **System**
- **Messaging**

<Info>
**Sender Convention**: The "Sender" refers to the acting user. In most bulk operations, the sender is filtered out of the recipient list unless explicitly opted into sender delivery.
</Info>

## Actions and Channels

<Tabs>
  <Tab title="Available Actions">
    - **Navigate**: Opens the linked entity
    - **Approve/Reject**: In-app-only actions that change notification status
    - **RSVP**: Response actions for events and meetings
    
    <Warning>
    Email notifications only include deep-link buttons for `navigate` actions. Interactive actions (approve/reject/RSVP) require recipients to open the app.
    </Warning>
  </Tab>
  
  <Tab title="Delivery Channels">
    Channels are registry candidates, with final delivery determined by:
    - User preferences
    - Per-reminder overrides
    - Permission-based visibility settings
  </Tab>
</Tabs>

## Setting Visibility

<Note>
Some notification types include `visibilityPermissions` in the registry. When present, `NotificationPreferenceService.getPreferences()` filters these types from the preference catalog for users without the required permissions. This is cosmetic only - delivery logic remains unchanged.
</Note>

### Activity Stakeholder Inbox

<CardGroup cols={1}>
  <Card title="Category: CRM" icon="clipboard-list">
    **Subcategory**: `activities`
  </Card>
</CardGroup>

Activity notifications are triggered when activities are logged or scheduled on leads or contacts. All notifications have **MEDIUM** priority and support both IN_APP and EMAIL channels.

<Steps>
  <Step title="Activity Logging">
    The following activity types trigger notifications:
    - `activity_call_logged`
    - `activity_email_logged` 
    - `activity_sms_logged`
    - `activity_note_added`
    - `activity_marked_important`
  </Step>
  
  <Step title="Activity Scheduling">
    Scheduled activities that trigger notifications:
    - `activity_meeting_scheduled`
    - `activity_site_visit_scheduled`
  </Step>
  
  <Step title="Recipients">
    Notifications go to lead + linked-contact stakeholders with `team.crm.view` or `team.crm.manage` permissions, excluding the performer.
  </Step>
</Steps>

<Warning>
Only activity creation and scheduling trigger notifications. Editing existing activities does NOT send notifications.
</Warning>

**Actions**: All activity types support "View record" action, which navigates using the payload `entityType` and `entityId`.
