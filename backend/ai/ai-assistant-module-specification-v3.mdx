# AI Assistant Module Specification v3

<Info>
**Last Updated:** 2026-06-03  
**Status:** Active Development  
**System Prompt Version:** `crm-assistant-v0.24-market-intelligence-routing`
</Info>

## Overview

The AI Assistant module provides conversational AI capabilities within the PropWise CRM, enabling users to interact with an intelligent assistant that can access and manipulate CRM data through natural language. The module implements a chat interface with streaming responses, tool execution capabilities, and comprehensive usage tracking.

The assistant operates within conversation scopes (global, lead, deal, contact, etc.) and can execute read-only and draft tools to help users with CRM tasks. It includes rate limiting, cost tracking, observability features, untrusted content fencing, permission key scrubbing, byte-capped result truncation, and configurable model parameters.

## Architecture

The module follows a layered architecture with clear separation of concerns:

- **Controllers**: Handle HTTP endpoints for chat, tool execution, and conversation management
- **Services**: Core business logic including orchestration, tool management, and provider abstraction
- **Entities**: Data models for conversations, messages, tool calls, and usage tracking
- **Tool System**: Extensible framework for registering and executing AI tools from other modules

<Note>
Key design decisions include provider-agnostic AI model interface supporting multiple LLM providers, streaming response architecture using Server-Sent Events, tool registry pattern allowing other modules to contribute capabilities, usage tracking and cost estimation for budget management, and in-memory rate limiting for Phase 0 (single-instance deployment).
</Note>

|-------|------|-------------|
| id | UUID | Primary key |

## API Endpoints

### POST /ai-assistant/chat

Start or continue a conversation with the AI assistant.

<CodeGroup>
```typescript Request
{
  message: string;           // User message (max 8000 chars)
  conversationId?: string;   // Existing conversation ID
  entityRef?: {             // Context entity reference
    type: AiConversationScopeType;
    id: string;
  };
  streamMode: boolean;      // Enable streaming response
  tier?: string;           // Model tier (lite/standard/max)
}
```

```typescript Response (Streaming)
// NDJSON chunks with assistant response
{"type": "chunk", "content": "Hello! How can I help you today?"}
{"type": "tool_call", "toolName": "searchLeads", "args": {...}}
{"type": "tool_result", "result": {...}}
{"type": "complete"}
```

```typescript Response (Non-streaming)
{
  conversation: AiConversation;
  message: AiMessage;
  assistantMessage?: AiMessage;
  toolCalls?: AiToolCall[];
}
```
</CodeGroup>

### GET /ai-assistant/conversations

List user's conversations with pagination.

<Tabs>
<Tab title="Parameters">
- `page`: Page number (default: 1)
- `limit`: Items per page (max: 100, default: 20)
- `scopeType`: Filter by conversation scope
- `scopeEntityId`: Filter by scoped entity ID
</Tab>

<Tab title="Response">
```typescript
{
  conversations: AiConversation[];
  pagination: {
    page: number;
    limit: number;
    total: number;
    totalPages: number;
  };
}
```
</Tab>
</Tabs>

### GET /ai-assistant/conversations/:id

Get a specific conversation with messages.

**Response:**
```typescript
{
  conversation: AiConversation;
  messages: AiMessage[];
  toolCalls: AiToolCall[];
}
```

### DELETE /ai-assistant/conversations/:id

Soft delete a conversation.

**Response:** `204 No Content`

### POST /ai-assistant/tools/execute

Execute a specific AI tool directly.

<CodeGroup>
```typescript Request
{
  toolName: string;
  args: Record<string, any>;
  conversationId?: string;
}
```

```typescript Response
{
  result: any;
  toolCall: AiToolCall;
  executionTimeMs: number;
}
```
</CodeGroup>

### GET /ai-assistant/tools

List available AI tools for the current user.

**Response:**
```typescript
{
  tools: Array<{
    name: string;
    description: string;
    parameters: JSONSchema;
    category: string;
  }>;
}
```

## Tool System

The AI Assistant uses an extensible tool system that allows other modules to register capabilities.

<Steps>
<Step title="Tool Registration">
Modules register tools using the `AiToolRegistry`:

```typescript
@Injectable()
export class MyModuleService {
  constructor(private toolRegistry: AiToolRegistry) {
    this.registerTools();
  }

  private registerTools() {
    this.toolRegistry.registerTool({
      name: 'searchLeads',
      description: 'Search for leads matching criteria',
      parameters: {
        type: 'object',
        properties: {
          query: { type: 'string' },
          limit: { type: 'number', maximum: 100 }
        }
      },
      category: 'crm',
      handler: this.searchLeads.bind(this)
    });
  }
}
```
</Step>

<Step title="Tool Execution">
Tools are executed through the `AiToolExecutionService` with context:

```typescript
async executeTool(
  toolName: string,
  args: any,
  context: AiToolExecutionContext
): Promise<any> {
  // Validate input
  // Execute tool logic
  // Return result
}
```
</Step>

<Step title="Permission Handling">
Tools automatically receive permission-scrubbed data through `formatResultForModel`:

```typescript
// Sensitive fields are removed before sending to the AI model
const scrubbed = formatResultForModel(result, context);
```
</Step>
</Steps>

### Timezone Handling

The assistant now properly handles viewer timezones across all date operations:

<Steps>
<Step title="Client Timezone Detection">
Controllers read the `Time-Zone` header via `@TimezoneParam()` and thread `viewerTimezone` into tool execution context.
</Step>

<Step title="System Prompt Enhancement">
`composeSystemPrompt` injects current time information with viewer-local instant, UTC instant, and IANA zone with timezone-aware instructions.
</Step>

<Step title="Date Parsing">
All date bounds are parsed using `parseClientDateTimeInZone(value, viewerTimezone)` with explicit offset preservation and naive string resolution in viewer zone.
</Step>
</Steps>

### Placeholder Input Hardening

<Tip>
Tools now use shared `tool-input-normalizers.util.ts` to handle nil-UUID sentinels and invalid numeric bounds, preventing fabricated filter values from hiding valid results.
</Tip>

### Eval Labs Stub

For testing and evaluation, the system includes `EvalStubPropwiseLabsService` which provides deterministic recorded real-estate payloads under the `AI_ASSISTANT_EVAL_MODE` + `AI_ASSISTANT_LABS_STUB` feature flags.

## Security & Permissions

### Permission Key Scrubbing

All tool results are automatically scrubbed of sensitive permission keys before being sent to the AI model:

- Database IDs are preserved for functionality
- Permission flags and internal metadata are removed
- Date objects and non-plain objects are preserved as-is

### Content Fencing

- User input is validated and sanitized
- Tool results are byte-capped to prevent excessive token usage
- Untrusted content is properly escaped in responses

### Rate Limiting

In-memory rate limiting with configurable requests per time window:

```typescript
// Default: 20 requests per minute per user
@RateLimit({ 
  requests: 20, 
  windowMs: 60000,
  keyGenerator: (req) => req.user.id
