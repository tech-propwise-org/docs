The AI module owns both the **AI Agent runtime** (PCD-64 entities/services for automated conversation handling) and the **organization-scoped knowledge base** that powers retrieval-augmented generation across the platform. It supports both the internal CRM assistant and customer-facing AI agents through a unified knowledge management system with OpenAI BYOK (Bring Your Own Key) integration.

<Info>
**Status:** Active — Fully Implemented and Operational

The module provides a complete AI infrastructure with 10 pre-configured agent templates, automated conversation handling, comprehensive action capabilities, and a flexible knowledge base supporting Documents, Pages, FAQ, and Snippets ingestion.

Last Updated: 2026-05-28
</Info>

## Architecture

The AI module follows a service-oriented architecture with **14+ specialized services** providing comprehensive AI capabilities across two major functional areas:

### AI Agent Runtime (Fully Implemented)

Complete runtime implementation with:

- **Full Service Architecture**: `AiAgentService`, `AiAgentRuntimeService`, `AiAgentResolverService`, `AiAgentContextBuilderService`, `AiAgentTokenBudgetService`, `AiAgentToolRegistryService`, `AiAgentActionService`, `AiAgentMediaProcessorService`, `AiAgentPricingService`, `AiAgentOptimizeService`, `AiAgentTemplateService`, `AiAgentKnowledgeBaseService`, `AiCreditUsageService`, and `AiActivityLogService`
- **Message Processing**: `AiAgentInboundListener` and `AiAgentExecuteWorker` for inbox handling
- **HTTP Surface**: Complete `/v1/ai-agents/*` REST endpoints via `AiAgentKnowledgeBaseController`
- **Workflow Integration**: Bidirectional integration with messaging automation and workflow systems
- **Template System**: 10 pre-configured agent templates seeded via `AiAgentTemplateSeeder`
- **Action Capabilities**: Conversation assignment, contact updates, lead lifecycle changes, workflow triggering, and team member notifications

### Knowledge Management

Multi-format ingestion and retrieval system:

- **Four Ingestion Shapes**: Documents (PDF/DOCX/TXT/Markdown), scraped Pages, FAQ rows, and freeform Snippets
- **OpenAI BYOK Integration**: Uses organization's own OpenAI API keys via `OpenAiProjectProvisioner`
- **Vector Storage**: `text-embedding-3-small` (768 dimensions) in PostgreSQL with pgvector
- **HTTP Surface**: `/v1/ai-knowledge-bases/*` endpoints for authenticated, org-scoped access
- **Stateless Design**: `EmbeddingService` takes OpenAI key as per-call argument for clean BYOK implementation

### Assignment Vocabulary (v0.11)

<Note>
The AI assistant uses "assignment"/"assignee"/"assigned to" terminology when communicating with users. While internal entities remain as `EntityStakeholder`, two boundary surfaces project data using assignee-named keys:

- **`getLeadStakeholders.execute()`** returns `assigneeCount`, `userAssigneeCount`, `teamAssigneeCount`, `primaryAssignee`, `assignees`, `assignedTeam`
- **Pre-hydrated lead context** projects `compactContext.lead.primaryAssignee` and `compactContext.lead.assignees`
</Note>

### Legacy Integration Layer

</Warning>

## Core Entities

### AI Agent Runtime Entities

#### AiAgent

Represents a configured AI agent for automated conversation handling.

| Field | Type | Description |
|-------|------|-------------|
| `id` | UUID | Primary key |
| `name` | string | Display name for the agent |
| `instructions` | string | AI behavior instructions |
| `isActive` | boolean | Whether agent is currently active |
| `aiMode` | AiModeEnum | Automation level (OFF/ASSIST/AUTO) |
| `organization` | Organization | Organization owner |
| `template` | AiAgentTemplate? | Optional template reference |
| `createdAt` | Date | Creation timestamp |
| `updatedAt` | Date | Last update timestamp |

#### AiAgentTemplate

Pre-configured agent templates for common use cases.

| Field | Type | Description |
|-------|------|-------------|
| `id` | UUID | Primary key |
| `name` | string | Template name |
| `description` | string | Template description |
| `instructions` | string | Default agent instructions |
| `category` | string | Template category |
| `isSystemTemplate` | boolean | Whether it's a system template |

### Knowledge Base Entities

#### KnowledgeBase

Represents a source of knowledge with four ingestion shapes: Documents, Pages, FAQ, Snippets.

| Field | Type | Description |
|-------|------|-------------|
| `id` | UUID | Primary key |
| `name` | string | Display name for the knowledge base |
| `type` | KnowledgeBaseType | DOCUMENT/PAGE/FAQ/SNIPPET |
| `url` | string? | URL for PAGE type |
| `filename` | string? | Original filename for DOCUMENT type |
| `mimeType` | string? | MIME type of document |
| `content` | string? | Direct content for FAQ/SNIPPET types |
| `organization` | Organization | Organization owner |
| `isDeleted` | boolean | Soft delete flag |
| `createdAt` | Date | Creation timestamp |
| `updatedAt` | Date | Last update timestamp |
| `chunks` | KnowledgeChunk[] | Associated knowledge chunks |

#### KnowledgeChunk

Represents processed text chunks with vector embeddings.

| Field | Type | Description |
|-------|------|-------------|
| `id` | UUID | Primary key |
| `knowledgeBase` | KnowledgeBase | Parent knowledge base |
| `text` | string | The actual text content |
| `embedding` | number[] | Vector embedding (768 dims) |
| `organization` | Organization | Organization owner |
| `isDeleted` | boolean | Soft delete flag |
| `createdAt` | Date | Creation timestamp |
| `updatedAt` | Date | Last update timestamp |

#### OrganizationLlmKey

Stores organization's OpenAI API keys for BYOK functionality. Provisioned by `OpenAiProjectProvisioner` against the platform admin key to ensure organizations have dedicated OpenAI projects for billing isolation.

| Field | Type | Description |
|-------|------|-------------|
| `id` | UUID | Primary key |
| `organization` | Organization | Organization owner |
| `provider` | LlmProvider | OPENAI |
| `apiKey` | string | Encrypted API key |
| `isActive` | boolean | Whether key is currently active |
| `projectId` | string | OpenAI project identifier for billing |
| `createdAt` | Date | Creation timestamp |
| `updatedAt` | Date | Last update timestamp |

## API Reference

### AI Agent Runtime API

**Base Path:** `/v1/ai-agents`  
**Authentication:** Bearer token required  
**Authorization:** Role-based access control

#### Create AI Agent

<CodeGroup>
```bash cURL
curl -X POST /v1/ai-agents \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Sales Assistant",
    "instructions": "Help qualify leads and schedule appointments",
    "aiMode": "AUTO",
    "templateId": "uuid"
  }'
```
```javascript JavaScript
const response = await fetch('/v1/ai-agents', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'Sales Assistant',
    instructions: 'Help qualify leads and schedule appointments',
    aiMode: 'AUTO',
    templateId: 'uuid'
  })
});
```
</CodeGroup>

**Request Body:**
```json
{
  "name": "string",
  "instructions": "string",
  "aiMode": "OFF|ASSIST|AUTO",
  "templateId": "uuid?"
}
```

**Response:**
```json
{
  "id": "uuid",
  "name": "Sales Assistant",
  "instructions": "Help qualify leads and schedule appointments",
  "aiMode": "AUTO",
  "isActive": true,
  "createdAt": "2026-05-28T10:00:00Z"
}
```

### Knowledge Base API

**Base Path:** `/v1/ai-knowledge-bases`  
**Authentication:** Bearer token required  
**Authorization:** Organization-scoped access

#### Upload Document

<CodeGroup>
```bash cURL
curl -X POST /v1/ai-knowledge-bases/documents \
  -H "Authorization: Bearer {token}" \
  -F "file=@document.pdf" \
  -F "name=Company Policies"
```
```javascript JavaScript
const formData = new FormData();
formData.append('file', file);
formData.append('name', 'Company Policies');

const response = await fetch('/v1/ai-knowledge-bases/documents', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${token}`
  },
  body: formData
});
```
</CodeGroup>

**Parameters:**
- `file`: File (PDF, DOCX, TXT, or Markdown)
- `name`: string (display name)

**Response:**
```json
{
  "id": "uuid",
  "name": "Company Policies",
  "type": "DOCUMENT",
  "filename": "document.pdf",
  "mimeType": "application/pdf",
  "createdAt": "2026-05-28T10:00:00Z"
}
```

## Business Rules

### AI Agent Runtime

<Steps>
  <Step title="Organization Isolation">
    AI agents are scoped to organizations with complete data isolation through RLS enforcement and tenant-aware processing
  </Step>
  <Step title="Template Inheritance">
    Agents created from templates inherit default instructions but can be fully customized with protected token preservation
  </Step>
  <Step title="AI Mode Enforcement">
    Agent automation level (OFF/ASSIST/AUTO) strictly controls when and how agents respond to conversations with workflow integration
  </Step>
  <Step title="Budget Constraints">
    AI operations respect both organizational and personal credit limits with real-time usage tracking and automatic cutoffs
  </Step>
  <Step title="Message Processing">
    Complete inbox handling through AiAgentInboundListener and AiAgentExecuteWorker with queue management and error recovery
  </Step>
</Steps>

### Knowledge Base Management

<Steps>
  <Step title="Multi-format Support">
    Four ingestion shapes (Documents, Pages, FAQ, Snippets) supported through unified entity structure with optimized processing
  </Step>
  <Step title="OpenAI BYOK Requirement">
    All embedding operations require valid OpenAI API key provisioned through OpenAiProjectProvisioner for billing isolation
  </Step>
  <Step title="Chunking Strategy">
    Automatic text splitting with optimal parameters for text-embedding-3-small model and 768-dimensional vector storage
  </Step>
  <Step title="Soft Delete Pattern">
    All entities use soft deletes to maintain audit trails while keeping interfaces clean and supporting compliance requirements
  </Step>
</Steps>
