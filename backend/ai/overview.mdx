The AI module owns both the **AI Agent runtime** (PCD-64 entities/services for automated conversation handling) and the **organization-scoped knowledge base** that powers retrieval-augmented generation across the platform. It supports both the internal CRM assistant and customer-facing AI agents through a unified knowledge management system with OpenAI BYOK (Bring Your Own Key) integration.

<Info>
The module provides a complete AI infrastructure with 10 pre-configured agent templates, automated conversation handling, comprehensive action capabilities, and a flexible knowledge base supporting Documents, Pages, FAQ, and Snippets ingestion.

Last Updated: 2026-05-28
</Info>

## Architecture

The AI module follows a service-oriented architecture with clear separation of concerns across two major functional areas:

### AI Agent Runtime

- **Conversation Automation**: Handles messaging conversations based on AI mode settings
- **Action Execution**: Performs conversation assignment, contact updates, lead lifecycle changes, workflow triggering, and team notifications
- **Media Processing**: Processes audio and image content through dedicated services
- **Budget Management**: Manages organizational and personal credit limits with usage tracking
- **Template System**: Provides 10 pre-configured agent templates for common real estate scenarios

### Knowledge Management

- **Multi-format Ingestion**: Supports Documents (PDF/DOCX/TXT/Markdown), scraped Pages, FAQ rows, and freeform Snippets
- **OpenAI BYOK Integration**: Uses organization's own OpenAI API keys for embedding generation
- **Vector Storage**: Stores embeddings using `text-embedding-3-small` (768 dimensions) in PostgreSQL with pgvector
- **Semantic Search**: Provides contextual knowledge retrieval for both internal assistant and AI agents

### Assignment Vocabulary (v0.11)

<Note>
The AI assistant uses "assignment"/"assignee"/"assigned to" terminology when communicating with users. While internal entities remain as `EntityStakeholder`, boundary surfaces project data using assignee-named keys (`assignees`, `primaryAssignee`, `assignedTeam`) to maintain consistent user-facing vocabulary.
</Note>

## Core Entities

### AI Agent Runtime Entities

#### AiAgent

Represents a configured AI agent for automated conversation handling.

| Field | Type | Description |
|-------|------|-------------|
| `id` | UUID | Primary key |
| `name` | string | Display name for the agent |
| `instructions` | string | AI behavior instructions |
| `isActive` | boolean | Whether agent is currently active |
| `aiMode` | AiModeEnum | Automation level (OFF/ASSIST/AUTO) |
| `organization` | Organization | Organization owner |
| `template` | AiAgentTemplate? | Optional template reference |
| `createdAt` | Date | Creation timestamp |
| `updatedAt` | Date | Last update timestamp |

#### AiAgentTemplate

Pre-configured agent templates for common use cases.

| Field | Type | Description |
|-------|------|-------------|
| `id` | UUID | Primary key |
| `name` | string | Template name |
| `description` | string | Template description |
| `instructions` | string | Default agent instructions |
| `category` | string | Template category |
| `isSystemTemplate` | boolean | Whether it's a system template |

### Knowledge Base Entities

#### KnowledgeBase

Represents a source of knowledge with four ingestion shapes: Documents, Pages, FAQ, Snippets.

| Field | Type | Description |
|-------|------|-------------|
| `id` | UUID | Primary key |
| `name` | string | Display name for the knowledge base |
| `type` | KnowledgeBaseType | DOCUMENT/PAGE/FAQ/SNIPPET |
| `url` | string? | URL for PAGE type |
| `filename` | string? | Original filename for DOCUMENT type |
| `mimeType` | string? | MIME type of document |
| `content` | string? | Direct content for FAQ/SNIPPET types |
| `organization` | Organization | Organization owner |
| `isDeleted` | boolean | Soft delete flag |
| `createdAt` | Date | Creation timestamp |
| `updatedAt` | Date | Last update timestamp |
| `chunks` | KnowledgeChunk[] | Associated knowledge chunks |

#### KnowledgeChunk

Represents processed text chunks with vector embeddings.

| Field | Type | Description |
|-------|------|-------------|
| `id` | UUID | Primary key |
| `knowledgeBase` | KnowledgeBase | Parent knowledge base |
| `text` | string | The actual text content |
| `embedding` | number[] | Vector embedding (768 dims) |
| `organization` | Organization | Organization owner |
| `isDeleted` | boolean | Soft delete flag |
| `createdAt` | Date | Creation timestamp |
| `updatedAt` | Date | Last update timestamp |

#### OrganizationLlmKey

Stores organization's OpenAI API keys for BYOK functionality.

| Field | Type | Description |
|-------|------|-------------|
| `id` | UUID | Primary key |
| `organization` | Organization | Organization owner |
| `provider` | LlmProvider | OPENAI |
| `apiKey` | string | Encrypted API key |
| `isActive` | boolean | Whether key is currently active |
| `createdAt` | Date | Creation timestamp |
| `updatedAt` | Date | Last update timestamp |

## API Reference

### AI Agent Runtime API

**Base Path:** `/v1/ai-agents`  
**Authentication:** Bearer token required  
**Authorization:** Role-based access control

#### Create AI Agent

<CodeGroup>
```bash cURL
curl -X POST /v1/ai-agents \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Sales Assistant",
    "instructions": "Help qualify leads and schedule appointments",
    "aiMode": "AUTO",
    "templateId": "uuid"
  }'
```
```javascript JavaScript
const response = await fetch('/v1/ai-agents', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'Sales Assistant',
    instructions: 'Help qualify leads and schedule appointments',
    aiMode: 'AUTO',
    templateId: 'uuid'
  })
});
```
</CodeGroup>

**Request Body:**
```json
{
  "name": "string",
  "instructions": "string",
  "aiMode": "OFF|ASSIST|AUTO",
  "templateId": "uuid?"
}
```

**Response:**
```json
{
  "id": "uuid",
  "name": "Sales Assistant",
  "instructions": "Help qualify leads and schedule appointments",
  "aiMode": "AUTO",
  "isActive": true,
  "createdAt": "2026-05-28T10:00:00Z"
}
```

### Knowledge Base API

**Base Path:** `/v1/ai-knowledge-bases`  
**Authentication:** Bearer token required  
**Authorization:** Organization-scoped access

#### Upload Document

<CodeGroup>
```bash cURL
curl -X POST /v1/ai-knowledge-bases/documents \
  -H "Authorization: Bearer {token}" \
  -F "file=@document.pdf" \
  -F "name=Company Policies"
```
```javascript JavaScript
const formData = new FormData();
formData.append('file', file);
formData.append('name', 'Company Policies');

const response = await fetch('/v1/ai-knowledge-bases/documents', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${token}`
  },
  body: formData
});
```
</CodeGroup>

**Parameters:**
- `file`: File (PDF, DOCX, TXT, or Markdown)
- `name`: string (display name)

**Response:**
```json
{
  "id": "uuid",
  "name": "Company Policies",
  "type": "DOCUMENT",
  "filename": "document.pdf",
  "mimeType": "application/pdf",
  "createdAt": "2026-05-28T10:00:00Z"
}
```

## Services

### AI Agent Services

#### AiAgentService
Primary service for AI agent management and configuration.

#### AiAgentRuntimeService
Handles real-time agent execution and conversation processing.

#### AiAgentResolverService
Determines which agent should handle incoming conversations.

#### AiAgentContextBuilderService
Builds context for agent responses including lead data and conversation history.

#### AiAgentTokenBudgetService
Manages token usage and budget constraints for AI operations.

#### AiAgentToolRegistryService
Registers and manages available tools/actions for AI agents.

#### AiAgentActionService
Executes agent actions like assignments, updates, and notifications.

#### AiAgentMediaProcessorService
Processes audio and image content for agent understanding.

#### AiAgentPricingService
Calculates costs for AI operations and credit usage.

#### AiAgentOptimizeService
Optimizes agent instructions while preserving protected tokens.

#### AiAgentTemplateService
Manages agent templates and template-based agent creation.

### Knowledge Base Services

#### AiAgentKnowledgeBaseService

Primary service for knowledge base operations supporting four ingestion shapes:

<Tabs>
  <Tab title="Documents">
    Upload and process PDF, DOCX, TXT, and Markdown files with automatic text extraction and chunking
  </Tab>
  <Tab title="Pages">
    Scrape and process web pages with Firecrawl integration for clean content extraction
  </Tab>
  <Tab title="FAQ">
    Direct entry of question-answer pairs for frequently asked questions
  </Tab>
  <Tab title="Snippets">
    Freeform text content for policies, procedures, and reference information
  </Tab>
</Tabs>

#### KnowledgeBaseService
Core CRUD operations for knowledge base entities with organization scoping.

#### EmbeddingService

Stateless service for generating vector embeddings using organization-provided OpenAI keys:

- **BYOK Integration**: Takes OpenAI API key as per-call parameter
- **Batch Processing**: Optimizes API calls for efficient embedding generation
- **Error Handling**: Graceful handling of API failures and retries
- **Vector Storage**: Efficient storage of 768-dimensional embeddings

<Warning>
The EmbeddingService requires valid OpenAI API keys configured via OrganizationLlmKey. All embedding costs are billed to the organization's OpenAI project.
</Warning>

#### AiCreditUsageService
Tracks and manages AI credit consumption across organizational and personal budgets.

#### AiActivityLogService
Comprehensive logging of AI agent activities with filtering and analytics capabilities.

### Legacy Integration Services

#### AIKnowledgeService

<Note>
This is a dev-only transitional service that provides search-only access to the new knowledge base structure for the internal CRM assistant. It will be deprecated once the assistant migrates to use AiAgentKnowledgeBaseService directly.
</Note>

Thin glue service with single public method `searchKnowledge` that bridges the AI assistant's knowledge search tool to the new KB structure.
