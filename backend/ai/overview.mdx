The AI module provides artificial intelligence capabilities for the PropWise CRM system, including knowledge management, document processing, embeddings, and system prompt management. It exposes authenticated internal APIs only; the previous API-key public system-prompt endpoint has been removed.

<Info>
The module is designed around a knowledge base system where documents and web content can be ingested, processed into chunks, embedded for semantic search, and used to provide contextual AI responses through system prompts.

Last Updated: 2026-04-26
</Info>

## Architecture

The AI module follows a service-oriented architecture with clear separation of concerns:

- **Knowledge Management**: Handles ingestion and storage of knowledge sources (documents, URLs)
- **Document Processing**: Converts various document formats (PDF, DOCX, TXT) into processable text
- **Embedding Generation**: Creates vector embeddings using Google Gemini API for semantic search
- **Web Scraping**: Integrates with Firecrawl service for URL content extraction

### Design Decisions

<AccordionGroup>
  <Accordion title="Multi-tenant Architecture">
    All entities are organization-scoped with RLS (Row Level Security) enforcement to ensure complete data isolation between organizations.
  </Accordion>
  
  <Accordion title="Chunking Strategy">
    Documents are split into 3000-character chunks with 200-character overlap for optimal embedding performance and semantic search accuracy.
  </Accordion>
  
  <Accordion title="Soft Deletes">
    All entities use `isDeleted` flag with MikroORM filters for data retention while maintaining clean user interfaces.
  </Accordion>
  
  <Accordion title="External Dependencies">
    Google Gemini for embeddings and Firecrawl for web scraping provide best-in-class AI and content extraction capabilities.
  </Accordion>
  
  <Accordion title="Internal Access Only">
    AI knowledge and prompt management are exposed through authenticated CRM APIs; external API-key access has been removed.
  </Accordion>
</AccordionGroup>

## Entities

### KnowledgeSource

Represents a source of knowledge that can be processed and chunked.

| Field | Type | Description |
|-------|------|-------------|
| `id` | UUID | Primary key |
| `name` | string | Display name for the source |
| `type` | KnowledgeSourceType | Either 'document' or 'url' |
| `url` | string? | URL if type is 'url' |
| `filename` | string? | Original filename if type is 'document' |
| `mimeType` | string? | MIME type of document |
| `organization` | Organization | Organization owner |
| `isDeleted` | boolean | Soft delete flag (default: false) |
| `createdAt` | Date | Creation timestamp |
| `updatedAt` | Date | Last update timestamp |
| `chunks` | KnowledgeChunk[] | Associated knowledge chunks |

### KnowledgeChunk

Represents a processed chunk of text from a knowledge source with its embedding.

| Field | Type | Description |
|-------|------|-------------|
| `id` | UUID | Primary key |
| `source` | KnowledgeSource | Parent knowledge source |
| `text` | string | The actual text content |
| `embedding` | number[] | Vector embedding for semantic search |
| `organization` | Organization | Organization owner |
| `isDeleted` | boolean | Soft delete flag (default: false) |
| `createdAt` | Date | Creation timestamp |
| `updatedAt` | Date | Last update timestamp |

### SystemPrompt

Stores system prompts used for AI interactions.

| Field | Type | Description |
|-------|------|-------------|
| `id` | UUID | Primary key |
| `prompt` | string | The system prompt text |
| `isActive` | boolean | Whether this prompt is active (default: true) |
| `organization` | Organization | Organization owner |
| `isDeleted` | boolean | Soft delete flag (default: false) |
| `createdAt` | Date | Creation timestamp |
| `updatedAt` | Date | Last update timestamp |

## API Reference

### Internal API (Authenticated)

**Base Path:** `/ai/knowledge`  
**Authentication:** Bearer token required  
**Authorization:** Role-based access control

#### Upload Document

<CodeGroup>
```bash cURL
curl -X POST /ai/knowledge/sources/upload \
  -H "Authorization: Bearer {token}" \
  -F "file=@document.pdf" \
  -F "name=Important Document"
```
```javascript JavaScript
const formData = new FormData();
formData.append('file', file);
formData.append('name', 'Important Document');

const response = await fetch('/ai/knowledge/sources/upload', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${token}`
  },
  body: formData
});
```
</CodeGroup>

**Parameters:**
- `file`: File (PDF, DOCX, or TXT)
- `name`: string (display name)

**Response:**
```json
{
  "id": "uuid",
  "name": "string",
  "type": "document",
  "filename": "string",
  "mimeType": "string",
  "createdAt": "2024-12-19T10:00:00Z"
}
```

#### Add URL Source

<CodeGroup>
```bash cURL

  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
```

## Services

### AIKnowledgeService

Primary service for knowledge management operations:

- Upload and process documents through multi-format support
- Add URL-based knowledge sources with Firecrawl integration
- Manage knowledge source lifecycle including soft deletes
- Handle document processing and chunking with optimal parameters
- Coordinate embedding generation for semantic search

<Tip>
The AIKnowledgeService orchestrates the entire knowledge ingestion pipeline from document upload through chunk creation and embedding generation.
</Tip>

### DocumentProcessorService

Processes various document formats into text with advanced capabilities:

<CardGroup cols={2}>
  <Card title="PDF Processing" icon="file-pdf">
    Uses `pdf-parse` library for reliable PDF text extraction with support for complex layouts
  </Card>
  <Card title="DOCX Processing" icon="file-word">
    Uses `mammoth` library for Microsoft Word document conversion with formatting preservation
  </Card>
  <Card title="TXT Processing" icon="file-text">
    Direct text extraction with automatic encoding detection
  </Card>
  <Card title="Text Chunking" icon="scissors">
    Intelligent splitting into 3000-character chunks with 200-character overlap for context preservation
  </Card>
</CardGroup>

**Key Features:**
- **Smart Chunking Logic**: Preserves sentence boundaries when possible
- **Batch Processing**: Handles large documents efficiently to avoid timeouts
- **Format Detection**: Automatically detects and processes different document types
- **Error Handling**: Graceful fallbacks for corrupted or unsupported files

**Library Dependencies:**
- **pdf-parse**: Primary PDF text extraction library providing reliable parsing of complex PDF structures
- **mammoth**: Microsoft Word document processing library enabling clean DOCX to text conversion with proper formatting preservation

**Text Processing Pipeline:**
- Document format detection and validation
- Content extraction using format-specific libraries
- Text normalization and cleaning
- Intelligent chunking with overlap for context preservation
- Quality validation before chunk storage

### EmbeddingService

Manages vector embedding generation using Google Gemini API with sophisticated processing:

- **Batch Processing**: Processes exactly 200 items per batch for optimal API performance
- **Rate Limiting**: Implements automatic retry logic with exponential backoff
- **Vector Storage**: Efficiently stores embeddings in PostgreSQL with vector extensions
- **Quality Assurance**: Validates embedding dimensions and handles API failures gracefully

<Warning>
The EmbeddingService requires Google Gemini API credentials to be properly configured. Ensure rate limits are monitored to avoid service interruptions.
</Warning>

### FirecrawlService

Integrates with Firecrawl service for comprehensive web content extraction:

- **Website Mapping**: Discovers all accessible URLs through intelligent site crawling
- **Content Scraping**: Extracts clean content and converts to structured markdown
- **API Integration**: Robust external API communication with comprehensive error handling
- **Content Validation**: Ensures scraped content meets quality standards before processing

<Info>
FirecrawlService can handle complex websites with dynamic content, JavaScript rendering, and various content management systems.
</Info>

## Business Rules

### Knowledge Sources

<Steps>
  <Step title="Organization Isolation">
    Users can only access knowledge sources within their organization through RLS enforcement at the database level
  </Step>
  <Step title="File Type Restrictions">
    Only PDF, DOCX, and TXT files are supported for document upload to ensure reliable processing
  </Step>
  <Step title="URL Validation">
    URLs must be accessible and return valid content before processing begins
  </Step>
  <Step title="Soft Delete Implementation">
    Deleted sources are flagged with `isDeleted` but data is retained for comprehensive audit trails
  </Step>
</Steps>

### System Prompts

<Warning>
Only one system prompt can be active per organization at a time. When a new prompt is activated, the previous one is automatically deactivated but retained for version history.
</Warning>

1. **Single Active Prompt**: Enforced at the application level to prevent conflicts
2. **Required Content**: System prompts cannot be empty and must contain meaningful instructions
3. **Version History**: All previous prompts are retained when deactivated for rollback capabilities
4. **Organization Scoping**: Each organization maintains its own independent prompt history

### Processing Rules

1. **Chunking Strategy**: Exactly 3000 characters per chunk with 200-character overlap to maintain context
2. **Embedding Generation**: All chunks must have embeddings generated before being available for search
3. **Batch Processing**: Large documents are processed in manageable batches to avoid system timeouts
4. **Error Recovery**: Failed processing attempts are logged and can be retried manually

## Patterns and Conventions

### Error Handling

- **Service-level Exceptions**: Caught and converted to appropriate HTTP status codes with descriptive messages
- **File Upload Errors**: Return `BadRequestException` with specific guidance for resolution
- **Missing Resources**: Return `NotFoundException` with helpful context
- **API Integration Errors**: Graceful degradation with retry mechanisms

### Data Access

- **Organization Boundaries**: All repository operations automatically respect organization boundaries through RLS
- **Soft Delete Pattern**: Implemented consistently across all entities with automated filtering
- **Audit Timestamps**: `createdAt` and `updatedAt` fields maintained automatically by MikroORM
- **Optimistic Locking**: Prevents concurrent modification conflicts

### Security

<Check>
The AI module implements comprehensive security measures including Bearer token authentication for internal endpoints and role-based authorization using `@CheckAccess` decorator with organization-level isolation.
</Check>

**Security Features:**
- **Rate Limiting**: Prevents abuse of embedding generation endpoints
- **Input Sanitization**: All user inputs are validated and sanitized
- **Audit Logging**: Complete security event logging for compliance

### Performance

- **Batch Processing**: Optimizes embedding generation through intelligent batching
- **Chunked Processing**: Handles large documents without memory issues
- **Async Operations**: All external API calls are non-blocking
- **Caching Strategy**: Frequently accessed system prompts are cached
- **Connection Pooling**: Efficient database connection management
