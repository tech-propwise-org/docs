## Overview

The AI Assistant Module provides intelligent conversational capabilities across CRM, real estate, and task management domains. This specification covers the latest improvements in token optimization, routing intelligence, and tool schema management.

## Recent Updates (v7)

### Centralized Access-Scope Contract

<Info>
**Version:** `crm-assistant-v0.44-centralized-access-scope`  
**Registry Phase:** `phase-1g-scope-describe-slim`
</Info>

The module now implements a centralized access-scope contract that significantly reduces token usage:

- **Single Source of Truth**: The paragraph-length `all`/`my`/`team` scope contract is now stated once in the `SYSTEM_PROMPT` "Access scope (`view` / `scope`) rules" block
- **Token Reduction**: ~10k characters net removed across five providers
- **Shared Constants**: Uses `CRM_SCOPE_VIEW_DESCRIBE` / `CRM_SCOPE_TEAMID_DESCRIBE` constants and `crmScopeViewDescribe(myClause)` helper

<CodeGroup>
```typescript src/modules/crm/shared/ai/scope-describe.constants.ts
export const CRM_SCOPE_VIEW_DESCRIBE = 
  "View scope parameter - references centralized access rules in system prompt";

export const CRM_SCOPE_TEAMID_DESCRIBE = 
  "Team ID parameter - references centralized access rules in system prompt";

export function crmScopeViewDescribe(myClause: string): string {
  return `${CRM_SCOPE_VIEW_DESCRIBE}. ${myClause}`;
}
```

```typescript assembleSystemPrompt
// Common block kept by assembleSystemPrompt for crm_records group
const ACCESS_SCOPE_RULES = `
Access scope (view / scope) rules:
- 'my': User's own records only
- 'team': User's team records (requires teamId)
- 'all': All accessible records based on permissions
`;
```
</CodeGroup>

### Enhanced Intent Routing

The intent router now properly handles active entity contexts as signals rather than overrides:

<Steps>
<Step title="Signal-Based Routing">
Active entity context is used as a signal, not an automatic override. The router now evaluates message content before applying entity context.
</Step>

<Step title="Conflict Resolution">
When message keywords conflict with active entity group, the system abstains and uses Tier B lite classifier for re-routing.
</Step>

<Step title="Agreement Handling">
When exactly one keyword family agrees with or complements the entity group, that group is selected immediately.
</Step>
</Steps>

## Error Handling

### Validation Patterns

<CodeGroup>
```typescript UUID Normalization
function normalizeOptionalUuid(value?: string): string | undefined {
  if (!value) return undefined;
  
  // Drop RFC 9562 reserved nil and max UUIDs
  if (value === '00000000-0000-0000-0000-000000000000' ||
      value === 'ffffffff-ffff-ffff-ffff-ffffffffffff') {
    return undefined;
  }
  
  return value;
}
```

```typescript Scope Validation  
if (view === 'team' && !teamId) {
  throw new BadRequestException("view='team' requires teamId");
}
```
