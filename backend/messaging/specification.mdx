# Messaging Module Specification

<Note>
Last Updated: 2026-03-30  
Status: Active
</Note>

## Overview

The Messaging module provides multi-channel communication capabilities for the PropWise CRM system. It supports conversation management across WhatsApp, Instagram, and Facebook Messenger channels, with features for message templates, outbound message queuing, per-conversation AI mode control, and PersonChannel-based contact resolution.

The module is designed for reliable message delivery using a transactional outbox pattern and supports both agent-to-customer and automated messaging workflows.

## Architecture

The messaging system follows a conversation-centric architecture with the following key design decisions:

<CardGroup cols={2}>
  <Card title="Channel Abstraction" icon="network-wired">
    Unified interface for WhatsApp, Instagram, and Facebook Messenger
  </Card>
  <Card title="Transactional Outbox" icon="box">
    Ensures reliable message delivery using the outbox pattern
  </Card>
  <Card title="PersonChannel-based Contact Resolution" icon="address-book">
    Conversations link to the CRM's PersonChannel entity (replacing the former ContactChannelIdentity)
  </Card>
  <Card title="AI Mode Control" icon="robot">
    Per-conversation AI mode (OFF, AUTO_REPLY, SUGGEST_ONLY, DRAFT) with cascade from ChannelAccount → Organization → default
  </Card>
</CardGroup>

### Additional Features

- **Template Management**: Supports pre-approved message templates for compliance
- **Conversation Threading**: Groups related messages into conversations with status tracking

#### Entity Filters

<Warning>
**Important Filtering Rules:**
- `isDeleted` (default=true): Excludes soft-deleted conversations
- `active` (default=true): Excludes archived conversations (`isArchived: false`). The `active` filter intentionally omits `isDeleted` to avoid a redundant WHERE clause — the separate `isDeleted` filter handles soft-delete exclusion.
- **Invariant: never disable `isDeleted` while keeping `active` — that would return soft-deleted records.**
</Warning>

<Tip>
When querying archived conversations (e.g. unarchive), disable the `active` filter: `{ filters: { active: false } }`.
</Tip>
