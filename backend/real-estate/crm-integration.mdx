The real estate module is structured as **six specialized modules** spanning three visibility scopes: **global** (no RLS), **org-private** (standard RLS), and **cross-org** (custom RLS). This page documents how these modules integrate with the CRM through property interests, external property data IDs, and inventory ownership links.

## Architecture overview

The external **Propwise Labs API** stores and serves all property data — both structural (DLD registry) and enrichment (images, payment plans, documents, amenities, pricing, geocoding where exposed). The CRM database stores only CRM-specific data (developer portal auth, ownership links, property interests, marketplace listings, secondary unit context).

<Note>
All modules consume property data exclusively through `PropwiseLabsService`. The CRM no longer opens a second MikroORM connection or maps Propwise Labs entities locally. The reference database configuration has been completely removed and replaced with Propwise Labs API configuration (`app.propwiseLabs.*`) including API credentials, timeout settings, and Bird Verify integration for developer portal contact verification.
</Note>

## Data ownership and access patterns

| Data Type                                                  | Source            | Access Pattern                                                                                         |
| ---------------------------------------------------------- | ----------------- | ------------------------------------------------------------------------------------------------------ |
| Projects, units, buildings, areas, communities, developers | Propwise Labs API | `PropwiseLabsService`                                                                                  |
| Images, payment plans, documents, amenities                | Propwise Labs API | `PropwiseLabsService`                                                                                  |
| Property data mutations                                    | Propwise Labs API | Change requests for create/delete/approval flows; direct wrappers for selected existing-resource edits |
| Developer auth/profile (DeveloperUser)                     | CRM DB            | Direct entity access                                                                                   |
| Off-plan building publication state                        | CRM DB            | `OffPlanBuildingPublication` stores app-owned building visibility by Propwise Labs building ID         |
| Secondary units (org-scoped inventory)                     | CRM DB            | Direct entity access                                                                                   |
| Marketplace listings & deals (SharedListing, SharedDeal)   | CRM DB            | Direct entity access                                                                                   |
| Property interests (Lead/Deal)                             | CRM DB            | Direct entity access, ref IDs as plain ints                                                            |
| Property matches                                           | CRM DB            | Direct entity access, ref IDs as plain ints                                                            |

## External property ID pattern

CRM entities that reference off-plan projects or units store **plain integer IDs** (`refProjectId`, `refUnitId`) from Propwise Labs. No foreign keys exist — this decouples the CRM from the property data source.

```typescript
// CRM entity referencing Propwise Labs data:
@Property({ type: 'int', nullable: true })
@Index()
refProjectId?: number;  // ID in Propwise Labs projects

@Property({ type: 'int', nullable: true })
@Index()
refUnitId?: number;     // ID in Propwise Labs units
```

### Lead to deal conversion workflow

When converting a lead to a deal, property interests are transferred through a structured process:

<Steps>
<Step title="Copy property interest">
All `LeadPropertyInterest` records are duplicated as `DealPropertyInterest` records
</Step>
<Step title="Transfer reference IDs">
`refProjectId` and `refUnitId` (plain integers) are copied directly without modification
</Step>
<Step title="Resolve CRM entities">
`secondaryUnit` and `sharedListing` (ManyToOne relationships) are resolved and validated from CRM database
</Step>
<Step title="Update status">
Original `LeadPropertyInterest` status is set to 'converted'
</Step>
</Steps>

### Contact and company relations

Property interests create indirect relationships between contacts and properties:

- Access a contact's property interests: `contact.person.leads -> lead.propertyInterests`
- Access a company's property deals: `company.deals -> deal.propertyInterests`

This enables tracking property preferences and transaction history at both individual and organizational levels.

export class Person extends TenantScopedEntity {

### Custom RLS implementation

The shared marketplace uses custom RLS that allows both participating organizations to access shared data:

```typescript
@Injectable()
export class SharedMarketplaceService {
  private applySharedRLS(qb: QueryBuilder<any>, entityAlias: string): QueryBuilder<any> {
    const tenantId = this.tenantContext.getTenantId();
    
    return qb.andWhere(
      `(${entityAlias}.listing_org_id = ? OR ${entityAlias}.target_org_id = ? OR ${entityAlias}.buying_org_id = ?)`,
      [tenantId, tenantId, tenantId]
    );
  }
}
```

## Property matching engine

The matching engine provides automated property recommendations using `PropwiseLabsService`:

```typescript
@Injectable()
export class PropertyMatchingService {
  constructor(
    private readonly propwiseLabsService: PropwiseLabsService,
    private readonly tenantContext: TenantContext,
    @InjectEntityManager() private readonly em: EntityManager,
  ) {}

  async findMatchingProperties(interest: LeadPropertyInterest): Promise<PropertyMatch[]> {
    const matches: PropertyMatch[] = [];

    // Match against Propwise Labs projects/units
    if (interest.budgetMin || interest.budgetMax) {
      const refMatches = await this.propwiseLabsService.searchProjects({
        priceMin: interest.budgetMin,
        priceMax: interest.budgetMax,
        interestType: interest.interestType,
      });
      
      matches.push(...refMatches.map(project => ({
        type: 'propwise_labs_project',
        refProjectId: project.id,
        matchScore: this.calculateMatchScore(interest, project),
      })));
    }

    // Match against inventory units
    const inventoryMatches = await this.findInventoryMatches(interest);
    matches.push(...inventoryMatches);

    // Match against shared marketplace listings
    const marketplaceMatches = await this.findMarketplaceMatches(interest);
    matches.push(...marketplaceMatches);

    return matches.sort((a, b) => b.matchScore - a.matchScore);
  }

  private calculateMatchScore(interest: LeadPropertyInterest, property: any): number {
    let score = 0;
    
    // Budget alignment
    if (interest.budgetMin && property.priceFrom >= interest.budgetMin) score += 20;
    if (interest.budgetMax && property.priceTo <= interest.budgetMax) score += 20;
    
    // Location preferences (if available)
    if (interest.preferredAreas?.includes(property.areaId)) score += 30;
    
    // Property type preferences
    if (interest.propertyTypes?.includes(property.propertyTypeId)) score += 20;
    
    // Interest type alignment
    if (interest.interestType === 'buy' && property.availableForSale) score += 10;
    if (interest.interestType === 'rent' && property.availableForRent) score += 10;
    
    return score;
  }
}
```

### Propwise Labs service integration

All property data access flows through the `PropwiseLabsService`:

```typescript
@Injectable()
export class PropwiseLabsService {
  private accessToken?: string;
  private tokenExpiresAt = 0;

  async getProject(id: number): Promise<Project | null> {
    const token = await this.getValidAccessToken();
    const response = await this.httpService.get(`/projects/${id}`, {
      headers: { Authorization: `Bearer ${token}` },
      params: { 
        include: 'developer,area,gallery,payment_plans,buildings,buildings.units'
      }
    }).toPromise();
    
    return response.data;
  }

  async submitChangeRequest(request: ChangeRequestData): Promise<ChangeRequest> {
    const token = await this.getValidAccessToken();
    const response = await this.httpService.post('/change-requests', request, {
      headers: { Authorization: `Bearer ${token}` }
    }).toPromise();
    
    return response.data;
  }

  private async getValidAccessToken(): Promise<string> {
    if (this.accessToken && Date.now() < this.tokenExpiresAt - 60000) {
      return this.accessToken;
    }
    
    const authResponse = await this.authenticate();
    this.accessToken = authResponse.access_token;
    this.tokenExpiresAt = Date.now() + (authResponse.expires_in * 1000);
    
    return this.accessToken;
  }
}
```
