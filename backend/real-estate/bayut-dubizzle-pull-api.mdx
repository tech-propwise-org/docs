This document provides the authoritative reference for the Bayut **Pull API** used to fetch lead and view data from Bayut, covering the external API contract defined by Bayut including endpoints, authorization, query parameters, and sample responses.
</Info>

## Overview

The Bayut Pull API allows clients to retrieve lead data and view statistics from the Bayut platform. This comprehensive API provides access to:

- **Lead Data**: Actual leads with contact information (`is_trulead=1`)
- **View Statistics**: View counts and engagement metrics (`is_trulead=0`)
- **Multiple Contact Methods**: Email, WhatsApp, SMS, and phone interactions
- **Target Types**: Listings, agents, and agencies
- **Call Logs**: Detailed call tracking information
- **Story Leads**: Engagement with property stories

<Note>
The API enables developers to effectively integrate Bayut lead data into their own applications or systems with proper authorization and query parameter configuration.
</Note>

## Endpoint

The base endpoint for all API requests:

```
www.bayut.com/api-v7/stats/website-client-leads
```

## Authorization

<Steps>
<Step title="Obtain API Key">
Each client is provided with a unique API key from Bayut.
</Step>

<Step title="Include Bearer Token">
Pass the API key as a bearer token in the `Authorization` header:

```http
Authorization: Bearer <API_KEY>
```
</Step>
</Steps>

<Warning>
Keep your API key secure and never expose it in client-side code or public repositories.
</Warning>

## Sample cURL Request

```bash
curl --location \
  'www.bayut.com/api-v7/stats/website-client-leads?timestamp=2025-03-10%2000%3A00%3A00&target=listing&type=email&is_trulead=1' \
  --header 'Authorization: Bearer <API_KEY>'
```

<Tip>
The `timestamp` value must be URL-encoded: `2025-03-10 00:00:00` becomes `2025-03-10%2000%3A00%3A00` where `%20` = space and `%3A` = colon.
</Tip>

## Query Parameters

| Parameter | Description | Possible Values |
|-----------|-------------|-----------------|
| `type` | The type of lead or view | `whatsapp`, `sms`, `phone`, `email`, `call_logs`, `story_leads` |
| `timestamp` | The date and time of the request | Format: `YYYY-MM-DD HH:MM:SS` |
| `target` | The entity for which data is fetched | `listing`, `agent`, `agency` |
| `is_trulead` | Whether to return leads or view counts | `0` (views), `1` (leads) |

<Info>
- `is_trulead=1` returns **actual leads** with contact details and messages
- `is_trulead=0` returns **view counts** and engagement statistics
</Info>

## View Data (is_trulead=0)

When `is_trulead=0`, the API returns view counts and engagement statistics instead of detailed lead information. This includes metrics for:

- WhatsApp listing and agent views
- SMS listing and agent views  
- Phone listing, agent, and agency views

<Check>
View data provides valuable insights into user engagement patterns and can help optimize your listing and agent performance.
</Check>

## Response Field Reference

<AccordionGroup>
<Accordion title="Lead Response Fields">

| Field | Type | Description |
|-------|------|-------------|
| `lead_id` | string | Unique identifier for the lead |
| `source` | string | Always "bayut" for this API |
| `date_time` | string | Timestamp of the lead in format "YYYY-MM-DD HH:MM:SS" |
| `listing_details` | object | Information about the property listing |
| `agent_details` | object | Information about the agent (when applicable) |
| `agency_details` | object | Information about the agency (when applicable) |
| `inquirer_details` | object | Contact information and message from the lead |

</Accordion>

<Accordion title="Listing Details Fields">

| Field | Type | Description |
|-------|------|-------------|
| `listing_id` | integer | Internal listing ID |
| `listing_reference` | string | External listing reference |
| `current_type` | string | Property type (e.g., "Apartment") |
| `listing_url` | string | URL to the listing page (story leads only) |

</Accordion>

<Accordion title="Inquirer Details Fields">

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Name of the person making the inquiry |
| `cell` | string | Phone number of the inquirer |
| `email` | string | Email address of the inquirer |
| `message` | string | Message content from the inquirer |

</Accordion>

<Accordion title="Delivery Notifications Fields">

| Field | Type | Description |
|-------|------|-------------|
| `status` | string | Delivery status: "sent", "delivered", "read" |
| `created_at` | string | Timestamp of the status update |

</Accordion>
</AccordionGroup>
