<Info>
This document provides the authoritative reference for the Bayut **Pull API** used to fetch lead and view data from Bayut. It describes the external API contract defined by Bayut including endpoint, authorization, query parameters, and sample responses for both leads (`is_trulead=1`) and views (`is_trulead=0`).
</Info>

## Overview

The Bayut Pull API allows clients to retrieve comprehensive lead data from the Bayut platform. This API provides access to:

- **Lead Data**: Actual contact information and inquiries from potential customers
- **View Counts**: Analytics data showing interaction metrics
- **Multiple Contact Types**: Email, WhatsApp, SMS, and phone interactions
- **Various Targets**: Listings, agents, and agencies

<Note>
Each client receives a unique API key for authentication. The API supports both real leads and view analytics through the `is_trulead` parameter.
</Note>

## Endpoint

The base endpoint for all API requests is:

```
www.bayut.com/api-v7/stats/website-client-leads
```

## Authorization

<Warning>
Each client is provided with a unique API key, which must be passed as a bearer token in the `Authorization` header.
</Warning>

```http
Authorization: Bearer <API_KEY>
```

## Basic request example

<CodeGroup>

```bash cURL
curl --location \
  'www.bayut.com/api-v7/stats/website-client-leads?timestamp=2025-03-10%2000%3A00%3A00&target=listing&type=email&is_trulead=1' \
  --header 'Authorization: Bearer <API_KEY>'
```

```javascript JavaScript
const response = await fetch(
  'www.bayut.com/api-v7/stats/website-client-leads?timestamp=2025-03-10%2000%3A00%3A00&target=listing&type=email&is_trulead=1',
  {
    headers: {
      'Authorization': 'Bearer <API_KEY>'
    }
  }
);
```

```python Python
import requests

url = "www.bayut.com/api-v7/stats/website-client-leads"
params = {
    "timestamp": "2025-03-10 00:00:00",
    "target": "listing",
    "type": "email",
    "is_trulead": "1"
}
headers = {
    "Authorization": "Bearer <API_KEY>"
}

response = requests.get(url, params=params, headers=headers)
```

</CodeGroup>

<Tip>
The `timestamp` value must be URL-encoded: `2025-03-10 00:00:00` becomes `2025-03-10%2000%3A00%3A00` where `%20` = space and `%3A` = colon.
</Tip>

## Query parameters

| Parameter | Description | Possible Values | Required |
|-----------|-------------|-----------------|----------|
| `type` | The type of lead/interaction | `whatsapp`, `sms`, `phone`, `email`, `call_logs`, `story_leads` | Yes |
| `timestamp` | The date and time of the request | Format: `YYYY-MM-DD HH:MM:SS` | Yes |
| `target` | The entity for which leads are fetched | `listing`, `agent`, `agency` | Yes* |
| `is_trulead` | Whether it returns leads or view counts | `0` (views), `1` (leads) | Yes |

<Note>
*The `target` parameter is not required for `call_logs` and `story_leads` types.
</Note>

- `is_trulead=1` returns **actual leads** with contact information
- `is_trulead=0` returns **view counts** for analytics

## View counts (is_trulead=0)

When `is_trulead=0`, the API returns view count analytics instead of actual lead data. This provides metrics on user interactions without personal contact information.

<Info>
View data is useful for analytics and understanding user engagement patterns with your listings, agents, or agency pages.
</Info>

## Integration considerations

<Steps>
  <Step title="Obtain API credentials">
    Contact Bayut to receive your unique API key for authentication.
  </Step>

  <Step title="Implement authentication">
    Always include the bearer token in the Authorization header for all requests.
  </Step>

  <Step title="Handle URL encoding">
    Properly encode timestamp parameters, especially spaces and colons in datetime strings.
  </Step>

  <Step title="Choose appropriate parameters">
    Use `is_trulead=1` for actual leads with contact information, `is_trulead=0` for analytics data.
  </Step>

  <Step title="Parse response data">
    Handle different response structures based on the lead type and target combination.
  </Step>
</Steps>

<Warning>
Remember to respect rate limits and implement appropriate error handling for production integrations. The API responses may contain sensitive personal information when `is_trulead=1`.
</Warning>
