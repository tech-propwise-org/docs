## Overview

The Bayut/Dubizzle Pull API provides a comprehensive solution for retrieving lead and view data from Bayut and Dubizzle real estate portals. This external API enables real estate professionals to:

- **Fetch Lead Data:** Retrieve actual leads with detailed inquirer information
- **Monitor View Counts:** Track property and agent view statistics
- **Access Multiple Channels:** Pull data from email, WhatsApp, SMS, and phone interactions
- **Target Specific Entities:** Focus on listings, agents, or agencies

<Note>
This API uses an external contract defined by Bayut. All endpoints, parameters, and response formats must be used exactly as specified in their documentation.
</Note>

## Authentication

### API Key Requirements

Each client receives a unique API key that must be included in every request as a bearer token.

<CodeGroup>
```bash cURL
curl --location 'www.bayut.com/api-v7/stats/website-client-leads' \
  --header 'Authorization: Bearer YOUR_API_KEY'
```

```javascript JavaScript
const response = await fetch('www.bayut.com/api-v7/stats/website-client-leads', {
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY'
  }
});
```

```python Python
import requests

headers = {
    'Authorization': 'Bearer YOUR_API_KEY'
}

response = requests.get('www.bayut.com/api-v7/stats/website-client-leads', headers=headers)
```
</CodeGroup>

## API Endpoint

The base endpoint for all lead and view data requests:

```
www.bayut.com/api-v7/stats/website-client-leads
```

## Query Parameters

<AccordionGroup>
  <Accordion title="Required Parameters">
    | Parameter | Description | Example |
    |---|---|---|
    | `timestamp` | Request date and time | `2025-03-10 00:00:00` |
    | `target` | Target entity type | `listing`, `agent`, `agency` |
    | `type` | Lead/view type | `email`, `whatsapp`, `sms`, `phone` |
    | `is_trulead` | Data type flag | `1` (leads) or `0` (views) |
  </Accordion>

  <Accordion title="Parameter Details">
    **Timestamp Format**
    - Format: `YYYY-MM-DD HH:MM:SS`
    - URL encoding required: spaces become `%20`, colons become `%3A`
    - Example: `2025-03-10%2000%3A00%3A00`

    **Target Values**
    - `listing`: Property-specific data
    - `agent`: Individual agent data
    - `agency`: Agency-level data

    **Type Values**
    - `email`: Email inquiries
    - `whatsapp`: WhatsApp messages
    - `sms`: SMS communications
    - `phone`: Phone calls
    - `call_logs`: Call log data (special type)
    - `story_leads`: Story-based leads (special type)

    **Is Trulead Flag**
    - `1`: Returns actual lead data with inquirer details
    - `0`: Returns view count statistics only
  </Accordion>
</AccordionGroup>

## View Data Retrieval

<Tip>
Use `is_trulead=0` to fetch view count statistics without detailed inquirer information. This provides aggregated data for analytics purposes.
</Tip>

View data endpoints follow the same query parameter structure but return count-based statistics rather than detailed lead information. The response format varies by target type and provides numerical data for tracking engagement metrics.

<Steps>
  <Step title="Set is_trulead parameter">
    Set `is_trulead=0` in your query parameters to retrieve view counts instead of lead details.
  </Step>
  
  <Step title="Choose target type">
    Select your target: `listing`, `agent`, or `agency` depending on which view statistics you need.
  </Step>
  
  <Step title="Specify view type">
    Choose from `whatsapp`, `sms`, `phone`, or `email` to filter view data by interaction type.
  </Step>
  
  <Step title="Process response">
    Handle the numerical view count data returned by the API for your analytics dashboard.
  </Step>
</Steps>

## Implementation Guidelines

### Error Handling

<CodeGroup>
```javascript JavaScript
try {
  const response = await fetch(url, { headers });
  
  if (!response.ok) {
    throw new Error(`API Error: ${response.status}`);
  }
  
  const data = await response.json();
  return data;
} catch (error) {
  console.error('Bayut API request failed:', error);
  throw error;
}
```

```python Python
import requests

try:
    response = requests.get(url, headers=headers)
    response.raise_for_status()
    return response.json()
except requests.exceptions.RequestException as e:
    print(f"Bayut API request failed: {e}")
    raise
```
</CodeGroup>
