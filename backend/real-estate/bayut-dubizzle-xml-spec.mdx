An XML integration is required to sync properties with Bayut and/or dubizzle in bulk from an external CRM. This document describes the complete specification for the unified XML feed format that PropWise generates to syndicate listings to both portals.

<Note>
This document describes the **external XML contract** defined by Bayut. For how PropWise produces this feed from internal data structures, see the Portal Syndication Specification.
</Note>

## XML Integration Overview

The XML feed allows clients to maintain their inventory on Bayut and/or dubizzle by providing property information in a structured format.

### Key Requirements

- The XML feed must be exposed as a **publicly accessible URL**
- The **Bayut system pulls** (reads) this URL on a schedule to sync updates
- Listings flow **one direction only**: CRM → portals (feed is source of truth)
- Portals never write back into the feed

### Unified Feed Structure

<Info>
Bayut and dubizzle now share a **single combined XML feed**. The portal(s) a property appears on is controlled per-property by the `<Portals>` tag.
</Info>

For existing integrations:
- **Retire** separate dubizzle feeds
- **Update** Bayut XML structure to use `<Portals>` attribute

## XML Document Structure

The XML structure consists of two main segments:

<Tabs>
  <Tab title="Header Tags">
    The root element must be named `<Properties>` with individual `<Property>` elements nested inside:

    ```xml
    <Properties>
      <Property> <!-- Property 1 attributes --> </Property>
      <Property> <!-- Property 2 attributes --> </Property>
    </Properties>
    ```
  </Tab>
  
  <Tab title="CDATA Wrapping">
    All textual values should be wrapped in `<![CDATA[ ... ]]>` to prevent XML parsing issues:

    ```xml
    <Property_Title><![CDATA[ Luxury 3BR Apartment in Marina ]]></Property_Title>
    <Property_Description><![CDATA[ Beautiful apartment with sea views & modern amenities ]]></Property_Description>
    ```
  </Tab>
</Tabs>

## Property Attributes

### Identity & Status Fields

| Tag | Description | Allowed Values |
|---|---|---|
| `<Property_Ref_No>` | Unique identifier from client's CRM | Must be unique. Duplicates marked **Inactive** |
| `<Permit_Number>` | Trakheesi/ADREC permit number | Required for listing to go live |
| `<Property_Status>` | Listing status | `live`, `deleted` |
| `<Last_Updated>` | Last modified timestamp | `YYYY-MM-DD HH:mm:ss` |

### Classification Fields

| Tag | Description | Allowed Values |
|---|---|---|
| `<Property_purpose>` | Sale or rental purpose | `Buy`, `Rent` |
| `<Property_Type>` | Property category | `Villa`, `Apartment`, `Office`, `Shop`, `Warehouse`, `Factory`, `Labour Camp`, `Other Commercial`, `Commercial Building`, `Residential Floor`, `Commercial Floor`, `Residential Land`, `Commercial Land`, `Townhouse`, `Residential Building`, `Hotel Apartment`, `Loft Apartment`, `Pent House` |
| `<Off_plan>` | Ready or off-plan status | `Yes`, `No` |

### Portal Targeting

The `<Portals>` tag controls which platform(s) publish the listing:

<CodeGroup>
```xml Both Portals
<Portals>
  <Portal><![CDATA[ Bayut ]]></Portal>
  <Portal><![CDATA[ dubizzle ]]></Portal>
</Portals>
```

```xml Bayut Only
<Portals>
  <Portal><![CDATA[ Bayut ]]></Portal>
</Portals>
```

```xml dubizzle Only
<Portals>
  <Portal><![CDATA[ dubizzle ]]></Portal>
</Portals>
```
</CodeGroup>

### Property Specifications

| Tag | Description | Values/Rules |
|---|---|---|
| `<Property_Size>` | Area of the property | Number |
| `<Property_Size_Unit>` | Area measurement unit | Preferred: `SQFT` |
| `<plotArea>` | Plot area in square feet | Number |
| `<Bedrooms>` | Number of bedrooms | `-1` (studio), `1`-`10`, `10+` |
| `<Bathrooms>` | Number of bathrooms | `1`-`10` |
| `<Furnished>` | Furnishing status | `Yes`, `No`, `Partly` |

### Features

Use nested `<Feature>` tags for property amenities:

```xml
<Features>
  <Feature><![CDATA[ Swimming Pool ]]></Feature>
  <Feature><![CDATA[ Gym ]]></Feature>
  <Feature><![CDATA[ Parking ]]></Feature>
</Features>
```

### Pricing Information

| Tag | Description | Required When |
|---|---|---|
| `<Price>` | Sale price or rent amount | Always |
| `<Rent_Frequency>` | Rental period | Only when `Property_purpose` is `Rent` |

**Rent Frequency Values**: `Daily`, `Weekly`, `Monthly`, `Yearly`

## Off-Plan Properties

<Warning>
Off-plan detail tags are required **only** when `<Off_plan>` is `Yes`.
</Warning>

### Required Off-Plan Fields

| Tag | Description | Required For |
|---|---|---|
| `<offplanDetails_saleType>` | Property sale type | All off-plan (`New`, `Resale`) |
| `<offplanDetails_dldWaiver>` | DLD waiver percentage | `New` properties (0-100) |
| `<offplanDetails_originalPrice>` | Original purchase price | `Resale` properties |
| `<offplanDetails_amountPaid>` | Amount already paid | `Resale` properties |

## Content & Media

### Listing Content

| Tag | Description |
|---|---|
| `<Property_Title>` | English listing title |
| `<Property_Description>` | English description |
| `<Property_Title_AR>` | Arabic listing title |
| `<Property_Description_AR>` | Arabic description |

### Media Assets

All URLs must be publicly accessible:

<CodeGroup>
```xml Images
<Images>
  <Image><![CDATA[ https://cdn.example.com/property/155352/img-1.jpg ]]></Image>
  <Image><![CDATA[ https://cdn.example.com/property/155352/img-2.jpg ]]></Image>
</Images>
```

```xml Videos
<Videos>
  <Video><![CDATA[ https://www.youtube.com/watch?v=example ]]></Video>
</Videos>
```

```xml Floor Plans
<Floor_Plans>
  <Floor_Plan><![CDATA[ https://cdn.example.com/floorplans/fp-1.pdf ]]></Floor_Plan>
</Floor_Plans>
```
</CodeGroup>

## Location Information

### Location Hierarchy

| Tag | Description | Example |
|---|---|---|
| `<City>` | City/Emirate | Dubai |
| `<Locality>` | Main area | Jumeirah Lakes Towers |
| `<Sub_Locality>` | Sub-area | Cluster C |
| `<Tower_Name>` | Building name | Pretoni Tower |

```xml
<City><![CDATA[ Dubai ]]></City>
<Locality><![CDATA[ Jumeirah Lakes Towers ]]></Locality>
<Sub_Locality><![CDATA[ Cluster C ]]></Sub_Locality>
<Tower_Name><![CDATA[ Pretoni Tower ]]></Tower_Name>
```

## Contact Information

| Tag | Description |
|---|---|
| `<Listing_Agent>` | Agent name for display |
| `<Listing_Agent_Phone>` | Agent contact number |
| `<Listing_Agent_Email>` | Agent email address |

## Field Requirements Summary

<AccordionGroup>
  <Accordion title="Required Fields">
    - `Property_Ref_No` (must be unique)
    - `Property_Status`
    - `Property_purpose`
    - `Property_Type`
    - `Permit_Number` (for listing to go live)
    - `Portals` (at least one portal)
    - `Price`
    - At least one `Image`
  </Accordion>

  <Accordion title="Conditional Fields">
    - `Rent_Frequency`: Required when `Property_purpose` is `Rent`
    - Off-plan fields: Required when `Off_plan` is `Yes`
      - `New` properties: `offplanDetails_dldWaiver` required
      - `Resale` properties: `offplanDetails_originalPrice` and `offplanDetails_amountPaid` required
  </Accordion>

  <Accordion title="Optional Fields">
    - `Property_Size`, `Property_Size_Unit`, `plotArea`
    - `Bedrooms`, `Bathrooms`, `Furnished`
    - `Features`, `Videos`, `Floor_Plans`
    - Arabic content (`Property_Title_AR`, `Property_Description_AR`)
    - Location details (`Sub_Locality`, `Tower_Name`)
  </Accordion>
</AccordionGroup>

    <Property_Status><![CDATA[ live ]]></Property_Status>
    <Permit_Number><![CDATA[ 1234567890 ]]></Permit_Number>
    
    <City><![CDATA[ Dubai ]]></City>
    <Locality><![CDATA[ Dubai Marina ]]></Locality>
