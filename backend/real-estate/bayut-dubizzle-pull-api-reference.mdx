# Bayut/Dubizzle Pull API Reference

<Info>
This document describes the **external API contract** defined by Bayut for the Pull API used to fetch lead and view data from Bayut, including both *leads* (`is_trulead=1`) and *views* (`is_trulead=0`), plus call logs and story leads.
</Info>

## Overview

The Bayut Pull API enables developers to retrieve lead data and view statistics from Bayut's platform. This comprehensive API provides access to:

- **Lead Data**: Actual leads with customer contact information and inquiries
- **View Statistics**: Count-based metrics for various interaction types
- **Call Logs**: Detailed phone call tracking and recordings
- **Story Leads**: Leads generated from property story interactions

<Note>
The API distinguishes between actual leads (`is_trulead=1`) and view counts (`is_trulead=0`) using the `is_trulead` parameter.
</Note>

## Endpoint

```
www.bayut.com/api-v7/stats/website-client-leads
```

## Authorization

<Warning>
Each client is provided with a unique API key, which must be passed as a **bearer token** in the `Authorization` header.
</Warning>

```http
Authorization: Bearer <API KEY>
```

## CURL Request Example

<CodeGroup>
```bash Basic Request
curl --location \
  'www.bayut.com/api-v7/stats/website-client-leads?timestamp=2025-03-10%2000%3A00%3A00&target=listing&type=email&is_trulead=1' \
  --header 'Authorization: Bearer <API KEY>'
```
</CodeGroup>

<Tip>
The `timestamp` value must be URL-encoded: `2025-03-10 00:00:00` becomes `2025-03-10%2000%3A00%3A00` where `%20` = space and `%3A` = colon.
</Tip>

## Query Parameters

| Parameter | Description | Possible Values |
|-----------|-------------|----------------|
| `type` | The type of lead | `whatsapp`, `sms`, `phone`, `email`, `call_logs`, `story_leads` |
| `timestamp` | The date and time of the request | Format: `YYYY-MM-DD HH:MM:SS` |
| `target` | The entity for which leads are fetched | `listing`, `agent`, `agency` |
| `is_trulead` | Whether it is a lead or just a views count | `0` (views), `1` (leads) |

When `is_trulead=1`, the API returns actual leads with detailed contact information and inquiry messages.

</Info>
