## Overview

The Bayut Pull API enables you to programmatically retrieve lead data and view statistics from the Bayut real estate portal. This comprehensive reference covers authentication, endpoints, query parameters, and response formats for both leads (`is_trulead=1`) and views (`is_trulead=0`).

<Info>
This document describes the **external API contract** defined by Bayut. All examples and field names reflect the official Bayut API specification.
</Info>

### Key capabilities

- Fetch leads by type (email, WhatsApp, SMS, phone)
- Retrieve view counts for listings, agents, and agencies
- Access call logs and story lead data
- Filter by timestamp and target entity

---

## Authentication

Each client receives a unique API key that must be included in every request as a bearer token.

<Warning>
Keep your API key secure. Do not expose it in client-side code or public repositories.
</Warning>

### Authorization header

```http
Authorization: Bearer <API_KEY>
```

Replace `<API_KEY>` with your unique bearer token provided by Bayut.

---

## Base endpoint

```
www.bayut.com/api-v7/stats/website-client-leads
```

All API requests use this base URL with query parameters to specify the data you want to retrieve.

---

## Making requests

### Basic CURL example

```bash
curl --location \
  'www.bayut.com/api-v7/stats/website-client-leads?timestamp=2025-03-10%2000%3A00%3A00&target=listing&type=email&is_trulead=1' \
  --header 'Authorization: Bearer <API_KEY>'
```

<Note>
The `timestamp` parameter must be URL-encoded: spaces become `%20` and colons become `%3A`.
Example: `2025-03-10 00:00:00` → `2025-03-10%2000%3A00%3A00`
</Note>

## Query parameters

<ParamField query="type" type="string" required>
  The type of lead to fetch.
  
  **Possible values:**
  - `whatsapp` – WhatsApp leads
  - `sms` – SMS leads
  - `phone` – Phone leads
  - `email` – Email leads
  - `call_logs` – Call log data
  - `story_leads` – Story-based leads
</ParamField>

<ParamField query="timestamp" type="string" required>
  The date and time threshold for retrieving leads.
  
  **Format:** `YYYY-MM-DD HH:MM:SS`
  
  **Example:** `2025-03-10 00:00:00` (must be URL-encoded in requests)
</ParamField>

<ParamField query="target" type="string" required>
  The entity for which leads are fetched.
  
  **Possible values:**
  - `listing` – Property listings
  - `agent` – Individual agents
  - `agency` – Real estate agencies
</ParamField>

<ParamField query="is_trulead" type="integer" required>
  Distinguishes between actual leads and view counts.
  
  **Possible values:**
  - `1` – Returns actual leads with contact information
  - `0` – Returns view count statistics only
</ParamField>

## Get leads (is_trulead=1)

When `is_trulead=1`, the API returns detailed lead information including contact details and messages from potential customers.

### Email listing leads

Retrieve email leads for specific property listings.

<CodeGroup>
```bash cURL
curl --location \
  'www.bayut.com/api-v7/stats/website-client-leads?timestamp=2025-03-10%2000%3A00%3A00&target=listing&type=email&is_trulead=1' \
  --header 'Authorization: Bearer <API_KEY>'
```

```javascript JavaScript
const response = await fetch(
  'www.bayut.com/api-v7/stats/website-client-leads?timestamp=2025-03-10%2000%3A00%3A00&target=listing&type=email&is_trulead=1',
  {
    headers: {
      'Authorization': 'Bearer <API_KEY>'
    }
  }
);
const data = await response.json();
```

```python Python
import requests

url = "www.bayut.com/api-v7/stats/website-client-leads"
params = {
    "timestamp": "2025-03-10 00:00:00",
    "target": "listing",
    "type": "email",
    "is_trulead": 1
}
headers = {
    "Authorization": "Bearer <API_KEY>"
}

response = requests.get(url, params=params, headers=headers)
data = response.json()
```
</CodeGroup>

**Response:**

```json
[
  {
    "lead_id": "email|dffb56c2-3fa0515851b",
    "source": "bayut",
    "date_time": "2025-04-04 12:01:29",
    "listing_details": {
      "listing_id": 1060074,
      "listing_reference": "axc-999922236",
      "current_type": "Apartment"
    },

      "name": "Jane Austin",

```

<AccordionGroup>
  <Accordion title="Response field reference">
    | Field | Type | Description |
    |-------|------|-------------|
    | `lead_id` | string | Unique identifier for the lead (format: `type\|uuid`) |
    | `source` | string | Always "bayut" for this API |
    | `date_time` | string | When the lead was generated (format: `YYYY-MM-DD HH:MM:SS`) |
    | `listing_details.listing_id` | integer | Internal listing identifier |
    | `listing_details.listing_reference` | string | External listing reference code |
    | `listing_details.current_type` | string | Property type (e.g., "Apartment", "Villa") |
    | `inquirer_details.name` | string | Lead's full name |
    | `inquirer_details.cell` | string | Lead's phone number |
    | `inquirer_details.email` | string | Lead's email address |
    | `inquirer_details.message` | string | Message from the lead |
  </Accordion>
</AccordionGroup>

## Best practices

<CardGroup cols={2}>
  <Card title="Rate limiting" icon="gauge-high">
    Implement exponential backoff for failed requests and respect any rate limit headers returned by the API.
  </Card>
  
  <Card title="Error handling" icon="triangle-exclamation">
    Always handle HTTP error responses (4xx, 5xx) gracefully and log errors for debugging.
  </Card>
  
  <Card title="Timestamp formatting" icon="clock">
    Remember to URL-encode timestamp values when constructing query strings programmatically.
  </Card>
  
  <Card title="Data freshness" icon="arrows-rotate">
    Use incremental timestamp queries to fetch only new leads since your last sync.
  </Card>
</CardGroup>

## Common use cases

<Steps>
  <Step title="Sync new leads daily">
    Set up a scheduled job to fetch leads from the last 24 hours using the current date minus one day as the timestamp parameter.
  </Step>
  
  <Step title="Track message delivery">
    For WhatsApp leads, monitor the `delivery_notifications` array to confirm when messages are delivered and read.
  </Step>
  
  <Step title="Analyze engagement">
    Use view count endpoints (`is_trulead=0`) to measure which listings, agents, or agencies generate the most interest.
  </Step>
  
  <Step title="Archive call recordings">
    Extract `call_recordingurl` from call logs to download and archive recordings for quality assurance or compliance.
  </Step>
</Steps>
