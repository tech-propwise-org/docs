**Path:** `src/modules/real-estate/inventory/`

The inventory module manages org-scoped secondary units (existing market properties that have been bought and sold) within PropWise's **six-module real estate system**. It provides unit number privacy controls and marketplace listing functionality while integrating with the reference database architecture.

## Architecture context

Inventory is one of six real estate modules spanning three visibility scopes:

<Tabs>
<Tab title="Global modules">
- **reference-data**: All property data (structural + enrichment)
- **developer-portal**: Auth + project management
- **map**: Geocoding utility (shared across modules)
</Tab>
<Tab title="Org-scoped modules">
- **inventory**: Secondary units (this module)
- **property-interest**: Lead/deal property interests + matching
</Tab>
<Tab title="Cross-org modules">
- **shared-marketplace**: Cross-organization listings + deals
</Tab>
</Tabs>

### Data ownership pattern

<Info>
Inventory follows PropWise's **cross-database reference pattern**: CRM entities store plain integer IDs (`refUnitId`, `refProjectId`) pointing to the reference database, with no cross-database foreign keys. All property details come through `ReferenceDataService`.
</Info>

| Data Type | Database | Access Pattern |
|-----------|----------|----------------|
| Property data (projects, units, buildings, areas) | Reference DB | `ReferenceDataService` |
| Gallery, payment plans, documents, amenities | Reference DB | `ReferenceDataService` |
| Geocoding, pricing, sale status, handover | Reference DB | `ReferenceDataService` |
| Secondary units (inventory records) | CRM DB | Direct entity access |
| Unit ownership, transactions, privacy settings | CRM DB | Direct entity access |

### Scope classification

- **Scope**: Org-scoped
- **RLS Behavior**: Standard RLS via TenantContext
- **Description**: Secondary units owned by an organization

### InventorySecurityService

Specialized security service for unit number privacy and access control with comprehensive permission checking:

```typescript
export class InventorySecurityService {
  // Core privacy methods
  canViewUnitNumber(user: User, unit: SecondaryUnit): boolean
  getVisibilityLevel(user: User, unit: SecondaryUnit): 'full' | 'basic' | 'restricted'
  
  // Access grant management
  async createAccessGrant(unitId: string, granteeId: string, permissions: string[], expiresAt?: Date): Promise<UnitAccessGrant>
  async revokeAccessGrant(grantId: string): Promise<void>
  async getActiveAccessGrants(unitId: string): Promise<UnitAccessGrant[]>
  
  // Permission checking
  async canModifyUnit(user: User, unit: SecondaryUnit): Promise<boolean>
  async canViewUnitDetails(user: User, unit: SecondaryUnit): Promise<boolean>
  async canManageOwnership(user: User, unit: SecondaryUnit): Promise<boolean>
  async canViewTransactions(user: User, unit: SecondaryUnit): Promise<boolean>
  
  // Privacy settings
  async updatePrivacySettings(unitId: string, settings: PrivacySettingsDto, user: User): Promise<void>
  getEffectivePrivacySettings(unit: SecondaryUnit, user: User): EffectivePrivacySettings
}
```

**Privacy Rules:**
- Unit number visible if: `isUnitNumberHidden === false` OR user has `real_estate.manage` permission OR user is `unitManager` OR user has active `UnitAccessGrant`
- Transaction history visible based on: ownership percentage, management role, or explicit access grants
- Financial data visibility controlled by: organization privacy policies and user access level

<Warning>
Always pass `canViewUnitNumber` to the DTO constructor. Forgetting this parameter defaults to `true`, which could expose hidden unit numbers.
</Warning>

## Entities

<AccordionGroup>
<Accordion title="SecondaryUnit">

**Primary Entity** - UUID PK, org-scoped with standard RLS

**Reference Integration:**
- `refUnitId` (number): Reference to unit in external property database
- `refBuildingId` (number): Reference to building in external property database  
- `refProjectId` (number): Reference to project in external property database
- `refAreaId` (number): Reference to area in external property database
- `refCommunityId` (number): Reference to community in external property database

**Denormalized Performance Fields:**
- `projectName` (string): Cached project name for display and search
- `buildingName` (string): Cached building name for display and search
- `areaName` (string): Cached area name for filtering
- `communityName` (string): Cached community name for filtering

**Core Unit Details:**
- `unitNumber` (string): Unit identifier (subject to privacy controls)
- `floor` (number): Floor level
- `rooms` (number): Number of rooms/bedrooms
- `propertySubType` (PropertySubType): Apartment, villa, townhouse, etc.
- `areaSqft` (number): Unit area in square feet
- `balconyArea` (number): Balcony area in square feet
- `isFreeHold` (boolean): Ownership type indicator

**Financial Information:**
- `askingPrice` (decimal): Sale price
- `rentPrice` (decimal): Rental price  
- `currency` (string): Price currency (AED, USD, etc.)
- `availableForSale` (boolean): Sale availability flag
- `availableForRent` (boolean): Rental availability flag
- `lastValuation` (decimal): Most recent valuation amount
- `lastValuationDate` (Date): Valuation date

**Status and Availability:**
- `status` (SecondaryUnitStatus): VACANT, RENTED, OWNER_OCCUPIED, UNDER_MAINTENANCE
- `occupancyStatus` (OccupancyStatus): Current occupancy state
- `availabilityDate` (Date): When unit becomes available

**Privacy and Marketplace:**
- `isUnitNumberHidden` (boolean, default: true): Privacy control
- `isListedOnMarketplace` (boolean, default: false): Marketplace visibility
- `listedAt` (Date): Marketplace listing timestamp
- `marketplaceSettings` (JSON): Marketplace-specific configuration

**Property Features:**
- `description` (text): Detailed unit description
- `furnishingType` (FurnishingType): FURNISHED, SEMI_FURNISHED, UNFURNISHED
- `parkingSpaces` (number): Number of parking spots
- `viewType` (ViewType): SEA, CITY, GARDEN, POOL, etc.
- `amenities` (string[]): Unit-specific amenities
- `condition` (PropertyCondition): EXCELLENT, GOOD, FAIR, NEEDS_RENOVATION

**Relationships:**
- `owner` (Contact): Current primary owner
- `unitManager` (User): Assigned property manager
- `organization` (Organization): Owning organization
- `createdBy` (User): Record creator
- `ownerships` (Collection<InventoryUnitOwnership>): All ownership records
- `transactions` (Collection<InventoryUnitTransaction>): Transaction history
- `accessGrants` (Collection<UnitAccessGrant>): Access permission grants

**Audit Fields:**
- `createdAt`, `updatedAt`, `deletedAt` (soft delete)

## Unit number privacy system

The `InventorySecurityService` implements comprehensive privacy controls:

<Tab title="Privacy rules">
```typescript
export class InventorySecurityService {
  canViewUnitNumber(user: User, unit: SecondaryUnit): boolean {
    // Unit number visible if any condition is true:
    
    // 1. Unit number is not hidden
    if (!unit.isUnitNumberHidden) return true;
    
    // 2. User has management permissions
    if (this.permissionService.hasPermission(user, 'real_estate.manage')) return true;
    
    // 3. User is the designated unit manager
    if (unit.unitManager?.id === user.id) return true;
    
    // 4. User has active access grant with unit number permission
    if (this.hasActiveAccessGrant(user, unit, 'VIEW_UNIT_NUMBER')) return true;
    
    // 5. User is an owner with sufficient ownership percentage
    if (this.isSignificantOwner(user, unit)) return true;
    
    return false;
  }
  
  getVisibilityLevel(user: User, unit: SecondaryUnit): VisibilityLevel {
    const canViewUnitNumber = this.canViewUnitNumber(user, unit);
    const canViewFinancials = this.canViewFinancialData(user, unit);
    const canViewTransactions = this.canViewTransactionHistory(user, unit);
    
    if (canViewUnitNumber && canViewFinancials && canViewTransactions) {
      return 'full';
    } else if (canViewUnitNumber || canViewFinancials) {
      return 'basic';
    }
    return 'restricted';
  }
}
```

## Transaction and ownership workflows

<Steps>
<Step title="Initial unit creation">
Create secondary unit with basic property details and establish initial ownership record. Reference IDs are validated against the property database and names are denormalized for performance.
</Step>

<Step title="Ownership establishment">
Create `InventoryUnitOwnership` record with ownership percentage, legal documentation, and acquisition details. Validate that total ownership percentages don't exceed 100%.
</Step>

<Step title="Transaction recording">
All financial activities (purchases, sales, refinancing) are tracked in `InventoryUnitTransaction` with complete financial breakdown including commissions, fees, and taxes.
</Step>

<Step title="Ownership transfers">
When ownership changes, previous ownership records are end-dated and new records created. Transaction records link ownership changes with financial details.
</Step>

<Step title="Status updates">
Unit status automatically updates based on occupancy, rental agreements, and marketplace activity. Status changes trigger notifications and workflow updates.
</Step>

<Step title="Marketplace integration">
Units can be listed on the shared marketplace with privacy controls and access grants for potential buyers/tenants. Inquiry handling creates temporary access permissions.
</Step>
</Steps>

```typescript
async transferOwnership(
  unitId: string, 
  transferData: OwnershipTransferDto, 
  user: User
) {
  return this.em.transactional(async () => {
    // Validate current ownership
    const currentOwnership = await this.getCurrentOwnership(unitId);
    if (!currentOwnership) {
      throw new BadRequestException('No current ownership found');
    }
    
    // Validate transfer percentage
    if (transferData.transferPercentage > currentOwnership.ownershipPercentage) {
      throw new BadRequestException('Cannot transfer more than owned percentage');
    }
    
    // Record transfer transaction
    const transaction = this.em.create(InventoryUnitTransaction, {
      unit: unitId,
      date: transferData.transferDate,
      value: transferData.transferValue || 0,
      transactionType: 'TRANSFER',
      sellerName: currentOwnership.owner.name,
      buyerName: transferData.newOwnerName,
      // ... legal documentation
    });
    
    // Update current ownership
    currentOwnership.ownershipPercentage -= transferData.transferPercentage;
    if (currentOwnership.ownershipPercentage <= 0) {
      currentOwnership.endDate = transferData.transferDate;
      currentOwnership.isActive = false;
    }
    
    // Create new ownership record
    const newOwnership = this.em.create(InventoryUnitOwnership, {
      unit: unitId,
      owner: transferData.newOwnerId,
      ownershipPercentage: transferData.transferPercentage,
      ownershipType: transferData.ownershipType,
      startDate: transferData.transferDate,
      acquisitionPrice: transferData.transferValue,
      // ... documentation details
    });
    
    await this.em.flush();
    return { transaction, newOwnership };
  });
}
```

```typescript TypeScript - Advanced search with privacy
    .andWhere('u.deletedAt IS NULL');

  criteria.criteria.forEach((criterion, index) => {
    const paramName = `param${index}`;
    
    switch (criterion.operator) {
      case 'BETWEEN':
        qb.andWhere(`u.${criterion.field} BETWEEN ?${index} AND ?${index + 1}`, 
                   [criterion.value[0], criterion.value[1]]);
        break;
      case 'IN':
        qb.andWhere(`u.${criterion.field} IN (?)`, [criterion.value]);
        break;
      case 'CONTAINS':
        if (criterion.field === 'amenities') {
          qb.andWhere(`u.amenities @> ?`, [criterion.value]);
        } else {
          qb.andWhere(`u.${criterion.field} ILIKE ?`, [`%${criterion.value}%`]);
        }
        break;
      // ... other operators
    }
  });
  
  // Apply sorting and pagination
  qb.orderBy(`u.${criteria.sortBy}`, criteria.sortOrder)
    .limit(criteria.limit)
    .offset((criteria.page - 1) * criteria.limit);
  
  const [units, total] = await qb.getResultAndCount();
  
  // Apply privacy controls to results
  const unitsWithPrivacy = units.map(unit => {
    const canViewUnitNumber = this.inventorySecurityService.canViewUnitNumber(user, unit);
    const visibilityLevel = this.inventorySecurityService.getVisibilityLevel(user, unit);
    return new SecondaryUnitDto(unit, canViewUnitNumber, visibilityLevel);
  });
  
  return {
    data: unitsWithPrivacy,
    meta: {
      total,
      page: criteria.page,
      limit: criteria.limit,
      totalPages: Math.ceil(total / criteria.limit)
    },
    appliedFilters: criteria.criteria,
    searchPerformed: new Date().toISOString()
  };
}
```
