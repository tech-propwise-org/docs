**Path:** `src/modules/real-estate/inventory/`

The inventory module manages org-scoped inventory unit claims and unit access requests within PropWise's **real estate system**. It connects to the Propwise Labs API through `PropwiseLabsService` and provides property data access patterns through standard RLS (Row Level Security) via TenantContext.

<Note>
This module is part of PropWise's comprehensive real estate specification that spans six active specialized modules across three visibility scopes: global (no RLS), org-private (standard RLS), and cross-org (custom RLS). The shared-marketplace module is retained but not mounted in the application.
</Note>

## Overview

The real estate module is structured as **six active specialized modules** spanning the global and org-private real-estate surfaces. The cross-org `shared-marketplace` code is retained in the repository but is not imported by `RealEstateModule`, so its controllers/providers are not mounted by the app.

The external **Propwise Labs API** stores and serves all property data — both structural (DLD registry) and enrichment (images, payment plans, documents, amenities, pricing, geocoding where exposed). The CRM database stores only CRM-specific data (developer portal auth, ownership links, property interests, marketplace listings, and org inventory context).

<Warning>
**Architecture:** All modules consume property data exclusively through `PropwiseLabsService`. The CRM no longer opens a second MikroORM connection or maps Propwise Labs entities locally. The reference database configuration has been completely removed and replaced with Propwise Labs API configuration (`app.propwiseLabs.*`) including API credentials, timeout settings, and Bird Verify integration for developer portal contact verification.
</Warning>

<Info>
**Portal syndication — self-contained `Listing` (2026 build):** The `portal-syndication` module's `Listing` is now self-contained: every field any portal (Bayut/dubizzle + Property Finder) needs lives on `Listing`, snapshotted from an `InventoryUnit` at create (linked mode) or entered directly (manual mode). The `inventoryUnit` FK is **optional**. Deleting an inventory unit cascades (event-driven, `inventory-unit.deleted` carrying the user's `removeLinkedListings` choice): **remove** (default) delists the unit's listings from all portals and archives them; **keep** unlinks them (`inventoryUnit = null`) so they survive as self-contained manual listings. Buildings auto-maintain `salesStatus` based on available units (`OUT_OF_STOCK` when no units remain `AVAILABLE`, reverting to `ON_SALE` when available units reappear). New `downpaymentType` enum distinguishes amount vs percentage downpayments.
</Info>

<Note>
**Listing approval workflow (Gate A):** a user without `real_estate.listing.publish` / `real_estate.manage` submits a listing for approval (`ListingStatus.PENDING_APPROVAL`). A `real_estate.manage` user approves (auto-publishes to the submitter's selected portals) or rejects (`ListingStatus.REJECTED` + `rejectionReason` persisted on the listing) from a manage-only **Requests** queue (`GET /portal-syndication/listings?type=requests` = pending + rejected). The submitter sees the rejection note and can edit + resubmit (clears the rejection) or delete. The listing **Delete** action soft-deletes (`isDeleted=true`) — there is no user-facing archived state (the `archived` status / `isArchived` columns remain only for the inventory-unit delete cascade). A `real_estate.manage` user may delete **any** listing (including pending requests and rejected ones) directly from the Requests queue.
</Note>

<Tip>
**Notifications:** submitting emits `LISTING_APPROVAL_REQUESTED` to all `real_estate.manage` approvers; approve / reject / delete emit `LISTING_APPROVED` / `LISTING_REJECTED` / `LISTING_DELETED` to the listing publisher (delete only when the deleter is not the publisher). See `NOTIFICATION_IMPLEMENTATION_GUIDE.md`. Full design: `PORTAL_SYNDICATION_SPECIFICATION.md` §1.5 (As-Built).
</Tip>

## Purpose

The inventory module serves as a critical component in PropWise's real estate ecosystem, providing:

- **Inventory unit claims**: Complete CRUD operations for organization-owned inventory units
- **Unit access requests**: Intra-org unit access request management
- **External API integration**: Seamless property data access through `PropwiseLabsService`
- **Standard RLS implementation**: Organization-level data isolation and security
- **Property data enrichment**: On-demand integration of structural and enrichment data from the Propwise Labs API
- **Business logic validation**: Comprehensive validation of reference project/unit relationships
- **Bulk operations**: Efficient handling of large-scale inventory operations

### External property ID pattern

<Tip>
`InventoryUnit` is the CRM boundary for org-owned secondary inventory. It stores plain integer IDs (`refProjectId`, `refBuildingId`, `refUnitId`, `refDeveloperId`, `refPropertyTypeId`, `refPropertySubTypeId`) plus a denormalized Labs snapshot, including the developer display name used by portal syndication and the canonical `propertyType` (`LayoutType` enum) used by portal syndication adapters. Labs raw property type/sub-type names are retained in `labsPropertyType` and `labsPropertySubType` for traceability. Lead interests can use inventory units as candidate matches while retaining search criteria. Deal property rows are concrete-only: they link either an org `InventoryUnit`, a direct Propwise Labs off-plan unit via `refProjectId` / `refBuildingId` / `refUnitId`, or a pending unit-access request before the inventory link is approved.
</Tip>

The inventory module relies on the Propwise Labs API configuration (`app.propwiseLabs.*`) for external property data access:

- **Timeout settings**: Request timeout configuration for external API calls
- **Bird Verify integration**: Contact verification for developer portal authentication

## Portal syndication integration

The inventory module provides foundation data for the portal syndication system:

### Self-contained listing architecture

<Info>
The portal syndication module's `Listing` entity is now self-contained with an optional `inventoryUnit` FK. All portal-required fields are either snapshotted from an `InventoryUnit` at creation (linked mode) or entered directly (manual mode).
</Info>

### Inventory unit deletion cascade

When deleting an inventory unit, the system handles linked portal listings through an event-driven cascade:

<Steps>
<Step title="User choice capture">
The deletion workflow captures the user's `removeLinkedListings` preference during the delete operation.
</Step>
<Step title="Event emission">
System emits `inventory-unit.deleted` event carrying the user's choice and unit details.
</Step>
<Step title="Portal listing cascade">
Portal syndication module processes the event:
- **Remove** (default): Delists from all portals and archives the listings
- **Keep**: Unlinks listings (`inventoryUnit = null`) so they survive as manual listings
</Step>
</Steps>

### Building sales status automation

Buildings automatically maintain their `salesStatus` based on available inventory units:

- `OUT_OF_STOCK`: When no units remain with `AVAILABLE` status
- `ON_SALE`: Reverts when available units reappear

### Approval workflow integration

<Note>
Users without `real_estate.listing.publish` or `real_estate.manage` permissions must submit listings for approval (`ListingStatus.PENDING_APPROVAL`). Users with `real_estate.manage` can approve/reject from the Requests queue, with automatic portal publishing on approval.
</Note>

## Service layer implementation

### InventoryService

The service layer handles business logic for inventory management:

```typescript
@Injectable()
export class InventoryService {
  constructor(
    private readonly em: EntityManager,
    private readonly propwiseLabsService: PropwiseLabsService,
    private readonly logger: Logger,
  ) {}

  async findAll(filters: InventoryFiltersDto, user: User): Promise<InventoryUnit[]> {
    const qb = this.em.createQueryBuilder(InventoryUnit, 'iu');
    
    // Standard RLS via organizationId
    qb.where({ organizationId: user.organizationId });
    
    if (filters.status) {
      qb.andWhere({ status: filters.status });
    }
    
    if (filters.priceMin !== undefined) {
      qb.andWhere({ price: { $gte: filters.priceMin } });
    }
    
    if (filters.priceMax !== undefined) {
      qb.andWhere({ price: { $lte: filters.priceMax } });
    }

    if (filters.bedrooms !== undefined) {
      qb.andWhere({ bedrooms: filters.bedrooms });
    }

    if (filters.bathrooms !== undefined) {
      qb.andWhere({ bathrooms: filters.bathrooms });
    }

    if (filters.furnished !== undefined) {
      qb.andWhere({ furnished: filters.furnished });
    }

    if (filters.refProjectId) {
      qb.andWhere({ refProjectId: filters.refProjectId });
    }

    if (filters.refBuildingId) {
      qb.andWhere({ refBuildingId: filters.refBuildingId });
    }

    if (filters.propertyType) {
      qb.andWhere({ propertyType: filters.propertyType });
    }

    // Search by name or description
    if (filters.search) {
      qb.andWhere({
        $or: [
          { name: { $ilike: `%${filters.search}%` } },
          { description: { $ilike: `%${filters.search}%` } },
        ],
      });
    }

    // Pagination
    if (filters.page !== undefined && filters.limit !== undefined) {
      qb.offset(filters.page * filters.limit).limit(filters.limit);
    }

    // Ordering
    qb.orderBy({ createdAt: 'DESC' });
    
    return qb.getResult();
  }

  async findOne(id: string, organizationId: string): Promise<InventoryUnit> {
    const unit = await this.em.findOne(InventoryUnit, { 
      id, 
      organizationId 
    });
    
    if (!unit) {
      throw new NotFoundException(`Inventory unit with ID ${id} not found`);
    }
    
    return unit;
  }

  async create(createDto: CreateInventoryUnitDto, user: User): Promise<InventoryUnit> {
    // Validate reference data if provided
    if (createDto.refProjectId) {
      await this.validateReferenceProject(createDto.refProjectId);
    }
    
    if (createDto.refBuildingId) {
      await this.validateReferenceBuilding(createDto.refBuildingId);
    }
    
    if (createDto.refUnitId) {
      await this.validateReferenceUnit(createDto.refUnitId);
    }

    if (createDto.refDeveloperId) {
      await this.validateReferenceDeveloper(createDto.refDeveloperId);
    }

    const unit = this.em.create(InventoryUnit, {
      ...createDto,
      organizationId: user.organizationId,
    });

    await this.em.persistAndFlush(unit);
    return unit;
  }

  async update(id: string, updateDto: UpdateInventoryUnitDto, user: User): Promise<InventoryUnit> {
    const unit = await this.findOne(id, user.organizationId);

    // Validate reference data if being updated
    if (updateDto.refProjectId !== undefined && updateDto.refProjectId !== unit.refProjectId) {
      await this.validateReferenceProject(updateDto.refProjectId);
    }
    
    if (updateDto.refBuildingId !== undefined && updateDto.refBuildingId !== unit.refBuildingId) {
      await this.validateReferenceBuilding(updateDto.refBuildingId);
    }
    
    if (updateDto.refUnitId !== undefined && updateDto.refUnitId !== unit.refUnitId) {
      await this.validateReferenceUnit(updateDto.refUnitId);
    }

    if (updateDto.refDeveloperId !== undefined && updateDto.refDeveloperId !== unit.refDeveloperId) {
      await this.validateReferenceDeveloper(updateDto.refDeveloperId);
    }

    this.em.assign(unit, updateDto);
    await this.em.flush();
    
    return unit;
  }

  async remove(id: string, organizationId: string): Promise<void> {
    const unit = await this.findOne(id, organizationId);
    await this.em.removeAndFlush(unit);
  }

  async enrichWithReferenceData(unit: InventoryUnit): Promise<InventoryUnit> {
    const enrichedData: any = {};
    
    // Fetch reference project data if available
    if (unit.refProjectId) {
      try {
        enrichedData.referenceProject = await this.propwiseLabsService.getProjectById(unit.refProjectId);
      } catch (error) {
        this.logger.warn(`Failed to fetch reference project ${unit.refProjectId}`, error);
        enrichedData.referenceProject = null;
      }
    }
    
    // Fetch reference building data if available
    if (unit.refBuildingId) {
      try {
        enrichedData.referenceBuilding = await this.propwiseLabsService.getBuildingById(unit.refBuildingId);
      } catch (error) {
        this.logger.warn(`Failed to fetch reference building ${unit.refBuildingId}`, error);
        enrichedData.referenceBuilding = null;
      }
    }
    
    // Fetch reference unit data if available
    if (unit.refUnitId) {
      try {
        enrichedData.referenceUnit = await this.propwiseLabsService.getUnitById(unit.refUnitId);
      } catch (error) {
        this.logger.warn(`Failed to fetch reference unit ${unit.refUnitId}`, error);
        enrichedData.referenceUnit = null;
      }
    }
    
    // Fetch reference developer data if available
    if (unit.refDeveloperId) {
      try {
        enrichedData.referenceDeveloper = await this.propwiseLabsService.getDeveloperById(unit.refDeveloperId);
      } catch (error) {
        this.logger.warn(`Failed to fetch reference developer ${unit.refDeveloperId}`, error);
        enrichedData.referenceDeveloper = null;
      }
    }
    
    unit.enrichedData = enrichedData;
    return unit;
  }

  async getStatistics(organizationId: string): Promise<InventoryStatistics> {
    const qb = this.em.createQueryBuilder(InventoryUnit, 'iu')
      .where({ organizationId });

    const total = await qb.clone().getCount();
    
    const statusCounts = await qb
      .select(['status', 'count(*) as count'])
      .groupBy('status')
      .execute();

    const avgPrice = await qb.clone()
      .where({ price: { $ne: null } })
      .getKnexQuery()
      .avg('price as avg_price')
      .first();

    return {
      total,
      statusDistribution: statusCounts.reduce((acc, item) => {
        acc[item.status] = parseInt(item.count);
        return acc;
      }, {}),
      averagePrice: avgPrice?.avg_price ? parseFloat(avgPrice.avg_price) : null,
    };
  }

  async bulkCreate(createDtos: CreateInventoryUnitDto[], user: User): Promise<{ created: number; errors: any[] }> {
    const results = { created: 0, errors: [] };

    for (const [index, createDto] of createDtos.entries()) {
      try {
        await this.create(createDto, user);
        results.created++;
      } catch (error) {
        results.errors.push({
          index,
          dto: createDto,
          error: error.message,
        });
      }
    }

    return results;
  }

  async bulkUpdateStatus(unitIds: string[], status: UnitStatus, organizationId: string): Promise<{ updated: number }> {
    const result = await this.em.nativeUpdate(
      InventoryUnit,
      { id: { $in: unitIds }, organizationId },
      { status, updatedAt: new Date() }
    );

    return { updated: result };
  }

  private async validateReferenceProject(refProjectId: number): Promise<void> {
    if (!refProjectId) return;
    
    try {
      await this.propwiseLabsService.getProjectById(refProjectId);
    } catch (error) {
      throw new BadRequestException(`Invalid reference project ID: ${refProjectId}`);
    }
  }

  private async validateReferenceBuilding(refBuildingId: number): Promise<void> {
    if (!refBuildingId) return;
    
    try {
      await this.propwiseLabsService.getBuildingById(refBuildingId);
    } catch (error) {
      throw new BadRequestException(`Invalid reference building ID: ${refBuildingId}`);
    }
  }

  private async validateReferenceUnit(refUnitId: number): Promise<void> {
    if (!refUnitId) return;
    
    try {
      await this.propwiseLabsService.getUnitById(refUnitId);
    } catch (error) {
      throw new BadRequestException(`Invalid reference unit ID: ${refUnitId}`);
    }
  }

  private async validateReferenceDeveloper(refDeveloperId: number): Promise<void> {
    if (!refDeveloperId) return;
    
    try {
      await this.propwiseLabsService.getDeveloperById(refDeveloperId);
    } catch (error) {
      throw new BadRequestException(`Invalid reference developer ID: ${refDeveloperId}`);
    }
  }
}
```

```typescript
@Controller('inventory')
@UseGuards(JwtAuthGuard)
@ApiTags('Inventory')
export class InventoryController {
  constructor(private readonly inventoryService: InventoryService) {}

  @Get()

    @Query() filters: InventoryFiltersDto,
    @CurrentUser() user: User,

    return this.inventoryService.findAll(filters, user);
  }

  @Get('statistics')

  }

  @Get(':id')

    @CurrentUser() user: User,
  ): Promise<InventoryUnit> {

  }

  @Patch(':id')
  @ApiOperation({ summary: 'Update inventory unit' })
  @ApiResponse({ status: 200, description: 'Inventory unit updated successfully' })
  @ApiResponse({ status: 404, description: 'Inventory unit not found' })
```

## Repository layer implementation

### InventoryRepository

The repository layer provides data access patterns for inventory units:

```typescript
@Repository(InventoryUnit)
export class InventoryRepository extends BaseRepository<InventoryUnit> {
  constructor(em: EntityManager) {
    super(em, InventoryUnit);
  }

  async findByOrganization(
    organizationId: string,
    filters?: Partial<InventoryFiltersDto>,
  ): Promise<InventoryUnit[]> {
    const qb = this.createQueryBuilder('unit')
      .where({ organizationId });

    if (filters?.status) {
      qb.andWhere({ status: filters.status });
    }

    if (filters?.refProjectId) {
      qb.andWhere({ refProjectId: filters.refProjectId });
    }

    if (filters?.refBuildingId) {
      qb.andWhere({ refBuildingId: filters.refBuildingId });
    }

    if (filters?.bedrooms !== undefined) {
      qb.andWhere({ bedrooms: filters.bedrooms });
    }

    if (filters?.furnished !== undefined) {
      qb.andWhere({ furnished: filters.furnished });
    }

    if (filters?.propertyType) {
      qb.andWhere({ propertyType: filters.propertyType });
    }

    return qb.getResult();
  }

  async findAvailable(organizationId: string): Promise<InventoryUnit[]> {
    return this.find({
      organizationId,
      status: UnitStatus.AVAILABLE,
    });
  }

  async findByRefProject(
    organizationId: string,
    refProjectId: number,
  ): Promise<InventoryUnit[]> {
    return this.find({
      organizationId,
      refProjectId,
    });
  }

  async findByRefBuilding(
    organizationId: string,
    refBuildingId: number,
  ): Promise<InventoryUnit[]> {
    return this.find({
      organizationId,
      refBuildingId,
    });
  }

  async countByStatus(organizationId: string): Promise<Record<UnitStatus, number>> {
    const result = await this.em.createQueryBuilder(InventoryUnit, 'unit')
      .select(['status', 'count(*) as count'])
      .where({ organizationId })
      .groupBy('status')
      .execute();

    return result.reduce((acc, item) => {
      acc[item.status] = parseInt(item.count);
      return acc;
    }, {} as Record<UnitStatus, number>);
  }

  async findExpiring(organizationId: string, days: number = 30): Promise<InventoryUnit[]> {
    const expiryDate = new Date();
    expiryDate.setDate(expiryDate.getDate() + days);

    return this.find({
      organizationId,
      availableFrom: { $lte: expiryDate },
      status: { $in: [UnitStatus.AVAILABLE, UnitStatus.RESERVED] },
    });
  }
}
```
