**Path:** `src/modules/real-estate/inventory/`

The inventory module manages org-scoped secondary units within PropWise's **six-module real estate system**. It serves as one of the org-scoped modules that uses standard RLS (Row Level Security) via TenantContext.

## Architecture context

Inventory is one of six real estate modules spanning three visibility scopes:

<Tabs>
<Tab title="Global modules">
- **reference-data**: All property data (structural + enrichment) — 3 controllers (general, projects, buildings)
- **developer-portal**: Auth + profile + project/building management via ref DB  
- **map**: MapService (2GIS geocoding) — shared across modules
</Tab>
<Tab title="Org-scoped modules">
- **inventory**: Secondary units (this module) — standard RLS
- **property-interest**: Lead/deal property interests + matching engine — standard RLS
</Tab>
<Tab title="Cross-org modules">
- **shared-marketplace**: Cross-organization listings + deals — custom RLS
</Tab>
</Tabs>

### Data ownership pattern

<Info>
Inventory follows PropWise's **cross-database reference pattern**: CRM entities store plain integer IDs (`refUnitId`, `refProjectId`) pointing to the reference database, with no cross-database foreign keys. All property details come through `ReferenceDataService`.
</Info>

| Data Type | Database | Access Pattern |
|-----------|----------|----------------|
| Projects, units, buildings, areas, communities, developers | Reference DB | `ReferenceDataService` |
| Gallery images, payment plans, documents, amenities | Reference DB | `ReferenceDataService` |
| Geocoding (lat/lng), pricing, sale status, handover | Reference DB | `ReferenceDataService` |
| Developer auth (DeveloperUser) | CRM DB | Direct entity access |
| Developer-to-project ownership (DeveloperProjectLink) | CRM DB | Direct entity access |
| **Secondary units (org-scoped inventory)** | **CRM DB** | **Direct entity access** |
| Marketplace listings & deals (SharedListing, SharedDeal) | CRM DB | Direct entity access |
| Property interests (Lead/Deal) | CRM DB | Direct entity access, ref IDs as plain ints |
| Property matches | CRM DB | Direct entity access, ref IDs as plain ints |

### SecondaryUnit entity

The core entity for org-scoped inventory management:

```typescript
@Entity({ tableName: 'secondary_units' })
@UseRequestContext()
export class SecondaryUnit extends BaseUuidEntity {
  @Property()
  name!: string;

  @Property({ nullable: true })
  description?: string;

  // Cross-database references (plain integer IDs)
  @Property({ type: 'int', nullable: true })
  @Index()
  refProjectId?: number;  // ID in reference DB projects table

  @Property({ type: 'int', nullable: true })
  @Index()
  refUnitId?: number;     // ID in reference DB units table

  // Organization scoping (standard RLS)
  @ManyToOne(() => Organization, { fieldName: 'organization_id' })
  @Index()
  organization!: Organization;

  @Property({ fieldName: 'organization_id' })
  organizationId!: string;

  // Unit-specific data
  @Enum(() => UnitStatus)
  status!: UnitStatus;

  @Property({ type: 'numeric', nullable: true })
  price?: number;

  @Property({ type: 'numeric', nullable: true })
  area?: number;

  @Property({ nullable: true })
  bedrooms?: number;

  @Property({ nullable: true })
  bathrooms?: number;

  // Metadata
  @Property({ nullable: true })
  notes?: string;

  @Property({ type: 'datetime', defaultRaw: 'CURRENT_TIMESTAMP' })
  createdAt!: Date;

  @Property({ type: 'datetime', defaultRaw: 'CURRENT_TIMESTAMP', onUpdate: () => new Date() })
  updatedAt!: Date;
}
```

## Module services

### InventoryService

Core business logic for secondary unit management:

```typescript
@Injectable()
export class InventoryService {
  constructor(
    private readonly em: EntityManager,
    private readonly referenceDataService: ReferenceDataService,
  ) {}

  async createSecondaryUnit(
    createData: CreateSecondaryUnitDto,
    organizationId: string,
  ): Promise<SecondaryUnit> {
    const unit = this.em.create(SecondaryUnit, {
      ...createData,
      organizationId,
    });

    await this.em.persistAndFlush(unit);
    return unit;
  }

  async getUnitsForOrganization(
    organizationId: string,
    filters?: InventoryFiltersDto,
  ): Promise<SecondaryUnit[]> {
    const qb = this.em.createQueryBuilder(SecondaryUnit, 'u');
    
    qb.where({ organizationId });

    if (filters?.status) {
      qb.andWhere({ status: filters.status });
    }

    if (filters?.minPrice) {
      qb.andWhere({ price: { $gte: filters.minPrice } });
    }

    if (filters?.maxPrice) {
      qb.andWhere({ price: { $lte: filters.maxPrice } });
    }

    if (filters?.minBedrooms) {
      qb.andWhere({ bedrooms: { $gte: filters.minBedrooms } });
    }

    if (filters?.maxBedrooms) {
      qb.andWhere({ bedrooms: { $lte: filters.maxBedrooms } });
    }

    if (filters?.minArea) {
      qb.andWhere({ area: { $gte: filters.minArea } });
    }

    if (filters?.maxArea) {
      qb.andWhere({ area: { $lte: filters.maxArea } });
    }

    return qb.getResult();
  }

  async getUnitWithPropertyData(unitId: string): Promise<SecondaryUnitWithPropertyData> {
    const unit = await this.em.findOneOrFail(SecondaryUnit, { id: unitId });
    
    let propertyData = null;
    if (unit.refUnitId) {
      propertyData = await this.referenceDataService.getUnit(unit.refUnitId);
    }
    
    return { ...unit, propertyDetails: propertyData };
  }

  async updateUnit(
    unitId: string,
    updateData: UpdateSecondaryUnitDto,
  ): Promise<SecondaryUnit> {
    const unit = await this.em.findOneOrFail(SecondaryUnit, { id: unitId });
    
    this.em.assign(unit, updateData);
    await this.em.flush();
    
    return unit;
  }

  async deleteUnit(unitId: string): Promise<void> {
    const unit = await this.em.findOneOrFail(SecondaryUnit, { id: unitId });
    await this.em.removeAndFlush(unit);
  }

  async getUnitsByStatus(
    organizationId: string,
    status: UnitStatus,
  ): Promise<SecondaryUnit[]> {
    return this.em.find(SecondaryUnit, {
      organizationId,
      status,
    });
  }

  async updateUnitStatus(
    unitId: string,
    status: UnitStatus,
  ): Promise<SecondaryUnit> {
    const unit = await this.em.findOneOrFail(SecondaryUnit, { id: unitId });
    unit.status = status;
    await this.em.flush();
    return unit;
  }

  async getBulkUnitsByStatus(
    organizationId: string,
    statuses: UnitStatus[],
  ): Promise<SecondaryUnit[]> {
    return this.em.find(SecondaryUnit, {
      organizationId,
      status: { $in: statuses },
    });
  }

  async bulkUpdateStatus(
    unitIds: string[],
    status: UnitStatus,
  ): Promise<SecondaryUnit[]> {
    const units = await this.em.find(SecondaryUnit, { id: { $in: unitIds } });
    
    units.forEach(unit => {
      unit.status = status;
    });
    
    await this.em.flush();
    return units;
  }
}
```

<Warning>
The `SecondaryUnitWithPropertyData` interface combines the CRM secondary unit data with property details from the reference database. This pattern allows for enriched responses without direct foreign key relationships.
</Warning>

## Module controllers

### InventoryController

HTTP endpoints for secondary unit management:

```typescript
@Controller('inventory')
@UseGuards(JwtAuthGuard, OrganizationGuard)
@ApiBearerAuth()
@ApiTags('Inventory')
export class InventoryController {
  constructor(private readonly inventoryService: InventoryService) {}

  @Post()
  @ApiOperation({ summary: 'Create a new secondary unit' })
  @ApiResponse({ status: 201, type: SecondaryUnit })
  async createUnit(
    @Body() createData: CreateSecondaryUnitDto,
    @Req() req: RequestWithUser,
  ): Promise<SecondaryUnit> {
    return this.inventoryService.createSecondaryUnit(
      createData,
      req.user.organizationId,
    );
  }

  @Get()
  @ApiOperation({ summary: 'Get organization secondary units' })
  @ApiResponse({ status: 200, type: [SecondaryUnit] })
  async getUnits(
    @Query() filters: InventoryFiltersDto,
    @Req() req: RequestWithUser,
  ): Promise<SecondaryUnit[]> {
    return this.inventoryService.getUnitsForOrganization(
      req.user.organizationId,
      filters,
    );
  }

  @Get(':id')
  @ApiOperation({ summary: 'Get secondary unit with property details' })
  async getUnit(@Param('id') id: string): Promise<SecondaryUnitWithPropertyData> {
    return this.inventoryService.getUnitWithPropertyData(id);
  }

  @Patch(':id')
  @ApiOperation({ summary: 'Update secondary unit' })
  @ApiResponse({ status: 200, type: SecondaryUnit })
  async updateUnit(
    @Param('id') id: string,
    @Body() updateData: UpdateSecondaryUnitDto,
  ): Promise<SecondaryUnit> {
    return this.inventoryService.updateUnit(id, updateData);
  }

  @Delete(':id')
  @ApiOperation({ summary: 'Delete secondary unit' })
  @ApiResponse({ status: 204 })
  async deleteUnit(@Param('id') id: string): Promise<void> {
    return this.inventoryService.deleteUnit(id);
  }

  @Patch(':id/status')
  @ApiOperation({ summary: 'Update secondary unit status' })
  @ApiResponse({ status: 200, type: SecondaryUnit })
  async updateUnitStatus(
    @Param('id') id: string,
    @Body() statusUpdate: { status: UnitStatus },
  ): Promise<SecondaryUnit> {
    return this.inventoryService.updateUnitStatus(id, statusUpdate.status);
  }

  @Get('status/:status')
  @ApiOperation({ summary: 'Get units by status' })
  @ApiResponse({ status: 200, type: [SecondaryUnit] })
  async getUnitsByStatus(
    @Param('status') status: UnitStatus,
    @Req() req: RequestWithUser,
  ): Promise<SecondaryUnit[]> {
    return this.inventoryService.getUnitsByStatus(
      req.user.organizationId,
      status,
    );
  }

  @Post('bulk/status')
  @ApiOperation({ summary: 'Bulk update unit statuses' })
  @ApiResponse({ status: 200, type: [SecondaryUnit] })
  async bulkUpdateStatus(
    @Body() bulkUpdate: { unitIds: string[]; status: UnitStatus },
  ): Promise<SecondaryUnit[]> {
    return this.inventoryService.bulkUpdateStatus(
      bulkUpdate.unitIds,
      bulkUpdate.status,
    );
  }
}
```

### Cross-database reference pattern

CRM entities that reference off-plan projects or units store **plain integer IDs** (`refProjectId`, `refUnitId`) pointing to the reference database. No cross-database foreign keys exist — this decouples the CRM from the property data source.

```typescript
// CRM entity referencing reference DB data:
@Property({ type: 'int', nullable: true })
@Index()
refProjectId?: number;  // ID in reference DB projects table

@Property({ type: 'int', nullable: true })
@Index()
refUnitId?: number;     // ID in reference DB units table
```

<Note>
If the reference database is replaced with an external property API, only `ReferenceDataService` needs to change — inventory module remains unchanged.
</Note>
