**Path:** `src/modules/real-estate/inventory/`

The inventory module manages org-scoped secondary units within PropWise's **six-module real estate system**. It serves as one of the org-scoped modules that uses standard RLS (Row Level Security) via TenantContext.

<Note>
This module is part of PropWise's comprehensive real estate specification that spans six specialized modules across three visibility scopes: global (no RLS), org-private (standard RLS), and cross-org (custom RLS).
</Note>

## Architecture context

Inventory is one of six real estate modules spanning three visibility scopes:

<Tabs>
<Tab title="Global modules">
- **reference-data**: All property data (structural + enrichment) — 3 controllers (general, projects, buildings)
- **developer-portal**: Auth + profile + project/building management via ref DB  
- **map**: MapService (2GIS geocoding) — shared across modules
</Tab>
<Tab title="Org-scoped modules">
- **inventory**: Secondary units (this module) — standard RLS
- **property-interest**: Lead/deal property interests + matching engine — standard RLS
</Tab>
<Tab title="Cross-org modules">
- **shared-marketplace**: Cross-organization listings + deals — custom RLS
</Tab>
</Tabs>

### Data ownership pattern

<Info>
Inventory follows PropWise's **cross-database reference pattern**: CRM entities store plain integer IDs (`refUnitId`, `refProjectId`) pointing to the reference database, with no cross-database foreign keys. All property details come through `ReferenceDataService`.
</Info>

| Data Type | Database | Access Pattern |
|-----------|----------|----------------|
| Projects, units, buildings, areas, communities, developers | Reference DB | `ReferenceDataService` |
| Gallery images, payment plans, documents, amenities | Reference DB | `ReferenceDataService` |
| Geocoding (lat/lng), pricing, sale status, handover | Reference DB | `ReferenceDataService` |
| Developer auth (DeveloperUser) | CRM DB | Direct entity access |
| Developer-to-project ownership (DeveloperProjectLink) | CRM DB | Direct entity access |
| **Secondary units (org-scoped inventory)** | **CRM DB** | **Direct entity access** |
| Marketplace listings & deals (SharedListing, SharedDeal) | CRM DB | Direct entity access |
| Property interests (Lead/Deal) | CRM DB | Direct entity access, ref IDs as plain ints |
| Property matches | CRM DB | Direct entity access, ref IDs as plain ints |

## Service layer implementation

### InventoryService

The service layer handles business logic for inventory management:

```typescript
@Injectable()
export class InventoryService {
  constructor(
    private readonly em: EntityManager,
    private readonly referenceDataService: ReferenceDataService,
    private readonly logger: Logger,
  ) {}

  async findAll(filters: InventoryFiltersDto, user: User): Promise<SecondaryUnit[]> {
    const qb = this.em.createQueryBuilder(SecondaryUnit, 'su');
    
    // Standard RLS via organizationId
    qb.where({ organizationId: user.organizationId });
    
    if (filters.status) {
      qb.andWhere({ status: filters.status });
    }
    
    if (filters.priceMin !== undefined) {
      qb.andWhere({ price: { $gte: filters.priceMin } });
    }
    
    if (filters.priceMax !== undefined) {
      qb.andWhere({ price: { $lte: filters.priceMax } });
    }

    if (filters.bedrooms !== undefined) {
      qb.andWhere({ bedrooms: filters.bedrooms });
    }

    if (filters.bathrooms !== undefined) {
      qb.andWhere({ bathrooms: filters.bathrooms });
    }

    if (filters.furnished !== undefined) {
      qb.andWhere({ furnished: filters.furnished });
    }

    if (filters.refProjectId) {
      qb.andWhere({ refProjectId: filters.refProjectId });
    }

    // Search by name or description
    if (filters.search) {
      qb.andWhere({
        $or: [
          { name: { $ilike: `%${filters.search}%` } },
          { description: { $ilike: `%${filters.search}%` } },
        ],
      });
    }

    // Pagination
    if (filters.page !== undefined && filters.limit !== undefined) {
      qb.offset(filters.page * filters.limit).limit(filters.limit);
    }

    // Ordering
    qb.orderBy({ createdAt: 'DESC' });
    
    return qb.getResult();
  }

  async findOne(id: string, organizationId: string): Promise<SecondaryUnit> {
    const unit = await this.em.findOne(SecondaryUnit, { 
      id, 
      organizationId 
    });
    
    if (!unit) {
      throw new NotFoundException(`Secondary unit with ID ${id} not found`);
    }
    
    return unit;
  }

  async create(createDto: CreateSecondaryUnitDto, user: User): Promise<SecondaryUnit> {
    // Validate reference data if provided
    if (createDto.refProjectId) {
      await this.validateReferenceProject(createDto.refProjectId);
    }
    
    if (createDto.refUnitId) {
      await this.validateReferenceUnit(createDto.refUnitId);
    }

    const unit = this.em.create(SecondaryUnit, {
      ...createDto,
      organizationId: user.organizationId,
    });

    await this.em.persistAndFlush(unit);
    return unit;
  }

  async update(id: string, updateDto: UpdateSecondaryUnitDto, user: User): Promise<SecondaryUnit> {
    const unit = await this.findOne(id, user.organizationId);

    // Validate reference data if being updated
    if (updateDto.refProjectId !== undefined && updateDto.refProjectId !== unit.refProjectId) {
      await this.validateReferenceProject(updateDto.refProjectId);
    }
    
    if (updateDto.refUnitId !== undefined && updateDto.refUnitId !== unit.refUnitId) {
      await this.validateReferenceUnit(updateDto.refUnitId);
    }

    this.em.assign(unit, updateDto);
    await this.em.flush();
    
    return unit;
  }

  async remove(id: string, organizationId: string): Promise<void> {
    const unit = await this.findOne(id, organizationId);
    await this.em.removeAndFlush(unit);
  }

  async enrichWithReferenceData(unit: SecondaryUnit): Promise<SecondaryUnit> {
    const enrichedData: any = {};
    
    // Fetch reference project data if available
    if (unit.refProjectId) {
      try {
        enrichedData.referenceProject = await this.referenceDataService.getProjectById(unit.refProjectId);
      } catch (error) {
        this.logger.warn(`Failed to fetch reference project ${unit.refProjectId}`, error);
        enrichedData.referenceProject = null;
      }
    }
    
    // Fetch reference unit data if available
    if (unit.refUnitId) {
      try {
        enrichedData.referenceUnit = await this.referenceDataService.getUnitById(unit.refUnitId);
      } catch (error) {
        this.logger.warn(`Failed to fetch reference unit ${unit.refUnitId}`, error);
        enrichedData.referenceUnit = null;
      }
    }
    
    unit.enrichedData = enrichedData;
    return unit;
  }

  async getStatistics(organizationId: string): Promise<InventoryStatistics> {
    const qb = this.em.createQueryBuilder(SecondaryUnit, 'su')
      .where({ organizationId });

    const total = await qb.clone().getCount();
    
    const statusCounts = await qb
      .select(['status', 'count(*) as count'])
      .groupBy('status')
      .execute();

    const avgPrice = await qb.clone()
      .where({ price: { $ne: null } })
      .getKnexQuery()
      .avg('price as avg_price')
      .first();

    return {
      total,
      statusDistribution: statusCounts.reduce((acc, item) => {
        acc[item.status] = parseInt(item.count);
        return acc;
      }, {}),
      averagePrice: avgPrice?.avg_price ? parseFloat(avgPrice.avg_price) : null,
    };
  }

  async bulkCreate(createDtos: CreateSecondaryUnitDto[], user: User): Promise<{ created: number; errors: any[] }> {
    const results = { created: 0, errors: [] };

    for (const [index, createDto] of createDtos.entries()) {
      try {
        await this.create(createDto, user);
        results.created++;
      } catch (error) {
        results.errors.push({
          index,
          dto: createDto,
          error: error.message,
        });
      }
    }

    return results;
  }

  async bulkUpdateStatus(unitIds: string[], status: UnitStatus, organizationId: string): Promise<{ updated: number }> {
    const result = await this.em.nativeUpdate(
      SecondaryUnit,
      { id: { $in: unitIds }, organizationId },
      { status, updatedAt: new Date() }
    );

    return { updated: result };
  }

  private async validateReferenceProject(refProjectId: number): Promise<void> {
    if (!refProjectId) return;
    
    try {
      await this.referenceDataService.getProjectById(refProjectId);
    } catch (error) {
      throw new BadRequestException(`Invalid reference project ID: ${refProjectId}`);
    }
  }

  private async validateReferenceUnit(refUnitId: number): Promise<void> {
    if (!refUnitId) return;
    
    try {
      await this.referenceDataService.getUnitById(refUnitId);
    } catch (error) {
      throw new BadRequestException(`Invalid reference unit ID: ${refUnitId}`);
    }
  }
}
```

### InventoryController

```typescript
@Controller('inventory')
@UseGuards(JwtAuthGuard)
@ApiTags('Inventory')
export class InventoryController {
  constructor(private readonly inventoryService: InventoryService) {}

  @Get()
  @ApiOperation({ summary: 'List secondary units with filtering' })
  @ApiResponse({ status: 200, description: 'Returns filtered list of secondary units' })
  async findAll(
    @Query() filters: InventoryFiltersDto,
    @CurrentUser() user: User,
  ): Promise<SecondaryUnit[]> {
    return this.inventoryService.findAll(filters, user);
  }

  @Get('statistics')
  @ApiOperation({ summary: 'Get inventory statistics' })
  @ApiResponse({ status: 200, description: 'Returns inventory statistics for the organization' })
  async getStatistics(
    @CurrentUser() user: User,
  ): Promise<InventoryStatistics> {
    return this.inventoryService.getStatistics(user.organizationId);
  }

  @Get(':id')
  @ApiOperation({ summary: 'Get secondary unit by ID with enriched data' })
  @ApiResponse({ status: 200, description: 'Returns secondary unit with reference data' })
  @ApiResponse({ status: 404, description: 'Secondary unit not found' })
  async findOne(
    @Param('id', ParseUUIDPipe) id: string,
    @CurrentUser() user: User,
  ): Promise<SecondaryUnit> {
    const unit = await this.inventoryService.findOne(id, user.organizationId);
    return this.inventoryService.enrichWithReferenceData(unit);
  }

  @Post()
  @ApiOperation({ summary: 'Create new secondary unit' })
  @ApiResponse({ status: 201, description: 'Secondary unit created successfully' })
  @ApiResponse({ status: 400, description: 'Invalid input data' })
  async create(
    @Body() createDto: CreateSecondaryUnitDto,
    @CurrentUser() user: User,
  ): Promise<SecondaryUnit> {
    return this.inventoryService.create(createDto, user);
  }

  @Patch(':id')
  @ApiOperation({ summary: 'Update secondary unit' })
  @ApiResponse({ status: 200, description: 'Secondary unit updated successfully' })
  @ApiResponse({ status: 404, description: 'Secondary unit not found' })
  async update(
    @Param('id', ParseUUIDPipe) id: string,
    @Body() updateDto: UpdateSecondaryUnitDto,
    @CurrentUser() user: User,
  ): Promise<SecondaryUnit> {
    return this.inventoryService.update(id, updateDto, user);
  }

  @Delete(':id')
  @ApiOperation({ summary: 'Delete secondary unit' })
  @ApiResponse({ status: 200, description: 'Secondary unit deleted successfully' })
  @ApiResponse({ status: 404, description: 'Secondary unit not found' })
  async remove(
    @Param('id', ParseUUIDPipe) id: string,
    @CurrentUser() user: User,
  ): Promise<{ message: string }> {
    await this.inventoryService.remove(id, user.organizationId);
    return { message: 'Secondary unit deleted successfully' };
  }

  @Post('bulk')
  @ApiOperation({ summary: 'Bulk create secondary units' })
  @ApiResponse({ status: 201, description: 'Secondary units created successfully' })
  async bulkCreate(
    @Body() createDtos: CreateSecondaryUnitDto[],
    @CurrentUser() user: User,

      updateData.unitIds,

  }
```

- Numeric fields (price, area) must be non-negative

### ReferenceDataService integration

<Note>

The inventory module integrates with ReferenceDataService for property data enrichment:

<Steps>
<Step title="Reference validation">
When creating or updating units with reference IDs, the service validates project/unit existence through ReferenceDataService.

<Step title="Data enrichment">

### Cross-database reference pattern

CRM entities that reference off-plan projects or units store **plain integer IDs** (`refProjectId`, `refUnitId`) pointing to the reference database. No cross-database foreign keys exist — this decouples the CRM from the property data source:

```typescript
// CRM entity referencing reference DB data:
@Property({ type: 'int', nullable: true })
@Index()
refProjectId?: number;  // ID in reference DB projects table

@Property({ type: 'int', nullable: true })
@Index()
refUnitId?: number;     // ID in reference DB units table
```

<Tip>
If the reference database is replaced with an external property API, only `ReferenceDataService` needs to change — the inventory module remains unchanged.
</Tip>
