**Path:** `src/modules/real-estate/property-interest/`

Manages lead and deal property interests and the matching engine. Links to reference DB projects/units via **plain integer IDs** (`refProjectId`, `refUnitId`) and to CRM DB entities (`SecondaryUnit`, `SharedListing`) via ManyToOne relations.

## Entities

<AccordionGroup>
<Accordion title="LeadPropertyInterest">

- UUID PK, org-scoped
- ManyToOne: Lead (required)
- **Reference DB links (plain ints):** `refProjectId?: number`, `refUnitId?: number`
- **CRM DB links (ManyToOne):** secondaryUnit, sharedListing
- Matching criteria: propertyType, layoutType, bedrooms, budgetMin/Max, sizeMin/Max
- Preferred locations: preferredAreas[], preferredCommunities[]
- Metadata: priority, isPrimary, notes

</Accordion>
<Accordion title="DealPropertyInterest">

- UUID PK, org-scoped, **OneToOne with Deal** (owner: true)
- **Reference DB links (plain ints):** `refProjectId?: number`, `refUnitId?: number`
- **CRM DB links (ManyToOne):** secondaryUnit, sharedListing
- originatingInterest: link to LeadPropertyInterest (for lineage)
- interestType: Buy, Sell, Rent, Landlord
- status: Active, Confirmed, Cancelled, Completed
- budgetMin/Max, agreedPrice, confirmedAt

</Accordion>
<Accordion title="PropertyMatch">

- UUID PK, org-scoped
- ManyToOne: LeadPropertyInterest
- **Matched entity IDs:**
  - `matchedRefProjectId?: number` (reference DB integer)
  - `matchedRefUnitId?: number` (reference DB integer)
  - `matchedSecondaryUnitId?: string` (CRM DB UUID)
  - `matchedSharedListingId?: string` (CRM DB UUID)
- matchScore (0-100), matchBreakdown (JSON)
- status: MatchStatus (NEW, VIEWED, SHORTLISTED, DISMISSED, CONTACTED)

</Accordion>
</AccordionGroup>

## Property reference types

| Reference | Type | Notes |
| --- | --- | --- |
| refProjectId | int | Plain int ID to reference DB projects table |
| refUnitId | int | Plain int ID to reference DB units table |
| secondaryUnit | SecondaryUnit (ManyToOne) | Org-scoped, CRM DB entity |
| sharedListing | SharedListing (ManyToOne) | Cross-org marketplace listing |

## Matching engine

The `PropertyMatchService` generates matches when a LeadPropertyInterest is created or updated. It uses `ReferenceDataService` for off-plan data (no direct entity access):

<Steps>
<Step title="Query sources in parallel">

- Internal inventory (SecondaryUnit) — org-scoped via TenantContext
- Off-plan directory (RefProject/RefUnit) — via `ReferenceDataService`
- Shared marketplace (SharedListing) — global via bypass

</Step>
<Step title="Score each candidate (0-100)">

| Criterion | Points | Scoring |
| --- | --- | --- |
| Budget match | 0-30 | exact=30, within 20%=20, within 50%=10 |
| Bedroom match | 0-25 | exact=25, +/-1=15, +/-2=5 |
| Area match | 0-20 | preferred area=20, preferred community=10 |
| Size match | 0-15 | within range=15, within 20%=10 |
| Type match | 0-10 | exact=10 |

</Step>
<Step title="Persist matches">
Save matches with score greater than 20, sorted by score descending.
</Step>
</Steps>

<Tip>
The `PropertyMatchService` avoids nested TenantContext calls by running off-plan queries (via `ReferenceDataService`) and shared listing queries (global, `executeWithBypass`) **first**, then entering `executeInOrg` for org-scoped secondary unit queries + match persistence.
</Tip>

## Response and input DTOs

Constructor-based DTOs: `LeadPropertyInterestDto`, `PropertyMatchDto`.

Input DTOs use `@IsInt()` for `refProjectId`/`refUnitId` and `@IsUUID()` for `secondaryUnitId`/`sharedListingId`.

## Access control

Property interest is **org-scoped** and uses `@UseGuards(AuthGuard, AccessGuard)` with `@CheckAccess`:

- **List/Create/Match**: `permissions: ['crm.view', 'crm.manage']`
- **Update/Delete**: `permissions: ['crm.manage']`

## API endpoints

```
POST   /property-interests/leads/:leadId          # Create for lead
GET    /property-interests/leads/:leadId          # List for lead
PATCH  /property-interests/:id                     # Update
DELETE /property-interests/:id                     # Soft delete
GET    /property-interests/leads/:leadId/matches  # Get matches
PATCH  /property-interests/matches/:id/status     # Update match status
```
