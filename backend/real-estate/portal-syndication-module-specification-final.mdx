## Overview

The `portal-syndication` module manages the syndication of property listings to external real estate portals like Property Finder, Bayut, and Dubizzle. This module acts as a bridge between the internal property inventory system and external marketing platforms, handling authentication, data transformation, and synchronization status tracking.

The module is part of `RealEstateModule` and is imported from `src/modules/real-estate/real-estate.module.ts`.

## Architecture

The module follows a layered architecture with clear separation between core components:

  <Card title="Sync Tracking" icon="clock">

### Design Decisions

<AccordionGroup>
  <Accordion title="Separation of Concerns">
    Listings are separate from inventory units to allow marketing-specific data and portal customization
  </Accordion>
  <Accordion title="Per-Portal Tracking">
    Each listing maintains independent sync state per portal for granular control
  </Accordion>
  <Accordion title="Encrypted Credentials">
    Portal API keys stored encrypted using AES-256-GCM for security
  </Accordion>
  <Accordion title="Mapping Tables">
    Cached lookups for portal-specific identifiers (agents, locations) to optimize API calls
  </Accordion>
</AccordionGroup>

## Entities

### Listing

Represents a property unit prepared for marketing across portals.

```typescript
interface Listing {
  id: string;
  title: string;
  description: string;
  price: number;
  location: string;
  bedrooms: number;
  bathrooms: number;
  size: number;
  amenities: string[];
  images: string[];
  furnishedStatus: FurnishedStatus;
  rentalPeriod?: RentalPeriod;
  finishingType: FinishingType;
  status: ListingStatus;
  isActive: boolean;
  inventoryUnit: string; // FK to inventory
  createdBy: string; // FK to User
  organization: string; // FK to Organization
  createdAt: Date;
  updatedAt: Date;
}
```

### PortalConfiguration

Stores per-organization portal credentials and settings.

```typescript
interface PortalConfiguration {
  id: string;
  portal: Portal;
  isEnabled: boolean;
  apiKey: string; // Encrypted
  apiSecret?: string; // Encrypted
  organizationId?: string; // Portal-specific org ID
  portalCredits?: number;
  lastCreditsCheckAt?: Date;
  organization: string; // FK to Organization
  createdAt: Date;
  updatedAt: Date;
}
```

### ListingPortalSync

Tracks sync state for each listing on each portal.

```typescript
interface ListingPortalSync {
  id: string;
  portal: Portal;
  isEnabled: boolean;
  status: ListingSyncStatus;
  portalListingId?: string;
  lastSyncedAt?: Date;
  lastError?: string;
  lastPublishCost?: {
    creditCost: number;
    checkedAt: Date;
  };
  listing: string; // FK to Listing
  organization: string; // FK to Organization
  createdAt: Date;
  updatedAt: Date;
}
```

### PfAgentMapping

Maps CRM users to Property Finder agent profiles.

```typescript
interface PfAgentMapping {
  id: string;
  pfAgentId: string;
  pfPublicProfileId: string;
  lastSyncedAt?: Date;
  user: string; // FK to User
  organization: string; // FK to Organization
}
```

### PfLocationMapping

Caches Property Finder location ID lookups.

```typescript
interface PfLocationMapping {
  id: string;
  locationName: string;
  pfLocationId: string;
  lastSyncedAt?: Date;
  organization: string; // FK to Organization
}
```

## API Endpoints

### Listing Controller

<Tabs>
  <Tab title="Query Listings">
    #### GET /listings
    Query listings with filtering and pagination.

    **Query Parameters:**
    - `page?: number` - Page number for pagination
    - `limit?: number` - Number of items per page
    - `status?: ListingStatus` - Filter by listing status
    - `search?: string` - Search term for title/description

    **Response:**
    ```typescript
    {
      data: ListingDto[];
      meta: {
        totalItems: number;
        currentPage: number;
        totalPages: number;
      };
    }
    ```
  </Tab>
  
  <Tab title="Create Listing">
    #### POST /listings
    Create a new listing.

    **Request Body:**
    ```typescript
    {
      title: string;
      description: string;
      price: number;
      location: string;
      bedrooms: number;
      bathrooms: number;
      size: number;
      amenities: string[];
      images: string[];
      furnishedStatus: FurnishedStatus;
      rentalPeriod?: RentalPeriod;
      finishingType: FinishingType;
      inventoryUnitId: string;
    }
    ```
  </Tab>
  
  <Tab title="Update & Delete">
    #### PATCH /listings/:id
    Update an existing listing.

    #### DELETE /listings/:id
    Soft delete a listing (sets status to ARCHIVED).

    #### GET /listings/:id/portal-sync
    Get portal sync status for a listing.
  </Tab>
</Tabs>

### Portal Configuration Controller

<Steps>
  <Step title="Get Configurations">
    **GET /portal-configurations**
    
    Get all portal configurations for the organization.
  </Step>
  
  <Step title="Create Configuration">
    **POST /portal-configurations**
    
    Create a new portal configuration.

    ```typescript
    {
      portal: Portal;
      apiKey: string;
      apiSecret?: string;
      organizationId?: string;
    }
    ```
  </Step>
  
  <Step title="Toggle Portal">
    **PATCH /portal-configurations/:portal/toggle**
    
    Enable/disable a portal.

    ```typescript
    {
      isEnabled: boolean;
    }
    ```
  </Step>
</Steps>
