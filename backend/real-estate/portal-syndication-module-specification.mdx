The `portal-syndication` module manages the syndication of property listings to external real estate portals like Property Finder, Bayut, and Dubizzle. This module acts as a bridge between the internal property inventory system and external marketing platforms, handling authentication, data transformation, and synchronization status tracking.

<Info>
  **Module path:** `src/modules/real-estate/portal-syndication/`  
  The module is part of `RealEstateModule` and is imported from `src/modules/real-estate/real-estate.module.ts`
</Info>

## Architecture

The module follows a layered architecture with clear separation between:

- **Listings**: Marketing layer that wraps inventory units with portal-specific metadata
- **Portal Configuration**: Per-organization portal credentials and settings
- **Sync Tracking**: Per-listing, per-portal synchronization state management
- **Adapters**: Portal-specific integration logic (future implementation)

### Design Decisions

<CardGroup cols={2}>
  <Card title="Separation of Concerns" icon="layer-group">

## Data Models

### Listing Entity

Represents a property unit prepared for marketing across portals.

```typescript
interface Listing {
  id: string;
  title: string;
  description: string;
  price: number;
  location: string;
  bedrooms: number;
  bathrooms: number;
  size: number;
  amenities: string[];
  images: string[];
  furnishedStatus: FurnishedStatus;
  rentalPeriod?: RentalPeriod;
  finishingType: FinishingType;
  status: ListingStatus;
  isActive: boolean;
  inventoryUnit: string; // FK to inventory
  createdBy: string; // FK to User
  organization: string; // FK to Organization
  createdAt: Date;
  updatedAt: Date;
}
```

### Portal Configuration Entity

Stores per-organization portal credentials and settings.

```typescript
interface PortalConfiguration {
  id: string;
  portal: Portal;
  isEnabled: boolean;
  apiKey: string; // Encrypted
  apiSecret?: string; // Encrypted
  organizationId?: string; // Portal-specific org ID
  portalCredits?: number;
  lastCreditsCheckAt?: Date;
  organization: string; // FK to Organization
  createdAt: Date;
  updatedAt: Date;
}
```

### Listing Portal Sync Entity

Tracks sync state for each listing on each portal.

```typescript
interface ListingPortalSync {
  id: string;
  portal: Portal;
  isEnabled: boolean;
  status: ListingSyncStatus;
  portalListingId?: string;
  lastSyncedAt?: Date;
  lastError?: string;
  lastPublishCost?: {
    creditCost: number;
    checkedAt: Date;
  };
  listing: string; // FK to Listing
  organization: string; // FK to Organization
  createdAt: Date;
  updatedAt: Date;
}
```

## API Endpoints

### Listing Controller

<Tabs>
  <Tab title="Query Listings">
    #### GET /listings

    Query listings with filtering and pagination.

    **Query Parameters:**
    - `page?: number`
    - `limit?: number` 
    - `status?: ListingStatus`
    - `search?: string`

    **Response:**
    ```json
    {
      "data": [
        {
          "id": "listing-123",
          "title": "Luxury 2BR Apartment",
          "price": 85000,
          "status": "ACTIVE"
        }
      ],
      "meta": {
        "totalItems": 150,
        "currentPage": 1,
        "totalPages": 15
      }
    }
    ```
  </Tab>

  <Tab title="Create Listing">
    #### POST /listings

    Create a new listing.

    **Request Body:**
    ```json
    {
      "title": "Luxury 2BR Apartment",
      "description": "Spacious apartment with city views",
      "price": 85000,
      "location": "Downtown",
      "bedrooms": 2,
      "bathrooms": 2,
      "size": 1200,
      "amenities": ["gym", "pool", "parking"],
      "images": ["image1.jpg", "image2.jpg"],
      "furnishedStatus": "FURNISHED",
      "rentalPeriod": "YEARLY",
      "finishingType": "LUXURY",
      "inventoryUnitId": "unit-456"
    }
    ```
  </Tab>

  <Tab title="Update & Delete">
    #### PATCH /listings/:id
    Update an existing listing.

    #### DELETE /listings/:id
    Soft delete a listing (sets status to ARCHIVED).

    #### GET /listings/:id/portal-sync
    Get portal sync status for a listing.
  </Tab>
</Tabs>

### Portal Configuration Controller

<Steps>
  <Step title="Get Portal Configurations">
    **GET /portal-configurations**  
    Get all portal configurations for the organization.
  </Step>

  <Step title="Create Configuration">
    **POST /portal-configurations**  
    Create a new portal configuration.

    ```json
    {
      "portal": "PROPERTY_FINDER",
      "apiKey": "your-api-key",
      "apiSecret": "your-api-secret",
      "organizationId": "portal-org-123"
    }
    ```
  </Step>

  <Step title="Toggle Portal">
    **PATCH /portal-configurations/:portal/toggle**  
    Enable/disable a portal.

    ```json
    {
      "isEnabled": true
    }
    ```
  </Step>
</Steps>

## Business Rules

### Listing Rules

<Check>Each listing must be associated with a valid inventory unit</Check>
<Check>Only active listings can be synced to portals</Check>
<Check>Price must be positive</Check>
<Check>At least one image is required for portal syndication</Check>

### Portal Configuration Rules

<Warning>
  Each organization can have one configuration per portal. API credentials are encrypted before storage and the portal must be enabled to sync listings.
</Warning>

### Sync Rules

<Note>
  Each listing can have one sync record per portal. Sync status transitions follow portal-specific workflows, failed syncs store error messages for debugging, and sync costs are tracked for billing purposes.
</Note>
