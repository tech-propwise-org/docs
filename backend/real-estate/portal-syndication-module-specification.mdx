## Overview

The `portal-syndication` module manages the syndication of property listings to external real estate portals like Property Finder, Bayut, and Dubizzle. This module acts as a bridge between the internal property inventory system and external marketing platforms, handling authentication, data transformation, and synchronization status tracking.

<Info>
**Module Path:** `src/modules/real-estate/portal-syndication/`

The module is part of `RealEstateModule` and is imported from `src/modules/real-estate/real-estate.module.ts`.
</Info>

<Note>
**Status:** Development - Phase 1  
**Last Updated:** January 2024
</Note>

## Architecture

The module follows a layered architecture with clear separation between components:

<CardGroup cols={2}>
  <Card title="Listings" icon="list">
    Marketing layer that wraps inventory units with portal-specific metadata
  </Card>
  <Card title="Portal Configuration" icon="gear">
    Per-organization portal credentials and settings management
  </Card>
  <Card title="Sync Tracking" icon="arrows-rotate">
    Per-listing, per-portal synchronization state management
  </Card>
  <Card title="Adapters" icon="plug">
    Portal-specific integration logic (future implementation)
  </Card>
</CardGroup>

### Design Decisions

<AccordionGroup>
  <Accordion title="Separation of Concerns">
    Listings are maintained separately from inventory units to allow marketing-specific data and portal customization without affecting core inventory management.
  </Accordion>
  
  <Accordion title="Per-Portal Tracking">
    Each listing maintains independent sync state per portal, enabling granular control over which listings appear on which platforms.
  </Accordion>
  
  <Accordion title="Encrypted Credentials">
    Portal API keys are stored encrypted using AES-256-GCM encryption to protect sensitive integration credentials.
  </Accordion>
  
  <Accordion title="Mapping Tables">
    Cached lookups for portal-specific identifiers (agents, locations) improve performance and reduce external API calls.
  </Accordion>
</AccordionGroup>

## Entities

### Listing

Represents a property unit prepared for marketing across portals.

```typescript
interface Listing {
  id: string;
  title: string;
  description: string;
  price: number;
  location: string;
  bedrooms: number;
  bathrooms: number;
  size: number;
  amenities: string[];
  images: string[];
  furnishedStatus: FurnishedStatus;
  rentalPeriod?: RentalPeriod;
  finishingType: FinishingType;
  status: ListingStatus;
  isActive: boolean;
  inventoryUnit: string; // FK to inventory
  createdBy: string; // FK to User
  organization: string; // FK to Organization
  createdAt: Date;
  updatedAt: Date;
}
```

<Check>
**Key Features:**
- Complete property marketing information
- Multi-portal image management
- Flexible amenity and feature tracking
- Active/inactive toggle for quick control
</Check>

### PortalConfiguration

Stores per-organization portal credentials and settings.

```typescript
interface PortalConfiguration {
  id: string;
  portal: Portal;
  isEnabled: boolean;
  apiKey: string; // Encrypted
  apiSecret?: string; // Encrypted
  organizationId?: string; // Portal-specific org ID
  portalCredits?: number;
  lastCreditsCheckAt?: Date;
  organization: string; // FK to Organization
  createdAt: Date;
  updatedAt: Date;
}
```

<Warning>
API credentials (`apiKey` and `apiSecret`) are encrypted at rest using AES-256-GCM encryption. Never log or expose these values in plain text.
</Warning>

### ListingPortalSync

Tracks sync state for each listing on each portal.

```typescript
interface ListingPortalSync {
  id: string;
  portal: Portal;
  isEnabled: boolean;
  status: ListingSyncStatus;
  portalListingId?: string;
  lastSyncedAt?: Date;
  lastError?: string;
  lastPublishCost?: {
    creditCost: number;
    checkedAt: Date;
  };
  listing: string; // FK to Listing
  organization: string; // FK to Organization
  createdAt: Date;
  updatedAt: Date;
}
```

<Tip>
The `lastError` field stores error messages for debugging failed sync attempts, making troubleshooting easier without checking application logs.
</Tip>

### PfAgentMapping

Maps CRM users to Property Finder agent profiles.

```typescript
interface PfAgentMapping {
  id: string;
  pfAgentId: string;
  pfPublicProfileId: string;
  lastSyncedAt?: Date;
  user: string; // FK to User
  organization: string; // FK to Organization
}
```

### PfLocationMapping

Caches Property Finder location ID lookups.

```typescript
interface PfLocationMapping {
  id: string;
  locationName: string;
  pfLocationId: string;
  lastSyncedAt?: Date;
  organization: string; // FK to Organization
}
```

<Info>
Mapping tables reduce API calls to external portals by caching frequently accessed reference data like agent profiles and location identifiers.
</Info>

## Business Rules

### Listing Rules

<AccordionGroup>
  <Accordion title="Inventory Unit Association">
    Each listing must be associated with a valid inventory unit. The unit must exist and belong to the same organization as the listing creator.
  </Accordion>
  
  <Accordion title="Active Listing Requirement">
    Only listings with `isActive: true` and `status: ACTIVE` can be synced to external portals. Draft or archived listings are excluded from syndication.
  </Accordion>
  
  <Accordion title="Price Validation">
    Price must be a positive number. Zero or negative prices are rejected to prevent invalid listings on external portals.
  </Accordion>
  
  <Accordion title="Image Requirement">
    At least one image is required for portal syndication. Most external portals reject listings without images.
  </Accordion>
</AccordionGroup>

### Portal Configuration Rules

<Steps>

<Step title="One Configuration Per Portal">
Each organization can have exactly one configuration per portal. Attempting to create a duplicate will result in an error.
</Step>

<Step title="Credential Encryption">
API credentials are encrypted using AES-256-GCM encryption before storage. The encryption happens automatically in the service layer.
</Step>

<Step title="Enabled Portal Requirement">
A portal must be enabled (`isEnabled: true`) before any listings can be synced to it.
</Step>

<Step title="Credit Balance Check">
For Property Finder, the credit balance is checked before submissions to prevent failed publishes due to insufficient credits.
</Step>

</Steps>

### Sync Rules

<Check>
**Sync State Management:**
- Each listing can have one sync record per portal
- Sync status transitions follow portal-specific workflows
- Failed syncs store error messages in `lastError` field
- Sync costs are tracked in `lastPublishCost` for billing
</Check>

### Security

<Warning>
**Security Implementation:**
- All endpoints protected by `AuthGuard` and `AccessGuard`
- Row-level security via organization filtering on all queries
- Sensitive credentials encrypted at rest using AES-256-GCM
- API documentation requires bearer token authentication
- No credential exposure in API responses or logs
</Warning>

## Example Workflows

### Publishing a New Listing

<Steps>

<Step title="Create Listing">
Create a new listing associated with an inventory unit:

```typescript
POST /listings
{
  "title": "Luxury Villa",
  "inventoryUnitId": "unit_123",
  // ... other fields
}
```
</Step>

<Step title="Upload Images">
Upload and associate images with the listing (handled by media module)
</Step>

<Step title="Activate Listing">
Update the listing to active status:

```typescript
PATCH /listings/lst_789
{
  "status": "ACTIVE",
  "isActive": true
}
```
</Step>

<Step title="Enable Portal Sync">
Configure which portals should receive the listing (handled via sync service)
</Step>

<Step title="Monitor Sync Status">
Check sync status across portals:

```typescript
GET /listings/lst_789/portal-sync
```
</Step>

</Steps>

### Configuring Portal Integration

<Steps>

<Step title="Add Portal Configuration">
```typescript
POST /portal-configurations
{
  "portal": "PROPERTY_FINDER",
  "apiKey": "your_key",
  "apiSecret": "your_secret"
}
```
</Step>

<Step title="Verify Configuration">
Check that the configuration was created and portal is enabled
</Step>

<Step title="Test Connection">
Validate credentials by checking credit balance or making a test API call
</Step>

<Step title="Enable Automatic Sync">
Enable the portal to begin automatic listing syndication
</Step>

</Steps>
