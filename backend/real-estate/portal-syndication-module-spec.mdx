## Overview

The `portal-syndication` module manages the syndication of property listings to external real estate portals like Property Finder, Bayut, and Dubizzle. This module acts as a bridge between the internal property inventory system and external marketing platforms, handling authentication, data transformation, and synchronization status tracking.

<Info>
**Module Path**: `src/modules/real-estate/portal-syndication/`  
**Parent Module**: `RealEstateModule` (imported from `src/modules/real-estate/real-estate.module.ts`)  
**Status**: Development - Phase 1  
**Last Updated**: January 2024
</Info>

## Architecture

The module follows a layered architecture with clear separation between core components:

  <Card title="Sync Tracking" icon="sync">

  </Card>

### Design Decisions

<Note>
Key architectural decisions that drive the module structure:

- **Separation of Concerns**: Listings are separate from inventory units to allow marketing-specific data
- **Per-Portal Tracking**: Each listing maintains independent sync state per portal
- **Encrypted Credentials**: Portal API keys stored encrypted using AES-256-GCM
- **Mapping Tables**: Cached lookups for portal-specific identifiers (agents, locations)
</Note>

## Entities

### Listing

Represents a property unit prepared for marketing across portals.

```typescript
interface Listing {
  id: string;
  title: string;
  description: string;
  price: number;
  location: string;
  bedrooms: number;
  bathrooms: number;
  size: number;
  amenities: string[];
  images: string[];
  furnishedStatus: FurnishedStatus;
  rentalPeriod?: RentalPeriod;
  finishingType: FinishingType;
  status: ListingStatus;
  isActive: boolean;
  inventoryUnit: string; // FK to inventory
  createdBy: string; // FK to User
  organization: string; // FK to Organization
  createdAt: Date;
  updatedAt: Date;
}
```

### PortalConfiguration

Stores per-organization portal credentials and settings.

```typescript
interface PortalConfiguration {
  id: string;
  portal: Portal;
  isEnabled: boolean;
  apiKey: string; // Encrypted
  apiSecret?: string; // Encrypted
  organizationId?: string; // Portal-specific org ID
  portalCredits?: number;
  lastCreditsCheckAt?: Date;
  organization: string; // FK to Organization
  createdAt: Date;
  updatedAt: Date;
}
```

### ListingPortalSync

Tracks sync state for each listing on each portal.

```typescript
interface ListingPortalSync {
  id: string;
  portal: Portal;
  isEnabled: boolean;
  status: ListingSyncStatus;
  portalListingId?: string;
  lastSyncedAt?: Date;
  lastError?: string;
  lastPublishCost?: {
    creditCost: number;
    checkedAt: Date;
  };
  listing: string; // FK to Listing
  organization: string; // FK to Organization
  createdAt: Date;
  updatedAt: Date;
}
```

### PfAgentMapping

Maps CRM users to Property Finder agent profiles.

```typescript
interface PfAgentMapping {
  id: string;
  pfAgentId: string;
  pfPublicProfileId: string;
  lastSyncedAt?: Date;
  user: string; // FK to User
  organization: string; // FK to Organization
}
```

### PfLocationMapping

Caches Property Finder location ID lookups.

```typescript
interface PfLocationMapping {
  id: string;
  locationName: string;
  pfLocationId: string;
  lastSyncedAt?: Date;
  organization: string; // FK to Organization
}
```

## API Endpoints

### Listing Controller

<AccordionGroup>
  <Accordion title="GET /listings">
    Query listings with filtering and pagination.

    **Query Parameters:**
    - `page?: number`
    - `limit?: number`
    - `status?: ListingStatus`
    - `search?: string`

    **Response:**
    ```typescript
    {
      data: ListingDto[];
      meta: {
        totalItems: number;
        currentPage: number;
        totalPages: number;
      };
    }
    ```
  </Accordion>

  <Accordion title="POST /listings">
    Create a new listing.

    **Request Body:**
    ```typescript
    {
      title: string;
      description: string;
      price: number;
      location: string;
      bedrooms: number;
      bathrooms: number;
      size: number;
      amenities: string[];
      images: string[];
      furnishedStatus: FurnishedStatus;
      rentalPeriod?: RentalPeriod;
      finishingType: FinishingType;
      inventoryUnitId: string;
    }
    ```
  </Accordion>

  <Accordion title="PATCH /listings/:id">
    Update an existing listing.
  </Accordion>

  <Accordion title="DELETE /listings/:id">
    Soft delete a listing (sets status to ARCHIVED).
  </Accordion>

  <Accordion title="GET /listings/:id/portal-sync">
    Get portal sync status for a listing.
  </Accordion>
</AccordionGroup>

## Business Rules

<Tabs>
  <Tab title="Listing Rules">
    <Steps>
      <Step title="Unit Association">
        Each listing must be associated with a valid inventory unit
      </Step>
      <Step title="Active Status">
        Only active listings can be synced to portals
      </Step>
      <Step title="Price Validation">
        Price must be positive
      </Step>
      <Step title="Image Requirement">
        At least one image is required for portal syndication
      </Step>
    </Steps>
  </Tab>

  <Tab title="Portal Configuration Rules">
    <Steps>
      <Step title="Single Configuration">
        Each organization can have one configuration per portal
      </Step>
      <Step title="Credential Security">
        API credentials are encrypted before storage
      </Step>
      <Step title="Enabled Requirement">
        Portal must be enabled to sync listings
      </Step>
      <Step title="Credit Validation">
        Credit balance checked before Property Finder submissions
      </Step>
    </Steps>
  </Tab>

  <Tab title="Sync Rules">
    <Steps>
      <Step title="Sync Records">
        Each listing can have one sync record per portal
      </Step>
      <Step title="Status Transitions">
        Sync status transitions follow portal-specific workflows
      </Step>
      <Step title="Error Tracking">
        Failed syncs store error messages for debugging
      </Step>
      <Step title="Cost Tracking">
        Sync costs are tracked for billing purposes
      </Step>
    </Steps>
