## Overview

The `portal-syndication` module manages the syndication of property listings to external real estate portals like Property Finder, Bayut, and Dubizzle. This module acts as a bridge between the internal property inventory system and external marketing platforms, handling authentication, data transformation, and synchronization status tracking.

<Info>
**Module path:** `src/modules/real-estate/portal-syndication/`

The module is part of `RealEstateModule` and is imported from `src/modules/real-estate/real-estate.module.ts`.
</Info>

<Note>
**Status:** Development - Phase 1  
**Last Updated:** January 2024
</Note>

## Architecture

The module follows a layered architecture with clear separation between different concerns:

### Design Decisions

<AccordionGroup>
  <Accordion title="Separation of Concerns">
    Listings are separate from inventory units to allow marketing-specific data and portal customization without affecting the core inventory system.
  </Accordion>
  
  <Accordion title="Per-Portal Tracking">
    Each listing maintains independent sync state per portal, enabling selective syndication and granular status tracking.
  </Accordion>
  
  <Accordion title="Encrypted Credentials">
    Portal API keys are stored encrypted using AES-256-GCM encryption to ensure security of sensitive authentication data.
  </Accordion>
  
  <Accordion title="Mapping Tables">
    Cached lookups for portal-specific identifiers (agents, locations) to improve performance and reduce API calls.
  </Accordion>
</AccordionGroup>

## Business Rules

### Listing Rules

<Steps>
  <Step title="Unit Association">
    Each listing must be associated with a valid inventory unit from the real estate inventory module.
  </Step>
  
  <Step title="Active Status Requirement">
    Only active listings can be synced to external portals. Draft or archived listings are excluded from syndication.
  </Step>
  
  <Step title="Price Validation">
    Listing price must be a positive number greater than zero.
  </Step>
  
  <Step title="Image Requirement">
    At least one image is required for portal syndication to meet external platform requirements.
  </Step>
</Steps>

### Portal Configuration Rules

<Check>
**One Configuration Per Portal**: Each organization can have only one configuration per portal type.
</Check>

<Check>
**Encrypted Storage**: API credentials are automatically encrypted before storage using AES-256-GCM encryption.
</Check>

<Check>
**Enablement Control**: Portal must be enabled in configuration to allow listing synchronization.
</Check>

<Check>
**Credit Monitoring**: Credit balance is checked before Property Finder submissions to prevent failed uploads.
</Check>

### Sync Rules

<Info>
**Independent Tracking**: Each listing can have one sync record per portal, enabling selective syndication strategies.
</Info>

- Sync status transitions follow portal-specific workflows
- Failed syncs store detailed error messages for debugging
- Sync costs are tracked for billing and reporting purposes

### Security

<Warning>
All security measures are strictly enforced:
</Warning>

- All endpoints protected by AuthGuard and AccessGuard
- Row-level security via organization filtering
- Sensitive credentials encrypted at rest
- API documentation includes bearer token authentication requirements

### Error Handling

The module implements structured error handling with:

- **Structured Responses**: Consistent error response format across all endpoints
- **Sync Failure Tracking**: Database storage of sync failures for debugging
- **Graceful Degradation**: Continued operation during portal connectivity issues
- **Detailed Logging**: Comprehensive logging for troubleshooting and monitoring
