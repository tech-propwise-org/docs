# Portal Syndication Module - Comprehensive Specification

<Info>
**Last Updated:** January 2024  
**Status:** Development - Phase 1  
**Module Path:** `src/modules/real-estate/portal-syndication/`
</Info>

## Overview

The `portal-syndication` module manages the syndication of property listings to external real estate portals like Property Finder, Bayut, and Dubizzle. This module acts as a bridge between the internal property inventory system and external marketing platforms, handling authentication, data transformation, and synchronization status tracking.

<Note>
The module is part of `RealEstateModule` and is imported from `src/modules/real-estate/real-estate.module.ts`.
</Note>

## Architecture

The module follows a layered architecture with clear separation between:
- **Listings**: Marketing layer that wraps inventory units with portal-specific metadata
- **Portal Configuration**: Per-organization portal credentials and settings
- **Sync Tracking**: Per-listing, per-portal synchronization state management
- **Adapters**: Portal-specific integration logic (future implementation)

### Design Decisions

<CardGroup cols={2}>
  <Card title="Separation of Concerns" icon="layer-group">

## Entities

### Listing

Represents a property unit prepared for marketing across portals.

```typescript
interface Listing {
  id: string;
  title: string;
  description: string;
  price: number;
  location: string;
  bedrooms: number;
  bathrooms: number;
  size: number;
  amenities: string[];
  images: string[];
  furnishedStatus: FurnishedStatus;
  rentalPeriod?: RentalPeriod;
  finishingType: FinishingType;
  status: ListingStatus;
  isActive: boolean;
  inventoryUnit: string; // FK to inventory
  createdBy: string; // FK to User
  organization: string; // FK to Organization
  createdAt: Date;
  updatedAt: Date;
}
```

### PortalConfiguration

Stores per-organization portal credentials and settings.

```typescript
interface PortalConfiguration {
  id: string;
  portal: Portal;
  isEnabled: boolean;
  apiKey: string; // Encrypted
  apiSecret?: string; // Encrypted
  organizationId?: string; // Portal-specific org ID
  portalCredits?: number;
  lastCreditsCheckAt?: Date;
  organization: string; // FK to Organization
  createdAt: Date;
  updatedAt: Date;
}
```

### ListingPortalSync

Tracks sync state for each listing on each portal.

```typescript
interface ListingPortalSync {
  id: string;
  portal: Portal;
  isEnabled: boolean;
  status: ListingSyncStatus;
  portalListingId?: string;
  lastSyncedAt?: Date;
  lastError?: string;
  lastPublishCost?: {
    creditCost: number;
    checkedAt: Date;
  };
  listing: string; // FK to Listing
  organization: string; // FK to Organization
  createdAt: Date;
  updatedAt: Date;
}
```

### PfAgentMapping

Maps CRM users to Property Finder agent profiles.

```typescript
interface PfAgentMapping {

  pfAgentId: string;
  pfPublicProfileId: string;
  lastSyncedAt?: Date;
  user: string; // FK to User
  organization: string; // FK to Organization

```

### PfLocationMapping

Caches Property Finder location ID lookups.

```typescript
interface PfLocationMapping {

  locationName: string;
  pfLocationId: string;
  lastSyncedAt?: Date;
  organization: string; // FK to Organization
}
```

## API Endpoints

### Listing Controller

#### GET /listings
Query listings with filtering and pagination.

**Query Parameters:**
- `page?: number`
- `limit?: number`
- `status?: ListingStatus`
- `search?: string`

**Response:**
```typescript
{
  data: ListingDto[];
  meta: {
    totalItems: number;
    currentPage: number;
    totalPages: number;
  };
}
```

#### POST /listings
Create a new listing.

**Request Body:**
```typescript
{
  title: string;
  description: string;
  price: number;
  location: string;
  bedrooms: number;
  bathrooms: number;
  size: number;
  amenities: string[];
  images: string[];
  furnishedStatus: FurnishedStatus;
  rentalPeriod?: RentalPeriod;
  finishingType: FinishingType;
  inventoryUnitId: string;
}
```

#### PATCH /listings/:id
Update an existing listing.

#### DELETE /listings/:id
Soft delete a listing (sets status to ARCHIVED).

#### GET /listings/:id/portal-sync
Get portal sync status for a listing.

### Portal Configuration Controller

#### GET /portal-configurations
Get all portal configurations for the organization.

#### POST /portal-configurations
Create a new portal configuration.

**Request Body:**
```typescript
{
  portal: Portal;
  apiKey: string;
  apiSecret?: string;
  organizationId?: string;
}
```

#### PATCH /portal-configurations/:portal/toggle
Enable/disable a portal.

**Request Body:**
```typescript
{
  isEnabled: boolean;
}
```

## Business Rules

<AccordionGroup>
  <Accordion title="Listing Rules">
    1. Each listing must be associated with a valid inventory unit
    2. Only active listings can be synced to portals
    3. Price must be positive
    4. At least one image is required for portal syndication
  </Accordion>

  <Accordion title="Portal Configuration Rules">
    1. Each organization can have one configuration per portal
    2. API credentials are encrypted before storage
    3. Portal must be enabled to sync listings
    4. Credit balance checked before Property Finder submissions
  </Accordion>

  <Accordion title="Sync Rules">
    1. Each listing can have one sync record per portal
    2. Sync status transitions follow portal-specific workflows
    3. Failed syncs store error messages for debugging
    4. Sync costs are tracked for billing purposes
  </Accordion>
</AccordionGroup>
