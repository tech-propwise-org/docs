## Overview

The Bayut Pull API provides a comprehensive interface for retrieving lead data and view statistics from the Bayut platform. This API enables developers to integrate Bayut lead information into their applications or systems by offering structured access to various types of leads and engagement metrics.

<Note>
This document describes the external API contract defined by Bayut, including endpoint specifications, authorization requirements, query parameters, and sample responses for both leads (`is_trulead=1`) and views (`is_trulead=0`).
</Note>

## Base endpoint

```
www.bayut.com/api-v7/stats/website-client-leads
```

## Authentication

Each client receives a unique API key that must be included as a bearer token in the Authorization header.

<CodeGroup>
```bash cURL
curl --location \
  'www.bayut.com/api-v7/stats/website-client-leads?timestamp=2025-03-10%2000%3A00%3A00&target=listing&type=email&is_trulead=1' \
  --header 'Authorization: Bearer <API_KEY>'
```

```http HTTP
GET /api-v7/stats/website-client-leads HTTP/1.1
Host: www.bayut.com
Authorization: Bearer <API_KEY>
```
</CodeGroup>

<Warning>
The timestamp value must be URL-encoded: `2025-03-10 00:00:00` becomes `2025-03-10%2000%3A00%3A00` where `%20` represents space and `%3A` represents colon.
</Warning>

## Query parameters

| Parameter | Description | Possible Values | Required |
|-----------|-------------|-----------------|----------|
| `type` | The type of lead or interaction | `whatsapp`, `sms`, `phone`, `email`, `call_logs`, `story_leads` | Yes |
| `timestamp` | The date and time of the request | Format: `YYYY-MM-DD HH:MM:SS` | Yes |
| `target` | The entity for which leads are fetched | `listing`, `agent`, `agency` | Yes* |
| `is_trulead` | Whether to return leads (1) or view counts (0) | `0`, `1` | Yes |

<Info>
*The `target` parameter is not required for `call_logs` and `story_leads` types.
</Info>

<Tip>
Use `is_trulead=1` to retrieve actual lead data with contact information. Use `is_trulead=0` to get view count statistics.
</Tip>

## Lead types (`is_trulead=1`)

When `is_trulead=1`, the API returns detailed lead information including contact details and inquiry messages.

## Integration guide

<Steps>
  <Step title="Obtain API credentials">
    Contact Bayut to receive your unique API key for authentication.
  </Step>

  <Step title="Set up authentication">
    Include your API key as a bearer token in the Authorization header for all requests.
  </Step>

  <Step title="Choose your data type">
    Decide whether you need lead data (`is_trulead=1`) or view statistics (`is_trulead=0`).
  </Step>

  <Step title="Configure query parameters">
    Set the appropriate `type`, `target`, and `timestamp` parameters based on your requirements.
  </Step>

  <Step title="Handle responses">
    Process the JSON responses according to the lead type structure documented above.
  </Step>
</Steps>
