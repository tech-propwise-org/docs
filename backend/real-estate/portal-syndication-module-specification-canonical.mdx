The Portal Syndication Module manages the syndication of property listings to external real estate portals like Property Finder, Bayut, and Dubizzle. This module acts as a bridge between the internal property inventory system and external marketing platforms, handling authentication, data transformation, and synchronization status tracking.

<Info>
**Module Path:** `src/modules/real-estate/portal-syndication/`

The module is part of `RealEstateModule` and is imported from `src/modules/real-estate/real-estate.module.ts`.
</Info>

## Architecture Overview

The module follows a layered architecture with clear separation between different concerns:

<CardGroup cols={2}>
  <Card title="Listings Layer" icon="list">
    Marketing layer that wraps inventory units with portal-specific metadata
  </Card>
  <Card title="Portal Configuration" icon="gear">
    Per-organization portal credentials and settings management
  </Card>
  <Card title="Sync Tracking" icon="arrows-rotate">
    Per-listing, per-portal synchronization state management
  </Card>
  <Card title="Portal Adapters" icon="plug">
    Portal-specific integration logic (future implementation)
  </Card>
</CardGroup>

### Design Decisions

<AccordionGroup>
  <Accordion title="Separation of Concerns">
    Listings are separate from inventory units to allow marketing-specific data without polluting the core inventory model.
  </Accordion>
  
  <Accordion title="Per-Portal Tracking">
    Each listing maintains independent sync state per portal to handle different portal requirements and workflows.
  </Accordion>
  
  <Accordion title="Encrypted Credentials">
    Portal API keys are stored encrypted using AES-256-GCM encryption for enhanced security.
  </Accordion>
  
  <Accordion title="Mapping Tables">
    Cached lookups for portal-specific identifiers (agents, locations) to optimize API performance.
  </Accordion>
</AccordionGroup>

## Core Entities

### Listing Entity

Represents a property unit prepared for marketing across portals.

```typescript
interface Listing {
  id: string;
  title: string;
  description: string;
  price: number;
  location: string;
  bedrooms: number;
  bathrooms: number;
  size: number;
  amenities: string[];
  images: string[];
  furnishedStatus: FurnishedStatus;
  rentalPeriod?: RentalPeriod;
  finishingType: FinishingType;
  status: ListingStatus;
  isActive: boolean;
  inventoryUnit: string; // FK to inventory
  createdBy: string; // FK to User
  organization: string; // FK to Organization
  createdAt: Date;
  updatedAt: Date;
}
```

### Portal Configuration Entity

Stores per-organization portal credentials and settings.

```typescript
interface PortalConfiguration {
  id: string;
  portal: Portal;
  isEnabled: boolean;
  apiKey: string; // Encrypted
  apiSecret?: string; // Encrypted
  organizationId?: string; // Portal-specific org ID
  portalCredits?: number;
  lastCreditsCheckAt?: Date;
  organization: string; // FK to Organization
  createdAt: Date;
  updatedAt: Date;
}
```

<Warning>
API keys and secrets are automatically encrypted before storage using AES-256-GCM encryption.
</Warning>

## API Endpoints

### Listing Management

<Steps>
  <Step title="Query Listings">
    **GET** `/listings`
    
    Query listings with filtering and pagination support.
    
    **Query Parameters:**
    ```typescript
    {
      page?: number;
      limit?: number;
      status?: ListingStatus;
      search?: string;
    }
    ```
    
    **Response:**
    ```typescript
    {
      data: ListingDto[];
      meta: {
        totalItems: number;
        currentPage: number;
        totalPages: number;
      };
    }
    ```
  </Step>
  
  <Step title="Create Listing">
    **POST** `/listings`
    
    Create a new property listing.
    
    **Request Body:**
    ```typescript
    {
      title: string;
      description: string;
      price: number;
      location: string;
      bedrooms: number;
      bathrooms: number;
      size: number;
      amenities: string[];
      images: string[];
      furnishedStatus: FurnishedStatus;
      rentalPeriod?: RentalPeriod;
      finishingType: FinishingType;
      inventoryUnitId: string;
    }
    ```
  </Step>
  
  <Step title="Update Listing">
    **PATCH** `/listings/:id`
    
    Update an existing listing with partial data.
  </Step>
  
  <Step title="Delete Listing">
    **DELETE** `/listings/:id`
    
    Soft delete a listing by setting status to ARCHIVED.
  </Step>
  
  <Step title="Get Portal Sync Status">
    **GET** `/listings/:id/portal-sync`
    
    Retrieve portal sync status for a specific listing.
  </Step>
</Steps>

## Business Rules

### Listing Requirements

<Check>
**Valid Listing Requirements:**
- Must be associated with a valid inventory unit
- Only active listings can be synced to portals
- Price must be positive
- At least one image is required for portal syndication
</Check>

### Portal Configuration Rules

<Note>
**Configuration Constraints:**
- Each organization can have one configuration per portal
- API credentials are encrypted before storage
- Portal must be enabled to sync listings
- Credit balance checked before Property Finder submissions
</Note>

### Synchronization Rules

<Tip>
**Sync State Management:**
- Each listing can have one sync record per portal
- Sync status transitions follow portal-specific workflows
- Failed syncs store error messages for debugging
- Sync costs are tracked for billing purposes
</Tip>

## Development Standards

### Data Transfer Objects

- Comprehensive validation using class-validator decorators
- Separate DTOs for create, update, and response operations
- Nested validation for complex objects

### Security Implementation

<Warning>
**Security Requirements:**
- All endpoints protected by AuthGuard and AccessGuard
- Row-level security via organization filtering
- Sensitive credentials encrypted at rest
- API documentation with bearer token authentication
</Warning>

### Error Handling Strategy

- Structured error responses with meaningful messages
- Sync failure tracking stored in database
- Graceful degradation for portal connectivity issues
- Comprehensive logging for debugging and monitoring
