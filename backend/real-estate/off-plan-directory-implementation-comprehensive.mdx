## Overview

The Off-Plan Directory is a comprehensive property listing system that displays published buildings from developer portal users. It features a card/map split view with rich filtering capabilities, 2GIS map integration, and detailed building views.

<Note>
The backend serves off-plan data through domain endpoints under `/off-plan/*`, which read Propwise Labs catalog data and apply CRM-owned visibility controls.
</Note>

## Architecture Overview

### Buildings as Primary Entity

The system uses **buildings** as the primary enrichment entity rather than projects:

- Buildings have their own `coverImageUrl`, `status`, `endDate`, `completionDate`, `paymentPlans`, `images`, `documents`, `amenities`
- Buildings can override inherited fields from projects (status, area, community, description)
- The off-plan directory displays **published buildings** based on CRM `is_published` visibility

<Info>
Publication is separate from Propwise Labs `building.status`. Developers can publish/unpublish buildings through the developer portal.
</Info>

### Data Flow

```
Frontend (CRM App)
  └── OffPlanApi → GET /off-plan/buildings              (card listing)
  └── OffPlanApi → GET /off-plan/buildings/:id          (detail view)
  └── OffPlanApi → GET /off-plan/buildings/:id/units/grouped
  └── OffPlanApi → GET /off-plan/buildings/map          (map markers)
  └── PropwiseLabsApi → GET /propwise-labs/developers?q= (developer options)
  └── PropwiseLabsApi → GET /propwise-labs/areas?q=      (area filter)
```

### Frontend Status Mapping

| Backend `building.status` | Frontend Status | Color  |
| ------------------------- | --------------- | ------ |
| `ACTIVE`                  | On Sale         | Orange |
| `PENDING`                 | EOI             | Purple |
| `FINISHED`                | Out of Stock    | Gray   |

```typescript
export interface OffPlanBuildingFilters {
  q?: string;
  status?: string;
  areaId?: number;
  developerId?: number;
  developerIds?: number[];
  propertySubTypeId?: number;
  priceMode?: 'unit' | 'sqft';
  minPrice?: number;
  maxPrice?: number;
  bedrooms?: string;
  completionBefore?: string;
  completionAfter?: string;
  page?: number;
  limit?: number;
  sortBy?: string;
  sortOrder?: 'asc' | 'desc';
}

export class OffPlanApi {
  static async searchBuildings(filters: OffPlanBuildingFilters) {
    return apiClient.get('/off-plan/buildings', { params: filters });
  }

  static async getBuildingDetail(id: number) {
    return apiClient.get(`/off-plan/buildings/${id}`);
  }

  static async getBuildingUnitsGrouped(buildingId: number) {
    return apiClient.get(`/off-plan/buildings/${buildingId}/units/grouped`);
  }

  static async getMapMarkers(filters?: MapMarkerFilters) {
    return apiClient.get('/off-plan/buildings/map', { params: filters });
  }
}
```
</Step>

## Map Integration

### 2GIS Map Features

The map integration provides:

- **Custom Markers**: Circular markers with developer logos and status-based border colors
- **Marker Popover**: Quick preview cards on marker hover/click
- **Cluster Support**: Automatic marker clustering for dense areas
- **Filter Sync**: Map markers update based on active filters
- **Responsive Design**: Adapts to different screen sizes

<Warning>
Ensure 2GIS API credentials are properly configured in the environment variables before implementing map features.

## Performance Considerations

### Optimization Strategies

<Check>**Infinite Scroll**: Implement virtual scrolling for large building lists</Check>
<Check>**Image Lazy Loading**: Load building images only when visible</Check>
<Check>**Map Clustering**: Group nearby markers for better performance</Check>
<Check>**Filter Debouncing**: Delay API calls during rapid filter changes</Check>
<Check>**Response Caching**: Cache frequently accessed building data</Check>

### Bundle Size Management

- Lazy load map components only when map view is active
- Split detail page components for faster initial page load
- Optimize image formats and sizes for building cards
- Use dynamic imports for heavy dependencies

<Tip>
Consider implementing a service worker for offline building browsing capabilities in areas with poor connectivity.
</Tip>
