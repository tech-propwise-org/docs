"Off-plan" is a **filter** applied to reference data queries, not a separate module. The `MapService` (2GIS geocoding) lives in `src/modules/real-estate/map/` as a shared utility.

## Filtering logic

Off-plan status is **derived** from the project/building's dates and status fields. Pass `?type=off-plan` to `/reference/projects` or `/reference/buildings` search endpoints:

```typescript
private applyOffPlanFilter(qb: QueryBuilder<RefProject | RefBuilding>) {
  qb.andWhere({
    $or: [
      { endDate: { $gt: new Date() } },
      { endDate: null },
    ],
  });
  qb.andWhere({ completionDate: null });
  qb.andWhere({
    status: { $in: ['active', 'pending', 'under_construction', 'announced'] },
  });
  qb.andWhere({ isPublished: true });
}
```

A project/building is considered "off-plan" when:
1. `endDate` is in the future OR null (not yet completed)
2. `completionDate` is null (not yet handed over)
3. `status` is one of: active, pending, under_construction, announced
4. `isPublished` is true

## API endpoints

```
GET /reference/projects?type=off-plan          # Off-plan project search
GET /reference/projects/map                    # Map markers with coordinates
GET /reference/projects/counts                 # Aggregated counts by status
GET /reference/projects/:id                    # Enriched project detail
GET /reference/projects/:id/units/grouped      # Units grouped by bedroom category
GET /reference/units/:id                       # Single unit detail
```

## Map integration (2GIS)

- **MapService** (`src/modules/real-estate/map/map.service.ts`): Shared geocoding utility via 2GIS API
- API: `https://catalog.api.2gis.com/3.0/items/geocode?q={address}&key={apiKey}&fields=items.point`
- Environment variable: `DGIS_API_KEY`
- Frontend uses 2GIS MapGL JS API (docs: https://docs.urbi.ae)
- Map project data is fetched via `ReferenceDataService.getMapProjects()`

<Tip>
The developer portal triggers geocoding via `POST /developer-portal/projects/:id/geocode`. The resulting lat/lng is stored on the `RefProject` entity in the reference database.
</Tip>
