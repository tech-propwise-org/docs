<Info>
**Purpose:** This document describes the external XML contract defined by Bayut for syndication to both Bayut and dubizzle portals. 

For how PropWise produces this feed from internal data (field mapping, portal selection, status logic), see the Portal Syndication Specification.
</Info>

## XML Integration Overview

An XML integration is required to sync properties with Bayut and/or dubizzle in bulk from an external CRM. It allows clients to maintain their inventory on both portals by providing property information in a unified XML feed.

<Note>
- The XML feed must be exposed as a **publicly accessible URL**
- The **Bayut system pulls** (reads) this URL on a schedule to sync updates
- Listings flow **one direction only**: CRM → portals. The feed is the source of truth for what is published
</Note>

### Unified Feed Architecture

Bayut and dubizzle now share a **single combined XML feed**. The portal(s) a property appears on is controlled per-property by the `<Portals>` tag.

<Warning>
Clients with existing integrations must:
- **Retire** the full & hourly feeds previously created for dubizzle
- **Update the Bayut XML structure** to use the `<Portals>` attribute to target each portal
</Warning>

## XML Structure

The XML feed consists of two main segments:

1. **Header tags** — the document envelope
2. **Property attributes** — per-property fields

### Root Structure

The root element must be named `<Properties>` with individual properties as nested `<Property>` elements:

```xml
<Properties>
  <Property> <!-- Property 1 attributes --> </Property>
  <Property> <!-- Property 2 attributes --> </Property>
</Properties>
```

<Tip>
All textual values should be wrapped in `<![CDATA[ ... ]]>` to prevent HTML, Arabic text, ampersands, and URLs from breaking XML parsing.
</Tip>

## Property Attributes

### Core Property Details

<AccordionGroup>
<Accordion title="Identity & Status Fields">

| Tag | Description | Allowed Values |
|---|---|---|
| `<Property_Ref_No>` | Unique identifier for the property in your CRM | Must be unique - duplicates are marked **Inactive** |
| `<Permit_Number>` | Trakheesi Permit (DLD) or Madhmoun Permit (ADREC) number | Required for listing to go live |
| `<Property_Status>` | Whether property is live or deleted | `live`, `deleted` |
| `<Last_Updated>` | Last updated date from your CRM | `YYYY-MM-DD HH:mm:ss` |

</Accordion>

<Accordion title="Classification Fields">

| Tag | Description | Allowed Values |
|---|---|---|
| `<Property_purpose>` | Sale or rental purpose | `Buy`, `Rent` |
| `<Property_Type>` | Type of property | `Villa`, `Apartment`, `Office`, `Shop`, `Warehouse`, `Factory`, `Labour Camp`, `Other Commercial`, `Commercial Building`, `Residential Floor`, `Commercial Floor`, `Residential Land`, `Commercial Land`, `Townhouse`, `Residential Building`, `Hotel Apartment`, `Loft Apartment`, `Pent House` |
| `<Off_plan>` | Ready or off-plan status | `Yes`, `No` |

</Accordion>

<Accordion title="Property Specifications">

| Tag | Description | Allowed Values |
|---|---|---|
| `<Property_Size>` | Numeric area value | Number |
| `<Property_Size_Unit>` | Area measurement unit | Preferred: `SQFT` |
| `<plotArea>` | Plot area in square feet | Number |
| `<Bedrooms>` | Number of bedrooms | `-1` (studio), `1`-`10`, `10+` |
| `<Bathrooms>` | Number of bathrooms | `1`-`10` |
| `<Furnished>` | Furnishing status | `Yes`, `No`, `Partly` |

</Accordion>
</AccordionGroup>

### Portal Targeting

The `<Portals>` tag controls which portal(s) the listing appears on:

<Tabs>
<Tab title="Both Portals">
```xml
<Portals>
  <Portal><![CDATA[ Bayut ]]></Portal>
  <Portal><![CDATA[ dubizzle ]]></Portal>
</Portals>
```
</Tab>
<Tab title="Bayut Only">
```xml
<Portals>
  <Portal><![CDATA[ Bayut ]]></Portal>
</Portals>
```
</Tab>
<Tab title="dubizzle Only">
```xml
<Portals>
  <Portal><![CDATA[ dubizzle ]]></Portal>
</Portals>
```
</Tab>
</Tabs>

### Features

Use nested `<Feature>` tags for property amenities:

```xml
<Features>
  <Feature><![CDATA[ Swimming Pool ]]></Feature>
  <Feature><![CDATA[ Gym ]]></Feature>
  <Feature><![CDATA[ Parking ]]></Feature>
</Features>
```

### Pricing

<AccordionGroup>
<Accordion title="Pricing Fields">

| Tag | Description | Required When |
|---|---|---|
| `<Price>` | Sale price or rent amount | Always |
| `<Rent_Frequency>` | Rental frequency | Only when `Property_purpose` is `Rent` |

**Rent Frequency Values:** `Daily`, `Weekly`, `Monthly`, `Yearly`

</Accordion>
</AccordionGroup>

### Off-Plan Properties

<Warning>
These tags are required **only** when `<Off_plan>` is `Yes`.
</Warning>

| Tag | Description | Required For |
|---|---|---|
| `<offplanDetails_saleType>` | Sale type | All off-plan (`New`, `Resale`) |
| `<offplanDetails_dldWaiver>` | DLD waiver percentage (0-100) | Off-plan `New` only |
| `<offplanDetails_originalPrice>` | Original property price | Off-plan `Resale` only |
| `<offplanDetails_amountPaid>` | Amount already paid | Off-plan `Resale` only |

### Content Fields

| Tag | Description |
|---|---|
| `<Property_Title>` | English listing title |
| `<Property_Description>` | English description |
| `<Property_Title_AR>` | Arabic listing title |
| `<Property_Description_AR>` | Arabic description |

### Media Assets

All URLs must be publicly accessible:

<CodeGroup>
```xml Images
<Images>
  <Image><![CDATA[ https://cdn.example.com/property/123/img-1.jpg ]]></Image>
  <Image><![CDATA[ https://cdn.example.com/property/123/img-2.jpg ]]></Image>
</Images>
```

```xml Videos
<Videos>
  <Video><![CDATA[ https://www.youtube.com/watch?v=example ]]></Video>
</Videos>
```

```xml Floor Plans
<Floor_Plans>
  <Floor_Plan><![CDATA[ https://cdn.example.com/floorplan-1.pdf ]]></Floor_Plan>
</Floor_Plans>
```
</CodeGroup>

### Location Details

Example for a property in Pretoni Tower, Cluster C, Jumeirah Lakes Towers, Dubai:

```xml
<City><![CDATA[ Dubai ]]></City>
<Locality><![CDATA[ Jumeirah Lakes Towers ]]></Locality>
<Sub_Locality><![CDATA[ Cluster C ]]></Sub_Locality>
<Tower_Name><![CDATA[ Pretoni Tower ]]></Tower_Name>
```

### Contact Information

| Tag | Description |
|---|---|
| `<Listing_Agent>` | Agent name displayed on listing |
| `<Listing_Agent_Phone>` | Agent contact number |
| `<Listing_Agent_Email>` | Agent email address |

## Validation Rules

<Steps>
<Step title="Required Fields">
- `<Property_Ref_No>` must be unique across all properties
- `<Permit_Number>` is required for listing to go live
- At least one publicly accessible `<Image>` should be provided
</Step>

<Step title="Conditional Requirements">
- `<Rent_Frequency>` required only when `<Property_purpose>` is `Rent`
- Off-plan detail tags required only when `<Off_plan>` is `Yes`
- For off-plan `New`: `offplanDetails_dldWaiver` required
- For off-plan `Resale`: `offplanDetails_originalPrice` and `offplanDetails_amountPaid` required
</Step>

<Step title="Data Handling">
- Duplicate `Property_Ref_No` values result in only one live instance
- `deleted` entries remain in feed for one crawl cycle to ensure portal delisting
- All text content should be CDATA-wrapped for XML safety
</Step>
</Steps>
