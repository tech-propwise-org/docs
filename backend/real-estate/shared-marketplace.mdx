**Path:** `src/modules/real-estate/shared-marketplace/`

Cross-organization property listing and the **deal-first** negotiation flow. One of **6 specialized real estate modules** that enables property listings and deals across organizations with custom RLS policies.

## Module overview

The real estate module is structured as **six specialized modules** spanning three visibility scopes: **global** (no RLS), **org-private** (standard RLS), and **cross-org** (custom RLS).

An external **reference database** (Neon "propwise-lab") stores **all property data** — both structural (DLD registry) and enrichment (gallery, payment plans, documents, amenities, pricing, geocoding). The CRM database stores only CRM-specific data (developer portal auth, ownership links, property interests, marketplace listings, secondary unit context).

<Info>
Architecture: All modules consume property data exclusively through `ReferenceDataService`. If the reference database is swapped for an external property API, only `ReferenceDataService` needs to change.
</Info>

### Cross-database reference pattern

CRM entities that reference off-plan projects or units store **plain integer IDs** (`refProjectId`, `refUnitId`) pointing to the reference database. No cross-database foreign keys exist — this decouples the CRM from the property data source:

```typescript
// SharedListing entity referencing reference DB data:
@Property({ type: 'int', nullable: true })
@Index()
refProjectId?: number;  // ID in reference DB projects table

@Property({ type: 'int', nullable: true })
@Index()
refUnitId?: number;     // ID in reference DB units table
```

## Reference data module

**Path:** `src/modules/real-estate/reference-data/`

### Purpose

Connects to a second PostgreSQL database (Neon "propwise-lab") containing **all property data**: DLD registry data (structural) plus enrichment data (gallery, payment plans, documents, amenities, sales contacts, pricing, geocoding). Provides read endpoints for the off-plan directory and write endpoints for the developer portal.

```typescript

  contextName: 'reference',
});
```

### Entity pattern

All reference entities are mapped to the external DB tables. They have **no UUID primary keys** (use `number` PKs matching the external DB), **no organization FK**, and use **soft delete** (`deletedAt` timestamp) for enrichment entities.

```typescript
@Entity({ tableName: 'projects' })
export class RefProject {
  @PrimaryKey()
  id!: number;

  @Property({ nullable: true })
  name?: string;

  @ManyToOne(() => RefDeveloper, { fieldName: 'developer_id', nullable: true })
  developer?: RefDeveloper;

  @ManyToOne(() => RefArea, { fieldName: 'area_id' })
  area!: RefArea;

  // Enrichment fields
  @Property({ type: 'double', nullable: true })
  latitude?: number;

  @Property({ fieldName: 'price_from', type: 'numeric', nullable: true })
  priceFrom?: number;

  @Property({ fieldName: 'is_published', default: false, nullable: true })
  isPublished?: boolean;

  // Enrichment collections
  @OneToMany(() => RefProjectGalleryImage, (img) => img.project)
  gallery = new Collection<RefProjectGalleryImage>(this);

  @OneToMany(() => RefPaymentPlan, (p) => p.project)
  paymentPlans = new Collection<RefPaymentPlan>(this);
  // ... documents, amenities
}
```

## Core principle: deal-first approach

The shared marketplace implements a unique **deal-first** negotiation model that prioritizes commitment over browsing:

<Steps>
<Step title="Unit number protection">
**Unit numbers are hidden** in SharedListing browsing to prevent direct contact bypass
</Step>

<Step title="Deal commitment required">
**Buyer agent must propose a SharedDeal** before seeing sensitive unit information
</Step>

<Step title="Automated deal creation">
On acceptance, the system **auto-creates internal Deal entities** in both organizations
</Step>

<Step title="Unit number revelation">
**Unit numbers are revealed only after** SharedDeal status becomes ACCEPTED
</Step>
</Steps>

This approach ensures all cross-organization transactions go through the proper negotiation and commission tracking workflow.

<Accordion title="SharedListing">

Cross-organization property listing entity with custom RLS policies.

**Key characteristics:**
- UUID primary key with **custom RLS** (readable by all, writable by owning org)
- ManyToOne relationships: SecondaryUnit, listedByOrganization, listedByAgent

**Relationship pattern:**
SharedListing → SecondaryUnit (org-scoped inventory) → Reference DB (via `refProjectId`/`refUnitId`)

<Accordion title="SharedDeal">

Cross-organization deal negotiation entity with dual-org visibility.

**Key characteristics:**
- UUID primary key with **custom RLS** (visible to both participating orgs)
- Links both transaction sides: listingOrganization/listingAgent, buyerOrganization/buyerAgent
- Auto-created deal references: listingDeal (Deal), buyerDeal (Deal) - populated on ACCEPT
- Negotiation state: status (SharedDealStatus), proposedPrice, agreedPrice, commissionSplit (JSON)
- Unit access control: **isUnitNumberRevealed** (false until accepted), unitNumberRevealedAt
- Message history: OneToMany relationship to SharedDealMessage[]

**SharedDealStatus enum:**
- PROPOSED - Initial buyer proposal
- COUNTER_OFFERED - Seller counter-proposal
- ACCEPTED - Deal accepted, triggers auto-deal creation
- REJECTED - Deal rejected by either party
- EXPIRED - Deal expired due to timeout

```typescript
@Entity()
export class SharedDeal {
  @PrimaryKey()
  id!: string; // UUID

  @ManyToOne(() => SharedListing)
  listing!: SharedListing;

  @ManyToOne(() => Organization)
  listingOrganization!: Organization;

  @ManyToOne(() => User)
  listingAgent!: User;

  @ManyToOne(() => Organization)
  buyerOrganization!: Organization;

  @ManyToOne(() => User)
  buyerAgent!: User;

  @Enum(() => SharedDealStatus)
  status: SharedDealStatus = SharedDealStatus.PROPOSED;

  @Property({ type: 'numeric' })
  proposedPrice!: number;

  @Property({ type: 'numeric', nullable: true })
  agreedPrice?: number;

  @Property({ type: 'json', nullable: true })
  commissionSplit?: {
    buyerAgent: number;
    sellerAgent: number;
  };

  @Property({ default: false })
  isUnitNumberRevealed: boolean = false;

  @Property({ nullable: true })
  unitNumberRevealedAt?: Date;

  @Property({ nullable: true })
  expiresAt?: Date; // Deal expiry

  // Auto-created deals (populated on acceptance)
  @ManyToOne(() => Deal, { nullable: true })
  listingDeal?: Deal;

  @ManyToOne(() => Deal, { nullable: true })
  buyerDeal?: Deal;

  @Property()
  createdAt: Date = new Date();

  @Property({ onUpdate: () => new Date() })
  updatedAt: Date = new Date();

  @OneToMany(() => SharedDealMessage, (msg) => msg.sharedDeal)
  messages = new Collection<SharedDealMessage>(this);
}
```

## SharedDeal accept flow

<Warning>
This is a critical flow that creates Deal entities in both organizations and reveals the unit number. It uses `TenantContext.executeInOrg()` to operate in each organization's context.
</Warning>

<Steps>
<Step title="Update shared deal status">
```typescript
// 1. Update SharedDeal status and reveal unit information
sharedDeal.status = SharedDealStatus.ACCEPTED;
sharedDeal.agreedPrice = latestProposedPrice;
sharedDeal.isUnitNumberRevealed = true;
sharedDeal.unitNumberRevealedAt = new Date();

// Create acceptance message
const acceptMessage = this.em.create(SharedDealMessage, {
  sharedDeal,
  type: SharedDealMessageType.ACCEPT,
  content: 'Deal accepted',
  terms: { price: sharedDeal.agreedPrice },
  createdBy: acceptingUser,
  organization: acceptingUser.organization,
});
```
</Step>

<Step title="Create Deal in listing organization">
```typescript
// 2. Auto-create Deal in LISTING organization context
const listingDeal = await this.tenantContext.executeInOrg(
  sharedDeal.listingOrganization.id, 
  async (em) => {
    const deal = em.create(Deal, {
      name: `Marketplace Sale - ${listing.projectName} ${listing.unitNumber}`,
      value: sharedDeal.agreedPrice,
      stage: DealStage.NEGOTIATION,
      source: DealSource.MARKETPLACE,
      refProjectId: listing.refProjectId,
      refUnitId: listing.refUnitId,
      createdBy: sharedDeal.listingAgent,
      organization: sharedDeal.listingOrganization,
    });

    // Create buyer stakeholder
    const buyerStakeholder = em.create(EntityStakeholder, {
      deal,
      stakeholderType: StakeholderType.BUYER,
      organization: sharedDeal.buyerOrganization,
      agent: sharedDeal.buyerAgent,
    });

    await em.persistAndFlush([deal, buyerStakeholder]);
    return deal;
  }
);
```
</Step>

<Step title="Create Deal in buyer organization">
```typescript
// 3. Auto-create Deal in BUYER organization context

  sharedDeal.buyerOrganization.id,
  async (em) => {
    const deal = em.create(Deal, {
      name: `Marketplace Purchase - ${listing.projectName} ${listing.unitNumber}`,
      value: sharedDeal.agreedPrice,
      stage: DealStage.NEGOTIATION,
      source: DealSource.MARKETPLACE,
      refProjectId: listing.refProjectId,
      refUnitId: listing.refUnitId,
      createdBy: sharedDeal.buyerAgent,
      organization: sharedDeal.buyerOrganization,

    // Create seller stakeholder
    const sellerStakeholder = em.create(EntityStakeholder, {
      deal,
      stakeholderType: StakeholderType.SELLER,
      organization: sharedDeal.listingOrganization,

    await em.persistAndFlush([deal, sellerStakeholder]);
```

## RLS for cross-org entities

Custom RLS policies enable cross-organization visibility while maintaining security:

```sql
-- SharedListing: readable by all orgs, writable by owning org only
CREATE POLICY "shared_listing_read" ON "shared_listing" 
FOR SELECT USING (true);

CREATE POLICY "shared_listing_write" ON "shared_listing" 
FOR ALL USING (
  listed_by_organization_id = current_setting('app.organization_id')::uuid
);

-- SharedDeal: visible to both participating organizations
CREATE POLICY "shared_deal_access" ON "shared_deal" 
FOR ALL USING (
  listing_organization_id = current_setting('app.organization_id')::uuid
  OR buyer_organization_id = current_setting('app.organization_id')::uuid
);

-- SharedDealMessage: inherits access from parent SharedDeal
CREATE POLICY "shared_deal_message_access" ON "shared_deal_message" 
FOR ALL USING (
  EXISTS (
    SELECT 1 FROM shared_deal sd 
    WHERE sd.id = shared_deal_id 
    AND (
      sd.listing_organization_id = current_setting('app.organization_id')::uuid
      OR sd.buyer_organization_id = current_setting('app.organization_id')::uuid
    )
  )
);

-- UnitAccessRequest: standard org-scoped RLS
CREATE POLICY "unit_access_request_policy" ON "unit_access_request" 
FOR ALL USING (
  organization_id = current_setting('app.organization_id')::uuid
);

-- UnitAccessGrant: standard org-scoped RLS
CREATE POLICY "unit_access_grant_policy" ON "unit_access_grant" 
FOR ALL USING (
  organization_id = current_setting('app.organization_id')::uuid
);
```

<Info>
The custom RLS policies allow SharedListing to be globally readable (for marketplace browsing) but only writable by the listing organization. SharedDeal entities are visible to both buyer and seller organizations for transparent negotiation.
</Info>

## Service architecture

The shared marketplace integrates with the reference data system through service dependency injection:

### SharedMarketplaceService

```typescript
@Injectable()
export class SharedMarketplaceService {
  constructor(
    private readonly referenceDataService: ReferenceDataService,
    @InjectEntityManager()
    private readonly em: EntityManager,
    private readonly tenantContext: TenantContext,
    private readonly notificationService: NotificationService,
  ) {}

  async createListing(
    secondaryUnit: SecondaryUnit, 
    createDto: CreateSharedListingDto,
    listedBy: User
  ): Promise<SharedListing> {
    // Enrich with reference data for denormalization
    const projectData = await this.referenceDataService.getProject(
      secondaryUnit.refProjectId
    );
    const unitData = await this.referenceDataService.getUnit(
      secondaryUnit.refUnitId
    );

    const listing = this.em.create(SharedListing, {
      ...createDto,
      secondaryUnit,
      listedByOrganization: listedBy.organization,
      listedByAgent: listedBy,
      refProjectId: secondaryUnit.refProjectId,
      refUnitId: secondaryUnit.refUnitId,
      // Denormalized data for fast browsing
      projectName: projectData?.name,
      areaName: projectData?.area?.name,
      communityName: projectData?.community?.name,
      unitType: unitData?.unitType?.name,
      bedrooms: unitData?.bedrooms,
      areaSqft: unitData?.areaSqft,
      unitNumber: unitData?.unitNumber,
    });

    await this.em.persistAndFlush(listing);
    return listing;
  }

  async proposeDeal(
    listing: SharedListing,
    proposeDto: ProposeDealDto,
    buyerAgent: User
  ): Promise<SharedDeal> {
    const expiresAt = proposeDto.expiresInDays 
      ? new Date(Date.now() + proposeDto.expiresInDays * 24 * 60 * 60 * 1000)
      : null;

    const deal = this.em.create(SharedDeal, {
      listing,
      listingOrganization: listing.listedByOrganization,
      listingAgent: listing.listedByAgent,
      buyerOrganization: buyerAgent.organization,
      buyerAgent,
      proposedPrice: proposeDto.proposedPrice,
      commissionSplit: proposeDto.commissionSplit,
      expiresAt,
    });

    // Create initial proposal message
    const message = this.em.create(SharedDealMessage, {
      sharedDeal: deal,
      type: SharedDealMessageType.PROPOSAL,
      content: proposeDto.message || 'Initial proposal',
      terms: { 
        price: proposeDto.proposedPrice,
        commissionSplit: proposeDto.commissionSplit 
      },
      createdBy: buyerAgent,
      organization: buyerAgent.organization,
    });

    await this.em.persistAndFlush([deal, message]);
    
    // Notify listing agent
    await this.notificationService.sendDealProposed(deal);
    
    return deal;
  }

  private async createListingDeal(sharedDeal: SharedDeal): Promise<Deal> {
    return this.tenantContext.executeInOrg(
      sharedDeal.listingOrganization.id,
      async (em) => {
        const deal = em.create(Deal, {
          name: `Marketplace Sale - ${sharedDeal.listing.projectName} ${sharedDeal.listing.unitNumber}`,
          value: sharedDeal.agreedPrice,
          stage: DealStage.NEGOTIATION,
          source: DealSource.MARKETPLACE,
          refProjectId: sharedDeal.listing.refProjectId,
          refUnitId: sharedDeal.listing.refUnitId,
          createdBy: sharedDeal.listingAgent,
          organization: sharedDeal.listingOrganization,
        });

        const buyerStakeholder = em.create(EntityStakeholder, {
          deal,
          stakeholderType: StakeholderType.BUYER,
          organization: sharedDeal.buyerOrganization,
          agent: sharedDeal.buyerAgent,
        });

        await em.persistAndFlush([deal, buyerStakeholder]);
        return deal;
      }
    );
  }

  private async createBuyerDeal(sharedDeal: SharedDeal): Promise<Deal> {
    return this.tenantContext.executeInOrg(
      sharedDeal.buyerOrganization.id,
      async (em) => {
        const deal = em.create(Deal, {
          name: `Marketplace Purchase - ${sharedDeal.listing.projectName} ${sharedDeal.listing.unitNumber}`,
          value: sharedDeal.agreedPrice,
          stage: DealStage.NEGOTIATION,
          source: DealSource.MARKETPLACE,
          refProjectId: sharedDeal.listing.refProjectId,
          refUnitId: sharedDeal.listing.refUnitId,
          createdBy: sharedDeal.buyerAgent,
          organization: sharedDeal.buyerOrganization,
        });

        const sellerStakeholder = em.create(EntityStakeholder, {
          deal,
          stakeholderType: StakeholderType.SELLER,
          organization: sharedDeal.listingOrganization,
          agent: sharedDeal.listingAgent,
        });

        await em.persistAndFlush([deal, sellerStakeholder]);
        return deal;
      }
    );
  }
}
```

### Cross-module data flow

```
Developer Portal → Reference DB → SharedListing → SharedDeal → Internal Deals
     ↓                ↓              ↓             ↓            ↓
Project/Unit     Property Data   Marketplace   Cross-Org    Org-specific
Management      (via Service)    Listing      Negotiation   Deal Records
```

### Integration points

- **Inventory module**: SharedListing.secondaryUnit relationship provides org-scoped context
- **Reference data module**: `ReferenceDataService` injection for property data enrichment  
- **Core CRM**: Auto-creation of Deal entities during SharedDeal acceptance flow
- **Organizations**: Cross-org visibility through custom RLS policies
- **MapService**: Shared geocoding utility across all real estate modules
- **Notification system**: Deal proposal, acceptance, and status change notifications
- **TenantContext**: Multi-organization context switching for deal auto-creation

### Deal lifecycle integration

1. **Listing creation**: SecondaryUnit (inventory) → SharedListing (marketplace)
2. **Deal proposal**: SharedListing → SharedDeal (cross-org negotiation)
3. **Deal acceptance**: SharedDeal → Deal entities (both organizations)
4. **Unit revelation**: Deal acceptance triggers unit number visibility
5. **Commission tracking**: Commission terms flow from SharedDeal to Deal entities

<Note>
The shared marketplace acts as the bridge between organization-scoped inventory and cross-organization deal negotiation, using the reference data system as the single source of truth for all property information.
</Note>
