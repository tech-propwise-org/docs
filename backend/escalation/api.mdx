# API Endpoints

The Escalation Module provides REST API endpoints for managing escalation rules, monitoring active trackers, and accessing escalation analytics. All endpoints enforce row-level security through organization-based permissions.

## Base URL

All escalation endpoints are prefixed with `/api/escalation`.

<Note>
All requests require proper authentication and organization context. Endpoints automatically filter data based on the user's organization membership.
</Note>

## Escalation Rules

### Create Escalation Rule

<CodeGroup>

```bash curl
curl -X POST /api/escalation/rules \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer {token}" \
  -d '{
    "name": "No First Contact Alert",
    "triggerType": "NO_FIRST_CONTACT",
    "triggerConfig": {
      "thresholdMinutes": 1440
    },
    "conditions": [
      {
        "field": "temperature",
        "operator": "eq",
        "value": "hot"
      }
    ],
    "respectBusinessHours": true,
    "priority": 1,
    "tiers": [
      {
        "tierOrder": 1,
        "delayMinutes": 0,
        "actions": [
          {
            "type": "NOTIFY_AGENT",
            "message": "Lead needs first contact"
          }
        ]
      }
    ]
  }'
```

```typescript TypeScript
const response = await fetch('/api/escalation/rules', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${token}`
  },
  body: JSON.stringify({
    name: 'No First Contact Alert',
    triggerType: 'NO_FIRST_CONTACT',
    triggerConfig: {
      thresholdMinutes: 1440
    },
    conditions: [
      {
        field: 'temperature',
        operator: 'eq',
        value: 'hot'
      }
    ],
    respectBusinessHours: true,
    priority: 1,
    tiers: [
      {
        tierOrder: 1,
        delayMinutes: 0,
        actions: [
          {
            type: 'NOTIFY_AGENT',
            message: 'Lead needs first contact'
          }
        ]
      }
    ]
  })
});
```

</CodeGroup>

**Request Body Parameters:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `name` | string | Yes | Human-readable rule name |
| `triggerType` | enum | Yes | `NO_FIRST_CONTACT` or `WENT_COLD` |
| `triggerConfig` | object | Yes | Trigger configuration (see below) |
| `conditions` | array | No | Applicability filters (empty = all leads) |
| `respectBusinessHours` | boolean | No | Default: true |
| `priority` | number | No | Evaluation order |
| `tiers` | array | Yes | Escalation tier definitions |

**Trigger Configuration:**

<Tabs>
  <Tab title="NO_FIRST_CONTACT">
    ```json
    {
      "thresholdMinutes": 1440
    }
    ```
  </Tab>
  <Tab title="WENT_COLD">
    ```json
    {
      "thresholdValue": 7,
      "thresholdUnit": "days"
    }
    ```
  </Tab>
</Tabs>

**Response:**
```json
{
  "id": "uuid",
  "name": "No First Contact Alert",
  "isActive": true,
  "triggerType": "NO_FIRST_CONTACT",
  "createdAt": "2024-01-01T00:00:00Z",
  "tiers": [...]
}
```


```typescript TypeScript
  }

```

</CodeGroup>

**Query Parameters:**

| Parameter | Type | Description |
|-----------|------|-------------|
| `isActive` | boolean | Filter by active status |
| `priority` | number | Filter by priority level |
| `triggerType` | enum | Filter by trigger type |

### Update Escalation Rule

<CodeGroup>

```bash curl
curl -X PUT /api/escalation/rules/{ruleId} \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer {token}" \
  -d '{
    "name": "Updated Rule Name",
    "isActive": false
  }'
```

```typescript TypeScript
const response = await fetch(`/api/escalation/rules/${ruleId}`, {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${token}`
  },
  body: JSON.stringify({
    name: 'Updated Rule Name',
    isActive: false
  })
});
```

</CodeGroup>

<Warning>
Deactivating a rule will cancel all active escalation trackers associated with it.
</Warning>

### Delete Escalation Rule

<CodeGroup>

```bash curl
  -H "Authorization: Bearer {token}"
```

```typescript TypeScript
const response = await fetch(`/api/escalation/rules/${ruleId}`, {
  method: 'DELETE',
  headers: {
    'Authorization': `Bearer ${token}`
  }
```

</CodeGroup>

<Warning>
Deleting a rule will cancel all active escalation trackers and perform a soft delete on the rule record.
</Warning>

## Escalation Trackers

### Get Active Trackers

<CodeGroup>

```bash curl
curl -X GET "/api/escalation/trackers?status=ACTIVE&leadId={leadId}" \
  -H "Authorization: Bearer {token}"
```

```typescript TypeScript
const response = await fetch('/api/escalation/trackers?status=ACTIVE&leadId=${leadId}', {
  headers: {
    'Authorization': `Bearer ${token}`
  }
});
```

</CodeGroup>

**Query Parameters:**

| Parameter | Type | Description |
|-----------|------|-------------|
| `status` | enum | `ACTIVE`, `RESOLVED`, `CANCELLED` |
| `leadId` | uuid | Filter by specific lead |
| `ruleId` | uuid | Filter by escalation rule |
| `limit` | number | Page size (default: 50) |
| `offset` | number | Pagination offset |

**Response:**
```json
{
  "trackers": [
    {
      "id": "uuid",
      "leadId": "uuid",
      "escalationRuleId": "uuid",

      "nextEscalationAt": "2024-01-01T01:00:00Z",
      "status": "ACTIVE",
      "history": [
        {
          "tierOrder": 1,
          "executedAt": "2024-01-01T00:00:00Z",
          "actions": ["NOTIFY_AGENT"]
        }
      ]
    }
  ],
  "total": 1,
  "hasMore": false
}
```

### Resolve Escalation Tracker

<CodeGroup>

```bash curl
curl -X POST /api/escalation/trackers/{trackerId}/resolve \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer {token}" \
  -d '{
    "reason": "Issue resolved by manual intervention"
  }'
```

```typescript TypeScript
const response = await fetch(`/api/escalation/trackers/${trackerId}/resolve`, {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${token}`
  },
  body: JSON.stringify({
    reason: 'Issue resolved by manual intervention'
  })
});
```

</CodeGroup>

**Request Body:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `reason` | string | No | Optional resolution reason |

<Info>
Manual resolution sets `resolvedBy` to `MANUAL` and stops further escalation processing for this tracker.
</Info>

## Tier Actions Reference

<AccordionGroup>
  <Accordion title="NOTIFY_AGENT">
    Sends notification to the lead's currently assigned agent.

    **Parameters:**
    - `message` (optional): Custom notification message

    **Auto-resolution:** None - purely informational
  </Accordion>

  <Accordion title="NOTIFY_ADMIN">
    Notifies all organization administrators.

    **Parameters:**
    - `message` (optional): Custom notification message

    **Behavior:** Self-resolving - queries users with `system.admin` permission. Skipped if no admins found.
  </Accordion>

  <Accordion title="NOTIFY_TEAM_LEAD">
    Notifies all team leaders of the lead's assigned team.

    **Parameters:**
    - `message` (optional): Custom notification message

    **Behavior:** Self-resolving - queries team members with `team.admin` permission. Skipped if no team or team leaders.
  </Accordion>

  <Accordion title="REDISTRIBUTE">
    Removes current stakeholders and reassigns the lead using the distribution engine.

    **Parameters:** None

    **Important:** Must be placed in the last tier only. If redistribution results in `ASSIGNED` status, the tracker is automatically resolved with `resolvedBy = REDISTRIBUTED`.
  </Accordion>

  <Accordion title="CHANGE_TEMPERATURE">
    Directly updates the lead's temperature without validation guards.

    **Parameters:**
    - `temperature`: `"hot"`, `"warm"`, or `"cold"`
  </Accordion>

  <Accordion title="ADD_TAG">
    Appends tags to the lead's tag list, deduplicating existing entries.

    **Parameters:**
    - `tagIds`: Array of tag UUID strings
  </Accordion>
</AccordionGroup>

## Permissions

<Info>
All escalation endpoints require authenticated users with appropriate organization membership. Individual permissions are checked based on the specific action:

- **Rule management**: Requires administrative permissions
- **Tracker viewing**: Available to all organization members
- **Manual resolution**: Requires lead access permissions
- **Analytics**: Administrative access required
</Info>

## Rate Limits

<Note>
Standard API rate limits apply to all escalation endpoints. Analytics endpoints have additional restrictions due to complex queries:

- Metrics endpoint: 10 requests per minute
- Action history endpoint: 20 requests per minute
- Other endpoints: Standard rate limits apply
</Note>
