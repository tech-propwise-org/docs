---
title: "Junction tables"
description: "Explicit junction tables (UserOrgRole, TeamMembershipRole) for multi-org role assignments with RLS integration."
---

The RBAC system uses explicit junction tables instead of simple pivot tables to support multi-organization context, metadata tracking, and RLS.

## Why explicit junction tables?

1. **Multi-org context** — A user can have different roles in different organizations. Simple pivot tables cannot store organization context.
2. **Metadata tracking** — Track who assigned the role (`assigned_by_id`), when it was assigned, and when removed.
3. **RLS compatibility** — Need `organization_id` for database-level filtering via RLS policies.
4. **Audit trail** — Track role assignment history and identify who made changes.

---

## UserOrgRole

Manages User ↔ Role assignments in Organization context.

```typescript
UserOrgRole
├── id: uuid (primary key)
├── user_id: uuid → User
├── role_id: uuid → Role (system role with organization = null)
├── organization_id: uuid → Organization (REQUIRED for RLS)
├── assigned_by_id?: uuid → User (who assigned this role)
├── Timestamps:
│   ├── created_at: timestamp
│   ├── updated_at: timestamp
│   └── is_deleted: boolean (soft delete)

Indexes:
├── (user_id, role_id, organization_id) [unique, where: is_deleted = false]
├── (user_id, organization_id)
└── (organization_id, role_id)
```

```typescript typescript
const userOrgRoles = await em.find(
  UserOrgRole,
  { user: userId, organization: organizationId, isDeleted: false },
  { populate: ['role', 'role.permissions'] },
);
const roles = userOrgRoles.map((uor) => uor.role);

const userOrgRole = em.create(UserOrgRole, {
  user, role, organization, assignedBy: currentUser,
});
await em.persistAndFlush(userOrgRole);
```

## TeamMembershipRole

Manages TeamMembership ↔ Role assignments.

```typescript
TeamMembershipRole
├── id: uuid (primary key)
├── team_membership_id: uuid → TeamMembership
├── role_id: uuid → Role
├── organization_id: uuid → Organization (REQUIRED for RLS, denormalized from TeamMembership)
├── assigned_by_id?: uuid → User (who assigned this role)
├── Timestamps:
│   ├── created_at: timestamp
│   ├── updated_at: timestamp
│   └── is_deleted: boolean (soft delete)

Indexes:
└── (team_membership_id, role_id) [unique]
```

```typescript typescript
const membership = await em.findOne(
  TeamMembership,
  { user: userId, team: teamId },
  { populate: ['teamMembershipRoles.role', 'teamMembershipRoles.role.permissions'] },
);
const roles = membership.teamMembershipRoles.getItems().map((tmr) => tmr.role);

const teamMembershipRole = em.create(TeamMembershipRole, {
  teamMembership: membership, role,
  organization: membership.organization, assignedBy: currentUser,
});
await em.persistAndFlush(teamMembershipRole);
```

## Organization field

Both junction tables include `organization_id` for RLS compatibility.

| Junction | How `organization_id` is set |
| --- | --- |
| **UserOrgRole** | Explicitly set when assigning role. Allows different roles per org. |
| **TeamMembershipRole** | Inherited from `TeamMembership.organization`. Denormalized for RLS. |

## RLS integration

### RLS-enabled tables

| Table | RLS Policy | Notes |
| --- | --- | --- |
| `user` | `organization_isolation_user` | Users visible only in current org context |
| `organization` | No RLS | Users can see all orgs they belong to |
| `role` | Special handling | System roles (org=NULL) visible to all, custom roles filtered by org |
| `permission` | No RLS | Global permission registry |
| `team` | `organization_isolation_team` | Teams filtered by organization |
| `team_membership` | `organization_isolation_team_membership` | Memberships filtered by organization |
| `user_org_role` | `organization_isolation_user_org_role` | Filtered by organization |
| `team_membership_role` | `organization_isolation_team_membership_role` | Filtered by organization |
| `invitation` | `organization_isolation_invitation` | Invitations filtered by organization |

### RLS policies for junction tables

```sql sql
ALTER TABLE "user_org_role" ENABLE ROW LEVEL SECURITY;
CREATE POLICY "organization_isolation_user_org_role" ON "user_org_role"
  USING (
    organization_id = current_setting('app.organization_id')::uuid
    OR current_setting('app.bypass_rls', true) = 'true'
  );
ALTER TABLE "user_org_role" FORCE ROW LEVEL SECURITY;

ALTER TABLE "team_membership_role" ENABLE ROW LEVEL SECURITY;
CREATE POLICY "organization_isolation_team_membership_role" ON "team_membership_role"
  USING (
    organization_id = current_setting('app.organization_id')::uuid
    OR current_setting('app.bypass_rls', true) = 'true'
  );
ALTER TABLE "team_membership_role" FORCE ROW LEVEL SECURITY;
```

### Role table special handling

The `Role` table requires special handling because system roles have `organization = NULL`:

```sql sql
-- Role RLS Policy (if needed)
-- Note: Currently Role table does NOT have RLS enabled
-- If we enable it, we need this logic:

DROP POLICY IF EXISTS "organization_isolation_role" ON "role";
CREATE POLICY "organization_isolation_role" ON "role"
  USING (
    organization_id = current_setting('app.organization_id')::uuid
    OR organization_id IS NULL  -- System roles visible to all
    OR current_setting('app.bypass_rls', true) = 'true'
  );
```

**Current status:** Role table does NOT have RLS enabled. System roles (`organization = NULL`) and custom roles are both fetched without RLS filtering. Application logic handles visibility.

### Bypassing RLS

```typescript typescript
import { BypassRLS } from '@decorators/rls.decorator';

@Injectable()
export class SystemService {
  @BypassRLS()
  async performSystemOperation() {
    const allUserOrgRoles = await this.em.find(UserOrgRole, {});
  }
}
```

### RLS context setting

The organization context is set automatically by `TenantMiddleware`:

```typescript typescript
@Injectable()
export class TenantMiddleware implements NestMiddleware {
  async use(req: Request, res: Response, next: NextFunction) {
    const organizationId = req.user?.organizationId;
    if (organizationId) {
      await this.em.execute(`SET LOCAL app.organization_id = '${organizationId}'`);
    }
    next();
  }
}
```

async function isAdmin(userId: string, organizationId: string): Promise<boolean> {
  const adminRoles = await em.find(
    UserOrgRole,
    { user: userId, organization: organizationId, isDeleted: false },
    { populate: ['role'] },
  );
  return adminRoles.some(
    (uor) => ['Owner', 'Admin'].includes(uor.role.name) && uor.role.scope === RoleScope.ORG,
  );
}

<Accordion title="Assign role to user">
```typescript typescript
async function assignOrgRole(
  userId: string, roleId: string, organizationId: string, assignedById: string,
): Promise<void> {
  const user = await em.findOneOrFail(User, { id: userId });
  const role = await em.findOneOrFail(Role, { id: roleId });
  const organization = await em.findOneOrFail(Organization, { id: organizationId });
  const assignedBy = await em.findOneOrFail(User, { id: assignedById });

  const existing = await em.findOne(UserOrgRole, {
    user: userId, role: roleId, organization: organizationId, isDeleted: false,
  });
  if (existing) throw new BadRequestException('Role already assigned to user');

  const userOrgRole = em.create(UserOrgRole, { user, role, organization, assignedBy });
  await em.persistAndFlush(userOrgRole);
}
```
</Accordion>
<Accordion title="Remove role from user">
```typescript typescript
async function removeOrgRole(
  userId: string, roleId: string, organizationId: string,
): Promise<void> {
  const userOrgRole = await em.findOne(UserOrgRole, {
    user: userId, role: roleId, organization: organizationId, isDeleted: false,
  });
  if (!userOrgRole) throw new NotFoundException('Role assignment not found');

  userOrgRole.isDeleted = true;
  userOrgRole.updatedAt = new Date();
  await em.persistAndFlush(userOrgRole);
}
```
