Teams organize users within an organization. Each team has its own roles and can restrict which org-scoped permissions are available to team members.

## Team entity

```typescript
Team
├── id: uuid
├── name: string
├── description?: string
├── organization_id: uuid → Organization
├── Relationships:
│   ├── memberships → TeamMembership[]
│   ├── roles → Role[] (team-specific roles)
│   └── availablePermissions → Permission[] (via team_available_permissions)
└── Timestamps: created_at, updated_at, is_deleted

Indexes:
└── (name, organization_id, is_deleted) [unique]
```

- Teams belong to exactly one organization
- Team names must be unique within organization
- `availablePermissions` defines which ORG permissions team roles can use

---

## Team membership

```typescript
TeamMembership
├── id: uuid
├── user_id: uuid → User
├── team_id: uuid → Team
├── organization_id: uuid → Organization
├── joined_at: timestamp
├── Relationships:
│   └── teamMembershipRoles → TeamMembershipRole[] (roles in this team)
└── Timestamps: created_at, updated_at, is_deleted

Indexes:
└── (user_id, team_id) [unique, where: is_deleted = false]
```

- User can be member of multiple teams
- One membership per user per team (enforced only among non-deleted records)
- `organization_id` inherited from team (for RLS)
- Access roles via `teamMembershipRoles` collection

## Available permissions

Team roles can have ORG-scoped permissions, but only if those permissions are in the team's `availablePermissions`:

```typescript typescript
// team_available_permissions junction table
Table team_available_permissions {
  team_id uuid [ref: > team.id]
  permission_id uuid [ref: > permission.id]
}

// When assigning ORG permission to team role, validate:
async function validateTeamRolePermissions(
  teamId: string, permissionIds: string[]
): Promise<void> {
  const team = await em.findOne(Team, { id: teamId }, { populate: ['availablePermissions'] });
  const availablePermissionIds = team.availablePermissions.getItems().map(p => p.id);

  for (const permissionId of permissionIds) {
    const permission = await em.findOne(Permission, { id: permissionId });
    if (permission.scope === PermissionScope.ORG) {
      if (!availablePermissionIds.includes(permissionId)) {
        throw new Error(`Permission ${permission.key} not available for team roles`);
      }
    }
  }
}
```

## Business rules

| Rule | Description |
| --- | --- |
| **One membership per team** | User can only have one membership per team |
| **Organization match** | `TeamMembership.organization` must match `Team.organization` |
| **Default role** | New team members get "Agent" role by default |
| **Team Leader minimum** | Each team must have at least one Team Leader |

<Info>
Many team endpoints use a `teamCheck` config in `@CheckAccess` that allows team leaders (with `team.admin` permission) to manage their own teams without needing org-level `system.admin`.
</Info>
