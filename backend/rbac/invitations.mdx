The invitation system enables secure user onboarding with predefined access controls, supporting both new user registration and existing user integration across multiple organizations within the RBAC framework.

<Info>
The invitation system provides secure token-based user onboarding with comprehensive role and team assignment capabilities, supporting both new user registration and existing user integration across multiple organizations.
</Info>

## Invitation Entity

The core entity that manages the invitation lifecycle and role assignments.

```typescript
Invitation
├── id: uuid
├── email: string
├── status: InvitationStatus (pending, completed, expired, cancelled)
├── token: string (secure, 64-character hex string)
├── expires_at?: timestamp (default: 7 days from creation)
├── invited_by_id: uuid → User (ManyToOne)
├── accepted_by_id?: uuid → User (ManyToOne, nullable)
├── accepted_at?: timestamp
├── organization_id: uuid → Organization (ManyToOne, required)
├── org_role_assignments: jsonb ({roleId, roleName}[])
├── team_id?: uuid → Team (ManyToOne, nullable, primary team assignment)
├── team_role_assignments: jsonb ({roleId, roleName, teamId, teamName, roleIds}[])
├── welcome_message?: string
├── Relationships:
│   ├── organization → Organization (ManyToOne)
│   ├── invitedBy → User (ManyToOne)
│   ├── acceptedBy? → User (ManyToOne, nullable)
│   └── team? → Team (ManyToOne, nullable)
└── Timestamps: created_at, updated_at

Indexes:
├── (invited_by_id)
├── (organization_id)
├── (token) [unique]
├── (status)
└── (email, organization_id) [unique, where: status = 'pending']
```

## Invitation Status Flow

The invitation lifecycle follows a state machine pattern with clear transitions and validation rules.

<Tabs>
<Tab title="pending">
**Initial State**: Invitation created and email sent  
**Transitions to**: completed, expired, cancelled  
**Actions Available**: Accept invitation, cancel invitation, extend expiry

```typescript
// Automatic expiry check
if (invitation.expiresAt && new Date() > invitation.expiresAt) {
  invitation.status = InvitationStatus.EXPIRED;
}
```
</Tab>

<Tab title="completed">
**Final State**: User successfully accepted invitation  
**Transitions from**: pending  
**Actions Available**: View completion details only

```typescript
// Completion tracking
invitation.status = InvitationStatus.COMPLETED;
invitation.acceptedBy = user;
invitation.acceptedAt = new Date();
```
</Tab>

<Tab title="expired">
**Final State**: Invitation expired without acceptance  
**Transitions from**: pending (automatic after expires_at)  
**Actions Available**: Create new invitation if needed

<Warning>
Expired invitations cannot be accepted. A new invitation must be created for the same user.
</Warning>
</Tab>

<Tab title="cancelled">
**Final State**: Invitation manually cancelled by admin  
**Transitions from**: pending  
**Actions Available**: Create new invitation if needed

```typescript
// Manual cancellation
invitation.status = InvitationStatus.CANCELLED;
await this.em.persistAndFlush(invitation);
```
</Tab>
</Tabs>

## Invitation Workflow

The complete invitation process from creation to acceptance with RBAC assignment.

<Steps>
<Step title="Admin creates invitation">
Administrator creates invitation specifying email, organizational roles, and team assignments with comprehensive validation of all role assignments and team memberships.

```typescript
const invitation = await invitationService.createInvitation({
  email: 'user@example.com',
  organizationId: 'org-123',
  orgRoleAssignments: [
    { roleId: 'basic-role-id', roleName: 'Basic' }
  ],
  teamRoleAssignments: [
    {
      teamId: 'team-123',
      teamName: 'Sales Team',
      roleIds: ['agent-role-id'],
      roleNames: ['Agent']
    }
  ],
  welcomeMessage: 'Welcome to our organization!',
  expiresAt: new Date(Date.now() + 7 * 24 * 60 * 60 * 1000) // 7 days
});
```
</Step>

<Step title="Email sent with secure link">
System automatically sends email with secure 64-character token link, different templates for new vs. existing users, and comprehensive invitation context.

```typescript
// Email contains secure invitation link
const invitationUrl = `${baseUrl}/accept-invitation/${invitation.token}`;

// Email template varies based on user existence
const template = await userExists ? 'invitation-existing-user' : 'invitation-new-user';

await this.emailService.sendEmail({
  to: invitation.email,
  template,
  context: {
    inviterName: invitation.invitedBy.firstName,
    organizationName: invitation.organization.name,
    invitationUrl,
    welcomeMessage: invitation.welcomeMessage,
    expiresAt: invitation.expiresAt
  }
});
```
</Step>

<Step title="User accepts invitation">
Two acceptance flows depending on whether the user already exists in the system:

**New User Flow** (Public endpoint):
```typescript
// Creates new user account and applies RBAC
const newUser = await invitationService.acceptInvitation(token, {
  firstName: 'John',
  lastName: 'Doe',
  password: 'securePassword',
  phoneNumber: '+1234567890'
});
```

**Existing User Flow** (Authenticated endpoint):
```typescript
// Adds existing user to new organization
await invitationService.acceptInvitationExisting(token, currentUser);
```
</Step>

<Step title="RBAC assignments applied">
System applies pre-configured roles and team assignments using RLS bypass context with comprehensive validation and audit logging.

```typescript
// Applied within RLS bypass context
await this.tenantContext.executeWithBypass(async (bypassEm) => {
  await this.applyInvitationRbacAssignments(bypassEm, user, invitation);
});
```
</Step>
</Steps>

## RBAC Assignment Implementation

The invitation acceptance process applies RBAC assignments through a series of database operations that must bypass RLS policies.

<Warning>
Invitation acceptance runs on public/identity-token endpoints that have **no RLS organization context**. All RBAC tables have RLS policies that deny access when `app.organization_id` is not set. The `applyInvitationRbacAssignments` method **must receive the bypassed EntityManager** from the calling `executeWithBypass` context.
</Warning>

### Assignment Steps

<Steps>
<Step title="Organization Membership">
Add user to organization if not already a member:

```typescript
async addUserToOrganization(
  bypassEm: EntityManager,
  user: User,
  invitation: Invitation
): Promise<void> {
  // Check existing membership
  const existingMembership = await bypassEm.findOne(OrganizationUser, {
    user: user.id,
    organization: invitation.organization.id
  });

  if (!existingMembership) {
    const orgUser = bypassEm.create(OrganizationUser, {
      user,
      organization: invitation.organization,
      addedBy: invitation.invitedBy,
      joinedAt: new Date()
    });
    await bypassEm.persist(orgUser);
  }
}
```
</Step>

<Step title="Organizational Role Assignments">
Apply organization-level roles from invitation:

```typescript
async applyOrgRoleAssignments(
  bypassEm: EntityManager,
  user: User,
  invitation: Invitation
): Promise<void> {
  for (const roleAssignment of invitation.orgRoleAssignments) {
    // Validate role exists and is assignable
    const role = await bypassEm.findOne(Role, {
      id: roleAssignment.roleId,
      scope: RoleScope.ORG
    });
    
    if (!role) {
      throw new BadRequestException(`Role ${roleAssignment.roleId} not found`);
    }

    const userOrgRole = bypassEm.create(UserOrgRole, {
      user,
      role,
      organization: invitation.organization,
      assignedBy: invitation.invitedBy
    });
    await bypassEm.persist(userOrgRole);
  }
}
```
</Step>

<Step title="Team Membership & Role Assignments">
Apply team memberships and team-level roles:

```typescript
async applyTeamRoleAssignments(
  bypassEm: EntityManager,
  user: User,
  invitation: Invitation
): Promise<void> {
  for (const teamAssignment of invitation.teamRoleAssignments) {
    // Verify team exists and belongs to organization
    const team = await bypassEm.findOne(Team, {
      id: teamAssignment.teamId,
      organization: invitation.organization.id
    });
    
    if (!team) {
      throw new BadRequestException(`Team ${teamAssignment.teamId} not found`);
    }

    // Create team membership
    const teamMembership = bypassEm.create(TeamMembership, {
      user,
      team,
      organization: invitation.organization,
      addedBy: invitation.invitedBy
    });
    await bypassEm.persist(teamMembership);
    
    // Assign team roles
    for (const roleId of teamAssignment.roleIds) {
      const role = await bypassEm.findOne(Role, {
        id: roleId,
        scope: RoleScope.TEAM
      });
      
      if (!role) {
        throw new BadRequestException(`Team role ${roleId} not found`);
      }

      const teamMembershipRole = bypassEm.create(TeamMembershipRole, {
        teamMembership,
        role,
        organization: invitation.organization,
        assignedBy: invitation.invitedBy
      });
      await bypassEm.persist(teamMembershipRole);
    }
  }
}
```
</Step>

<Step title="Invitation Completion">
Mark invitation as completed and set acceptance metadata:

```typescript
async completeInvitation(
  bypassEm: EntityManager,
  invitation: Invitation,
  user: User
): Promise<void> {
  invitation.status = InvitationStatus.COMPLETED;
  invitation.acceptedBy = user;
  invitation.acceptedAt = new Date();
  
  await bypassEm.persistAndFlush(invitation);
}
```
</Step>
</Steps>

## Token-Based Security

The invitation system uses cryptographically secure tokens for authorization and validation.

### Token Generation

```typescript
class InvitationService {
  private generateSecureToken(): string {
    // Generate 64-character hex string (256 bits of entropy)
    return crypto.randomBytes(32).toString('hex');
  }
  
  async createInvitation(dto: CreateInvitationDto): Promise<Invitation> {
    const token = this.generateSecureToken();
    
    // Ensure token uniqueness (extremely unlikely to collide)
    const existingInvitation = await this.em.findOne(Invitation, { token });
    if (existingInvitation) {
      throw new ConflictException('Token collision detected, please retry');
    }
    
    // Create invitation with secure token
    const invitation = this.em.create(Invitation, {
      ...dto,
      token,
      status: InvitationStatus.PENDING,
      expiresAt: dto.expiresAt || new Date(Date.now() + 7 * 24 * 60 * 60 * 1000)
    });
    
    return invitation;
  }
}
```

### Token Validation

```typescript
async validateInvitationToken(token: string): Promise<Invitation> {
  const invitation = await this.em.findOne(
    Invitation,
    { token },
    {
      populate: [
        'organization:name,id',
        'invitedBy:firstName,lastName',
        'team:name,id'
      ]
    }
  );

  if (!invitation) {
    throw new NotFoundException('Invitation not found');
  }

  if (invitation.status !== InvitationStatus.PENDING) {
    throw new BadRequestException(
      `Invitation is ${invitation.status} and cannot be accepted`
    );
  }

  if (invitation.expiresAt && new Date() > invitation.expiresAt) {
    // Automatically mark as expired
    invitation.status = InvitationStatus.EXPIRED;
    await this.em.persistAndFlush(invitation);
    throw new BadRequestException('Invitation has expired');
  }

  return invitation;
}
```

The invitation system fully supports users participating in multiple organizations with different roles.

### Cross-Organization User Scenarios

<Tabs>

```typescript
// User doesn't exist, creates new account
const newUser = await this.authService.register({
  email: invitation.email,

  lastName: userData.lastName,
  password: userData.password
});

// Apply RBAC assignments for first organization

```
</Tab>

<Tab title="Existing User - New Organization">
Existing user receives invitation to join additional organization:

```typescript
// User exists, add to new organization
const existingUser = await this.em.findOne(User, { email: invitation.email });

if (existingUser) {
  // Check if already member of this organization
  const existingMembership = await this.em.findOne(OrganizationUser, {
    user: existingUser.id,
    organization: invitation.organization.id
  });
  
  if (existingMembership) {
    throw new ConflictException('User is already a member of this organization');
  }
  
  // Apply RBAC assignments for new organization
  await this.applyInvitationRbacAssignments(bypassEm, existingUser, invitation);
}
```
</Tab>

<Tab title="Organization Switching">
Users can switch between organizations they belong to:

```typescript
class UserService {
  async switchOrganization(userId: string, organizationId: string): Promise<void> {
    // Verify user is member of target organization
    const membership = await this.em.findOne(OrganizationUser, {
      user: userId,
      organization: organizationId
    });
    
    if (!membership) {
      throw new ForbiddenException('User is not a member of this organization');
    }
    
    // Update selected organization
    const user = await this.em.findOneOrFail(User, userId);
    user.selectedOrganizationId = organizationId;
    await this.em.persistAndFlush(user);
  }
}
```
</Tab>

## Business Rules

### Invitation Business Rules

<AccordionGroup>
<Accordion title="Email Uniqueness">
- Only one pending invitation per email per organization
- Users can have multiple pending invitations across different organizations
- Completed or expired invitations don't block new invitations
</Accordion>

<Accordion title="Expiration Policy">
- Invitations expire after 7 days by default (configurable)
- Expired invitations cannot be accepted or modified
- System provides cleanup job for expired invitations
- Admins can extend invitation expiry before expiration
</Accordion>

<Accordion title="Role Assignment Validation">
- All role assignments validated at invitation creation
- Roles must exist and be appropriate for their context (ORG/TEAM scope)
- Team role assignments require valid team membership creation
- ORG-scoped permissions in team roles must be in team available permissions
</Accordion>

<Accordion title="Permission Inheritance">
- Team members inherit team-scoped permissions from team roles
- ORG-scoped permissions in team roles require team availability
- Organizational roles provide organization-wide permissions
- Permission conflicts resolved through union (most permissive)
</Accordion>

<Accordion title="Cancellation Rules">
- Only pending invitations can be cancelled
- Cancellation immediately prevents acceptance
- Cancelled invitations cannot be reactivated
- Admin who created invitation or organization owners can cancel
</Accordion>
</AccordionGroup>

## Query Patterns

### Common Invitation Queries

<CodeGroup>

```typescript Get pending invitations for organization
async getPendingInvitations(
  organizationId: string,
  options: PaginationOptions
): Promise<{ invitations: Invitation[], total: number }> {
  const [invitations, total] = await this.em.findAndCount(
    Invitation,
    { 
      organization: organizationId,
      status: InvitationStatus.PENDING
    },
    {
      populate: [
        'invitedBy:firstName,lastName,email',
        'team:name,id',
        'organization:name'
      ],
      orderBy: { createdAt: 'DESC' },
      limit: options.limit,
      offset: options.offset
    }
  );
  
  return { invitations, total };
}
```

```typescript Get invitation by token with validation
async getInvitationByToken(token: string): Promise<Invitation | null> {
  const invitation = await this.em.findOne(
    Invitation,
    { token },
    {
      populate: [
        'organization:name,id,email',
        'invitedBy:firstName,lastName,email',
        'team:name,id,description'
      ]
    }
  );
  
  // Auto-expire if past expiration date
  if (invitation?.expiresAt && new Date() > invitation.expiresAt && 
      invitation.status === InvitationStatus.PENDING) {
    invitation.status = InvitationStatus.EXPIRED;
    await this.em.persistAndFlush(invitation);
  }
  
  return invitation;
}
```

```typescript Get user invitation history
async getUserInvitationHistory(
  email: string,
  options: PaginationOptions

  const [invitations, total] = await this.em.findAndCount(
    Invitation,

      ],

```

```typescript Cleanup expired invitations
async cleanupExpiredInvitations(): Promise<number> {
  const expiredInvitations = await this.em.find(Invitation, {
    status: InvitationStatus.PENDING,
    expiresAt: { $lt: new Date() }
  });
  
  for (const invitation of expiredInvitations) {
    invitation.status = InvitationStatus.EXPIRED;
  }
  
  await this.em.flush();
  return expiredInvitations.length;
}
```

### Accepting Invitations

```typescript
// Public endpoint for new users
@Post('accept/:token')
async acceptInvitation(
  @Param('token') token: string,
  @Body() userData: AcceptInvitationDto
): Promise<{ user: User; token: string }> {
  const user = await this.invitationService.acceptInvitation(token, userData);
  const authToken = await this.authService.generateTokens(user);
  return { user, token: authToken.accessToken };
}

// Authenticated endpoint for existing users
@Post('accept-existing/:token')
@UseGuards(JwtAuthGuard)
async acceptInvitationExisting(
  @Param('token') token: string,
  @CurrentUser() user: User
): Promise<{ success: boolean }> {
  await this.invitationService.acceptInvitationExisting(token, user);
  return { success: true };
}
```
