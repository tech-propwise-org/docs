The invitation system operates within a comprehensive Role-Based Access Control (RBAC) architecture that supports multi-tenant organizations with sophisticated permission management.

</Step>

<Step title="Team-level permissions">

## Invitation System Architecture

The invitation system follows these core principles:

1. **Secure Token-Based Invitations**: Uses cryptographically secure 64-character hex tokens
2. **Pre-Configured RBAC Assignments**: Invitations contain complete role and team assignments
3. **Multi-Organization Support**: Users can be invited to multiple organizations with different roles
4. **Dual User Flows**: Supports both new user registration and existing user integration
5. **RLS Bypass Integration**: Invitation acceptance bypasses Row-Level Security for RBAC assignments

## Invitation Entity

The core entity that manages the invitation lifecycle and role assignments within the RBAC framework.

```typescript
Invitation
├── id: uuid
├── email: string
├── status: InvitationStatus (pending, completed, expired, cancelled)
├── token: string (secure, 64-character hex string)
├── expires_at?: timestamp (default: 7 days from creation)
├── invited_by_id: uuid → User (ManyToOne)
├── accepted_by_id?: uuid → User (ManyToOne, nullable)
├── accepted_at?: timestamp
├── organization_id: uuid → Organization (ManyToOne, required)
├── org_role_assignments: jsonb ({roleId, roleName}[])
├── team_id?: uuid → Team (ManyToOne, nullable, primary team assignment)
├── team_role_assignments: jsonb ({roleId, roleName, teamId, teamName, roleIds}[])
├── welcome_message?: string
├── Relationships:
│   ├── organization → Organization (ManyToOne)
│   ├── invitedBy → User (ManyToOne)
│   ├── acceptedBy? → User (ManyToOne, nullable)
│   └── team? → Team (ManyToOne, nullable)
└── Timestamps: created_at, updated_at

Indexes:
├── (invited_by_id)
├── (organization_id)
├── (token) [unique]
├── (status)
└── (email, organization_id) [unique, where: status = 'pending']
```

## Invitation Status Flow

The invitation lifecycle follows a state machine pattern with clear transitions and validation rules.

<Tabs>
<Tab title="pending">
**Initial State**: Invitation created and email sent  
**Transitions to**: completed, expired, cancelled  
**Actions Available**: Accept invitation, cancel invitation, extend expiry

```typescript
// Automatic expiry check
if (invitation.expiresAt && new Date() > invitation.expiresAt) {
  invitation.status = InvitationStatus.EXPIRED;
}
```
</Tab>

<Tab title="completed">
**Final State**: User successfully accepted invitation  
**Transitions from**: pending  
**Actions Available**: View completion details only

```typescript
// Completion tracking
invitation.status = InvitationStatus.COMPLETED;
invitation.acceptedBy = user;
invitation.acceptedAt = new Date();
```
</Tab>

<Tab title="expired">
**Final State**: Invitation expired without acceptance  
**Transitions from**: pending (automatic after expires_at)  
**Actions Available**: Create new invitation if needed

<Warning>
Expired invitations cannot be accepted. A new invitation must be created for the same user.
</Warning>
</Tab>

<Tab title="cancelled">
**Final State**: Invitation manually cancelled by admin  
**Transitions from**: pending  
**Actions Available**: Create new invitation if needed

```typescript
// Manual cancellation
invitation.status = InvitationStatus.CANCELLED;
await this.em.persistAndFlush(invitation);
```
</Tab>
</Tabs>

## RBAC Assignment Implementation

The invitation acceptance process applies RBAC assignments through the explicit junction table architecture that supports multi-organization user membership.

<Warning>
Invitation acceptance runs on public/identity-token endpoints that have **no RLS organization context**. All RBAC tables have RLS policies that deny access when `app.organization_id` is not set. The `applyInvitationRbacAssignments` method **must receive the bypassed EntityManager** from the calling `executeWithBypass` context.
</Warning>

### Junction Table Architecture

The RBAC system uses explicit junction tables to support multi-organization membership:

- **`UserOrgRole`**: User ↔ Role assignments in Organization context
- **`TeamMembershipRole`**: TeamMembership ↔ Role assignments
- **`OrganizationUser`**: User ↔ Organization membership

### Assignment Steps

<Steps>
<Step title="Organization Membership">
Add user to organization via `OrganizationUser` junction table:

```typescript
async addUserToOrganization(
  bypassEm: EntityManager,
  user: User,
  invitation: Invitation
): Promise<void> {
  // Check existing membership
  const existingMembership = await bypassEm.findOne(OrganizationUser, {
    user: user.id,
    organization: invitation.organization.id
  });

  if (!existingMembership) {
    const orgUser = bypassEm.create(OrganizationUser, {
      user,
      organization: invitation.organization,
      addedBy: invitation.invitedBy,
      joinedAt: new Date()
    });
    await bypassEm.persist(orgUser);
  }
}
```
</Step>

<Step title="Organizational Role Assignments">
Apply organization-level roles via `UserOrgRole` junction table:

```typescript
async applyOrgRoleAssignments(
  bypassEm: EntityManager,
  user: User,
  invitation: Invitation
): Promise<void> {
  for (const roleAssignment of invitation.orgRoleAssignments) {
    // Validate role exists and is assignable
    const role = await bypassEm.findOne(Role, {
      id: roleAssignment.roleId,
      scope: RoleScope.ORG,
      $or: [
        { organization: invitation.organization.id }, // Custom role
        { organization: null }                        // System role
      ]
    });
    
    if (!role) {
      throw new BadRequestException(`Role ${roleAssignment.roleId} not found`);
    }

    // Check for existing assignment
    const existingAssignment = await bypassEm.findOne(UserOrgRole, {
      user: user.id,
      role: role.id,
      organization: invitation.organization.id
    });

    if (!existingAssignment) {
      const userOrgRole = bypassEm.create(UserOrgRole, {
        user,
        role,
        organization: invitation.organization,
        assignedBy: invitation.invitedBy
      });
      await bypassEm.persist(userOrgRole);
    }
  }
}
```
</Step>

<Step title="Team Membership & Role Assignments">
Apply team memberships and team-level roles via `TeamMembership` and `TeamMembershipRole`:

```typescript
async applyTeamRoleAssignments(
  bypassEm: EntityManager,
  user: User,
  invitation: Invitation
): Promise<void> {
  for (const teamAssignment of invitation.teamRoleAssignments) {
    // Verify team exists and belongs to organization
    const team = await bypassEm.findOne(Team, {
      id: teamAssignment.teamId,
      organization: invitation.organization.id
    });
    
    if (!team) {
      throw new BadRequestException(`Team ${teamAssignment.teamId} not found`);
    }

    // Create team membership
    let teamMembership = await bypassEm.findOne(TeamMembership, {
      user: user.id,
      team: team.id
    });

    if (!teamMembership) {
      teamMembership = bypassEm.create(TeamMembership, {
        user,
        team,
        organization: invitation.organization,
        addedBy: invitation.invitedBy
      });
      await bypassEm.persist(teamMembership);
    }
    
    // Assign team roles
    for (const roleId of teamAssignment.roleIds) {
      const role = await bypassEm.findOne(Role, {
        id: roleId,
        scope: RoleScope.TEAM,
        $or: [
          { organization: invitation.organization.id }, // Custom team role
          { organization: null }                        // System team role
        ]
      });
      
      if (!role) {
        throw new BadRequestException(`Team role ${roleId} not found`);
      }

      // Check for existing team role assignment
      const existingTeamRole = await bypassEm.findOne(TeamMembershipRole, {
        teamMembership: teamMembership.id,
        role: role.id
      });

      if (!existingTeamRole) {
        const teamMembershipRole = bypassEm.create(TeamMembershipRole, {
          teamMembership,
          role,
          organization: invitation.organization,
          assignedBy: invitation.invitedBy
        });
        await bypassEm.persist(teamMembershipRole);
      }
    }
  }
}
```
</Step>

<Step title="Invitation Completion">
Mark invitation as completed and set acceptance metadata:

```typescript

  user: User
```

```typescript
interface UserPermissions {
  organizationPermissions: Permission[];
  teamPermissions: TeamPermissionMapping[];
  allPermissions: string[]; // Union of all permission keys
}

// Permission resolution service
async resolveUserPermissions(userId: string, orgId: string): Promise<UserPermissions> {
  const [orgPerms, teamPerms] = await Promise.all([
    this.getOrganizationPermissions(userId, orgId),
    this.getTeamPermissions(userId, orgId)
  ]);

  return {
    organizationPermissions: orgPerms,
    teamPermissions: teamPerms,
    allPermissions: [...new Set([
      ...orgPerms.map(p => p.key),
      ...teamPerms.flatMap(tp => tp.permissions.map(p => p.key))
    ])]
  };
}
```

## Token-Based Security

The invitation system uses cryptographically secure tokens for authorization and validation.

### Token Generation

```typescript
class InvitationService {
  private generateSecureToken(): string {
    // Generate 64-character hex string (256 bits of entropy)
    return crypto.randomBytes(32).toString('hex');
  }
  
  async createInvitation(dto: CreateInvitationDto): Promise<Invitation> {
    const token = this.generateSecureToken();
    
    // Ensure token uniqueness (extremely unlikely to collide)
    const existingInvitation = await this.em.findOne(Invitation, { token });
    if (existingInvitation) {
      throw new ConflictException('Token collision detected, please retry');
    }
    
    // Create invitation with secure token
    const invitation = this.em.create(Invitation, {
      ...dto,
      token,
      status: InvitationStatus.PENDING,
      expiresAt: dto.expiresAt || new Date(Date.now() + 7 * 24 * 60 * 60 * 1000)
    });
    
    return invitation;
  }
}
```

### Token Validation

```typescript
async validateInvitationToken(token: string): Promise<Invitation> {
  const invitation = await this.em.findOne(
    Invitation,
    { token },
    {
      populate: [
        'organization:name,id',
        'invitedBy:firstName,lastName',
        'team:name,id'
      ]
    }
  );

  if (!invitation) {
    throw new NotFoundException('Invitation not found');
  }

  if (invitation.status !== InvitationStatus.PENDING) {
    throw new BadRequestException(
      `Invitation is ${invitation.status} and cannot be accepted`
    );
  }

  if (invitation.expiresAt && new Date() > invitation.expiresAt) {
    // Automatically mark as expired
    invitation.status = InvitationStatus.EXPIRED;
    await this.em.persistAndFlush(invitation);
    throw new BadRequestException('Invitation has expired');
  }

  return invitation;
}
```

## Business Rules

### Invitation Business Rules

<AccordionGroup>
<Accordion title="Email Uniqueness">
- Only one pending invitation per email per organization
- Users can have multiple pending invitations across different organizations
- Completed or expired invitations don't block new invitations
- Enforced via unique index on `(email, organization_id)` where `status = 'pending'`
</Accordion>

<Accordion title="Expiration Policy">
- Invitations expire after 7 days by default (configurable)
- Expired invitations cannot be accepted or modified
- System provides cleanup job for expired invitations
- Admins can extend invitation expiry before expiration
</Accordion>

<Accordion title="Role Assignment Validation">
- All role assignments validated at invitation creation
- Roles must exist and be appropriate for their context (ORG/TEAM scope)
- System roles (organization = NULL) available to all organizations
- Custom roles only available within their organization
- Team role assignments require valid team membership creation
</Accordion>

<Accordion title="Permission Inheritance">
- Team members inherit team-scoped permissions from team roles
- ORG-scoped permissions in team roles require team availability
- Organizational roles provide organization-wide permissions
- Permission conflicts resolved through union (most permissive)
</Accordion>

<Accordion title="RBAC Integration Rules">
- Invitation RBAC assignments bypass RLS during acceptance
- All assignments create explicit junction table records
- Multi-organization support via separate role assignments per org
- User can have different roles in different organizations
</Accordion>

<Accordion title="Cancellation Rules">
- Only pending invitations can be cancelled
- Cancellation immediately prevents acceptance
- Cancelled invitations cannot be reactivated
- Admin who created invitation or organization owners can cancel
</Accordion>
</AccordionGroup>

### InvitationService

Core business logic for invitation management with comprehensive RBAC integration:

```typescript
@Injectable()
export class InvitationService {
  constructor(
    @InjectRepository(Invitation) private readonly em: EntityManager,
    private readonly authService: AuthService,
    private readonly emailService: EmailService,
    private readonly tenantContext: TenantContext
  ) {}

  async createInvitation(dto: CreateInvitationDto): Promise<Invitation> {
    // Validate role assignments against RBAC system
    await this.validateRoleAssignments(
      dto.organizationId, 
      dto.orgRoleAssignments, 
      dto.teamRoleAssignments
    );

    const token = this.generateSecureToken();
    const invitation = this.em.create(Invitation, {
      ...dto,
      token,
      status: InvitationStatus.PENDING,
      expiresAt: dto.expiresAt || new Date(Date.now() + 7 * 24 * 60 * 60 * 1000)
    });

    await this.em.persistAndFlush(invitation);
    await this.sendInvitationEmail(invitation);
    
    return invitation;
  }

  async acceptInvitation(token: string, userData: AcceptInvitationDto): Promise<User> {
    const invitation = await this.validateInvitationToken(token);
    
    // Create new user account
    const user = await this.authService.register({
      email: invitation.email,
      ...userData
    });

    // Apply RBAC assignments with RLS bypass
    await this.tenantContext.executeWithBypass(async (bypassEm) => {
      await this.applyInvitationRbacAssignments(bypassEm, user, invitation);
    });

    return user;
  }

  async acceptInvitationExisting(token: string, user: User): Promise<Invitation> {
    const invitation = await this.validateInvitationToken(token);
    
    if (invitation.email !== user.email) {
      throw new BadRequestException('Invitation email does not match user email');
    }

    // Apply RBAC assignments for existing user
    await this.tenantContext.executeWithBypass(async (bypassEm) => {
      await this.applyInvitationRbacAssignments(bypassEm, user, invitation);
    });

    return invitation;
  }
}
```

```typescript
  @ApiOperation({ summary: 'Accept invitation (new user)' })
  async acceptInvitation(
    @Param('token') token: string,
    @Body() userData: AcceptInvitationDto
  ): Promise<{ user: User; token: string; permissions: string[] }> {
    const user = await this.invitationService.acceptInvitation(token, userData);
    const authToken = await this.authService.generateTokens(user);
    
    // Get user's permissions in the new organization
    const invitation = await this.invitationService.getInvitationByToken(token);
    const permissions = await this.rbacService.getUserPermissions(
      user.id,
      invitation.organization.id
    );
    
    return { 
      user, 
      token: authToken.accessToken,
      permissions: permissions.allPermissions
    };
  }

    // Get updated permissions in the new organization

      user.id,
      invitation.organization.id
```

```typescript Validate role assignments for invitation
async validateRoleAssignments(
  organizationId: string,
  orgRoleAssignments: OrgRoleAssignment[],
  teamRoleAssignments: TeamRoleAssignment[]
): Promise<void> {
  // Validate org roles
  for (const roleAssignment of orgRoleAssignments) {
    const role = await this.em.findOne(Role, {
      id: roleAssignment.roleId,
      scope: RoleScope.ORG,
      $or: [
        { organization: organizationId }, // Custom role
        { organization: null }           // System role
      ]
    });
    
    if (!role) {
      throw new BadRequestException(`Invalid org role: ${roleAssignment.roleId}`);
    }
  }
  
  // Validate team roles and teams
  for (const teamAssignment of teamRoleAssignments) {
    const team = await this.em.findOne(Team, {
      id: teamAssignment.teamId,
      organization: organizationId
    });
    
    if (!team) {
      throw new BadRequestException(`Invalid team: ${teamAssignment.teamId}`);
    }
    
    for (const roleId of teamAssignment.roleIds) {
      const role = await this.em.findOne(Role, {
        id: roleId,
        scope: RoleScope.TEAM,
        $or: [
          { organization: organizationId }, // Custom team role
          { organization: null }           // System team role
        ]
      });
      
      if (!role) {
        throw new BadRequestException(`Invalid team role: ${roleId}`);
      }
    }
  }
}
```

```typescript Get user permissions after invitation acceptance
async getUserPermissionsAfterInvitation(
  userId: string,
  organizationId: string
): Promise<UserPermissions> {
  // Get organization permissions
  const orgPermissions = await this.em.createQueryBuilder(Permission, 'p')
    .innerJoin('p.roles', 'r')
    .innerJoin('r.userOrgRoles', 'uor')
    .where('uor.user = :userId', { userId })
    .andWhere('uor.organization = :orgId', { orgId: organizationId })
    .getMany();

  // Get team permissions
  const teamPermissions = await this.em.createQueryBuilder(Permission, 'p')
    .innerJoin('p.roles', 'r')
    .innerJoin('r.teamMembershipRoles', 'tmr')
    .innerJoin('tmr.teamMembership', 'tm')
    .innerJoin('tm.team', 't')
    .where('tm.user = :userId', { userId })
    .andWhere('tm.organization = :orgId', { orgId: organizationId })
    .getMany();

  return {
    organizationPermissions: orgPermissions,
    teamPermissions: this.groupPermissionsByTeam(teamPermissions),
    allPermissions: [...new Set([
      ...orgPermissions.map(p => p.key),
      ...teamPermissions.map(p => p.key)
    ])]
  };
}
```

## RLS Integration

The invitation system operates within Row-Level Security (RLS) constraints and requires careful integration patterns:

### RLS Bypass for Invitation Acceptance

```typescript
class InvitationService {
  async acceptInvitation(token: string, userData: AcceptInvitationDto): Promise<User> {
    // Validate invitation (public access, no RLS context)
    const invitation = await this.validateInvitationToken(token);
    
    // Create user (public registration)
    const user = await this.authService.register({
      email: invitation.email,
      ...userData
    });
    
    // Apply RBAC assignments with RLS bypass
    await this.tenantContext.executeWithBypass(async (bypassEm) => {
      await this.applyInvitationRbacAssignments(bypassEm, user, invitation);
    });
    
    return user;
  }

  private async applyInvitationRbacAssignments(
    bypassEm: EntityManager,
    user: User,
    invitation: Invitation
  ): Promise<void> {
    // All RBAC operations use bypassed EntityManager
    // This ensures RLS policies don't block the assignments
    
    await this.addUserToOrganization(bypassEm, user, invitation);
    await this.applyOrgRoleAssignments(bypassEm, user, invitation);
    await this.applyTeamRoleAssignments(bypassEm, user, invitation);
    await this.completeInvitation(bypassEm, invitation, user);
    
    await bypassEm.flush();
  }
}
```
