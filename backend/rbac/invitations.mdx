The invitation system enables secure user onboarding with predefined access controls, supporting both new user registration and existing user integration across multiple organizations within the complete RBAC framework.

<Info>
The invitation system provides secure token-based user onboarding with comprehensive role and team assignment capabilities, supporting both new user registration and existing user integration across multiple organizations.
</Info>

<Warning>
Invitation acceptance runs on public/identity-token endpoints that have **no RLS organization context**. All RBAC tables have RLS policies that deny access when `app.organization_id` is not set. The invitation system **must use RLS bypass context** for all RBAC assignments.
</Warning>

## Core invitation entity

The invitation system manages the complete user onboarding lifecycle through the core `Invitation` entity:

```typescript
Invitation
├── id: uuid
├── email: string
├── status: InvitationStatus (pending, completed, expired, cancelled)
├── token: string (secure, 64-character hex string)
├── expires_at?: timestamp (default: 7 days from creation)
├── invited_by_id: uuid → User (ManyToOne)
├── accepted_by_id?: uuid → User (ManyToOne, nullable)
├── accepted_at?: timestamp
├── organization_id: uuid → Organization (ManyToOne, required)
├── org_role_assignments: jsonb ({roleId, roleName}[])
├── team_id?: uuid → Team (ManyToOne, nullable, primary team assignment)
├── team_role_assignments: jsonb ({roleId, roleName, teamId, teamName, roleIds}[])
├── welcome_message?: string
├── Relationships:
│   ├── organization → Organization (ManyToOne)
│   ├── invitedBy → User (ManyToOne)
│   ├── acceptedBy? → User (ManyToOne, nullable)
│   └── team? → Team (ManyToOne, nullable)
└── Timestamps: created_at, updated_at

Indexes:
├── (invited_by_id)
├── (organization_id)
├── (token) [unique]
├── (status)
└── (email, organization_id) [unique, where: status = 'pending']
```

## Invitation status flow

The invitation lifecycle is managed through a well-defined status progression:

<Tabs>
<Tab title="pending">
**Initial state** when invitation is created and email sent:
- **Transitions to**: completed, expired, cancelled
- **Actions available**: Accept invitation, cancel invitation, extend expiry

```typescript
if (invitation.expiresAt && new Date() > invitation.expiresAt) {
  invitation.status = InvitationStatus.EXPIRED;
}
```
</Tab>

<Tab title="completed">
**Final state** when user successfully accepts invitation:
- **Transitions from**: pending
- **Actions available**: View completion details only

```typescript
invitation.status = InvitationStatus.COMPLETED;
invitation.acceptedBy = user;
invitation.acceptedAt = new Date();
```
</Tab>

<Tab title="expired">
**Final state** when invitation expires without acceptance:
- **Transitions from**: pending (automatic after expires_at)
- **Actions available**: Create new invitation if needed

<Warning>
Expired invitations cannot be accepted. A new invitation must be created.
</Warning>
</Tab>

<Tab title="cancelled">
**Final state** when invitation is manually cancelled:
- **Transitions from**: pending
- **Actions available**: Create new invitation if needed

```typescript

```
</Tab>

## RBAC assignment application

The core logic for applying invitation-based role assignments with proper RBAC integration:

```typescript
private async applyInvitationRbacAssignments(
  bypassEm: EntityManager,
  user: User,
  invitation: Invitation
): Promise<void> {
  // 1. Add user to organization
  await this.addUserToOrganization(bypassEm, user, invitation);
  
  // 2. Apply organizational role assignments
  for (const roleAssignment of invitation.orgRoleAssignments) {
    const userOrgRole = bypassEm.create(UserOrgRole, {
      user,
      role: { id: roleAssignment.roleId },
      organization: invitation.organization,
      assignedBy: invitation.invitedBy
    });
    await bypassEm.persist(userOrgRole);
  }
  
  // 3. Apply team role assignments
  for (const teamAssignment of invitation.teamRoleAssignments) {
    // Create team membership
    const teamMembership = bypassEm.create(TeamMembership, {
      user,
      team: { id: teamAssignment.teamId },
      organization: invitation.organization,
      addedBy: invitation.invitedBy,
      joinedAt: new Date()
    });
    await bypassEm.persist(teamMembership);

    // Apply team roles
    for (const roleId of teamAssignment.roleIds) {
      const teamMembershipRole = bypassEm.create(TeamMembershipRole, {
        teamMembership,
        role: { id: roleId },
        organization: invitation.organization,
        assignedBy: invitation.invitedBy
      });
      await bypassEm.persist(teamMembershipRole);
    }
  }
  
  // 4. Complete invitation
  invitation.status = InvitationStatus.COMPLETED;
  invitation.acceptedBy = user;
  invitation.acceptedAt = new Date();
  
  await bypassEm.flush();
}

private async addUserToOrganization(
  bypassEm: EntityManager,
  user: User,
  invitation: Invitation
): Promise<void> {
  // Check if user is already a member
  const existingMembership = await bypassEm.findOne(OrganizationUser, {
    user,
    organization: invitation.organization
  });

  if (!existingMembership) {
    // Generate unique public ID for this organization
    const publicId = await this.generateUserPublicId(
      bypassEm, 
      invitation.organization.id
    );
    
    // Create organization membership
    const orgUser = bypassEm.create(OrganizationUser, {
      user,
      organization: invitation.organization,
      publicId,
      fallbackAvatarColor: this.generateFallbackAvatarColor(),
      addedBy: invitation.invitedBy,
      joinedAt: new Date()
    });
    await bypassEm.persist(orgUser);
  }
  
  // Update user's selected organization if not set
  if (!user.selectedOrganizationId) {
    user.selectedOrganizationId = invitation.organization.id;
    await bypassEm.persist(user);
  }
}
```

## Token security and validation

The invitation system implements comprehensive token security measures:

```typescript
private async validateInvitationToken(token: string): Promise<Invitation> {
  // Find invitation by token
  const invitation = await this.em.findOne(
    Invitation,
    { token },
    { populate: ['organization', 'invitedBy', 'team'] }
  );

  if (!invitation) {
    throw new BadRequestException('Invalid invitation token');
  }

  // Check status
  if (invitation.status !== InvitationStatus.PENDING) {
    throw new BadRequestException('Invitation is no longer valid');
  }

  // Check expiry
  if (invitation.expiresAt && new Date() > invitation.expiresAt) {
    invitation.status = InvitationStatus.EXPIRED;
    await this.em.persistAndFlush(invitation);
    throw new BadRequestException('Invitation has expired');
  }

  return invitation;
}

private generateSecureToken(): string {
  // 64-character hex string (256 bits of entropy)
  return crypto.randomBytes(32).toString('hex');
}

private async isTokenUnique(token: string): Promise<boolean> {
  const existing = await this.em.findOne(Invitation, { token });
  return !existing;
}

private async generateUniqueToken(): Promise<string> {
  let token: string;
  let attempts = 0;
  const maxAttempts = 10;

  do {
    token = this.generateSecureToken();
    attempts++;
    
    if (attempts > maxAttempts) {
      throw new Error('Failed to generate unique token after maximum attempts');
    }
  } while (!(await this.isTokenUnique(token)));

  return token;
}
```

<Note>
**Token Security Features**:
- 256 bits of entropy using crypto.randomBytes(32)
- Globally unique tokens enforced by database constraint
- Automatic expiry validation during token checks
- Single-use tokens that cannot be reused after acceptance
- Retry mechanism for token generation to handle collisions
</Note>

## RLS bypass requirements

The invitation system requires special handling of Row-Level Security (RLS) policies:

<Warning>
**Critical RLS Considerations**:

1. **Public endpoint context**: Invitation acceptance endpoints have no organization context
2. **RLS policy blocking**: All RBAC tables deny access when `app.organization_id` is NULL
3. **Bypass requirement**: RBAC assignments **must** use bypass context during invitation acceptance
4. **Security implications**: Bypass context removes multi-tenant isolation temporarily
</Warning>

### RLS bypass implementation

```typescript
class InvitationService {
  async acceptInvitation(token: string, userData: AcceptInvitationDto): Promise<User> {
    // Public endpoint - no RLS context
    const invitation = await this.validateInvitationToken(token);
    
    const user = await this.authService.register({
      email: invitation.email,
      ...userData
    });
    
    // Apply RBAC assignments with RLS bypass
    await this.tenantContext.executeWithBypass(async (bypassEm) => {
      await this.applyInvitationRbacAssignments(bypassEm, user, invitation);
    });
    
    return user;
  }
  
  // Helper method to execute queries with proper bypass context
  private async withBypass<T>(operation: (em: EntityManager) => Promise<T>): Promise<T> {
    return this.tenantContext.executeWithBypass(operation);
  }
}

// Tenant context service implementation
@Injectable()
export class TenantContext {
  constructor(@Inject(EntityManager) private readonly em: EntityManager) {}

  async executeWithBypass<T>(operation: (em: EntityManager) => Promise<T>): Promise<T> {
    const bypassEm = this.em.fork();
    
    // Disable RLS for this entity manager instance
    await bypassEm.getConnection().execute('SET row_security = off');
    
    try {
      const result = await operation(bypassEm);
      await bypassEm.flush();
      return result;
    } finally {
      // Re-enable RLS
      await bypassEm.getConnection().execute('SET row_security = on');
    }
  }
}
```

### RLS policy implications

```sql
-- Example RLS policy on RBAC tables
CREATE POLICY user_org_roles_tenant_isolation ON user_org_roles
FOR ALL TO authenticated
USING (organization_id = current_setting('app.organization_id')::uuid);

-- During invitation acceptance (no org context):
-- current_setting('app.organization_id') = NULL
-- Policy evaluates to: organization_id = NULL (fails for all records)
-- Result: Access denied to all RBAC assignments

-- Solution: Use bypass context with elevated privileges
```

<Accordion title="Get invitation with complete role details">
```sql
SELECT i.*,
       o.name as organization_name,
       u.first_name || ' ' || u.last_name as invited_by_name,
       t.name as primary_team_name,
       COALESCE(
         json_agg(DISTINCT jsonb_build_object(
           'id', r_org.id,
           'name', r_org.name,
           'scope', r_org.scope,
           'isSystem', r_org.organization_id IS NULL
         )) FILTER (WHERE r_org.id IS NOT NULL), 
         '[]'::json
       ) as org_roles,
       COALESCE(
         json_agg(DISTINCT jsonb_build_object(
           'teamId', team_info.team_id,
           'teamName', team_info.team_name,
           'roles', team_info.roles
         )) FILTER (WHERE team_info.team_id IS NOT NULL),
         '[]'::json
       ) as team_assignments
FROM invitations i
INNER JOIN organizations o ON i.organization_id = o.id
INNER JOIN users u ON i.invited_by_id = u.id
LEFT JOIN teams t ON i.team_id = t.id
LEFT JOIN jsonb_array_elements(i.org_role_assignments) ora(role_assignment) ON true
LEFT JOIN roles r_org ON r_org.id = (ora.role_assignment->>'roleId')::uuid
LEFT JOIN LATERAL (
  SELECT tra.team_assignment->>'teamId' as team_id,
         tra.team_assignment->>'teamName' as team_name,
         json_agg(jsonb_build_object(
           'id', r_team.id,
           'name', r_team.name,
           'scope', r_team.scope
         )) as roles
  FROM jsonb_array_elements(i.team_role_assignments) tra(team_assignment)
  LEFT JOIN jsonb_array_elements_text(tra.team_assignment->'roleIds') role_id ON true
  LEFT JOIN roles r_team ON r_team.id = role_id::uuid
  GROUP BY tra.team_assignment->>'teamId', tra.team_assignment->>'teamName'
) team_info ON true
WHERE i.token = $1
GROUP BY i.id, o.name, u.first_name, u.last_name, t.name;
```
</Accordion>

LEFT JOIN user_org_roles uor ON uor.user_id = ou.user_id 
  AND uor.organization_id = ou.organization_id
WHERE ou.user_id = $1
  AND ou.organization_id = $2
GROUP BY ou.organization_id, o.name;

@Injectable()
export class InvitationService {
  constructor(
    @InjectRepository(Invitation) private readonly em: EntityManager,
    private readonly authService: AuthService,
    private readonly emailService: EmailService,
    private readonly tenantContext: TenantContext,
    private readonly rbacService: RbacService,
    private readonly auditService: AuditService
  ) {}

  async createInvitation(dto: CreateInvitationDto): Promise<Invitation> {
    // Validate role assignments
    await this.validateRoleAssignments(
      dto.organizationId, 
      dto.orgRoleAssignments, 
      dto.teamRoleAssignments
    );

    // Check for existing pending invitation
    const existingInvitation = await this.em.findOne(Invitation, {
      email: dto.email,
      organizationId: dto.organizationId,
      status: InvitationStatus.PENDING
    });

    if (existingInvitation) {
      throw new ConflictException('Pending invitation already exists for this email');
    }

    // Generate secure token and create invitation
    const token = await this.generateUniqueToken();
    const invitation = this.em.create(Invitation, {
      ...dto,
      token,
      status: InvitationStatus.PENDING,
      expiresAt: dto.expiresAt || new Date(Date.now() + 7 * 24 * 60 * 60 * 1000)
    });

    await this.em.persistAndFlush(invitation);
    
    // Send invitation email
    try {
      await this.sendInvitationEmail(invitation);
    } catch (error) {
      // Log email failure but don't fail invitation creation
      console.error('Failed to send invitation email:', error);
      
      // Optionally update invitation status to indicate email failure
      invitation.emailSent = false;
      await this.em.persistAndFlush(invitation);
    }
    
    return invitation;
  }

  async acceptInvitation(token: string, userData: AcceptInvitationDto): Promise<User> {
    const invitation = await this.validateInvitationToken(token);
    
    // Check if user already exists
    const existingUser = await this.authService.findUserByEmail(invitation.email);
    if (existingUser) {
      throw new ConflictException('User already exists. Use existing user acceptance flow.');
    }
    
    // Create new user
    const user = await this.authService.register({
      email: invitation.email,
      firstName: userData.firstName,
      lastName: userData.lastName,
      password: userData.password,
      phoneNumber: userData.phoneNumber,
      languages: userData.languages
    });

    // Apply RBAC assignments with bypass context
    await this.tenantContext.executeWithBypass(async (bypassEm) => {
      await this.applyInvitationRbacAssignments(bypassEm, user, invitation);
    });

    // Audit log
    await this.auditService.logInvitationAcceptance({
      invitationId: invitation.id,
      userId: user.id,
      organizationId: invitation.organizationId,
      timestamp: new Date()
    });

    return user;
  }

  async acceptInvitationExisting(token: string, user: User): Promise<Invitation> {
    const invitation = await this.validateInvitationToken(token);
    
    if (invitation.email !== user.email) {
      throw new BadRequestException('Email mismatch between invitation and authenticated user');
    }

    // Check if user is already a member of the organization
    const existingMembership = await this.em.findOne(OrganizationUser, {
      user,
      organization: invitation.organization
    });

    // Apply RBAC assignments
    await this.tenantContext.executeWithBypass(async (bypassEm) => {
      await this.applyInvitationRbacAssignments(bypassEm, user, invitation);
    });

    // Audit log
    await this.auditService.logInvitationAcceptance({
      invitationId: invitation.id,
      userId: user.id,
      organizationId: invitation.organizationId,
      wasExistingUser: true,
      timestamp: new Date()
    });

    return invitation;
  }

  private async validateRoleAssignments(
    organizationId: string,
    orgRoleAssignments: OrgRoleAssignment[],
    teamRoleAssignments: TeamRoleAssignment[]
  ): Promise<void> {
    // Validate organizational roles
    const orgRoleIds = orgRoleAssignments.map(ra => ra.roleId);
    if (orgRoleIds.length > 0) {
      const validOrgRoles = await this.em.find(Role, {
        id: { $in: orgRoleIds },
        scope: RoleScope.ORG,
        $or: [
          { organization: organizationId }, // Custom role
          { organization: null }           // System role
        ],
        isDeleted: false
      });
      
      if (validOrgRoles.length !== orgRoleIds.length) {
        const invalidIds = orgRoleIds.filter(id => 
          !validOrgRoles.find(role => role.id === id)
        );
        throw new BadRequestException(`Invalid org roles: ${invalidIds.join(', ')}`);
      }
    }
    
    // Validate team roles and teams
    for (const teamAssignment of teamRoleAssignments) {
      // Validate team exists
      const team = await this.em.findOne(Team, {
        id: teamAssignment.teamId,
        organization: organizationId,
        isDeleted: false
      });
      
      if (!team) {
        throw new BadRequestException(`Invalid team: ${teamAssignment.teamId}`);
      }
      
      // Validate team roles
      const validTeamRoles = await this.em.find(Role, {
        id: { $in: teamAssignment.roleIds },
        scope: RoleScope.TEAM,
        $or: [
          { organization: organizationId },
          { organization: null }
        ],
        isDeleted: false
      });
      
      if (validTeamRoles.length !== teamAssignment.roleIds.length) {
        const invalidIds = teamAssignment.roleIds.filter(id =>
          !validTeamRoles.find(role => role.id === id)
        );
        throw new BadRequestException(`Invalid team roles: ${invalidIds.join(', ')}`);
      }
    }
  }

  private async sendInvitationEmail(invitation: Invitation): Promise<void> {
    const template = await this.emailService.getTemplate('invitation');
    
    const emailData = {
      to: invitation.email,
      subject: `Invitation to join ${invitation.organization.name}`,
      template: 'invitation',
      context: {
        organizationName: invitation.organization.name,
        invitedByName: `${invitation.invitedBy.firstName} ${invitation.invitedBy.lastName}`,
        welcomeMessage: invitation.welcomeMessage,
        acceptUrl: `${process.env.FRONTEND_URL}/invitations/accept/${invitation.token}`,
        expiresAt: invitation.expiresAt,
        orgRoles: invitation.orgRoleAssignments,
        teamAssignments: invitation.teamRoleAssignments
      }
    };

    await this.emailService.send(emailData);
  }

  async cancelInvitation(invitationId: string, cancelledById: string): Promise<void> {
    const invitation = await this.em.findOne(Invitation, {
      id: invitationId,
      status: InvitationStatus.PENDING
    });

    if (!invitation) {
      throw new NotFoundException('Pending invitation not found');
    }

    invitation.status = InvitationStatus.CANCELLED;
    invitation.cancelledBy = { id: cancelledById } as User;
    invitation.cancelledAt = new Date();

    await this.em.persistAndFlush(invitation);

    // Audit log
    await this.auditService.logInvitationCancellation({
      invitationId,
      cancelledById,
      organizationId: invitation.organizationId,
      timestamp: new Date()
    });
  }

  async cleanupExpiredInvitations(): Promise<number> {
    const expiredInvitations = await this.em.find(Invitation, {
      status: InvitationStatus.PENDING,
      expiresAt: { $lte: new Date() }
    });

    for (const invitation of expiredInvitations) {
      invitation.status = InvitationStatus.EXPIRED;
    }

    if (expiredInvitations.length > 0) {
      await this.em.flush();
    }

    return expiredInvitations.length;
  }

  async resendInvitation(invitationId: string, requestedById: string): Promise<void> {
    const invitation = await this.em.findOne(
      Invitation,
      { id: invitationId, status: InvitationStatus.PENDING },
      { populate: ['organization', 'invitedBy'] }
    );

    if (!invitation) {
      throw new NotFoundException('Pending invitation not found');
    }

    // Check if invitation has expired
    if (invitation.expiresAt && new Date() > invitation.expiresAt) {
      throw new BadRequestException('Cannot resend expired invitation');
    }

    // Generate new token for security
    invitation.token = await this.generateUniqueToken();
    invitation.resentAt = new Date();
    invitation.resentBy = { id: requestedById } as User;

    await this.em.persistAndFlush(invitation);
    await this.sendInvitationEmail(invitation);

    // Audit log
    await this.auditService.logInvitationResend({
      invitationId,
      resentById: requestedById,
      organizationId: invitation.organizationId,
      timestamp: new Date()
    });
  }
}

async acceptInvitation(token: string, userData: AcceptInvitationDto): Promise<User> {
  try {
    const invitation = await this.validateInvitationToken(token);
    
    // Check various error conditions
    if (invitation.status === InvitationStatus.COMPLETED) {
      throw new InvitationAlreadyAcceptedException();
    }
    
    if (invitation.expiresAt && new Date() > invitation.expiresAt) {
      throw new InvitationExpiredException();
    }
    
    const existingUser = await this.authService.findUserByEmail(invitation.email);
    if (existingUser && !existingUser.isDeleted) {
      throw new ConflictException('User already exists. Use existing user flow.');
    }
    
    // Proceed with acceptance logic...
    
  } catch (error) {
    // Log the error with context
    this.logger.error('Failed to accept invitation', {
      token: token.substring(0, 8) + '...',
      email: userData.email,
      error: error.message,
      stack: error.stack
    });
    
    // Re-throw known exceptions
    if (error instanceof HttpException) {
      throw error;
    }
    
    // Wrap unexpected errors
    throw new InternalServerErrorException('Failed to process invitation acceptance');
  }
}
