The invitation system allows admins to onboard new or existing users into an organization with pre-configured org roles and team assignments.

## Invitation entity

```typescript
Invitation
├── id: uuid
├── email: string
├── status: InvitationStatus (pending, completed, expired, cancelled)
├── token: string (secure token for acceptance)
├── expires_at?: timestamp (nullable)
├── invited_by_id: uuid → User (ManyToOne)
├── accepted_by_id?: uuid → User (ManyToOne, nullable)
├── accepted_at?: timestamp
├── organization_id: uuid → Organization (ManyToOne)
├── org_role_assignments: jsonb ({roleId, roleName}[])
├── team_id?: uuid → Team (ManyToOne, nullable)
├── team_role_assignments: jsonb ({roleId, roleName, teamId, teamName}[])
├── welcome_message?: string
└── Timestamps: created_at, updated_at

Indexes:
├── (invited_by_id)
└── (email, organization_id) [unique, where: status = 'pending']
```

<Note>
RBAC assignments are stored in two separate JSONB fields (`orgRoleAssignments` and `teamRoleAssignments`) rather than a single field. Each includes the role name alongside the ID for display purposes without requiring a join. The `team` ManyToOne relationship stores the primary team assignment separately.
</Note>

## Invitation flow

<Steps>
<Step title="Admin creates invitation">
```json
POST /invitations
{
  "email": "newuser@example.com",
  "orgRoles": ["admin-role-uuid"],
  "teamAssignments": [
    { "teamId": "sales-team-uuid", "roleIds": ["agent-role-uuid"] }
  ],
  "welcomeMessage": "Welcome to the team!"
}
```

Creates invitation with a secure token, `expires_at = 7 days`, `status = 'pending'`, and RBAC assignment snapshots.
</Step>
<Step title="Email sent with invitation link">
The system sends an email containing a link like `https://app.propwise.com/accept-invitation?token=xxx`.
</Step>
<Step title="User accepts invitation">

**New user** (public endpoint):

`POST /invitations/:token/accept` with `firstName`, `lastName`, `password`, `confirmPassword`. System creates account, adds to org, applies RBAC assignments.

**Existing user** (identity token endpoint):

`POST /invitations/:token/accept-existing` requires `IdentityTokenGuard`. System adds existing user to org and applies RBAC assignments.
</Step>
</Steps>

## Endpoints

| Method | Endpoint | Auth | Description |
| --- | --- | --- | --- |
| POST | `/invitations` | `system.admin` | Create invitation |
| GET | `/invitations` | `system.admin` | List invitations (paginated) |
| GET | `/invitations/:token` | Public | Get invitation by token |
| DELETE | `/invitations/:id` | `system.admin` | Cancel invitation |
| POST | `/invitations/:token/accept` | Public | Accept invitation (new user) |
| POST | `/invitations/:token/reject` | Public | Reject invitation (new user) |
| POST | `/invitations/:token/accept-existing` | IdentityToken | Accept invitation (existing user) |
| POST | `/invitations/:token/reject-existing` | IdentityToken | Reject invitation (existing user) |

## RBAC assignment on acceptance

<Warning>
Invitation acceptance runs on public/identity-token endpoints that have **no RLS organization context**. All RBAC tables have RLS policies that deny access when `app.organization_id` is not set. The `applyInvitationRbacAssignments` method **must receive the bypassed EntityManager** from the calling `executeWithBypass` context — never `this.em` or injected repositories.
</Warning>

```typescript typescript
private async applyInvitationRbacAssignments(
  em: EntityManager, user: User, invitation: Invitation,
): Promise<void> {
  if (invitation.orgRoleAssignments.length > 0) {
    const roles = await em.find(Role, { id: { $in: roleIds } });
    const userOrgRoles = rolesToCreate.map((role) =>
      em.create(UserOrgRole, { user, role, organization, assignedBy }),
    );
    em.persist(userOrgRoles);
  } else {
    const basicRole = await em.findOne(Role, { name: 'Basic', isSystem: true });
    if (basicRole) {
      em.persist(em.create(UserOrgRole, { user, role: basicRole, organization, assignedBy }));
    }
  }

  if (invitation.teamRoleAssignments.length > 0) {
    const team = await em.findOne(Team, { id: teamId, organization: organizationId });
    const membership = em.create(TeamMembership, { user, team, organization });
    em.persist(membership);

    const teamRoles = await em.find(Role, { id: { $in: teamRoleIds }, scope: 'TEAM' });
    const teamMembershipRoles = teamRoles.map((role) =>
      em.create(TeamMembershipRole, { teamMembership: membership, role, organization, assignedBy }),
    );
    em.persist(teamMembershipRoles);
  }

  await em.flush();
}
```

Both callers pass the bypassed `em`:

```typescript typescript
// In acceptInvitation (new users - public endpoint):
return this.tenantContext.executeWithBypass(async (em) => {
  // ... create user with em ...
  await this.applyInvitationRbacAssignments(em, user, invitation);
  // ... update invitation with em ...
});

// In acceptInvitationForExistingUser (identity token endpoint):
return this.tenantContext.executeWithBypass(async (em) => {
  // ... add user to org with em ...
  await this.applyInvitationRbacAssignments(em, user, invitation);
  // ... update invitation with em ...
});
```

## Business rules

| Rule | Description |
| --- | --- |
| **Expiration** | Invitations expire after 7 days |
| **One-time use** | Invitation token can only be used once |
| **Email uniqueness** | Cannot invite email that already exists in organization |
| **Role validation** | roleIds in invitation must be valid for organization |
| **Team validation** | teamId in invitation must exist in organization |
