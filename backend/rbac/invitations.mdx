The invitation system enables secure user onboarding with predefined access controls, supporting both new user registration and existing user integration across multiple organizations within the complete RBAC framework.

<Info>
The invitation system provides secure token-based user onboarding with comprehensive role and team assignment capabilities, supporting both new user registration and existing user integration across multiple organizations.
</Info>

<Warning>
Invitation acceptance runs on public/identity-token endpoints that have **no RLS organization context**. All RBAC tables have RLS policies that deny access when `app.organization_id` is not set. The invitation system **must use RLS bypass context** for all RBAC assignments.
</Warning>

## Core invitation entity

The invitation system manages the complete user onboarding lifecycle through the core `Invitation` entity:

```typescript
Invitation
├── id: uuid
├── email: string
├── status: InvitationStatus (pending, completed, expired, cancelled)
├── token: string (secure, 64-character hex string)
├── expires_at?: timestamp (default: 7 days from creation)
├── invited_by_id: uuid → User (ManyToOne)
├── accepted_by_id?: uuid → User (ManyToOne, nullable)
├── accepted_at?: timestamp
├── organization_id: uuid → Organization (ManyToOne, required)
├── org_role_assignments: jsonb ({roleId, roleName}[])
├── team_id?: uuid → Team (ManyToOne, nullable, primary team assignment)
├── team_role_assignments: jsonb ({roleId, roleName, teamId, teamName, roleIds}[])
├── welcome_message?: string
├── Relationships:
│   ├── organization → Organization (ManyToOne)
│   ├── invitedBy → User (ManyToOne)
│   ├── acceptedBy? → User (ManyToOne, nullable)
│   └── team? → Team (ManyToOne, nullable)
└── Timestamps: created_at, updated_at

Indexes:
├── (invited_by_id)
├── (organization_id)
├── (token) [unique]
├── (status)
└── (email, organization_id) [unique, where: status = 'pending']
```

## Invitation status flow

The invitation lifecycle is managed through a well-defined status progression:

<Tabs>
<Tab title="pending">
**Initial state** when invitation is created and email sent:
- **Transitions to**: completed, expired, cancelled
- **Actions available**: Accept invitation, cancel invitation, extend expiry

```typescript
if (invitation.expiresAt && new Date() > invitation.expiresAt) {
  invitation.status = InvitationStatus.EXPIRED;
}
```
</Tab>

<Tab title="completed">
**Final state** when user successfully accepts invitation:
- **Transitions from**: pending
- **Actions available**: View completion details only

```typescript
invitation.status = InvitationStatus.COMPLETED;
invitation.acceptedBy = user;
invitation.acceptedAt = new Date();
```
</Tab>

<Tab title="expired">
**Final state** when invitation expires without acceptance:
- **Transitions from**: pending (automatic after expires_at)
- **Actions available**: Create new invitation if needed

<Warning>
Expired invitations cannot be accepted. A new invitation must be created.
</Warning>
</Tab>

<Tab title="cancelled">
**Final state** when invitation is manually cancelled:
- **Transitions from**: pending
- **Actions available**: Create new invitation if needed

```typescript

await this.em.persistAndFlush(invitation);
```
</Tab>
</Tabs>

## RBAC assignment application

The core logic for applying invitation-based role assignments with proper RBAC integration:

```typescript
private async applyInvitationRbacAssignments(
  bypassEm: EntityManager,
  user: User,
  invitation: Invitation
): Promise<void> {
  // 1. Add user to organization
  await this.addUserToOrganization(bypassEm, user, invitation);
  
  // 2. Apply organizational role assignments
  for (const roleAssignment of invitation.orgRoleAssignments) {
    const userOrgRole = bypassEm.create(UserOrgRole, {
      user,
      role: { id: roleAssignment.roleId },
      organization: invitation.organization,
      assignedBy: invitation.invitedBy
    });
    await bypassEm.persist(userOrgRole);
  }
  
  // 3. Apply team role assignments
  for (const teamAssignment of invitation.teamRoleAssignments) {
    // Create team membership
    const teamMembership = bypassEm.create(TeamMembership, {
      user,
      team: { id: teamAssignment.teamId },
      organization: invitation.organization,
      addedBy: invitation.invitedBy,
      joinedAt: new Date()
    });
    await bypassEm.persist(teamMembership);

    // Apply team roles
    for (const roleId of teamAssignment.roleIds) {
      const teamMembershipRole = bypassEm.create(TeamMembershipRole, {
        teamMembership,
        role: { id: roleId },
        organization: invitation.organization,
        assignedBy: invitation.invitedBy
      });
      await bypassEm.persist(teamMembershipRole);
    }
  }
  
  // 4. Complete invitation
  invitation.status = InvitationStatus.COMPLETED;
  invitation.acceptedBy = user;
  invitation.acceptedAt = new Date();
  
  await bypassEm.flush();
}
```

## Token security and validation

The invitation system implements comprehensive token security measures:

```typescript
private async validateInvitationToken(token: string): Promise<Invitation> {
  // Find invitation by token
  const invitation = await this.em.findOne(
    Invitation,
    { token },
    { populate: ['organization', 'invitedBy', 'team'] }
  );

  if (!invitation) {
    throw new BadRequestException('Invalid invitation token');
  }

  // Check status
  if (invitation.status !== InvitationStatus.PENDING) {
    throw new BadRequestException('Invitation is no longer valid');
  }

  // Check expiry
  if (invitation.expiresAt && new Date() > invitation.expiresAt) {
    invitation.status = InvitationStatus.EXPIRED;
    await this.em.persistAndFlush(invitation);
    throw new BadRequestException('Invitation has expired');
  }

  return invitation;
}

private generateSecureToken(): string {
  // 64-character hex string (256 bits of entropy)
  return crypto.randomBytes(32).toString('hex');
}
```

<Note>
**Token Security Features**:
- 256 bits of entropy using crypto.randomBytes(32)
- Globally unique tokens enforced by database constraint
- Automatic expiry validation during token checks
- Single-use tokens that cannot be reused after acceptance
</Note>

- Only one pending invitation per email per organization
- Users can have multiple pending invitations across different organizations
- Completed or expired invitations don't block new invitations
- Enforced via unique index on `(email, organization_id)` where `status = 'pending'`

## RLS bypass requirements

The invitation system requires special handling of Row-Level Security (RLS) policies:

<Warning>
**Critical RLS Considerations**:

1. **Public endpoint context**: Invitation acceptance endpoints have no organization context
2. **RLS policy blocking**: All RBAC tables deny access when `app.organization_id` is NULL
3. **Bypass requirement**: RBAC assignments **must** use bypass context during invitation acceptance
4. **Security implications**: Bypass context removes multi-tenant isolation temporarily
</Warning>

### RLS bypass implementation

```typescript
class InvitationService {
  async acceptInvitation(token: string, userData: AcceptInvitationDto): Promise<User> {
    // Public endpoint - no RLS context
    const invitation = await this.validateInvitationToken(token);
    
    const user = await this.authService.register({
      email: invitation.email,
      ...userData
    });
    
    // Apply RBAC assignments with RLS bypass
    await this.tenantContext.executeWithBypass(async (bypassEm) => {
      await this.applyInvitationRbacAssignments(bypassEm, user, invitation);
    });
    
    return user;
  }
}
```

### RLS policy implications

```sql
-- Example RLS policy on RBAC tables
CREATE POLICY user_org_roles_tenant_isolation ON user_org_roles
FOR ALL TO authenticated
USING (organization_id = current_setting('app.organization_id')::uuid);

-- During invitation acceptance (no org context):
-- current_setting('app.organization_id') = NULL
-- Policy evaluates to: organization_id = NULL (fails for all records)
-- Result: Access denied to all RBAC assignments

-- Solution: Use bypass context with elevated privileges
```

## Query patterns and optimization

Common database operations for invitation management and RBAC integration:

<AccordionGroup>
<Accordion title="Find pending invitations by organization">
```sql
SELECT i.*, u.first_name || ' ' || u.last_name as invited_by_name,
       o.name as organization_name
FROM invitations i
INNER JOIN users u ON i.invited_by_id = u.id
INNER JOIN organizations o ON i.organization_id = o.id
WHERE i.organization_id = $1 
  AND i.status = 'pending'
  AND (i.expires_at IS NULL OR i.expires_at > NOW())
ORDER BY i.created_at DESC;
```

<Accordion title="Validate invitation role assignments">
```sql
-- Validate organizational roles
SELECT r.id, r.name, r.scope
FROM roles r

  AND r.scope = 'ORG'
  AND (r.organization_id = $2 OR r.organization_id IS NULL)
  AND r.is_deleted = false;

WHERE r.id = ANY($1::uuid[])  -- roleIds from invitation
  AND r.scope = 'TEAM'
  AND (r.organization_id = $2 OR r.organization_id IS NULL)
  AND r.is_deleted = false;
```

</Accordion>

<Accordion title="Cleanup expired invitations">
```sql
UPDATE invitations 
SET status = 'expired'
WHERE status = 'pending'
  AND expires_at IS NOT NULL
  AND expires_at < NOW();

-- Return count
SELECT COUNT(*) FROM invitations 
WHERE status = 'expired' 
  AND updated_at >= $1;  -- timestamp from update
```
</Accordion>

<Accordion title="Get invitation with complete role details">
```sql
SELECT i.*,
       json_agg(DISTINCT jsonb_build_object(
         'id', r_org.id,
         'name', r_org.name,
         'scope', r_org.scope,
         'isSystem', r_org.is_system
       )) FILTER (WHERE r_org.id IS NOT NULL) as org_roles,
       json_agg(DISTINCT jsonb_build_object(
         'teamId', t.id,
         'teamName', t.name,
         'roles', team_roles.roles
       )) FILTER (WHERE t.id IS NOT NULL) as team_assignments
FROM invitations i
LEFT JOIN jsonb_array_elements(i.org_role_assignments) ora(role_assignment) ON true
LEFT JOIN roles r_org ON r_org.id = (ora.role_assignment->>'roleId')::uuid
LEFT JOIN jsonb_array_elements(i.team_role_assignments) tra(team_assignment) ON true
LEFT JOIN teams t ON t.id = (tra.team_assignment->>'teamId')::uuid
LEFT JOIN LATERAL (
  SELECT json_agg(jsonb_build_object(
    'id', r_team.id,
    'name', r_team.name,
    'scope', r_team.scope
  )) as roles
  FROM jsonb_array_elements_text(tra.team_assignment->'roleIds') role_id
  INNER JOIN roles r_team ON r_team.id = role_id::uuid
) team_roles ON true
WHERE i.token = $1
GROUP BY i.id;
```
</Accordion>
</AccordionGroup>

## Service implementation

Core business logic with comprehensive RBAC integration:

```typescript
@Injectable()
export class InvitationService {
  constructor(
    @InjectRepository(Invitation) private readonly em: EntityManager,
    private readonly authService: AuthService,
    private readonly emailService: EmailService,
    private readonly tenantContext: TenantContext,
    private readonly rbacService: RbacService
  ) {}

  async createInvitation(dto: CreateInvitationDto): Promise<Invitation> {
    // Validate role assignments
    await this.validateRoleAssignments(
      dto.organizationId, 
      dto.orgRoleAssignments, 
      dto.teamRoleAssignments
    );

    // Check for existing pending invitation
    const existingInvitation = await this.em.findOne(Invitation, {
      email: dto.email,
      organizationId: dto.organizationId,
      status: InvitationStatus.PENDING
    });

    if (existingInvitation) {
      throw new ConflictException('Pending invitation already exists for this email');
    }

    // Generate secure token and create invitation
    const token = this.generateSecureToken();
    const invitation = this.em.create(Invitation, {
      ...dto,
      token,
      status: InvitationStatus.PENDING,
      expiresAt: dto.expiresAt || new Date(Date.now() + 7 * 24 * 60 * 60 * 1000)
    });

    await this.em.persistAndFlush(invitation);
    await this.sendInvitationEmail(invitation);
    
    return invitation;
  }

  private async validateRoleAssignments(
    organizationId: string,
    orgRoleAssignments: OrgRoleAssignment[],
    teamRoleAssignments: TeamRoleAssignment[]
  ): Promise<void> {
    // Validate organizational roles
    for (const roleAssignment of orgRoleAssignments) {
      const role = await this.em.findOne(Role, {
        id: roleAssignment.roleId,
        scope: RoleScope.ORG,
        $or: [
          { organization: organizationId }, // Custom role
          { organization: null }           // System role
        ],
        isDeleted: false
      });
      
      if (!role) {
        throw new BadRequestException(`Invalid org role: ${roleAssignment.roleId}`);
      }
    }
    
    // Validate team roles and teams
    for (const teamAssignment of teamRoleAssignments) {
      const team = await this.em.findOne(Team, {
        id: teamAssignment.teamId,
        organization: organizationId,
        isDeleted: false
      });
      
      if (!team) {
        throw new BadRequestException(`Invalid team: ${teamAssignment.teamId}`);
      }
      
      for (const roleId of teamAssignment.roleIds) {
        const role = await this.em.findOne(Role, {
          id: roleId,
          scope: RoleScope.TEAM,
          $or: [
            { organization: organizationId },
            { organization: null }
          ],
          isDeleted: false
        });
        
        if (!role) {
          throw new BadRequestException(`Invalid team role: ${roleId}`);
        }
      }
    }
  }

  private generateSecureToken(): string {
    return crypto.randomBytes(32).toString('hex');
  }
}
```

```typescript
    const invitation = await this.invitationService.acceptInvitationExisting(token, user);
    
    const permissions = await this.rbacService.getUserPermissions(
      user.id,
      invitation.organization.id
    );
    
    return { 
      success: true, 
      invitation,
      permissions: permissions.allPermissions
    };
  }

  @Patch(':id/cancel')
  @UseGuards(JwtAuthGuard, PermissionGuard)
  @RequirePermissions('invitations:cancel')
  async cancelInvitation(@Param('id') id: string): Promise<{ success: boolean }> {
    await this.invitationService.cancelInvitation(id);
    return { success: true };
  }

  @Delete('cleanup-expired')
  @UseGuards(JwtAuthGuard, PermissionGuard)
  @RequirePermissions('invitations:cleanup')
  async cleanupExpiredInvitations(): Promise<{ cleaned: number }> {
    const cleaned = await this.invitationService.cleanupExpiredInvitations();
    return { cleaned };
  }
}
```
