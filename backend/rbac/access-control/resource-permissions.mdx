---
title: "Resource permissions"
description: "Layer 2 of the access control system: computed per-resource permission flags returned in API responses so the frontend knows what the user can do."
---

This layer computes **what actions the current user can perform** on each resource and attaches that information to the API response. The frontend reads these permissions directly for UI decisions (show/hide buttons, enable/disable actions) without duplicating access logic.

## Which entities have resource permissions

### Entities WITH resource permissions

These are **CRM business entities** with complex access rules involving stakeholders, teams, and creators:

| Entity | Has `permissions`? | Has `assignedTeam`? | Reason |
|--------|-------------------|---------------------|--------|
| **Lead** | Yes | Yes | Has EntityStakeholder, team assignment, creator access |
| **Deal** | Yes | Yes | Has EntityStakeholder, team assignment, creator access |
| **Contact** | Yes | No | Derived access from related Leads/Deals |
| **Company** | Yes | No | Derived access from related Contacts/Deals |

### Entities WITHOUT resource permissions

These are **configuration/admin entities** with simple permission-based access:

| Entity | Why no resource permissions |
|--------|----------------------------|
| User | Admin-only access (`system.admin`) |
| Team | Admin or team membership access |
| Role | Admin-only access (`system.admin`, `system.owner`) |
| Commission Payment | Recipient-based or `sales.manage` permission |
| Entity Transfer | Sender/recipient-based access |
| Invitation | Admin-only access |
| Activity | Read-only timeline data |
| Stage History | Read-only audit data |

## PermissionComputeService

**Location:** `src/modules/auth/services/permission-compute.service.ts`

This service computes permissions using the existing `AccessService` methods:

```typescript
@Injectable()
export class PermissionComputeService {
  constructor(private readonly accessService: AccessService) {}

  // Single resource methods
  async computeLeadPermissionsInTransaction(em, user, leadId): Promise<ResourcePermissionsDto>;
  async computeDealPermissionsInTransaction(em, user, dealId): Promise<ResourcePermissionsDto>;
  async computeContactPermissionsInTransaction(em, user, contactId): Promise<ResourcePermissionsDto>;
  async computeCompanyPermissionsInTransaction(em, user, companyId): Promise<ResourcePermissionsDto>;

  // Optimized bulk methods (used by services for list views)
  async computeAllBulkLeadPermissionsWithTeamsInTransaction(
    em, user, leadIds, preloadedLeads?
  ): Promise<{ permissions: Map<string, ResourcePermissionsDto> }>;

  async computeAllBulkDealPermissionsWithTeamsInTransaction(
    em, user, dealIds, preloadedDeals?
  ): Promise<{ permissions: Map<string, ResourcePermissionsDto> }>;

  // Standard bulk methods (3 queries per action type)
  async computeBulkDealPermissionsInTransaction(em, user, dealIds): Promise<Map<string, ResourcePermissionsDto>>;
  async computeBulkContactPermissionsInTransaction(em, user, contactIds): Promise<Map<string, ResourcePermissionsDto>>;
  async computeBulkCompanyPermissionsInTransaction(em, user, companyIds): Promise<Map<string, ResourcePermissionsDto>>;

  // Team assignment helpers
  async getLeadAssignedTeamInTransaction(em, leadId): Promise<TeamBasicDto | null>;
  async getBulkLeadAssignedTeamsInTransaction(em, leadIds): Promise<Map<string, TeamBasicDto>>;
  async getDealAssignedTeamInTransaction(em, dealId): Promise<TeamBasicDto | null>;
  async getBulkDealAssignedTeamsInTransaction(em, dealIds): Promise<Map<string, TeamBasicDto>>;
}
```

**Permission derivation rules:**

- `canAssign` = `canView && canManageStakeholders(user)` where `canManageStakeholders` checks for `crm.manage`, `system.admin`, `system.owner`, or `team_crm.manage`
- `canTransfer` = `canEdit`
- `canArchive` = `canDelete`
- `canUnarchive` = `canDelete`
- For Contacts/Companies: no `canAssign` or `canTransfer` (these are derived-access entities)

  ],

</Step>

<Step title="Inject PermissionComputeService">
```typescript
import { PermissionComputeService } from '@/modules/auth/services/permission-compute.service';

@Injectable()
export class LeadService {
  constructor(
    private readonly permissionComputeService: PermissionComputeService,
  ) {}
}
```
</Step>

<Step title="Update findOne (single resource)">
```typescript
async findOne(user: RequestUser, id: string): Promise<LeadDto> {
  return this.tenantContext.executeReadOnly(user.organizationId, async (em) => {
    const lead = await em.findOne(Lead, { id }, { populate: [...] });

    if (!lead) {
      throw new NotFoundException('Lead not found');
    }

    const [permissions, assignedTeam] = await Promise.all([
      this.permissionComputeService.computeLeadPermissionsInTransaction(em, user, lead.id),
      this.permissionComputeService.getLeadAssignedTeamInTransaction(em, lead.id),
    ]);

    return new LeadDto(lead, { permissions, assignedTeam });
  });
}
```
</Step>

<Step title="Update findAll (bulk)">
```typescript
async findAll(user: RequestUser, filters: FindLeadsDto): Promise<LeadDto[]> {
  return this.tenantContext.executeReadOnly(user.organizationId, async (em) => {
    const leads = await em.find(Lead, filters, { populate: [...] });

    if (leads.length === 0) return [];

    const leadIds = leads.map((l) => l.id);

    const [permissionsMap, teamsMap] = await Promise.all([
      this.permissionComputeService.computeBulkLeadPermissionsInTransaction(em, user, leadIds),
      this.permissionComputeService.getBulkLeadAssignedTeamsInTransaction(em, leadIds),
    ]);

    return leads.map((lead) =>
      new LeadDto(lead, {
        permissions: permissionsMap.get(lead.id),
        assignedTeam: teamsMap.get(lead.id),
      }),
    );
  });
}
```
</Step>
</Steps>

## Lead & Deal service implementation reference

Both `LeadService` and `DealService` have `PermissionComputeService` injected and use optimized bulk permission computation for all list views.

<Tabs>
<Tab title="Lead service">
### `enrichLeadsWithPermissions()`

The `LeadService` helper computes permissions and fetches primary stakeholders in an optimized pass:

```typescript
private async enrichLeadsWithPermissions(
  em: EntityManager,
  user: any,
  leads: Lead[],
): Promise<LeadDto[]> {
  if (!leads || leads.length === 0) return [];

  const leadIds = leads.map((lead) => lead.id);

  const { permissions: permissionsMap } =
    await this.permissionComputeService.computeAllBulkLeadPermissionsWithTeamsInTransaction(
      em, user, leadIds, leads,
    );

  const primaryStakeholdersMap =
    await this.stakeholderService.getBulkPrimaryStakeholdersInTransaction(
      em, StakeholderEntityType.LEAD, leadIds,
    );

  return leads.map((lead) =>
    new LeadDto(lead, {
      permissions: permissionsMap.get(lead.id),
      primaryStakeholder: primaryStakeholdersMap.has(lead.id)
        ? new EntityStakeholderDto(primaryStakeholdersMap.get(lead.id)!)
        : undefined,
    }),
  );
}
```

Used in: `getLeadsForKanbanByStage()` (stages + archived), `getLeadsForKanbanByStageUnassigned()`, `getLeadsForList()`, `findAllMy()`.
</Tab>

## Performance considerations

<Tip>
Always use the optimized bulk methods for list views. They compute all permission types in a single pass with 2-4 queries total.
</Tip>

```typescript
// 2-4 queries total (optimized single-pass)
const { permissions } = await this.permissionComputeService
  .computeAllBulkLeadPermissionsWithTeamsInTransaction(em, user, leadIds, leads);

const { permissions } = await this.permissionComputeService
  .computeAllBulkDealPermissionsWithTeamsInTransaction(em, user, dealIds, deals);
```

<Warning>
Never loop over individual permission computations. This causes N+1 queries.

```typescript
// 300-500 queries for 100 items
for (const lead of leads) {
  const permissions = await this.permissionComputeService
    .computeLeadPermissionsInTransaction(em, user, lead.id);
}
```
</Warning>

**Query count comparison:**

| Operation | Method | Database queries |
|-----------|--------|-----------------|
| 1 lead/deal | Single method | 3-5 queries |
| 100 leads/deals | Loop (N+1) | 300-500 queries |
| 100 leads/deals | Optimized bulk (`computeAll*`) | 2-4 queries |

All permission checks happen within the same EntityManager transaction, maintaining RLS context.
