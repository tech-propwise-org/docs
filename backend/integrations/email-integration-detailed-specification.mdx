# Email Integration Detailed Specification

This specification covers Gmail and Outlook **mail** OAuth, token storage, refresh, and API surfaces for both personal and organization-level integrations. Calendar OAuth is documented separately in the Calendar System Deep Dive.

<Note>
This specification is scoped to email integrations only. Calendar OAuth functionality is documented in `CALENDAR_SYSTEM_DEEP_DIVE.md`.
</Note>

## OAuth Flow (Tenant Session)

The OAuth flow supports both personal and organization-level mailbox integrations with comprehensive security checks.

<Steps>
<Step title="User Selection">
User has selected an organization and the SPA holds a **tenant JWT**.
</Step>

<Step title="Personal OAuth Initiation">
**`POST /v1/email-integrations/oauth/:provider/authorize`** (`provider` = `gmail` | `outlook`) for **personal** mailbox integration (`EmailIntegrationLevel.PERSONAL`). Uses standard `/v1` API authentication.
</Step>

<Step title="Organization OAuth Initiation">
**`POST /v1/email-integrations/organization/oauth/:provider/authorize`** for **workspace** mailbox integration (`EmailIntegrationLevel.ORGANIZATION`). Requires **`SYSTEM_ADMIN`** or **`SYSTEM_OWNER`** permissions via `@CheckAccess`.
</Step>

<Step title="Authorization URL Generation">
Backend returns **`{ authorizationUrl }`** containing the Google/Microsoft authorize URL with **HMAC-signed** `state` parameter using `SignedPayloadService` with purpose `email.oauth.state`. The payload includes:
- `userId`
- `organizationId` 
- **`tokenVersion`** (prevents stale OAuth popups after auth reset)
- `provider`
- Optional **`level`** (omitted defaults to `PERSONAL` for legacy compatibility)
</Step>

<Step title="Browser Authorization">
Browser opens `authorizationUrl` in a popup window.
</Step>

<Step title="Provider Callback">
Provider redirects to **public** callback endpoints (no `/v1` prefix):
- **`GET /auth/gmail/callback`**
- **`GET /auth/outlook/callback`**
</Step>

<Step title="Gmail Access Verification">
<Warning>
Gmail-specific step: Before saving to `email_integrations`, backend calls **`GET .../gmail/v1/users/me/profile`** with the new access token. If Google returns **400** or **403**, the integration **is not persisted** and the callback returns `oauth-error` with `error: EMAIL_OAUTH_GMAIL_ACCESS_BLOCKED_CODE`.
</Warning>

This prevents saving integrations for Google accounts that authenticate but don't expose Gmail API access. Outlook skips this verification step.
</Step>

<Step title="Callback Access Re-check">
Backend performs comprehensive access validation before token exchange and again during DB transaction:
- **`User.isActive`** status
- Matching **`User.tokenVersion`** from signed state
- Active **`user_org_roles`** row for `(userId, organizationId)`
- For **`ORGANIZATION`** level: current **`system.admin`** or **`system.owner`** permissions

Uses **`AccessService.assertOAuthCallbackOrgAccessInTransaction`**. Failure results in `oauth-error` with `OAUTH_CALLBACK_TENANT_ACCESS_DENIED_MESSAGE`.
</Step>

<Step title="Integration Persistence">
**ORGANIZATION** rows store:
- **`user_id = null`**
- **`connected_by_id`** = OAuth actor
- **`email`** normalized to lowercase
- Uniqueness constraint: **`(organization_id, provider, lower(email))`** among active org rows

**Cross-level mailbox conflicts**: A mailbox address cannot exist at both PERSONAL and ORGANIZATION levels in the same organization.

**Redundant reconnect prevention**: If OAuth email matches existing row's email and the row is healthy (active, no sync errors, valid access token), the connection is rejected.
</Step>

<Step title="Response Handling">
Response HTML uses **`postMessage`** with **`app.postMessageTargetOrigin`** (from `FRONTEND_URL`). SPA invalidates **`queryKeys.emailIntegrations`** after success and handles specific error codes like `GMAIL_ACCESS_BLOCKED`.
</Step>
</Steps>

### Gmail Access Verification Details

<Info>
**Constant**: `EMAIL_OAUTH_GMAIL_ACCESS_BLOCKED_CODE` in `src/common/constants/email-oauth.constants.ts` must stay synchronized with frontend constant in `email-integration.api.ts`.
</Info>

**Required Scopes**:
- **`gmail.send`** - for outbound email sending
- **`gmail.metadata`** - for `users.getProfile` access probe
- **`userinfo.email`** - for reading profile mailbox address

<CodeGroup>

```bash Gmail Configuration
GMAIL_REDIRECT_URI=${BACKEND_URL}/auth/gmail/callback
# Optional override
GMAIL_EMAIL_REDIRECT_URI=custom_uri_here
```

```bash Outlook Configuration
OUTLOOK_REDIRECT_URI=${BACKEND_URL}/auth/outlook/callback  
# Optional override
OUTLOOK_EMAIL_REDIRECT_URI=custom_uri_here
```
</CodeGroup>

<Warning>
Register these exact redirect URIs in your Google Cloud OAuth client and Azure app configurations.
</Warning>

### Callback Error Handling

Token exchange failures and IdP error parameters are logged server-side with provider HTTP status and truncated body previews. The popup `postMessage` payload uses generic error strings for user display rather than raw API responses.

**OAuth State TTL**: `EMAIL_OAUTH_STATE_TTL_MS` (15 minutes), aligned with calendar UX patterns.

## API Endpoints & DTOs

### Personal Integration Management

<Tabs>
<Tab title="List Personal Integrations">
**`GET /v1/email-integrations`**

Returns **PERSONAL** mailboxes for the current user only. Admins cannot view other members' personal integrations.

**Response**: `{ items, nextCursor }`
- **Ordering**: `connected_at DESC, id ASC` (keyset pagination)  
- **Pagination**: `?cursor=` with opaque cursor string
- **Limits**: Default 20, max 50 (via `EMAIL_INTEGRATION_PAGE_DEFAULT/MAX`)
</Tab>

<Tab title="Get Send Accounts">
**`GET /v1/email-integrations/send-accounts`**

Returns mailboxes allowed as **From** addresses in the composer:
- **`personal`**: Caller's PERSONAL rows
- **`organization`**: ORGANIZATION rows (requires `SYSTEM_ADMIN` or `SYSTEM_OWNER`)

**Pagination Parameters**:
- `cursorPersonal`, `cursorOrganization`
- Use literal `__exhausted__` when a stream is completed
</Tab>
</Tabs>

### Organization Integration Management

<Tabs>
<Tab title="List Organization Integrations">
**`GET /v1/email-integrations/organization`**

Returns **ORGANIZATION** mailboxes with `@CheckAccess` (admin/owner required).

Uses `findOrganizationMailboxesPage` with `populate(['connectedBy'])` for OAuth actor details.
</Tab>

<Tab title="Authorize Organization OAuth">
**`POST /v1/email-integrations/organization/oauth/:provider/authorize`**

Initiates OAuth for workspace mailbox (`@CheckAccess` required).
</Tab>
</Tabs>

## Access Control

### Read/Mutate by ID

**Endpoints**: `GET/PUT/DELETE /v1/email-integrations/:id`, `POST /v1/email-integrations/:id/send`

**Access Rules**:
- **PERSONAL**: Only when `user_id` matches JWT user (admins cannot access other members' integrations)
- **ORGANIZATION**: Requires `SYSTEM_ADMIN` or `SYSTEM_OWNER`
- Missing access returns **404** (same as unknown ID)

Uses `EmailIntegrationService.getAccessibleIntegrationFindWhere` with secondary `assertCanAccessIntegration` check.

### Send Access Control

<AccordionGroup>
<Accordion title="Personal Send Access">
Only the mailbox owner may use `POST .../id/send` for `level=PERSONAL` rows (`user_id` must match JWT user).
</Accordion>

<Accordion title="Organization Send Access">  
Requires `SYSTEM_ADMIN` or `SYSTEM_OWNER` permissions via `EmailIntegrationService.assertCanAccessIntegration` and `sendWithIntegration`.
</Accordion>
</AccordionGroup>

## Data Model

### Integration Levels

<Tabs>
<Tab title="PERSONAL Level">
- **`user_id`**: NOT NULL
- **`connected_by_id`**: NULL  
- **Uniqueness**: `(user_id, organization_id, provider)` where `is_deleted = false` and `level = 'PERSONAL'`
</Tab>

<Tab title="ORGANIZATION Level">
- **`user_id`**: NULL
- **`connected_by_id`**: NOT NULL (OAuth actor)
- **Uniqueness**: `(organization_id, provider, lower(email))` where `is_deleted = false` and `level = 'ORGANIZATION'`
- **Default Flag**: Optional `is_default` with at most one default per organization
</Tab>
</Tabs>

### Cross-Level Uniqueness

<Warning>
Active rows cannot mix PERSONAL and ORGANIZATION for the same `organization_id + lower(email)` combination.
</Warning>

**Implementation**: Migration `Migration20260521171000_PreventEmailIntegrationCrossLevelMailboxConflicts` installs `email_integrations_cross_level_mailbox_conflict_trigger` using transaction advisory locks to prevent concurrent conflicts.

### Token Storage

<Check>
All tokens are encrypted using AES-256-GCM via shared `EncryptionService`:
</Check>

- **`encrypted_access_token`**
- **`encrypted_refresh_token`**  
- **Encryption Key**: 64 hex characters via `ENCRYPTION_KEY` / `app.encryption.key`

### Additional Fields

- **`email`**: Provider mailbox/profile email (normalized to lowercase on write)
- **`level`**: Postgres check constraint + application validation
- **Migration**: `Migration20260508150000_EmailIntegrationOrganizationLevel` introduces level support
