## Overview

The integrations module provides connectivity to external services and platforms within PropWise CRM. It currently supports email provider integrations (Gmail and Outlook) via OAuth2 authentication, and Tiptap AI for rich text editing capabilities. The module handles OAuth flows, token management, email sending, and secure access to third-party services.

## Architecture

The integrations module is organized into three main sub-modules:

<CardGroup cols={3}>
  <Card title="Email Integration" icon="envelope">
    Manages email provider connections and email sending functionality
  </Card>
  <Card title="OAuth" icon="key">
    Calendar OAuth controllers only (google-calendar, outlook-calendar). Mail OAuth lives in email-integration
  </Card>
  <Card title="Tiptap AI" icon="wand-magic-sparkles">
    Provides JWT token generation for Tiptap AI services
  </Card>
</CardGroup>

### Key Design Decisions

<AccordionGroup>
  <Accordion title="Provider-agnostic email integration">
    Uses enum-based provider differentiation to support multiple email services
  </Accordion>
  
  <Accordion title="Token refresh automation">
    Implements scheduled token refresh to maintain continuous integration connectivity
  </Accordion>
  
  <Accordion title="Tenant-aware integrations">
    All integrations are scoped to organizations using tenant context
  </Accordion>
  
  <Accordion title="Secure token storage">
    OAuth access/refresh tokens are **AES-GCM encrypted** at rest (`encryptedAccessToken` / `encryptedRefreshToken`)
  </Accordion>
  
  <Accordion title="Personal vs workspace mailboxes">
    **`EmailIntegrationLevel`** distinguishes **PERSONAL** (owned by **`user_id`**) and **ORGANIZATION** (`user_id` null, **`connected_by_id`** stores OAuth actor). Partial unique indexes enforce one active personal row per `(user, organization, provider)` and one org mailbox per `(organization, provider, lower(email))`; a cross-level trigger rejects the same normalized mailbox address from being connected as both personal and workspace in the same organization.
  </Accordion>
</AccordionGroup>

## Entities

### EmailIntegration

Represents a connection to Gmail or Outlook for sending mail. Either a **personal** mailbox (**`level=PERSONAL`**, **`user`** required) or a **workspace** mailbox (**`level=ORGANIZATION`**, **`user`** null, **`connectedBy`** set).

```typescript
interface EmailIntegration {
  id: string;
  email: string;
  provider: EmailProvider;
  level: 'PERSONAL' | 'ORGANIZATION';
  isDefault?: boolean; // ORGANIZATION: optional default sender for admins
  encryptedAccessToken?: string;
  encryptedRefreshToken?: string;
  scopes?: string;
  externalAccountId?: string;
  lastSyncErrorAt?: Date;
  lastSyncErrorMessage?: string;
  tokenExpiresAt?: Date;
  metadata?: Record<string, any>;
  isActive: boolean;
  lastUsedAt?: Date;
  user?: User | null; // PERSONAL owner; null for ORGANIZATION
  connectedBy?: User | null; // OAuth actor for ORGANIZATION
  organization: Organization;
  isDeleted: boolean;
  createdAt: Date;
  updatedAt: Date;
}
```

<Info>
**Unique constraints (partial, where `is_deleted = false`):**

- **PERSONAL:** `(user_id, organization_id, provider)`
- **ORGANIZATION:** `(organization_id, provider, lower(email))`; at most one **`is_default`** org mailbox per organization.
- **Cross-level mailbox rule:** the same `organization_id + lower(email)` cannot appear in both **PERSONAL** and **ORGANIZATION** rows, regardless of provider. The same mailbox may still be connected by multiple personal users.
</Info>

### EmailProvider Enum

```typescript
enum EmailProvider {
  OUTLOOK = 'outlook',
  GMAIL = 'gmail',
}
```

## API Endpoints

### Email Integration Management

#### GET /email-integrations

Lists **PERSONAL** email integrations for the current user.

**Response:**
```typescript
{
  data: EmailIntegration[];
}
```

#### GET /email-integrations/send-accounts

Mailboxes usable as **From** in the composer: caller's personal integrations plus **ORGANIZATION** rows when the caller has **`SYSTEM_ADMIN`** or **`SYSTEM_OWNER`**.

**Response:**
```typescript
{
  data: EmailIntegration[];
}
```

#### GET /email-integrations/organization

Lists **ORGANIZATION** mailboxes (**SYSTEM_ADMIN** / **SYSTEM_OWNER**).

**Response:**
```typescript
{
  data: EmailIntegration[];
}
```

<Warning>
Requires `SYSTEM_ADMIN` or `SYSTEM_OWNER` role.
</Warning>

#### POST /v1/email-integrations/organization/oauth/:provider/authorize

Starts OAuth for a workspace mailbox (same `@CheckAccess` as **`GET …/organization`**).

**Parameters:**
- `provider` - Email provider (`gmail` | `outlook`)

**Response:**
```typescript
{
  authUrl: string; // OAuth authorization URL
}
```

#### POST /v1/email-integrations/oauth/:provider/authorize

Starts OAuth for **personal** mail (`EmailIntegrationLevel.PERSONAL`).

**Parameters:**
- `provider` - Email provider (`gmail` | `outlook`)

**Response:**
```typescript
{
  authUrl: string; // OAuth authorization URL
}
```

<Note>
Creating integrations by POSTing plaintext OAuth tokens is **not supported**. Use the authorize endpoint and provider callback only.
</Note>

#### PUT /email-integrations/:id

Updates an existing email integration.

**Parameters:**
- `id` - Email integration ID

**Request Body:**
```typescript
{
  isDefault?: boolean;
  isActive?: boolean;
}
```

**Response:**
```typescript
{
  data: EmailIntegration;
}
```

#### DELETE /email-integrations/:id

Soft deletes an email integration.

**Parameters:**
- `id` - Email integration ID

**Response:**
```typescript
{
  message: string;
}
```

#### POST /email-integrations/:id/send

Sends an email through the specified integration.

**Parameters:**
- `id` - Email integration ID

**Request Body:**
```typescript
{
  to: string[];
  subject: string;
  body: string;
  isHtml?: boolean;
  cc?: string[];
  bcc?: string[];
  attachments?: Array<{
    filename: string;
    content: string; // base64
    contentType: string;
  }>;
}
```

**Response:**
```typescript
{
  success: boolean;
  messageId?: string;
}
```

### OAuth Authentication

<Tip>
See **`Docs/EMAIL_INTEGRATION_SPECIFICATION.md`** for the canonical mail OAuth flow (tenant `POST /v1/email-integrations/oauth/:provider/authorize`, signed state bound to `userId` / `organizationId` / `tokenVersion`, public **`GET /auth/gmail/callback`** / **`GET /auth/outlook/callback`**, encrypted tokens).
</Tip>

<Steps>
  <Step title="Initiate OAuth Flow">
    User clicks connect button, frontend calls authorize endpoint
  </Step>
  
  <Step title="Redirect to Provider">
    Backend generates signed state token and redirects to provider OAuth URL
  </Step>
  
  <Step title="User Grants Permissions">
    User authenticates with provider and grants requested permissions
  </Step>
  
  <Step title="Provider Callback">
    Provider redirects to callback URL with authorization code and state
  </Step>
  
  <Step title="Token Exchange">
    Backend validates state, exchanges code for access/refresh tokens
  </Step>
  
  <Step title="Store Integration">
    Tokens are encrypted and stored, user redirected to frontend
  </Step>
</Steps>

### Tiptap AI Integration

#### GET /tiptap-ai/token

Generates a short-lived JWT token for Tiptap AI services.

**Response:**
```typescript
{
  token: string; // JWT valid for 1 hour
}
```

<Info>
The JWT token is valid for **1 hour** and must be refreshed before expiration.
</Info>

## Business Rules

### Email Integrations

<Check>
**One Integration Per Provider**: At most one active row per `(user, organization, provider)` — reconnect updates tokens on the same row
</Check>

<Check>
**Token Expiration**: Expired tokens are automatically refreshed via scheduled jobs
</Check>

<Check>
**Provider Validation**: Only supported email providers (Gmail, Outlook) are allowed
</Check>

<Check>
**Organization Scoping**: All integrations are scoped to the user's current organization
</Check>

<Check>
**Soft Deletion**: Integrations are soft-deleted to maintain audit history
</Check>

<Warning>
**No Personal/Workspace Split**: A normalized mailbox address may be connected by multiple **PERSONAL** users, but cannot be connected at **PERSONAL** and **ORGANIZATION** levels in the same organization at the same time
</Warning>

### OAuth Flows

<Tabs>
  <Tab title="State Validation">
    OAuth state parameter must contain valid user, organization, provider, and token-version context
  </Tab>
  
  <Tab title="Secure Redirects">
    Only whitelisted redirect URLs are allowed
  </Tab>
  
  <Tab title="Token Exchange">
    Authorization codes must be exchanged for tokens immediately
  </Tab>
  
  <Tab title="Error Handling">
    OAuth errors redirect to frontend with appropriate error messages
  </Tab>
</Tabs>

### Token Management

<AccordionGroup>
  <Accordion title="Automatic Refresh">
    Tokens are refreshed 5 minutes before expiration
  </Accordion>
  
  <Accordion title="Retry Logic">
    Failed refresh attempts are retried with exponential backoff
  </Accordion>
  
  <Accordion title="Encryption">
    All stored tokens are encrypted at rest using AES-GCM
  </Accordion>
  
  <Accordion title="Expiration Handling">
    Expired tokens trigger re-authentication flows
  </Accordion>
</AccordionGroup>

### Calendar Conferencing (Google Meet / Teams)

The calendar drivers can request a conferencing link when a CRM Event meeting channel carries `conferenceType` (see `Docs/CALENDAR_SYSTEM_DEEP_DIVE.md` → "Create event" for the full write-back flow). The driver `CalendarEventPayload` gained `conferenceType` (`google_meet` | `teams`) and `conferenceRequestId` (stable CRM event id):

<CodeGroup>
```typescript Google Meet Driver
// Google driver (google_meet)
// createEvent adds conferenceDataVersion: 1 to the request params
// toGoogleEvent emits conferenceData.createRequest
{
  requestId: conferenceRequestId,
  conferenceSolutionKey: {
    type: 'hangoutsMeet'
  }
}
// The existing calendar.events scope is sufficient — no re-consent
// The mapper already reads hangoutLink → meetingLink
```

```typescript Microsoft Teams Driver
// Outlook driver (teams)
// toGraphEvent sets:
{
  isOnlineMeeting: true,
  onlineMeetingProvider: 'teamsForBusiness'
}
// Skips manual link-in-body append
// The existing Calendars.ReadWrite scope is sufficient
// The mapper already reads onlineMeeting.joinUrl → meetingLink
```
</CodeGroup>

<Warning>
**Consumer `outlook.com` accounts cannot create Teams meetings** — Graph errors are caught per-integration and surfaced as a sync error without failing event creation.
</Warning>

<Info>
Only the **matching** provider receives the conference request; other connected providers get the minted link as description text. Drivers remain DB-free and pure.
</Info>

### Security

<AccordionGroup>

  <Accordion title="JWT Expiration">

  <Accordion title="CSRF Protection">
