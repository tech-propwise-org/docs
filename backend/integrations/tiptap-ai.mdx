The Tiptap AI integration provides secure authentication for AI-powered writing features within PropWise CRM. This service generates short-lived JWT tokens that enable access to Tiptap's AI writing capabilities while maintaining proper security boundaries.

## Overview

The Tiptap AI integration is part of the broader integrations module and handles authentication for AI-enhanced text editing features. It generates JWT tokens with a 1-hour expiration to provide secure, time-limited access to Tiptap's AI services.

<Note>
  Tiptap AI tokens are automatically scoped to the current user's organization context for security and data isolation.
</Note>

## Token Generation

### GET /tiptap-ai/token

Generates a short-lived JWT token for accessing Tiptap AI services.

<Steps>
<Step title="Authenticate Request">
Ensure the user is authenticated and has valid session credentials.
</Step>

<Step title="Generate JWT Token">
The system creates a JWT token valid for 1 hour with appropriate user and organization context.
</Step>

<Step title="Return Token">
The token is returned immediately for use in Tiptap AI API calls.
</Step>
</Steps>

#### Response Format

```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

#### Response Schema

| Field | Type | Description |
|-------|------|-------------|
| `token` | `string` | JWT token valid for 1 hour |

## Security Considerations

<Warning>
  JWT tokens have a 1-hour expiration for security. Your application should request new tokens as needed rather than storing them long-term.
</Warning>

### Token Properties

- **Expiration**: 1 hour from generation
- **Scope**: Organization-specific access
- **Usage**: Single-purpose for Tiptap AI services
- **Refresh**: No refresh mechanism - request new tokens as needed

### Best Practices

<CardGroup cols={2}>
<Card title="Token Lifecycle" icon="clock">
Request tokens just before use and avoid long-term storage
</Card>

<Card title="Error Handling" icon="shield-check">
Implement fallback mechanisms for expired or invalid tokens
</Card>

<Card title="Rate Limiting" icon="gauge">
Cache tokens when possible to avoid excessive API calls
</Card>

<Card title="Security" icon="lock">
Never expose tokens in client-side code or logs
</Card>
</CardGroup>

## Implementation Example

Here's a complete example of integrating Tiptap AI token generation in a React component:

```jsx
import { useState, useEffect } from 'react';
import { Editor, EditorContent } from '@tiptap/react';
import { AIExtension } from '@tiptap/extension-ai';

function AIEnabledEditor() {
  const [editor, setEditor] = useState(null);
  const [aiToken, setAiToken] = useState(null);

  useEffect(() => {
    async function initializeEditor() {
      try {
        // Get Tiptap AI token
        const response = await fetch('/api/tiptap-ai/token');
        const { token } = await response.json();
        setAiToken(token);

        // Initialize editor with AI capabilities
        const editorInstance = new Editor({
          extensions: [
            AIExtension.configure({
              token: token,
              onTokenExpired: async () => {
                // Refresh token when expired
                const newResponse = await fetch('/api/tiptap-ai/token');
                const { token: newToken } = await newResponse.json();
                setAiToken(newToken);
                return newToken;
              }
            })
          ]
        });

        setEditor(editorInstance);
      } catch (error) {
        console.error('Failed to initialize AI editor:', error);
      }
    }

    initializeEditor();

    return () => editor?.destroy();
  }, []);

  return (
    <div>
      {editor && <EditorContent editor={editor} />}
    </div>
  );
}
```

## Integration Architecture

The Tiptap AI integration fits within the broader integrations module architecture:

```
integrations/
├── email-integration/    # Email provider integrations

└── tiptap-ai/          # AI writing service authentication
```

<Info>
  The Tiptap AI integration leverages the same authentication and tenant context infrastructure used by other integrations in the module.
</Info>

## Dependencies

The Tiptap AI integration requires the following modules:

- **AuthModule**: Provides authentication guards and user context
- **TenantContextModule**: Manages organization scoping for token generation
- **ConfigModule**: Supplies Tiptap AI configuration and secrets

<Tip>
  Tokens are automatically scoped to the authenticated user's current organization, ensuring proper data isolation in multi-tenant environments.
</Tip>
