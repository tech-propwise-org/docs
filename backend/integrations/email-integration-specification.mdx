## Overview

The Email Integration system supports both **Gmail** and **Outlook** mail OAuth flows, enabling users to connect personal mailboxes and administrators to connect organization-wide mailboxes. This specification covers OAuth authorization, token management, access control, and API endpoints.

<Note>
Calendar OAuth is documented separately in the Calendar System specification. This document focuses exclusively on **mail** integration.
</Note>

## Integration Levels

Email integrations operate at two distinct levels:

<CardGroup cols={2}>
  <Card title="Personal Mailbox" icon="user">
    Connected by individual users for their own use. Each user can connect their own mailboxes independently.
  </Card>
  <Card title="Organization Mailbox" icon="building">
    Connected by system administrators or owners. Shared across the workspace and usable by admins/owners for sending.
  </Card>
</CardGroup>

### Level Characteristics

| Aspect | Personal (`PERSONAL`) | Organization (`ORGANIZATION`) |
|--------|----------------------|-------------------------------|
| **Database Field** | `user_id` NOT NULL | `user_id` NULL |
| **Connected By** | N/A | `connected_by_id` stores OAuth actor |
| **Access Control** | Owner only | `SYSTEM_ADMIN` or `SYSTEM_OWNER` |
| **Uniqueness** | Per user + org + provider | Per org + provider + email (lowercase) |
| **Default Flag** | Not applicable | Optional `is_default` (one per org) |

<Warning>
A mailbox email address cannot exist at both levels within the same organization. The system enforces cross-level uniqueness via database triggers.
</Warning>

## OAuth Flow

<Steps>
  <Step title="Initiate OAuth">
    User selects an organization and the SPA holds a tenant JWT.
    
    **Personal mailbox:**
    ```http
    POST /v1/email-integrations/oauth/:provider/authorize
    ```
    
    **Organization mailbox:**
    ```http
    POST /v1/email-integrations/organization/oauth/:provider/authorize
    ```
    
    Provider must be `gmail` or `outlook`. Organization endpoint requires `SYSTEM_ADMIN` or `SYSTEM_OWNER` role.
  </Step>

  <Step title="Receive Authorization URL">
    Backend returns an authorization URL with HMAC-signed state:
    
    ```json
    {
      "authorizationUrl": "https://accounts.google.com/o/oauth2/v2/auth?..."
    }
    ```
    
    The signed state payload includes:
    - `userId` - OAuth actor
    - `organizationId` - Current organization
    - `tokenVersion` - Auth epoch for invalidation
    - `provider` - `gmail` or `outlook`
    - `level` - `PERSONAL` or `ORGANIZATION` (omitted = `PERSONAL` for legacy)
  </Step>

  <Step title="User Authorization">
    Browser opens the authorization URL in a popup window. User authenticates with the provider and grants permissions.
  </Step>

  <Step title="Provider Callback">
    Provider redirects to public callback endpoint:
    
    - **Gmail:** `GET /auth/gmail/callback`
    - **Outlook:** `GET /auth/outlook/callback`
    
    Backend verifies signed state, exchanges authorization code for tokens.
  </Step>

  <Step title="Gmail Access Verification">
    <Tip>
    This step applies **only to Gmail** integrations and prevents saving integrations for accounts without Gmail API access.
    </Tip>
    
    Before persisting the integration, backend calls:
    
    ```http
    GET https://gmail.googleapis.com/gmail/v1/users/me/profile
    ```
    
    If Google returns **400** or **403**, the integration is **not saved**. The callback HTML posts an `oauth-error` message with:
    
    ```typescript
    {
      error: "GMAIL_ACCESS_BLOCKED",
      message: "This Google account cannot access Gmail API"
    }
    ```
  </Step>

  <Step title="Callback Access Re-check">
    Before saving tokens, backend re-validates OAuth actor against current state:
    
    - User is active (`User.isActive`)
    - User's `tokenVersion` matches signed state
    - Active `user_org_roles` row exists for `(userId, organizationId)`
    - For `ORGANIZATION` level: actor has `system.admin` or `system.owner`
    
    If validation fails, popup receives `oauth-error` with `OAUTH_CALLBACK_TENANT_ACCESS_DENIED_MESSAGE`.
  </Step>

  <Step title="Save Integration">
    `connectInTransaction` runs:
    
    - Encrypts access and refresh tokens (AES-256-GCM)
    - Normalizes email to lowercase
    - Checks cross-level mailbox conflicts
    - Checks redundant reconnect conditions
    - Upserts `email_integrations` row
  </Step>

  <Step title="PostMessage Response">
    Callback HTML sends `postMessage` to SPA with `app.postMessageTargetOrigin`:
    
    ```javascript
    window.opener.postMessage({
      type: 'oauth-success',
      level: 'PERSONAL' // or 'ORGANIZATION'
    }, targetOrigin);
    ```
    
    <Warning>
    Never uses `*` as target origin. Always validates against `FRONTEND_URL`.
    </Warning>
  </Step>

  <Step title="SPA Updates">
    Frontend invalidates `queryKeys.emailIntegrations` and refetches:
    
    - `GET /v1/email-integrations/send-accounts` (composer)
    - Integration lists refresh automatically
    
    Validates `event.source` is the OAuth popup before processing message.
  </Step>
</Steps>

### OAuth State Security

- **TTL:** 15 minutes (`EMAIL_OAUTH_STATE_TTL_MS`)
- **Signing:** `SignedPayloadService` with purpose `email.oauth.state`
- **Token Version:** Invalidates open popups after password reset or logout-all

### Mailbox Conflict Rules

<AccordionGroup>
  <Accordion title="Cross-Level Conflicts">
    A mailbox address cannot exist at both `PERSONAL` and `ORGANIZATION` levels in the same organization:
    
    - Attempting to connect a **personal** mailbox fails if an active **organization** row has the same normalized email
    - Attempting to connect an **organization** mailbox fails if any active **personal** row has that email
    - Provider is intentionally ignored for this rule
    - Enforced by `email_integrations_cross_level_mailbox_conflict_trigger`
  </Accordion>

  <Accordion title="Redundant Reconnect Prevention">
    OAuth fails if the mailbox is already connected and healthy:
    
    **Healthy integration:** Active, no `last_sync_error_at`, access token present, `token_expires_at` beyond buffer
    
    **Unhealthy integration:** Allows re-OAuth to repair:
    - Inactive row
    - Has sync error
    - Missing access token
    - Unknown expiry
    - Token expired or inside buffer (`EMAIL_RECONNECT_ACCESS_TOKEN_BUFFER_MINUTES`)
    
    Switching to a different email address saves only if it doesn't conflict with the other level.
  </Accordion>

  <Accordion title="Same-Level Uniqueness">
    **Personal:** Partial unique on `(user_id, organization_id, provider)` where `is_deleted = false` and `level = 'PERSONAL'`
    
    **Organization:** Partial unique on `(organization_id, provider, lower(email))` where `is_deleted = false` and `level = 'ORGANIZATION'`
    
    Multiple users may each connect the same mailbox address as personal integrations.
  </Accordion>
</AccordionGroup>

**Gmail:**
- `gmail.send` - Outbound email sending
- `gmail.metadata` - Profile access verification
- `userinfo.email` - Profile email address

**Outlook:**
- Standard mail scopes (see provider documentation)

## API Reference

### List Personal Integrations

```http
GET /v1/email-integrations
```

Returns personal mailboxes for the current user only. Never lists another member's personal mailboxes, even for `SYSTEM_OWNER` or `SYSTEM_ADMIN`.

**Query Parameters:**

| Parameter | Type | Default | Max | Description |
|-----------|------|---------|-----|-------------|
| `limit` | integer | 20 | 50 | Page size |
| `cursor` | string | - | - | Opaque pagination cursor |

**Response:**

```json
{
  "items": [
    {
      "id": "uuid",
      "level": "PERSONAL",
      "provider": "gmail",
      "email": "user@example.com",
      "connectedAt": "2024-01-15T10:30:00Z"
    }
  ],
  "nextCursor": "eyJjb25uZWN0ZWRBdCI6IjIwMjQtMD..."
}
```

**Ordering:** `connected_at DESC`, `id ASC` (keyset pagination)

### List Organization Integrations

```http
GET /v1/email-integrations/organization
```

Returns organization-level mailboxes. Requires `SYSTEM_ADMIN` or `SYSTEM_OWNER`.

**Query Parameters:** Same as personal list

**Response:**

```json
{
  "items": [
    {
      "id": "uuid",
      "level": "ORGANIZATION",
      "provider": "outlook",
      "email": "support@company.com",
      "isDefault": true,
      "connectedBy": {
        "id": "uuid",
        "email": "admin@company.com",
        "firstName": "Admin",
        "lastName": "User"
      },
      "connectedAt": "2024-01-15T10:30:00Z"
    }
  ],
  "nextCursor": "..."
}
```

**Ordering:** `is_default DESC`, `connected_at DESC`, `id ASC` (offset pagination)

<Note>
The `connectedBy` field is populated when the `connectedBy` relation is loaded. It shows which admin/owner ran the OAuth flow.
</Note>

### Get Send Accounts

```http
GET /v1/email-integrations/send-accounts
```

Returns mailboxes available as **From** addresses in the composer. Returns two separate streams:

- `personal` - Caller's personal integrations
- `organization` - Organization mailboxes (only if caller has `SYSTEM_ADMIN` or `SYSTEM_OWNER`)

**Query Parameters:**

| Parameter | Type | Description |
|-----------|------|-------------|
| `limit` | integer | Page size (default 20, max 50) |
| `cursorPersonal` | string | Personal stream cursor |
| `cursorOrganization` | string | Organization stream cursor |

<Warning>
When one stream is exhausted, pass the literal string `__exhausted__` for that stream's cursor on subsequent requests. This prevents the server from restarting that stream.
</Warning>

**Response:**

```json
{
  "personal": {
    "items": [...],
    "nextCursor": "..."
  },
  "organization": {
    "items": [...],
    "nextCursor": "..."
  }
}
```

### Get Integration by ID

```http
GET /v1/email-integrations/:id
```

Retrieves a specific integration. Access control:

- **Personal:** Returns 404 unless `user_id` matches JWT user
- **Organization:** Returns 404 unless caller has `SYSTEM_ADMIN` or `SYSTEM_OWNER`

**Response:**

```json
{
  "id": "uuid",
  "level": "PERSONAL",
  "provider": "gmail",
  "email": "user@example.com",
  "connectedAt": "2024-01-15T10:30:00Z",
  "tokenExpiresAt": "2024-01-15T11:30:00Z"
}
```

### Update Integration

```http
PUT /v1/email-integrations/:id
```

Updates integration metadata. Same access control as GET.

**Request Body:**

```json
{
  "metadata": {
    "customField": "value"
  }
}
```

### Delete Integration

```http
DELETE /v1/email-integrations/:id
```

Soft-deletes an integration (sets `is_deleted = true`). Same access control as GET.

**Response:** `204 No Content`

### Send Email

```http
POST /v1/email-integrations/:id/send
```

Sends an email using the specified integration.

**Access Control:**

- **Personal:** Only mailbox owner (`user_id` matches JWT user)
- **Organization:** `SYSTEM_ADMIN` or `SYSTEM_OWNER` only

**Request Body:**

```json
{
  "to": ["recipient@example.com"],
  "cc": ["cc@example.com"],
  "bcc": ["bcc@example.com"],
  "subject": "Email subject",
  "body": "Email body content",
  "isHtml": true
}
```

**Response:**

```json
{
  "messageId": "provider-message-id",
  "sentAt": "2024-01-15T10:35:00Z"
}
```

## Data Model

### EmailIntegration Entity

```typescript
{
  id: string; // UUID
  level: 'PERSONAL' | 'ORGANIZATION';
  provider: 'gmail' | 'outlook';
  email: string; // Normalized to lowercase
  
  // Personal integrations
  userId?: string; // NOT NULL for PERSONAL
  
  // Organization integrations
  connectedById?: string; // NOT NULL for ORGANIZATION
  isDefault?: boolean; // ORGANIZATION only
  
  // OAuth tokens (encrypted AES-256-GCM)
  encryptedAccessToken: string;
  encryptedRefreshToken?: string;
  tokenExpiresAt?: Date;
  
  // Sync tracking
  lastSyncAt?: Date;
  lastSyncErrorAt?: Date;
  
  // Metadata
  connectedAt: Date;
  isDeleted: boolean;
  organizationId: string;
}
```

### Database Constraints

<AccordionGroup>
  <Accordion title="Personal Uniqueness">
    ```sql
    CREATE UNIQUE INDEX email_integrations_personal_unique
    ON email_integrations (user_id, organization_id, provider)
    WHERE is_deleted = false AND level = 'PERSONAL';
    ```
  </Accordion>

  <Accordion title="Organization Uniqueness">
    ```sql
    CREATE UNIQUE INDEX email_integrations_org_mailbox_unique
    ON email_integrations (organization_id, provider, lower(email))
    WHERE is_deleted = false AND level = 'ORGANIZATION';
    ```
  </Accordion>

  <Accordion title="Organization Default">
    ```sql
    CREATE UNIQUE INDEX email_integrations_org_default_unique
    ON email_integrations (organization_id)
    WHERE is_deleted = false 
      AND level = 'ORGANIZATION' 
      AND is_default = true;
    ```
  </Accordion>

  <Accordion title="Cross-Level Conflict Trigger">
    ```sql
    CREATE TRIGGER email_integrations_cross_level_mailbox_conflict_trigger
    BEFORE INSERT OR UPDATE ON email_integrations
    FOR EACH ROW EXECUTE FUNCTION 
    prevent_email_integration_cross_level_mailbox_conflicts();
    ```
    
    Takes advisory lock per `(organization_id, lower(email))` and checks opposite level.
  </Accordion>

  <Accordion title="Level Constraints">
    ```sql
    -- PERSONAL must have user_id
    CHECK (level != 'PERSONAL' OR user_id IS NOT NULL)
    
    -- ORGANIZATION must have connected_by_id
    CHECK (level != 'ORGANIZATION' OR connected_by_id IS NOT NULL)
    
    -- ORGANIZATION cannot have user_id
    CHECK (level != 'ORGANIZATION' OR user_id IS NULL)
    ```
  </Accordion>
</AccordionGroup>

## Token Management

### Encryption

Tokens are encrypted using AES-256-GCM via `EncryptionService`:

- **Key:** `ENCRYPTION_KEY` environment variable (64 hex characters)
- **Fields:** `encrypted_access_token`, `encrypted_refresh_token`
- **Algorithm:** AES-256-GCM with unique IV per token

### Refresh Strategy

<Steps>
  <Step title="Check Expiry Buffer">
    Before using an access token, check if `token_expires_at` is within the configured buffer (`EMAIL_RECONNECT_ACCESS_TOKEN_BUFFER_MINUTES`).
  </Step>

  <Step title="Attempt Refresh">
    If expired or near expiry, use `encrypted_refresh_token` to obtain new access token from provider.
  </Step>

  <Step title="Update Database">
    Save new `encrypted_access_token` and `token_expires_at`. If refresh fails, set `last_sync_error_at`.
  </Step>

  <Step title="Require Re-OAuth">
    If refresh token is invalid or expired, mark integration as unhealthy. User must re-run OAuth flow.
  </Step>
</Steps>

### Health Check

An integration is considered **healthy** when:

<Check>Active (`is_deleted = false`)</Check>
<Check>No sync error (`last_sync_error_at` is null)</Check>
<Check>Access token present</Check>
<Check>`token_expires_at` is known and beyond buffer</Check>

Use `integrationNeedsRepairOrTokenRefresh()` to evaluate health status.

## Error Handling

### OAuth Callback Errors

| Error Code | Cause | User Action |
|------------|-------|-------------|
| `GMAIL_ACCESS_BLOCKED` | Gmail API access verification failed | Check Google account settings, ensure Gmail is enabled |
| `OAUTH_CALLBACK_TENANT_ACCESS_DENIED` | User lost org access during OAuth | Re-check organization membership |
| Token exchange failure | Provider returned error | Check provider credentials, retry |
| State verification failure | Invalid or expired state | Restart OAuth flow |

### PostMessage Error Format

```javascript
{
  type: 'oauth-error',
  error: 'GMAIL_ACCESS_BLOCKED',
  message: 'This Google account does not have Gmail API access'
}
```

### API Errors

<Tabs>
  <Tab title="404 Not Found">
    Integration ID doesn't exist or caller lacks access:
    
    ```json
    {
      "statusCode": 404,
      "message": "Integration not found"
    }
    ```
  </Tab>
  
  <Tab title="403 Forbidden">
    Insufficient permissions for organization operations:
    
    ```json
    {
      "statusCode": 403,
      "message": "Requires SYSTEM_ADMIN or SYSTEM_OWNER role"
    }
    ```
  </Tab>
  
  <Tab title="409 Conflict">
    Cross-level mailbox conflict or redundant reconnect:
    
    ```json
    {
      "statusCode": 409,
      "message": "This mailbox is already connected at a different level"
    }
    ```
  </Tab>
</Tabs>

### Frontend Error Handling

```typescript
// Validate postMessage source
if (event.source !== oauthPopupWindow) {
  return; // Ignore messages from unknown sources
}

// Handle Gmail access blocked
if (event.data.error === 'GMAIL_ACCESS_BLOCKED') {
  showAlert({
    title: 'Gmail Access Blocked',
    description: event.data.message
  });
}

// Always refetch on popup close
oauthPopupWindow.addEventListener('close', () => {
  queryClient.invalidateQueries(queryKeys.emailIntegrations);
});
```

## Best Practices

  <Card title="Security" icon="shield">

    - Rotate `ENCRYPTION_KEY` securely

  <Card title="Performance" icon="gauge-high">

    - Handle popup blockers gracefully

  <Card title="UX" icon="users">
    - Indicate default organization mailbox
