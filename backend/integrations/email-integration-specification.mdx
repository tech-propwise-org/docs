## OAuth Flow Overview

The email integration system supports two connection levels:
- **PERSONAL**: Individual user mailboxes connected by the user themselves
- **ORGANIZATION**: Workspace-wide shared mailboxes requiring admin/owner privileges

<Steps>

<Step title="User authenticates with tenant JWT">
User has selected an organization and the SPA holds a tenant JWT for authenticated API access.
</Step>

<Step title="Initiate OAuth authorization">
Choose the appropriate endpoint based on integration level:

**Personal Mailbox:**
```http
POST /v1/email-integrations/oauth/:provider/authorize
```

**Organization Mailbox:**
```http
POST /v1/email-integrations/organization/oauth/:provider/authorize
```

<Info>
The `:provider` parameter accepts `gmail` or `outlook`. Organization-level OAuth requires `SYSTEM_ADMIN` or `SYSTEM_OWNER` role via `@CheckAccess`.
</Info>
</Step>

<Step title="Receive authorization URL">
Backend returns an authorization URL with HMAC-signed state:

```json
{
  "authorizationUrl": "https://accounts.google.com/o/oauth2/v2/auth?..."
}
```

The signed `state` parameter includes:
- `userId`
- `organizationId`
- `tokenVersion` (invalidates OAuth popups after password reset/logout)
- `provider`
- `level` (optional: `PERSONAL` | `ORGANIZATION`, defaults to `PERSONAL` for legacy tokens)

<Warning>
The `tokenVersion` ensures that password resets or auth epoch changes invalidate already-open OAuth popups before callback persistence.
</Warning>
</Step>

<Step title="Open OAuth popup">
Browser opens the authorization URL in a popup window where the user grants permissions.
</Step>

<Step title="Provider callback">
Provider redirects to public callback endpoints (no `/v1` prefix):

- **Gmail**: `GET /auth/gmail/callback`
- **Outlook**: `GET /auth/outlook/callback`
</Step>

<Step title="Backend validates and exchanges token">
Backend performs the following operations:

1. Verifies HMAC-signed `state` parameter
2. Exchanges authorization `code` via provider driver
3. **Gmail only**: Verifies API access (see Gmail Access Verification below)
4. Re-checks OAuth actor authorization (see Callback Access Re-check below)
5. Encrypts and stores tokens in transaction
</Step>

<Step title="Return result via postMessage">
Callback HTML uses `postMessage` with `app.postMessageTargetOrigin` (from `FRONTEND_URL`) to communicate result to the SPA.

<Warning>
Never use `*` as the target origin. Always validate `event.source` matches the OAuth popup.
</Warning>

Payload includes:
- Success/error status
- Optional `level` field for the SPA
- Error code and message if applicable
</Step>

<Step title="SPA handles callback">
On success, the SPA should:
1. Invalidate `queryKeys.emailIntegrations` cache
2. Refetch `GET /v1/email-integrations/send-accounts` to update composer
3. Handle any error dialogs (e.g., Gmail access blocked)
4. Refetch when popup closes to catch dropped messages
</Step>

</Steps>

## Gmail Access Verification

Before persisting a Gmail integration, the backend verifies API access to prevent saving broken integrations.

<CodeGroup>

```typescript Backend Constant
// src/common/constants/email-oauth.constants.ts
export const EMAIL_OAUTH_GMAIL_ACCESS_BLOCKED_CODE = 'GMAIL_ACCESS_BLOCKED';
```

```typescript Frontend Constant
// email-integration.api.ts
export const EMAIL_OAUTH_GMAIL_ACCESS_BLOCKED_CODE = 'GMAIL_ACCESS_BLOCKED';
```

</CodeGroup>

### Verification Process

1. **When**: Immediately after `exchangeCode`, before `connectInTransaction`
2. **What**: Backend calls `GET .../gmail/v1/users/me/profile` with the new access token
3. **Why**: Some Google accounts authenticate via OAuth but don't expose Gmail API access
4. **Outcome**: 
   - If Google returns **400** or **403**, integration is **not persisted**
   - Callback HTML posts `oauth-error` with error code and message
   - Outlook skips this verification step

<Warning>
The constant name must stay synchronized between backend and frontend codebases to ensure proper error handling in the SPA.
</Warning>

### Required OAuth Scopes

Gmail OAuth must include:
- `gmail.send` - For outbound email sending
- `gmail.metadata` - For `users.getProfile` access probe
- `userinfo.email` - To read profile mailbox address

## Callback Access Re-check

Before token persistence, the backend re-validates OAuth actor authorization in the same database transaction.

### Validation Checks

<AccordionGroup>

<Accordion title="User Status Validation">
- `User.isActive` must be `true`
- `User.tokenVersion` must match version in signed state
- Active (non-deleted) `user_org_roles` row for `(userId, organizationId)` must exist
</Accordion>

<Accordion title="Organization-Level Additional Checks">
For `ORGANIZATION` level connections only, the actor must still hold:
- `system.admin` OR
- `system.owner`

Verified via role/permission join (same effective rule as `@CheckAccess` on org mailbox authorize).
</Accordion>

</AccordionGroup>

<Check>
Implemented in `AccessService.assertOAuthCallbackOrgAccessInTransaction`
</Check>

### Failure Handling

If validation fails:
- Popup receives `oauth-error` with `OAUTH_CALLBACK_TENANT_ACCESS_DENIED_MESSAGE`
- Tokens are **not** written to database
- OAuth flow must be restarted

Otherwise, `connectInTransaction` proceeds to encrypt tokens and upsert the integration row.

## Data Model and Constraints

### Integration Levels

<Tabs>
  <Tab title="PERSONAL">
    **Personal mailboxes** connected by individual users for their own use.
    
    - `user_id` NOT NULL
    - `connected_by_id` NULL
    - Unique constraint: `(user_id, organization_id, provider)` where `is_deleted = false` and `level = 'PERSONAL'`
    - Multiple users may connect the **same** mailbox address as PERSONAL in the same organization
  </Tab>
  
  <Tab title="ORGANIZATION">
    **Workspace mailboxes** shared across the organization, requiring admin privileges.
    
    - `user_id` NULL
    - `connected_by_id` NOT NULL (OAuth actor who connected the mailbox)
    - Unique constraint: `(organization_id, provider, lower(email))` where `is_deleted = false` and `level = 'ORGANIZATION'`
    - Optional `is_default` flag (at most one default per organization)
    - Mailbox `email` normalized to lowercase on write
  </Tab>
</Tabs>

### Cross-Level Uniqueness

A mailbox address cannot exist at both PERSONAL and ORGANIZATION levels within the same organization.

<Warning>
The provider is intentionally ignored for this cross-level conflict rule. An `example@company.com` Gmail integration blocks the same address as an Outlook integration at the opposite level.
</Warning>

**Enforcement:**

`connectInTransaction` rejects:
- **PERSONAL** OAuth when an active **ORGANIZATION** row has the same normalized `email`
- **ORGANIZATION** OAuth when any active **PERSONAL** row has the same normalized `email`

**Database trigger:**

Migration `Migration20260521171000_PreventEmailIntegrationCrossLevelMailboxConflicts` installs `email_integrations_cross_level_mailbox_conflict_trigger`, which:
- Takes a transaction advisory lock per organization/mailbox
- Checks the opposite level before insert
- Prevents concurrent inserts from creating split personal/workspace connections

<Info>
Existing personal-personal duplicates (same address, different users) remain valid.
</Info>

### Redundant Reconnect Prevention

`connectInTransaction` rejects OAuth attempts when the integration is already healthy:

**Healthy criteria:**
- Row is active (not deleted)
- No `last_sync_error_at` timestamp
- Access token present
- `token_expires_at` beyond configured pre-expiry buffer

<Tip>
Buffer configured via `EMAIL_RECONNECT_ACCESS_TOKEN_BUFFER_MINUTES` and checked by `integrationNeedsRepairOrTokenRefresh`.
</Tip>

**Allowed reconnect scenarios:**
- Row is inactive
- Sync error exists
- Access token missing
- Token expiry unknown
- Token expired or inside buffer

**Result:** Redundant reconnect throws error, popup receives `oauth-error`. Switching to a **different** address saves only when the new address doesn't conflict with the other level.

### Token Storage

All tokens are encrypted at rest using AES-256-GCM:

- `encrypted_access_token`
- `encrypted_refresh_token`

**Encryption configuration:**
- Service: Shared `EncryptionService`
- Key: `ENCRYPTION_KEY` environment variable or `app.encryption.key` (64 hex characters)

### Display Fields

- `email` - Provider mailbox/profile email (normalized to lowercase on write)
- `provider` - `gmail` or `outlook`
- `level` - `PERSONAL` or `ORGANIZATION`
- `is_default` - Optional, organization-level only
- `connected_at` - Timestamp of initial connection
- `token_expires_at` - Access token expiration

## API Endpoints

### List Personal Integrations

```http
GET /v1/email-integrations
```

Returns **PERSONAL** mailboxes for the current user only. Never lists another member's personal mailboxes, even for `SYSTEM_OWNER`/`SYSTEM_ADMIN`.

**Query Parameters:**

| Parameter | Type | Default | Max | Description |
|-----------|------|---------|-----|-------------|
| `limit` | integer | 20 | 50 | Page size |
| `cursor` | string | - | - | Opaque pagination cursor |

**Response:**

```json
{
  "items": [
    {
      "id": "uuid",
      "email": "user@example.com",
      "provider": "gmail",
      "level": "PERSONAL",
      "connectedAt": "2024-01-15T10:30:00Z"
    }
  ],
  "nextCursor": "opaque_string_or_null"
}
```

<Info>
Ordering: `connected_at DESC`, `id ASC` (keyset pagination). Constants defined in `email-integration-list.constants.ts`: `EMAIL_INTEGRATION_PAGE_DEFAULT` (20) and `EMAIL_INTEGRATION_PAGE_MAX` (50).
</Info>

### List Send Accounts

```http
GET /v1/email-integrations/send-accounts
```

Returns mailboxes allowed as **From** addresses in the composer:
- `personal` stream: Caller's PERSONAL rows
- `organization` stream: ORGANIZATION rows (if caller has `SYSTEM_ADMIN` or `SYSTEM_OWNER`)

**Query Parameters:**

| Parameter | Type | Description |
|-----------|------|-------------|
| `cursorPersonal` | string | Pagination cursor for personal stream |
| `cursorOrganization` | string | Pagination cursor for organization stream |
| `limit` | integer | Page size (same bounds as `GET /`) |

**Stream exhaustion:**

When one stream is exhausted before the other, pass literal `__exhausted__` (`EMAIL_INTEGRATION_MAILBOX_STREAM_EXHAUSTED_QUERY` constant) for that stream's cursor on subsequent requests.

<Warning>
Failure to send `__exhausted__` will cause the server to restart the exhausted stream.
</Warning>

**Response:**

```json
{
  "personal": {
    "items": [...],
    "nextCursor": "cursor_or_null"
  },
  "organization": {
    "items": [...],
    "nextCursor": "cursor_or_null"
  }
}
```

**Ordering:**
- Personal: Keyset pagination (`connected_at DESC`, `id ASC`)
- Organization: Offset pagination (`is_default DESC`, `connected_at DESC`, `id ASC`)

### List Organization Integrations

```http
GET /v1/email-integrations/organization
```

Returns **ORGANIZATION** mailboxes only. Requires `SYSTEM_ADMIN` or `SYSTEM_OWNER` via `@CheckAccess`.

**Query Parameters:**

| Parameter | Type | Default | Max | Description |
|-----------|------|---------|-----|-------------|
| `limit` | integer | 20 | 50 | Page size |
| `cursor` | string | - | - | Pagination cursor |

**Response:**

```json
{
  "items": [
    {
      "id": "uuid",
      "email": "support@company.com",
      "provider": "outlook",
      "level": "ORGANIZATION",
      "isDefault": true,
      "connectedBy": {
        "id": "user_uuid",
        "email": "admin@company.com",
        "firstName": "Admin",
        "lastName": "User"
      },
      "connectedAt": "2024-01-15T10:30:00Z"
    }
  ],
  "nextCursor": "opaque_string_or_null"
}
```

<Check>
Uses `findOrganizationMailboxesPage` with `populate(['connectedBy'])` to include OAuth actor details.
</Check>

### Get Integration by ID

```http
GET /v1/email-integrations/:id
```

Retrieve a specific email integration by UUID.

**Access control:**
- **PERSONAL**: Caller must be the mailbox owner (`user_id` matches JWT user)
- **ORGANIZATION**: Caller must have `SYSTEM_ADMIN` or `SYSTEM_OWNER`

<Note>
Missing access returns **404** (indistinguishable from unknown ID). Implemented via `EmailIntegrationService.getAccessibleIntegrationFindWhere` + `assertCanAccessIntegration`.
</Note>

### Update Integration

```http
PUT /v1/email-integrations/:id
```

Update integration metadata (e.g., display preferences).

**Request Body:**

```json
{
  "metadata": {
    "displayName": "Support Mailbox"
  }
}
```

<Info>
SPA types in `email-integration.api.ts` should mirror `UpdateEmailIntegrationDto.metadata` structure.
</Info>

**Access control:** Same as GET (personal owner or org admin/owner).

### Delete Integration

```http
DELETE /v1/email-integrations/:id
```

Soft-delete an email integration (sets `is_deleted = true`).

**Access control:** Same as GET (personal owner or org admin/owner).

### Send Email

```http
POST /v1/email-integrations/:id/send
```

Send an email using the specified integration as the From address.

**Access control:**
- **PERSONAL**: Only the mailbox owner (`user_id` matches JWT user)
- **ORGANIZATION**: `SYSTEM_ADMIN` or `SYSTEM_OWNER` only

**Request Body:**

```json
{
  "to": ["recipient@example.com"],
  "subject": "Email subject",
  "body": "Email body content",
  "cc": ["cc@example.com"],
  "bcc": ["bcc@example.com"]
}
```

<Check>
Enforced by `EmailIntegrationService.assertCanAccessIntegration` + `sendWithIntegration`.
</Check>

### Start OAuth (Personal)

```http
POST /v1/email-integrations/oauth/:provider/authorize
```

Initiate OAuth flow for a personal mailbox connection.

**Path Parameters:**
- `provider` - `gmail` or `outlook`

**Response:**

```json
{
  "authorizationUrl": "https://accounts.google.com/o/oauth2/v2/auth?..."
}
```

### Start OAuth (Organization)

```http
POST /v1/email-integrations/organization/oauth/:provider/authorize
```

Initiate OAuth flow for an organization mailbox connection.

**Path Parameters:**
- `provider` - `gmail` or `outlook`

**Authorization:** Requires `SYSTEM_ADMIN` or `SYSTEM_OWNER` (`@CheckAccess`)

**Response:**

```json
{
  "authorizationUrl": "https://accounts.microsoft.com/oauth2/v2.0/authorize?..."
}
```

## DTO Reference

### EmailIntegrationDto

```typescript
{
  id: string;                    // UUID
  email: string;                 // Normalized mailbox address
  provider: 'gmail' | 'outlook';
  level: 'PERSONAL' | 'ORGANIZATION';
  isDefault?: boolean;           // Organization-level only
  connectedAt: string;           // ISO 8601 timestamp
  user?: UserDto;                // Only when relation populated
  connectedBy?: UserDto;         // Only when relation populated (org-level)
  tokenExpiresAt?: string;       // ISO 8601 timestamp
  lastSyncErrorAt?: string;      // ISO 8601 timestamp
}
```

<Warning>
`EmailIntegrationDto.user` is included only when the `user` relation is **populated**. Never deserialize `UserDto` from an uninitialized MikroORM reference. `GET /v1/email-integrations` usually returns integrations **without** nested `user`.
</Warning>

## Environment Configuration

### Redirect URIs

<Tabs>
  <Tab title="Gmail">
    **Primary:** `GMAIL_REDIRECT_URI`  
    **Override:** `GMAIL_EMAIL_REDIRECT_URI`
    
    **Default:**
    ```bash
    ${BACKEND_URL or http://localhost:$PORT}/auth/gmail/callback
    ```
    
    Accessed via: `app.gmail.emailRedirectUri`
  </Tab>
  
  <Tab title="Outlook">
    **Primary:** `OUTLOOK_REDIRECT_URI`  
    **Override:** `OUTLOOK_EMAIL_REDIRECT_URI`
    
    **Default:**
    ```bash
    ${BACKEND_URL or http://localhost:$PORT}/auth/outlook/callback
    ```
    
    Accessed via: `app.outlook.emailRedirectUri`
  </Tab>
</Tabs>

<Info>
`PORT` defaults to `3002` in app config and `.env.example`. Set `BACKEND_URL` when the API public origin differs from `localhost:$PORT`.
</Info>

<Warning>
Register these exact redirect URIs on:
- Google Cloud OAuth client (for `GMAIL_*` credentials)
- Azure app registration (for `OUTLOOK_*` credentials)

Mismatched redirect URIs will cause OAuth to fail.
</Warning>

**Implementation:**

`EmailOAuthCallbackController` serves:
- `GET /auth/gmail/callback`
- `GET /auth/outlook/callback`

These routes must match the `app.*.emailRedirectUri` defaults and environment configuration.

## Error Handling

### Callback Errors

Token exchange failures and IdP `error` query parameters are logged server-side with:
- Provider HTTP status
- Truncated body preview

The popup `postMessage` payload uses short, generic error strings for users — **not** raw Axios/`response.data` bodies.

### Common Error Codes

<CardGroup cols={2}>

<Card title="GMAIL_ACCESS_BLOCKED" icon="ban">
Gmail API access denied despite successful OAuth authentication
</Card>

<Card title="OAUTH_CALLBACK_TENANT_ACCESS_DENIED" icon="shield-xmark">
User lost organization access or required role during OAuth flow
</Card>

<Card title="MAILBOX_CONFLICT" icon="triangle-exclamation">
Attempted to connect mailbox that exists at opposite level
</Card>

<Card title="REDUNDANT_RECONNECT" icon="rotate">
Attempted to reconnect already-healthy integration
</Card>

</CardGroup>

### SPA Error Handling

When receiving `oauth-error` postMessage:

```typescript
if (error === EMAIL_OAUTH_GMAIL_ACCESS_BLOCKED_CODE) {
  // Show AlertDialog explaining Gmail access issue
  showGmailAccessBlockedDialog(message);
} else {
  // Show generic error message
  showErrorToast(message);
}
```

<Tip>
Always refetch when the popup closes to update UI in case the postMessage was dropped (e.g., LAN vs `localhost` backend URL mismatch).
</Tip>

## Configuration Constants

### OAuth State TTL

```typescript
EMAIL_OAUTH_STATE_TTL_MS = 15 minutes
```

Signed OAuth state expires after 15 minutes, aligned with calendar UX expectations.

### Token Refresh Buffer

```typescript
EMAIL_RECONNECT_ACCESS_TOKEN_BUFFER_MINUTES
```

Grace period before token expiry when reconnect is allowed. Used by `integrationNeedsRepairOrTokenRefresh` to determine integration health.

### Pagination Defaults

```typescript
// email-integration-list.constants.ts
EMAIL_INTEGRATION_PAGE_DEFAULT = 20
EMAIL_INTEGRATION_PAGE_MAX = 50
```

### Stream Exhaustion Marker


Literal string sent as cursor when a mailbox stream (personal or organization) has no more pages.

## Security Considerations

<AccordionGroup>

<Accordion title="Token Encryption">
All OAuth tokens are encrypted at rest using AES-256-GCM via shared `EncryptionService`. The encryption key must be 64 hex characters and properly secured in production environments.
</Accordion>

<Accordion title="State Signing">
OAuth state parameters are HMAC-signed via `SignedPayloadService` with purpose `email.oauth.state`. This prevents tampering and ensures callbacks can only be completed with valid, unmodified state.
</Accordion>

<Accordion title="Token Version Validation">
The `tokenVersion` in signed state prevents OAuth flows from completing after password resets, logout-all events, or auth epoch bumps. Callbacks re-verify this version before persisting tokens.
</Accordion>

<Accordion title="postMessage Target Origin">
Callback HTML always uses `app.postMessageTargetOrigin` (from `FRONTEND_URL`) — never `*`. The SPA must validate `event.source` matches the OAuth popup.
</Accordion>

<Accordion title="Access Control">
Personal integrations are strictly isolated per user. Organization integrations require admin/owner roles checked at authorization time and re-verified at callback time within the persistence transaction.
</Accordion>

<Accordion title="Cross-Level Conflict Prevention">
Database trigger with advisory locks prevents race conditions when concurrent OAuth flows attempt to create conflicting personal/organization integrations for the same mailbox.
</Accordion>

</AccordionGroup>
