## OAuth Flow Overview

The email integration OAuth flow supports both personal and organization-level mailbox connections through a tenant-scoped session system.

### Flow Steps

<Steps>
<Step title="User Selection">
User selects an organization and the SPA holds a **tenant JWT** for authentication.
</Step>

<Step title="OAuth Authorization Request">
Choose the appropriate endpoint based on integration level:
- **Personal**: `POST /v1/email-integrations/oauth/:provider/authorize`
- **Organization**: `POST /v1/email-integrations/organization/oauth/:provider/authorize`

Where `provider` is either `gmail` or `outlook`.
</Step>

<Step title="Authorization URL Generation">
Backend returns `{ authorizationUrl }` containing:
- Google/Microsoft authorize URL
- HMAC-signed `state` parameter with payload including:
  - `userId`
  - `organizationId` 
  - `tokenVersion`
  - `provider`
  - `level` (PERSONAL or ORGANIZATION)
</Step>

<Step title="Provider Authentication">
Browser opens the authorization URL in a popup window for user authentication.
</Step>

<Step title="Callback Processing">
Provider redirects to public callback endpoints:
- `GET /auth/gmail/callback`
- `GET /auth/outlook/callback`

Backend verifies the signed `state` and exchanges the authorization `code`.
</Step>

<Step title="Access Verification">
For Gmail integrations, backend performs access verification by calling `GET .../gmail/v1/users/me/profile` to ensure the account has Gmail API access.
</Step>

<Step title="Token Storage">
After successful verification, tokens are encrypted and stored in the database with appropriate uniqueness constraints.
</Step>

<Step title="Response Handling">
Callback HTML uses `postMessage` to communicate results back to the parent window.
</Step>
</Steps>

## Integration Levels

<Tabs>
<Tab title="Personal Integration">
- **Level**: `EmailIntegrationLevel.PERSONAL`
- **Access**: Available to any authenticated user
- **Storage**: `user_id` NOT NULL, `connected_by_id` NULL
- **Uniqueness**: `(user_id, organization_id, provider)` where not deleted
- **Usage**: User's personal mailbox for sending/receiving emails
</Tab>

<Tab title="Organization Integration">
- **Level**: `EmailIntegrationLevel.ORGANIZATION`  
- **Access**: Requires `SYSTEM_ADMIN` or `SYSTEM_OWNER` role
- **Storage**: `user_id` NULL, `connected_by_id` NOT NULL (OAuth actor)
- **Uniqueness**: `(organization_id, provider, lower(email))` where not deleted
- **Usage**: Shared workspace mailbox accessible to authorized users
</Tab>
</Tabs>

### Cross-Level Conflict Prevention

<Warning>
A mailbox email address cannot exist at both PERSONAL and ORGANIZATION levels within the same organization. The system enforces this through database triggers and application logic.
</Warning>

## Gmail Access Verification

Gmail integrations require additional verification to ensure API access is available.

### Verification Process

<CodeGroup>
```typescript Verification Check
// Called immediately after token exchange, before persistence
const profileResponse = await gmail.users.getProfile({
  userId: 'me',
  access_token: newAccessToken
});

// If this fails with 400/403, integration is not saved
```
</CodeGroup>

<Info>
Gmail OAuth must include both `gmail.send` and `gmail.metadata` scopes. The `gmail.metadata` scope enables the profile access verification, while `userinfo.email` provides the mailbox address.
</Info>

### Error Handling

When Gmail access is blocked:
- Integration is **not** persisted to database
- Callback returns `oauth-error` with `EMAIL_OAUTH_GMAIL_ACCESS_BLOCKED_CODE`
- Frontend displays appropriate error dialog

## Environment Configuration

### Redirect URIs

<AccordionGroup>
<Accordion title="Gmail Configuration">
- **Environment Variable**: `GMAIL_REDIRECT_URI`
- **Default**: `${BACKEND_URL}/auth/gmail/callback`
- **Override**: `GMAIL_EMAIL_REDIRECT_URI` (optional)
- **Fallback**: `http://localhost:${PORT}/auth/gmail/callback`
</Accordion>

<Accordion title="Outlook Configuration">
- **Environment Variable**: `OUTLOOK_REDIRECT_URI`  
- **Default**: `${BACKEND_URL}/auth/outlook/callback`
- **Override**: `OUTLOOK_EMAIL_REDIRECT_URI` (optional)
- **Fallback**: `http://localhost:${PORT}/auth/outlook/callback`
</Accordion>
</AccordionGroup>

<Tip>
Register these exact redirect URIs in your Google Cloud OAuth client and Azure app configurations.
</Tip>

## API Endpoints

### List Endpoints

<CardGroup cols={2}>
<Card title="Personal Integrations" icon="user">
`GET /v1/email-integrations`

Lists personal mailboxes for the current user only. Includes pagination support with `nextCursor`.
</Card>

<Card title="Organization Integrations" icon="building">
`GET /v1/email-integrations/organization`

Lists organization mailboxes (admin/owner access required). Includes `connectedBy` relation when populated.
</Card>
</CardGroup>

### Send Accounts Endpoint

```http
GET /v1/email-integrations/send-accounts
```

Returns mailboxes available as "From" addresses in the email composer:
- `personal`: Caller's personal integrations
- `organization`: Organization integrations (admin/owner only)

**Pagination Parameters:**
- `cursorPersonal`: Cursor for personal stream
- `cursorOrganization`: Cursor for organization stream
- `limit`: Items per page (default: 20, max: 50)

<Note>
When a stream is exhausted, pass `__exhausted__` as the cursor value to prevent restarting that stream.
</Note>

### Management Endpoints

<Tabs>
<Tab title="Get Integration">
```http
GET /v1/email-integrations/:id
```
Retrieve integration by ID with access control validation.
</Tab>

<Tab title="Update Integration">
```http
PUT /v1/email-integrations/:id
```
Update integration metadata and settings.
</Tab>

<Tab title="Delete Integration">
```http

```
Soft delete integration (sets `is_deleted = true`).
</Tab>

<Tab title="Send Email">
```http
POST /v1/email-integrations/:id/send
```
Send email using the specified integration.
</Tab>
</Tabs>

## Data Model

### Core Fields

```typescript
interface EmailIntegration {
  id: string;
  level: 'PERSONAL' | 'ORGANIZATION';
  provider: 'gmail' | 'outlook';
  email: string; // Normalized to lowercase
  
  // Personal integrations
  user_id?: string; // NOT NULL for PERSONAL
  
  // Organization integrations  
  connected_by_id?: string; // NOT NULL for ORGANIZATION
  is_default?: boolean; // Organization only
  
  // Token storage (encrypted)
  encrypted_access_token: string;
  encrypted_refresh_token?: string;
  token_expires_at?: Date;
  
  // Metadata
  organization_id: string;
  connected_at: Date;
  last_sync_error_at?: Date;
  is_deleted: boolean;
}
```

### Uniqueness Constraints

<Check>
**Personal Level**: `(user_id, organization_id, provider)` where `is_deleted = false` and `level = 'PERSONAL'`
</Check>

<Check>
**Organization Level**: `(organization_id, provider, lower(email))` where `is_deleted = false` and `level = 'ORGANIZATION'`
</Check>

<Check>
**Default Organization**: At most one `is_default = true` per `organization_id`
</Check>

### Security Features

- **Token Encryption**: All tokens stored using AES-256-GCM encryption
- **Cross-Level Prevention**: Database triggers prevent email conflicts between levels
- **Access Validation**: OAuth callback re-checks user permissions before token storage
- **State Signing**: OAuth state parameters signed with HMAC for tampering protection

## Error Handling

### OAuth Callback Errors

<Warning>
All OAuth errors are logged server-side with provider HTTP status and response previews. Client-side errors use generic, user-friendly messages.
</Warning>

**Common Error Codes:**
- `GMAIL_ACCESS_BLOCKED`: Gmail API access unavailable for authenticated account
- `OAUTH_CALLBACK_TENANT_ACCESS_DENIED`: User permissions changed during OAuth flow
- `oauth-error`: Generic OAuth flow failure

### Access Control

- **404 Response**: Returned when user lacks access to requested integration
- **Personal Access**: Only integration owner can access personal mailboxes
- **Organization Access**: Requires `SYSTEM_ADMIN` or `SYSTEM_OWNER` role
- **Send Access**: Same rules apply for email sending operations

## Frontend Integration

### PostMessage Handling

```javascript
// Validate message source and origin
if (event.source !== oauthPopup) return;
if (event.origin !== expectedOrigin) return;

// Handle success/error
switch (event.data.type) {
  case 'oauth-success':
    // Invalidate email integration queries
    queryClient.invalidateQueries(queryKeys.emailIntegrations);
    break;
  case 'oauth-error':
    if (event.data.error === 'GMAIL_ACCESS_BLOCKED') {
      // Show Gmail access blocked dialog
    }
    break;
}
```

<Tip>
Always invalidate email integration queries after successful OAuth to ensure UI reflects new connections.
</Tip>
