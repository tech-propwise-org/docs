This page lists all enum types used across the CRM module.

## Person and contact enums

```sql
-- Contact method preference
CREATE TYPE contact_method AS ENUM (
  'email', 'phone', 'whatsapp', 'sms', 'in_person'
);

-- Contact status in CRM workflow
CREATE TYPE contact_status AS ENUM (
  'active', 'inactive', 'churned', 'blocked'
);

-- Original source tracking
CREATE TYPE original_source AS ENUM (
  'website', 'social_media', 'referral', 'advertisement',
  'cold_call', 'walk_in', 'event', 'partner', 'other'
);

-- Timezone preference
CREATE TYPE timezone AS ENUM (
  'UTC', 'America/New_York', 'America/Chicago', 'America/Denver', 
  'America/Los_Angeles', 'Europe/London', 'Europe/Paris', 
  'Europe/Berlin', 'Asia/Dubai', 'Asia/Riyadh', 'Asia/Tokyo', 
  'Asia/Shanghai', 'Asia/Mumbai', 'Australia/Sydney'
);

-- Person merge status
CREATE TYPE merge_status AS ENUM (
  'pending', 'completed', 'failed', 'cancelled'
);

-- Merge operation status
CREATE TYPE person_merge_status AS ENUM (
  'initiated', 'in_progress', 'completed', 'failed', 'cancelled'
);

-- Merge operation type
CREATE TYPE merge_operation_type AS ENUM (
  'field_selection', 'contact_info_merge', 'channel_reassignment',
  'entity_reassignment', 'reference_update', 'cleanup'
);

-- Merge validation result
CREATE TYPE merge_validation_result AS ENUM (
  'valid', 'invalid_primary_person', 'invalid_secondary_person',
  'same_person', 'different_organization', 'already_merged'
);

-- Entity verification status
CREATE TYPE verification_status AS ENUM (
  'unverified', 'verified', 'rejected', 'pending'
);
```

<Note>
The Person entity uses a unified language array format (`UserLanguageEntry`) that includes both language code and proficiency level in a single entry, rather than separate language and proficiency enums.
</Note>
