The activity system uses a lightweight `activity` table as a timeline index. Full content is stored in separate channel tables. Notes are activities with `type='note'`.

## Architecture

```
ACTIVITY (lightweight timeline index)
├── target_entity_type → 'lead' | 'deal' | 'contact' | 'company'
├── target_entity_id → UUID of the target entity
├── contact_id (nullable) → for bubble-up aggregation
├── activity_type + reference_id → points to channel table
├── summary (for quick timeline display)
└── is_pinned, is_important
              │
              ▼ reference_id points to:
┌──────────┬──────────┬──────────┬──────────┬──────────┬──────────┐
│ call_log │ email_   │ sms_log  │ meeting  │ site_    │ note     │
│          │ log      │          │          │ visit    │          │
│ Content  │ Content  │ Content  │ Content  │ Content  │ Content  │
│ ONLY     │ ONLY     │ ONLY     │ ONLY     │ ONLY     │ ONLY     │
│ (no FKs) │ (no FKs) │ (no FKs) │ (no FKs) │ (no FKs) │ (no FKs) │
└──────────┴──────────┴──────────┴──────────┴──────────┴──────────┘
```

<Info>
  Outcome, duration, and other detailed fields are stored on the **channel
  tables**, not on `activity` itself. Activity is purely a lightweight index for
  timeline display.
</Info>

## Activity table

```
activity
├── target_entity_type (lead | deal | contact | company)
├── target_entity_id (UUID)
├── contact_id (UUID, nullable) — for bubble-up visibility
├── activity_type (call, email, sms, whatsapp, meeting, site_visit, note, task, other)
├── reference_id → channel table record (nullable for note/task)
├── summary (text for quick display: "Called +1234567890 • 5 min • Connected")
├── is_pinned, is_important
├── performed_by_id, created_by_id, created_at
```

### Activity type classification

- **Communication types** (bubble up): call, email, sms, whatsapp, meeting, site_visit
- **Internal types** (stay on entity): note, task

---

## Visibility rules

| Target type          | contactId logic                                    | Visible on                                                |
| -------------------- | -------------------------------------------------- | --------------------------------------------------------- |
| Lead                 | Auto-filled from `lead.person.contact` (if exists) | Lead page + Contact page (if contact exists)              |
| Deal (communication) | Required from UI (selected contact)                | Deal page + Contact page                                  |
| Deal (internal note) | `null`                                             | Deal page only                                            |
| Contact              | `contact.id`                                       | Contact page + Company pages (via `contact_company_role`) |
| Company              | `null`                                             | Company page only                                         |

### Visibility queries

```sql
-- Lead page timeline
WHERE target_entity_type = 'lead' AND target_entity_id = :leadId
ORDER BY is_pinned DESC, created_at DESC

-- Deal page timeline
WHERE target_entity_type = 'deal' AND target_entity_id = :dealId
ORDER BY is_pinned DESC, created_at DESC

-- Contact page timeline (aggregates from leads, deals, direct activities)
WHERE contact_id = :contactId
ORDER BY is_pinned DESC, created_at DESC

-- Company page timeline (direct + from all company contacts)
WHERE (target_entity_type = 'company' AND target_entity_id = :companyId)
   OR contact_id IN (
     SELECT contact_id FROM contact_company_role
     WHERE company_id = :companyId AND is_active = true
   )
ORDER BY is_pinned DESC, created_at DESC
```

## Activity creation workflow

<Steps>
<Step title="Create channel record">
Create the content record in the appropriate channel table (e.g., `call_log`). This stores the full data — phone number, direction, outcome, duration, notes.
</Step>
<Step title="Create activity record">
Create an `activity` record pointing to the channel record via `reference_id`. The system auto-fills `contact_id` for bubble-up visibility based on the target entity.

```
activity:
├── target_entity_type: 'lead'
├── target_entity_id: lead-123
├── contact_id: contact-456 (auto-filled from lead.person.contact)
├── activity_type: 'call'
├── reference_id: call-log-uuid
├── summary: "Called +1234567890 • 5 min • Connected"
```

</Step>
<Step title="Timeline display">
The timeline UI shows `activity.summary` (fast query, ordered by `created_at`). Clicking to expand fetches full channel details via `reference_id` on demand.
</Step>
</Steps>

## Contact ID resolution logic

The system automatically populates `contactId` based on the target entity:

```typescript
// Company → null (no bubble up)
if (targetType === "company") return null;

// Contact → use contact itself
if (targetType === "contact") return targetId;

// Lead → get from lead.person.contact (if exists)
if (targetType === "lead") {
  const lead = await em.findOne(Lead, targetId, {
    populate: ["person.contact"],
  });
  return lead?.person?.contact?.id ?? null;
}

// Deal (communication) → require selectedContactId from UI
// Deal (internal note) → null
if (targetType === "deal") {
  if (isCommunicationActivity(activityType)) {
    if (!selectedContactId) throw new BadRequestException("Contact required");
    return selectedContactId;
  }
  return null;
}
```

## Notes system

Notes are activities with `type='note'`. Polymorphism is handled by the Activity table, not the note table.

### Note table

```
note
├── title (optional)
├── body (text content)
├── attachments[] (JSONB array: {url, filename, size, mimeType})
├── is_pinned
├── is_private
├── organization_id, created_by_id, created_at
```

### Note creation

Creating a note involves two records:

1. **Create note** (content only): title, body, attachments, privacy settings
2. **Create activity**: `target_entity_type`, `target_entity_id`, `activity_type: 'note'`, `reference_id: note-uuid`

### Note visibility

Notes are "internal" activities that do **not** bubble up from Lead/Deal to Contact.

| Target  | Note visibility                                    |
| ------- | -------------------------------------------------- |
| Lead    | Lead page only (`contactId = null`, internal type) |
| Deal    | Deal page only (`contactId = null`, internal type) |
| Contact | Contact page + Company pages (`contactId` set)     |
| Company | Company page only (`contactId = null`)             |

---

## Notes query pattern

Notes are retrieved via the activity table, not directly from the `note` table:

```sql
SELECT n.*, a.target_entity_type, a.target_entity_id
FROM activity a
JOIN note n ON a.reference_id = n.id
WHERE a.target_entity_type = :type AND a.target_entity_id = :id
  AND a.activity_type = 'note'
  AND a.is_deleted = false
ORDER BY n.is_pinned DESC, a.created_at DESC;
```

## Business rules

### Activity rules

| Rule                  | Description                                                        |
| --------------------- | ------------------------------------------------------------------ |
| Target required       | `target_entity_type` + `target_entity_id` are required             |
| Contact auto-resolved | `contact_id` auto-filled for bubble-up visibility                  |
| Reference optional    | `reference_id` points to channel table (null for note/task)        |
| Summary required      | `summary` (max 500 chars) for quick timeline display               |
| Channel is truth      | Full data (outcome, duration) lives in channel table, not activity |
| Archive block         | Cannot create activities for archived entities                     |

### Note rules

| Rule                | Description                                                       |
| ------------------- | ----------------------------------------------------------------- |
| Content only        | Note stores title, body, attachments only (no entity references)  |
| Entity via activity | Entity targeting handled by Activity table                        |
| Internal type       | Notes are "internal" activities — do not bubble up from Lead/Deal |
| Private option      | `is_private=true` limits visibility to creator only               |
