## Overview

The Lead Capture module provides a universal, source-agnostic pipeline that transforms any **captured** lead from Property Finder, Bayut/dubizzle, Meta Ads, website forms, and other sources into a CRM `Lead` with consistent source fields, person reuse, deduplication, and assignment.

<Note>
**Module Location:** `src/modules/crm/lead-capture/`

**Core Principle:** Reuse, don't rebuild. This layer orchestrates existing CRM primitives rather than reimplementing them.
</Note>

<Info>
**Implementation Status: BUILT** - The module ships with `LeadCaptureService`, source adapters, async transport via pg-boss queue, and all supporting infrastructure.
</Info>

## Scope and Architecture

### In Scope
- Canonical captured-lead contract
- Source-adapter pattern for extensibility
- Orchestration of source-field mapping + identity resolution + deduplication + assignment
- Async transport (queue + worker)

### Out of Scope
Meta Ads and website-form ingestion are **not** designed here but the contract reserves necessary fields (`externalLeadId`, `assignmentHint`) for future implementation without refactoring.

## Canonical Contract

### CapturedLeadInput Interface

```typescript
interface CapturedLeadInput {
  orgId: string;
  source: LeadSource;                  // Bayut | PropertyFinder | Dubizzle | Website | Instagram
  sourceDetail?: string;               // Campaign / form / "WhatsApp message"
  inboundChannel?: InboundChannel;     // call | whatsapp | email | sms
  referenceId?: string;                // Listing / campaign / form id
  externalLeadId?: string;             // Source's own lead id

  identity: {
    firstName?: string;
    lastName?: string;
    email?: string;
    phone?: string;
    channel?: { type: ChannelType; identifier: string };
  };

  intent?: LeadIntent;
  propertyRef?: { listingId?: string; refUnitId?: number };
  assignmentHint?: CaptureAssignmentConfig;
  capturedAt?: Date;
  rawPayload: unknown;                 // Stored verbatim for audit/replay
}
```

### CaptureResult Interface

```typescript
interface CaptureResult {
  leadId: string;
  personId: string;
  outcome: 'created' | 'attached' | 'skipped';
  assignment: 'agent' | 'team_pool' | 'org_pool' | 'queued' | 'none';
}
```

### LeadCaptureSource Interface

```typescript
interface LeadCaptureSource {
  readonly source: LeadSource;
  toCapturedInput(raw: unknown, ctx: { orgId: string }): CapturedLeadInput | CapturedLeadInput[];
}
```

## Capture Pipeline

The `LeadCaptureService.capture(input)` method processes leads through a structured pipeline:

<Steps>
<Step title="Idempotency Guard">
Check the `captured_lead` ledger by `(orgId, source, externalLeadId)`. If present, return the prior `CaptureResult` to handle webhook retries safely.
</Step>

<Step title="Source-field Normalization">
Derive `leadSource`, `sourceDetail`, `referenceId`, and any `sourceChannel`/`sourceConversation` from the input.
</Step>

<Step title="Identity Resolution">
Resolve or create the `Person` using existing `PersonService` methods with match-priority ordering.
</Step>

<Step title="Deduplication">
Decide whether to create a new lead or attach to an existing open lead using `findDuplicateLeadMatchInTransaction`.
</Step>

<Step title="Assignment">
Execute assignment directive through stakeholder or distribution services.
</Step>

<Step title="Record Creation">
Create the lead via `createLeadInTransaction` or attach re-inquiry activity, write the `captured_lead` ledger row, and return `CaptureResult`.
</Step>
</Steps>

<Note>
**System Actor:** The pipeline runs with the per-org System service user as `createdBy`. This ensures memberless orgs can still capture leads instead of dropping them.
</Note>

## Source Field Policy

<Tabs>
<Tab title="Lead Fields">
| Field | Rule |
|-------|------|
| `Lead.leadSource` | `input.source` - The portal/origin of this inquiry |
| `Lead.sourceDetail` | `input.sourceDetail` - Human-readable campaign/context |
| `Lead.inboundChannel` | `input.inboundChannel` - Structured delivery channel (first-touch only) |
| `Lead.referenceId` | `input.referenceId` - Attribution + dedup key |
| `Lead.sourceChannel` | Resolved `PersonChannel` for messaging identities |
| `Lead.sourceConversation` | Set when captured from messaging conversation |
</Tab>

<Tab title="Person Fields">
| Field | Rule |
|-------|------|
| `Person.originalSource` | Set on FIRST creation only (first-touch attribution) |
</Tab>
</Tabs>

### InboundChannel Vocabulary

The `InboundChannel` enum defines structured delivery channels:

<CodeGroup>
```typescript InboundChannel Enum
export enum InboundChannel {
  EMAIL = 'email',
  WHATSAPP = 'whatsapp', 
  CALL = 'call',
  SMS = 'sms'
}
```
</CodeGroup>

<Tip>
`InboundChannel` is distinct from `leadSource`: 
- `leadSource` identifies the portal/origin
- `inboundChannel` identifies how the inquiry was delivered
</Tip>

### Worker Implementation

The `LeadIngestionWorker` processes queued leads with:
- Automatic retries on failure
- Idempotency through the `captured_lead` ledger
- Tenant context isolation
- Error bubbling for monitoring

## Source Adapters

Source modules implement the `LeadCaptureSource` interface:

### Property Finder Adapter
- **Source:** `PropertyFinder`
- **Payload:** Reads documented `WHPayloadLead` webhook envelope
- **Mapping:** `payload.channel` → `InboundChannel` (1:1 mapping)

### Bayut/Dubizzle Adapter  
- **Source:** `Bayut` (splits dubizzle per-lead)
- **Payload:** Processes unified Bayut/dubizzle format
- **Mapping:** Infers channel from inquiry context

<Warning>
Source adapters live in `real-estate/portal-syndication` and depend on `crm/lead-capture`, maintaining proper dependency direction.
</Warning>

## Database Schema

### Captured Lead Ledger

```sql
CREATE TABLE captured_lead (
  id UUID PRIMARY KEY,
  org_id UUID NOT NULL,
  source TEXT NOT NULL,
  external_lead_id TEXT,
  inbound_channel TEXT,
  lead_id UUID,
  person_id UUID,
  outcome TEXT NOT NULL,
  raw_payload JSONB NOT NULL,
  captured_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  
  UNIQUE(org_id, source, external_lead_id) WHERE external_lead_id IS NOT NULL
);
```

### Settings Entity

```sql  
CREATE TABLE lead_capture_settings (
  id UUID PRIMARY KEY,
  org_id UUID NOT NULL UNIQUE,
  default_assignment_strategy TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```
