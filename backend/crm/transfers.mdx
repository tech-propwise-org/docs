The CRM transfer system manages formal transfers of leads and deals between agents or teams, with comprehensive approval workflows, commission splitting, and integration with the broader stakeholder management system.

<Note>
**Vocabulary note:** This doc uses "stakeholder" because it describes the engineering subsystem (entity name, table, services). The **user-facing and AI-facing** terms are "Assignment" (the concept) and "Assignee" (the assigned user or team). See `Docs/STAKEHOLDER_SYSTEM.md` → "Vocabulary" for the full policy and which surfaces switched (modal copy, notification copy, audit timeline labels, AI tool descriptions, AI tool boundary JSON keys).
</Note>

<Info>
**Unified inbound lead capture:** Leads arriving from external sources (Property Finder, Bayut/dubizzle, and future Meta/website) are ingested through the source-agnostic `crm/lead-capture` module — `LeadCaptureService.capture()` reuses `PersonService`, `LeadService.createLeadInTransaction`/`findDuplicateLeadMatchInTransaction`, `EntityStakeholderService`, and the `DistributionEngine`. The lead-capture module owns the `CapturedLeadInput` contract, `LeadCaptureSourceRegistry` for adapter registration, org-default `LeadCaptureSettings`, the `CapturedLead` idempotency ledger, and the source-agnostic `lead-ingestion` pg-boss queue + `LeadIngestionWorker`. Full design: `Docs/LEAD_CAPTURE_SPECIFICATION.md`.
</Info>

## Transfer system

The transfer system enables formal ownership transfers of leads and deals with sophisticated commission splitting and approval workflows.

### Entity transfer schema

The `entity_transfer` table manages transfer requests with comprehensive tracking and validation:

```sql
entity_transfer
├── Entity reference:
│   ├── entity_type (lead | deal)
│   └── entity_id
├── Transfer parties:
│   ├── from_user_id (nullable), from_team_id (nullable)
│   └── to_user_id (nullable), to_team_id (nullable)
├── Commission handling:
│   ├── from_commission_total (snapshot of sender's commission at request time)
│   └── from_keeps_percentage (e.g., 30%)
├── Workflow tracking:
│   ├── status (pending, approved, rejected, cancelled)
│   ├── reason, reject_reason
│   ├── requested_by_id, requested_at
│   ├── approved_by, approved_at, rejected_by, rejected_at
│   ├── cancelled_at, cancel_reason, cancelled_by_system
├── Audit fields:
│   ├── created_at, updated_at
│   └── organization_id
└── Database constraints:
    ├── Unique partial index: (entity_type, entity_id) WHERE status = 'pending'
    ├── Check: Either from_user_id OR from_team_id (not both)
    ├── Check: Either to_user_id OR to_team_id (not both)
    ├── Check: from_keeps_percentage BETWEEN 0 AND 100
    └── Check: from_commission_total >= 0
```

<Note>
**Polymorphic entity pattern:**
- `entity_type` + `entity_id` references any transferable entity
- Unified transfer logic works across leads and deals
- Future entity types (contacts, projects) can use same system
- Foreign key relationships maintained through application logic
</Note>

### Transfer workflow states

The transfer lifecycle follows a strict state machine:

```
┌─────────┐    request     ┌─────────┐    approve    ┌──────────┐
│         ├───────────────→│ pending ├──────────────→│ approved │
│  (none) │                └────┬────┘               └──────────┘
│         │                     │
└─────────┘                     │ reject / cancel
                                ↓
                          ┌──────────┐
                          │ rejected │
                          │    or    │
                          │cancelled │
                          └──────────┘
```

**State descriptions:**
- `pending` - Transfer requested, awaiting approval
- `approved` - Transfer executed, commission split completed
- `rejected` - Transfer denied by approver with reason
- `cancelled` - Transfer withdrawn or auto-cancelled

<Warning>
**Auto-cancellation triggers:**
- Entity deleted or archived
- Transferor loses stakeholder status
- Commission percentage changes significantly
- Entity status becomes non-transferable
</Warning>

### Commission split mechanics

The transfer system uses snapshot-based commission splitting to ensure predictable outcomes:

**Calculation formula:**
```
to_commission = from_commission_total × (100% - from_keeps_percentage) / 100%
```

**Transfer scenarios:**
- 100% commission, 30% retention → 30% kept, 70% transferred
- 60% commission, 20% retention → 48% kept, 12% transferred
- 50% commission, 0% retention → 0% kept, 50% transferred (primary status only)

**Primary stakeholder rules:**
- 0% retention → Primary status transfers to recipient
- >0% retention → Primary status remains with transferor (unless recipient has higher commission)
- Commission amounts rounded to 2 decimal places for consistency
- Total stakeholder commission cannot exceed 100%

<Info>
The `from_commission_total` snapshot prevents race conditions where commission changes between request and approval would cause incorrect splits.
</Info>

### Transfer execution workflow

<Steps>
<Step title="Pre-execution validation">
Transfer eligibility validation:

```javascript
const validateTransferEligibility = async (transferRequest) => {
  const { entityType, entityId, fromUserId, toUserId, fromKeepsPercentage } = transferRequest;
  
  // Check self-transfer
  if (fromUserId === toUserId) {
    throw new ValidationError('Cannot transfer to yourself');
  }
  
  // Validate entity type
  if (!['lead', 'deal'].includes(entityType)) {
    throw new ValidationError('Only leads and deals can be transferred');
  }
  
  // Check existing pending transfers
  const existingPending = await EntityTransfer.findOne({
    where: { 
      entity_type: entityType, 
      entity_id: entityId, 
      status: 'pending' 
    }
  });
  
  if (existingPending) {
    throw new ValidationError('Entity already has a pending transfer');
  }
  
  // Validate stakeholder status and commission
  const fromStakeholder = await EntityStakeholder.findOne({
    where: { 
      entity_type: entityType, 
      entity_id: entityId, 
      user_id: fromUserId,
      status: 'active'
    }
  });
  
  if (!fromStakeholder || fromStakeholder.commission_percentage === 0) {
    throw new ValidationError('Transferor must have commission stake in entity');
  }
  
  return true;
};
```
</Step>

<Step title="Atomic execution">
Transfer approval transaction:

```sql
BEGIN;
  -- 1. Update transferor stakeholder
  UPDATE entity_stakeholder 
  SET 
    commission_percentage = :from_commission_total * :from_keeps_percentage / 100,
    is_primary = CASE WHEN :from_keeps_percentage = 0 THEN false ELSE is_primary END,
    updated_at = NOW()
  WHERE entity_type = :entity_type 
    AND entity_id = :entity_id 
    AND user_id = :from_user_id;
    
  -- 2. Create or update recipient stakeholder
  INSERT INTO entity_stakeholder (
    entity_type, entity_id, user_id, commission_percentage, 
    is_primary, role, status, assigned_at, assigned_by_id, organization_id
  ) VALUES (
    :entity_type, :entity_id, :to_user_id,
    :from_commission_total * (100 - :from_keeps_percentage) / 100,
    CASE WHEN :from_keeps_percentage = 0 THEN true ELSE false END,
    'owner', 'active', NOW(), :approved_by_id, :organization_id
  )
  ON CONFLICT (entity_type, entity_id, user_id) 
  DO UPDATE SET 
    commission_percentage = commission_percentage + EXCLUDED.commission_percentage,
    is_primary = CASE 
      WHEN EXCLUDED.commission_percentage > entity_stakeholder.commission_percentage 
      THEN EXCLUDED.is_primary 
      ELSE entity_stakeholder.is_primary 
    END;
    
  -- 3. Update transfer status
  UPDATE entity_transfer 
  SET 
    status = 'approved',
    approved_by = :approved_by_id,
    approved_at = NOW(),
    updated_at = NOW()
  WHERE id = :transfer_id;
  
COMMIT;
```

<Info>
The atomic transaction ensures either complete success or complete rollback. No partial states are possible, maintaining data integrity even during system failures.
</Info>
</Step>

<Step title="Post-execution integration">
System integration updates:

```javascript
const executePostTransferIntegration = async (transfer) => {
  // 1. Update commission payments for closed deals
  if (transfer.entity_type === 'deal') {
    await CommissionPaymentService.recalculateForEntity('deal', transfer.entity_id, {
      reason: `Transfer approved: ${transfer.id}`,
      recalculated_by: transfer.approved_by
    });
  }
  
  // 2. Create activity record
  await Activity.create({
    entity_type: transfer.entity_type,
    entity_id: transfer.entity_id,
    activity_type: 'transfer',
    activity_subtype: 'approved',
    subject: `Ownership transferred`,
    description: `Transfer approved. Commission split: ${transfer.from_keeps_percentage}% retained, ${100 - transfer.from_keeps_percentage}% transferred.`,
    created_by: transfer.approved_by,
    activity_data: {
      transfer_id: transfer.id,
      commission_split: {
        from_keeps: transfer.from_keeps_percentage,
        to_receives: 100 - transfer.from_keeps_percentage,
        total_amount: transfer.from_commission_total
      }
    }
  });
  
  // 3. Update entity assignment if primary stakeholder changed
  if (transfer.from_keeps_percentage === 0) {
    const entity = await getEntity(transfer.entity_type, transfer.entity_id);
    await entity.update({
      assigned_to_id: transfer.to_user_id,
      assigned_at: new Date(),
      assigned_by_id: transfer.approved_by
    });
  }
  
  // 4. Send notifications
  await notifyTransferCompletion(transfer);
};
```
</Step>

<Accordion title="Business rules and constraints">
**Transfer validation rules:**

```javascript
const validateTransferBusinessRules = async (entityType, entityId, transferRequest) => {
  // Rule 1: Entity must be in transferable state
  const entity = await getEntity(entityType, entityId);
  const nonTransferableStates = {
    lead: ['archived', 'converted', 'deleted'],
    deal: ['won', 'lost', 'cancelled', 'closed']
  };
  
  if (nonTransferableStates[entityType].includes(entity.status)) {
    throw new BusinessRuleError(`Cannot transfer ${entityType} in ${entity.status} state`);
  }
  
  // Rule 2: Maximum transfer frequency (prevents abuse)
  const recentTransfers = await EntityTransfer.count({
    where: {
      entity_type: entityType,
      entity_id: entityId,
      status: 'approved',
      approved_at: { [Op.gte]: new Date(Date.now() - 30 * 24 * 60 * 60 * 1000) }
    }
  });
  
  if (recentTransfers >= 3) {
    throw new BusinessRuleError('Maximum transfer frequency exceeded (3 per month)');
  }
  
  // Rule 3: Commission retention limits
  if (transferRequest.from_keeps_percentage > 75) {
    throw new BusinessRuleError('Maximum commission retention is 75%');
  }
  
  return true;
};
```

```javascript
class TransferService {
  async requestTransfer({
    entityType,
    entityId,
    fromUserId,
    toUserId,
    fromKeepsPercentage,
    reason,
    requestedById,
    organizationId
  }) {
    const transaction = await sequelize.transaction();
    
    try {
      // Validation
      await this.validateTransferEligibility({
        entityType, entityId, fromUserId, toUserId, fromKeepsPercentage
      });
      
      await this.validateTransferBusinessRules(entityType, entityId, {
        from_keeps_percentage: fromKeepsPercentage
      });
      
      // Get current stakeholder commission snapshot
      const fromStakeholder = await EntityStakeholder.findOne({
        where: {
          entity_type: entityType,
          entity_id: entityId,
          user_id: fromUserId,
          status: 'active'
        },
        transaction
      });
      
      // Create transfer request
      const transfer = await EntityTransfer.create({
        entity_type: entityType,
        entity_id: entityId,
        from_user_id: fromUserId,
        to_user_id: toUserId,
        from_commission_total: fromStakeholder.commission_percentage,
        from_keeps_percentage: fromKeepsPercentage,
        reason,
        status: 'pending',
        requested_by_id: requestedById,
        requested_at: new Date(),
        organization_id: organizationId
      }, { transaction });
      
      // Create notification for approver
      await this.notifyApprover(transfer, transaction);
      
      await transaction.commit();
      
      return transfer;
      
    } catch (error) {
      await transaction.rollback();
      throw error;
    }
  }
}
```

<Accordion title="Approve transfer">
**Transfer approval execution:**

```javascript
async approveTransfer(transferId, approvedBy) {
  const transaction = await sequelize.transaction();
  
  try {
    // Lock transfer for update
    const transfer = await EntityTransfer.findByPk(transferId, {
      transaction,
      lock: true
    });
    
    if (transfer.status !== 'pending') {
      throw new Error('Transfer is not in pending status');
    }
    
    // Validate current stakeholder state hasn't changed
    const fromStakeholder = await EntityStakeholder.findOne({
      where: {
        entity_type: transfer.entity_type,
        entity_id: transfer.entity_id,
        user_id: transfer.from_user_id,
        status: 'active'
      },
      transaction,
      lock: true
    });
    
    if (!fromStakeholder || 
        Math.abs(fromStakeholder.commission_percentage - transfer.from_commission_total) > 0.01) {
      throw new Error('Stakeholder commission has changed since transfer request');
    }
    
    // Execute stakeholder updates
    await this.executeStakeholderUpdates(transfer, transaction);
    
    // Update transfer status
    await transfer.update({
      status: 'approved',
      approved_by: approvedBy,
      approved_at: new Date()
    }, { transaction });
    
    await transaction.commit();
    
    // Post-commit integration
    await this.executePostTransferIntegration(transfer);
    
    return transfer;
    
  } catch (error) {
    await transaction.rollback();
    throw error;
  }
}
```

<Accordion title="Commission calculation consistency">
**Snapshot-based commission calculations:**

```javascript
const calculateCommissionPayments = async (entityType, entityId) => {
  const transaction = await sequelize.transaction();
  
  try {
    // 1. Lock entity for value snapshot
    const entity = await getEntity(entityType, entityId, {
      transaction,
      lock: true
    });
    
    // 2. Get current stakeholders with lock
    const stakeholders = await EntityStakeholder.findAll({
      where: {
        entity_type: entityType,
        entity_id: entityId,
        status: 'active',
        commission_percentage: { [Op.gt]: 0 }
      },
      transaction,
      lock: true
    });
    
    // 3. Validate total commission doesn't exceed 100%
    const totalCommission = stakeholders.reduce(
      (sum, s) => sum + s.commission_percentage, 0
    );
    
    if (totalCommission > 100) {
      throw new Error(`Commission total exceeds 100%: ${totalCommission}%`);
    }
    
    // 4. Cancel existing pending payments
    await CommissionPayment.update(
      { status: 'cancelled', cancelled_reason: 'Recalculation' },
      {
        where: {
          source_entity_type: entityType,
          source_entity_id: entityId,
          status: 'pending'
        },
        transaction
      }
    );
    
    // 5. Create new payments with locked values
    const payments = stakeholders.map(stakeholder => ({
      stakeholder_id: stakeholder.id,
      amount: entity.value * (stakeholder.commission_percentage / 100),
      currency: entity.currency,
      source_entity_type: entityType,
      source_entity_id: entityId,
      status: 'pending',
      due_date: calculateDueDate(stakeholder.payment_schedule, entity),
      calculation_date: new Date(),
      calculation_note: `${stakeholder.commission_percentage}% of ${entity.value}`
    }));
    
    await CommissionPayment.bulkCreate(payments, { transaction });
    
    await transaction.commit();
    
    return payments;
    
  } catch (error) {
    await transaction.rollback();
    throw error;
  }
};
```

### Commission payment processing

The transfer system triggers commission recalculation for completed deals:

```
commission_payment
├── public_id (COMM-ADNS-001)
├── stakeholder_id → entity_stakeholder
├── Financial Details: amount, currency, exchange_rate
├── Source Reference: source_entity_type, source_entity_id
├── Calculation: calculation_note, calculation_date, calculated_by_id
├── Payment Processing: payment_method, payment_account_id, payment_reference
├── Status Tracking: status, processed_at, failed_at, failure_reason
├── Schedule: due_date, payment_schedule
└── Audit: organization_id, created_at, updated_at
```

**Payment lifecycle:**
- `pending` - Calculated but not yet processed
- `processing` - Payment initiated
- `completed` - Successfully processed
- `failed` - Processing failed (with reason)
- `cancelled` - Cancelled before processing

-- Get pending transfers with entity and party details
SELECT 
  et.*,
  -- Entity details
  CASE 
    WHEN et.entity_type = 'lead' THEN l.public_id
    WHEN et.entity_type = 'deal' THEN d.public_id
  END as entity_public_id,
  -- From party
  fu.first_name || ' ' || fu.last_name as from_user_name,
  -- To party
  tu.first_name || ' ' || tu.last_name as to_user_name,
  -- Commission calculation
  ROUND(et.from_commission_total * et.from_keeps_percentage / 100, 2) as from_retains,
  ROUND(et.from_commission_total * (100 - et.from_keeps_percentage) / 100, 2) as to_receives
FROM entity_transfer et
LEFT JOIN leads l ON et.entity_type = 'lead' AND et.entity_id = l.id
LEFT JOIN deals d ON et.entity_type = 'deal' AND et.entity_id = d.id
LEFT JOIN users fu ON et.from_user_id = fu.id
LEFT JOIN users tu ON et.to_user_id = tu.id
WHERE et.organization_id = :org_id
  AND et.status = 'pending'
  AND et.requested_at >= NOW() - INTERVAL '30 days'
ORDER BY et.requested_at ASC;

### Transfer validation rules

<AccordionGroup>
<Accordion title="Eligibility constraints">
**Transfer eligibility validation:**

```javascript
const validateTransferEligibility = async (transferRequest) => {
  const rules = [];
  
  // Rule 1: No self-transfer
  if (transferRequest.fromUserId === transferRequest.toUserId) {
    rules.push({
      rule: 'no_self_transfer',
      violated: true,
      message: 'Cannot transfer to yourself',
      severity: 'error'
    });
  }
  
  // Rule 2: Transferor must be active stakeholder
  const fromStakeholder = await EntityStakeholder.findOne({
    where: {
      entity_type: transferRequest.entityType,
      entity_id: transferRequest.entityId,
      user_id: transferRequest.fromUserId,
      status: 'active'
    }
  });
  
  if (!fromStakeholder || fromStakeholder.commission_percentage === 0) {
    rules.push({
      rule: 'active_stakeholder_required',
      violated: true,
      message: 'Transferor must have active commission stake',
      severity: 'error'
    });
  }
  
  // Rule 3: Entity must be in transferable state
  const entity = await getEntity(transferRequest.entityType, transferRequest.entityId);
  const nonTransferableStates = {
    lead: ['archived', 'converted', 'deleted'],
    deal: ['won', 'lost', 'cancelled', 'closed']
  };
  
  if (nonTransferableStates[transferRequest.entityType].includes(entity.status)) {
    rules.push({
      rule: 'transferable_state',
      violated: true,
      message: `Cannot transfer ${transferRequest.entityType} in ${entity.status} state`,
      severity: 'error'
    });
  }
  
  // Rule 4: No existing pending transfer
  const existingPending = await EntityTransfer.findOne({
    where: {
      entity_type: transferRequest.entityType,
      entity_id: transferRequest.entityId,
      status: 'pending'
    }
  });
  
  if (existingPending) {
    rules.push({
      rule: 'no_concurrent_transfers',
      violated: true,
      message: 'Entity already has a pending transfer',
      severity: 'error'
    });
  }
  
  return {
    valid: rules.filter(r => r.severity === 'error').length === 0,
    rules: rules
  };
};
```
</Accordion>

<Accordion title="Commission constraints">
**Commission splitting validation:**

```javascript
const validateCommissionSplit = async (transferRequest) => {
  const rules = [];
  
  // Rule 1: Retention percentage must be 0-75%
  if (transferRequest.fromKeepsPercentage < 0 || transferRequest.fromKeepsPercentage > 75) {
    rules.push({
      rule: 'retention_limit',
      violated: true,
      message: 'Commission retention must be between 0% and 75%',
      severity: 'error'
    });
  }
  
  // Rule 2: Calculate impact on total commission
  const allStakeholders = await EntityStakeholder.findAll({
    where: {
      entity_type: transferRequest.entityType,
      entity_id: transferRequest.entityId,
      status: 'active'
    }
  });
  
  const fromStakeholder = allStakeholders.find(s => s.user_id === transferRequest.fromUserId);
  const toStakeholder = allStakeholders.find(s => s.user_id === transferRequest.toUserId);
  
  const newFromCommission = fromStakeholder.commission_percentage * 
    (transferRequest.fromKeepsPercentage / 100);
  const transferredAmount = fromStakeholder.commission_percentage - newFromCommission;
  const newToCommission = (toStakeholder?.commission_percentage || 0) + transferredAmount;
  
  const totalAfterTransfer = allStakeholders
    .filter(s => s.user_id !== transferRequest.fromUserId && s.user_id !== transferRequest.toUserId)
    .reduce((sum, s) => sum + s.commission_percentage, 0) + 
    newFromCommission + newToCommission;
  
  if (totalAfterTransfer > 100) {
    rules.push({
      rule: 'total_commission_limit',
      violated: true,
      message: `Transfer would result in ${totalAfterTransfer}% total commission, exceeding 100% limit`,
      severity: 'error'
    });
  }
  
  // Rule 3: Warn on large transfers
  if (transferredAmount > 50) {
    rules.push({
      rule: 'large_transfer_warning',
      violated: false,
      message: `Large commission transfer: ${transferredAmount}%`,
      severity: 'warning'
    });
  }
  
  return {
    valid: rules.filter(r => r.severity === 'error').length === 0,
    rules: rules
  };
};
```

const validateTransferFrequency = async (entityType, entityId, fromUserId) => {
  // Check transfer frequency for entity
  const recentTransfers = await EntityTransfer.count({
    where: {
      entity_type: entityType,
      entity_id: entityId,

    }

  if (recentTransfers >= 3) {
    throw new BusinessRuleError(
      'Maximum transfer frequency exceeded (3 per month per entity)'
    );
  }
  
  // Check user's outbound transfer frequency
  const userRecentTransfers = await EntityTransfer.count({
    where: {
      from_user_id: fromUserId,
      status: 'approved',
      approved_at: { [Op.gte]: new Date(Date.now() - 7 * 24 * 60 * 60 * 1000) }
    }
  });
  
  if (userRecentTransfers >= 10) {
    throw new BusinessRuleError(
      'Maximum user transfer frequency exceeded (10 per week)'
    );
  }
  
  return true;
};

  async publishTransferEvent(eventType, transfer, eventData = {}) {
    const event = {
      event_type: eventType,

      entity_id: transfer.id,
      event_data: {
        ...eventData,
        transfer_id: transfer.id,

        }

      user_id: transfer.requested_by_id,

      timestamp: new Date().toISOString(),

    await EventBus.publish(event);

        break;

      case 'transfer.rejected':
        await this.handleTransferRejected(event);
        break;
      case 'transfer.cancelled':
        await this.handleTransferCancelled(event);
