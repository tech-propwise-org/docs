The CRM transfer system manages formal transfers of leads and deals between agents or teams, with comprehensive approval workflows, commission splitting, and integration with the broader stakeholder management system.

## Transfer system overview

The transfer system enables formal reassignment of entity ownership with sophisticated commission splitting and approval workflows:

<Tabs>
<Tab title="Core concepts">
**Entity transfers:**
- Formal ownership transfer of leads and deals between users or teams
- Integrated approval workflow with configurable authorization
- Commission splitting with percentage-based retention for transferor
- Atomic operations ensuring data consistency during transfers

**Key principles:**
- Snapshot-based commission calculation prevents race conditions
- Auto-cancellation protection for invalid states
- History tracking for audit compliance
- Integration with broader stakeholder management system
</Tab>

<Tab title="Transfer workflow">
<Steps>
<Step title="Transfer request">
Agent A requests transfer of Lead #1 to Agent B with 30% commission retention
</Step>

<Step title="Approval process">
Authorized approver reviews request with full context and commission impact
</Step>

<Step title="Atomic execution">
System performs stakeholder updates, commission redistribution, and history tracking in single transaction
</Step>

<Step title="Notification">
All parties notified of completed transfer with updated ownership and commission structure
</Step>
</Steps>
</Tab>

<Tab title="Commission mechanics">
**Calculation formula:**
```
to_commission = from_commission_total × (100% - from_keeps_percentage) / 100%
```

**Examples:**
- 100% commission, 30% retention → 30% kept, 70% transferred
- 60% commission, 20% retention → 48% kept, 12% transferred  
- 50% commission, 0% retention → 0% kept, 50% transferred (primary status only)
</Tab>
</Tabs>

## Entity transfer schema

The `entity_transfer` table manages transfer requests with comprehensive tracking and validation:

<AccordionGroup>
<Accordion title="Transfer entity structure">
```sql
entity_transfer
├── Entity reference:
│   ├── entity_type (lead | deal)
│   └── entity_id
├── Transfer parties:
│   ├── from_user_id (nullable), from_team_id (nullable)
│   └── to_user_id (nullable), to_team_id (nullable)
├── Commission handling:
│   ├── from_commission_total (snapshot of sender's commission at request time)
│   └── from_keeps_percentage (e.g., 30%)
├── Workflow tracking:
│   ├── status (pending, approved, rejected, cancelled)
│   ├── reason, reject_reason
│   ├── requested_by_id, requested_at
│   ├── approved_by, approved_at, rejected_by, rejected_at
├── Audit fields:
│   ├── created_at, updated_at
│   └── organization_id
└── Database constraints:
    ├── Unique partial index: (entity_type, entity_id) WHERE status = 'pending'
    ├── Check: Either from_user_id OR from_team_id (not both)
    ├── Check: Either to_user_id OR to_team_id (not both)
    ├── Check: from_keeps_percentage BETWEEN 0 AND 100
    └── Check: from_commission_total >= 0
```

**Key constraints:**
- Only one pending transfer per entity (enforced by partial unique index)
- Mutually exclusive user/team selection for both parties
- Commission percentage and total validation
- Organization-scoped operations
</Accordion>

<Accordion title="Commission snapshot protection">
**Purpose:** The `from_commission_total` field captures the exact commission percentage at request time, protecting against concurrent modifications.

```javascript
const requestTransfer = async (entityType, entityId, fromUserId, toUserId, fromKeepsPercentage) => {
  // Get current stakeholder commission (snapshot)
  const fromStakeholder = await EntityStakeholder.findOne({
    where: { 
      entity_type: entityType, 
      entity_id: entityId, 
      user_id: fromUserId,
      status: 'active'
    }
  });
  
  if (!fromStakeholder) {
    throw new Error('Transferor is not a stakeholder of this entity');
  }
  
  // Create transfer with commission snapshot
  const transfer = await EntityTransfer.create({
    entity_type: entityType,
    entity_id: entityId,
    from_user_id: fromUserId,
    to_user_id: toUserId,
    from_commission_total: fromStakeholder.commission_percentage, // SNAPSHOT
    from_keeps_percentage: fromKeepsPercentage,
    status: 'pending',
    requested_by_id: currentUserId,
    reason: 'Workload redistribution'
  });
  
  return transfer;
};
```

**Protection scenarios:**
- Prevents invalidation if transferor's commission changes during pending period
- Guards against race conditions between transfer and other stakeholder operations
- Ensures transfer terms remain valid even with concurrent modifications
- Provides audit trail of commission state at request time
</Accordion>

## Transfer approval workflow

Comprehensive approval system with authorization controls and atomic execution:

<Tabs>
<Tab title="Request validation">
<Warning>
**Transfer eligibility validation:**
- Cannot transfer to self (same user or team)
- Must have valid from/to targets (user or team, not both)
- Transferor must be current stakeholder with sufficient commission
- Entity must be in transferable status (active lead/deal)
- All parties must be in same organization
- No existing pending transfers for the entity
</Warning>

```javascript
const validateTransferEligibility = async (transferRequest) => {
  const { entityType, entityId, fromUserId, toUserId, fromKeepsPercentage } = transferRequest;
  
  // Check self-transfer
  if (fromUserId === toUserId) {
    throw new ValidationError('Cannot transfer to yourself');
  }
  
  // Check existing pending transfers
  const existingPending = await EntityTransfer.findOne({
    where: { 
      entity_type: entityType, 
      entity_id: entityId, 
      status: 'pending' 
    }
  });
  
  if (existingPending) {
    throw new ValidationError('Entity already has a pending transfer');
  }
  
  // Validate stakeholder status and commission
  const fromStakeholder = await EntityStakeholder.findOne({
    where: { entity_type: entityType, entity_id: entityId, user_id: fromUserId }
  });
  
  if (!fromStakeholder || fromStakeholder.commission_percentage === 0) {
    throw new ValidationError('Transferor must have commission stake in entity');
  }
  
  // Validate entity status
  const entity = await getEntity(entityType, entityId);
  if (!entity.isTransferable()) {
    throw new ValidationError(`${entityType} is not in transferable state`);
  }
  
  return true;
};
```
</Tab>

<Tab title="Authorization controls">
**Approval authority matrix:**

| Transfer Type | Approver Requirements | Additional Conditions |
|---------------|----------------------|----------------------|
| User → User | Team Lead or Manager | Both users in same team |
| User → Team | Team Lead | Target team in same department |
| Team → User | Department Manager | User qualified for team workload |
| Team → Team | Department Manager | Cross-team coordination required |
| High Value Deal | Senior Manager + Finance | Deal value > $100K threshold |

```javascript
const validateApprovalAuthority = async (transfer, approverId) => {
  const approver = await User.findById(approverId, { include: ['roles', 'teams'] });
  const entity = await getEntity(transfer.entity_type, transfer.entity_id);
  
  // Check role-based permissions
  const requiredRoles = getRequiredApprovalRoles(transfer);
  const hasRequiredRole = approver.roles.some(role => requiredRoles.includes(role.name));
  
  if (!hasRequiredRole) {
    throw new AuthorizationError('Insufficient approval authority for this transfer type');
  }
  
  // Check value-based requirements
  if (transfer.entity_type === 'deal' && entity.value > 100000) {
    const hasFinanceApproval = await checkFinanceApproval(transfer.id);
    if (!hasFinanceApproval) {
      throw new AuthorizationError('High value deals require finance approval');
    }
  }
  
  return true;
};
```
</Tab>

<Tab title="Atomic execution">
**Transfer approval transaction:**

```sql
BEGIN;
  -- 1. Update transferor stakeholder
  UPDATE entity_stakeholder 
  SET 
    commission_percentage = :from_commission_total * :from_keeps_percentage / 100,
    is_primary = CASE WHEN :from_keeps_percentage = 0 THEN false ELSE is_primary END,
    updated_at = NOW()
  WHERE entity_type = :entity_type 
    AND entity_id = :entity_id 
    AND user_id = :from_user_id;
    
  -- 2. Create or update recipient stakeholder
  INSERT INTO entity_stakeholder (
    entity_type, entity_id, user_id, commission_percentage, 
    is_primary, role, status, assigned_at, assigned_by_id, organization_id
  ) VALUES (
    :entity_type, :entity_id, :to_user_id,
    :from_commission_total * (100 - :from_keeps_percentage) / 100,
    CASE WHEN :from_keeps_percentage = 0 THEN true ELSE false END,
    'owner', 'active', NOW(), :approved_by_id, :organization_id
  )
  ON CONFLICT (entity_type, entity_id, user_id) 
  DO UPDATE SET 
    commission_percentage = commission_percentage + EXCLUDED.commission_percentage,
    is_primary = CASE 
      WHEN EXCLUDED.commission_percentage > entity_stakeholder.commission_percentage 
      THEN EXCLUDED.is_primary 
      ELSE entity_stakeholder.is_primary 
    END;
    
  -- 3. Update transfer status
  UPDATE entity_transfer 
  SET 
    status = 'approved',
    approved_by = :approved_by_id,
    approved_at = NOW(),
    updated_at = NOW()
  WHERE id = :transfer_id;
  
  -- 4. Create stakeholder history records
  INSERT INTO entity_stakeholder_history (
    stakeholder_id, action, old_values, new_values, 
    changed_by, changed_at, change_reason
  ) VALUES 
    (:from_stakeholder_id, 'transfer_out', :old_from_values, :new_from_values, 
     :approved_by_id, NOW(), 'Transfer approved'),
    (:to_stakeholder_id, 'transfer_in', :old_to_values, :new_to_values, 
     :approved_by_id, NOW(), 'Transfer approved');
     
COMMIT;
```

<Info>
The atomic transaction ensures either complete success or complete rollback. No partial states are possible, maintaining data integrity even during system failures.
</Info>
</Tab>
</Tabs>

## Commission splitting mechanics

Sophisticated commission calculation with retention options and edge case handling:

<AccordionGroup>
<Accordion title="Basic commission calculations">
**Standard transfer scenarios:**

```javascript
const calculateCommissionSplit = (fromCommissionTotal, fromKeepsPercentage) => {
  const fromKeeps = fromCommissionTotal * (fromKeepsPercentage / 100);
  const toReceives = fromCommissionTotal * ((100 - fromKeepsPercentage) / 100);
  
  return {
    fromKeeps: Number(fromKeeps.toFixed(2)),
    toReceives: Number(toReceives.toFixed(2)),
    totalTransferred: fromCommissionTotal
  };
};

// Examples
calculateCommissionSplit(100, 30); // { fromKeeps: 30, toReceives: 70, totalTransferred: 100 }
calculateCommissionSplit(60, 20);  // { fromKeeps: 48, toReceives: 12, totalTransferred: 60 }
calculateCommissionSplit(50, 0);   // { fromKeeps: 0, toReceives: 50, totalTransferred: 50 }
```

**Edge cases:**
- **Zero retention (0%):** Complete transfer including primary stakeholder status
- **Full retention (100%):** Invalid - would result in no actual transfer
- **Partial commission:** Transferor retains proportional commission based on snapshot
</Accordion>

<Accordion title="Multi-stakeholder scenarios">
**Complex stakeholder arrangements:**

Consider Deal #123 with multiple stakeholders:
- User A: 60% commission (primary)
- User B: 25% commission  
- User C: 15% commission

**User A transfers to User D with 40% retention:**

```javascript
// Before transfer
const beforeTransfer = {
  userA: { commission: 60, isPrimary: true },
  userB: { commission: 25, isPrimary: false },
  userC: { commission: 15, isPrimary: false }
};

// After transfer (A keeps 40% = 24%, D gets 60% = 36%)
const afterTransfer = {
  userA: { commission: 24, isPrimary: false },  // Reduced, no longer primary
  userB: { commission: 25, isPrimary: false },  // Unchanged
  userC: { commission: 15, isPrimary: false },  // Unchanged
  userD: { commission: 36, isPrimary: true }    // New primary stakeholder
};
```

**Business rules:**
- Primary status transfers to recipient if transferor retains <50% of original commission
- Total commission across all stakeholders still cannot exceed 100%
- Other stakeholders unaffected by transfer
</Accordion>

<Accordion title="Commission payment impact">
**Payment recalculation:**

When transfers occur after deal closure but before commission payment:

```javascript
const recalculateCommissionPayments = async (dealId, transferId) => {
  const deal = await Deal.findById(dealId);
  const transfer = await EntityTransfer.findById(transferId);
  
  if (deal.status !== 'won') {
    // No payments to recalculate for open deals
    return;
  }
  
  // Find pending commission payments
  const pendingPayments = await CommissionPayment.findAll({
    where: { 
      source_entity_type: 'deal',
      source_entity_id: dealId,
      status: 'pending'
    }
  });
  
  for (const payment of pendingPayments) {
    const newStakeholder = await EntityStakeholder.findOne({
      where: { id: payment.stakeholder_id }
    });
    
    // Recalculate payment amount based on new commission percentage
    const newAmount = deal.value * (newStakeholder.commission_percentage / 100);
    
    await payment.update({
      amount: newAmount,
      recalculated_at: new Date(),
      recalculation_reason: `Transfer ${transferId} - commission split`
    });
  }
};
```

<Note>
Commission payments are only recalculated for pending payments. Completed payments remain unchanged to preserve financial integrity.
</Note>
</Accordion>

```javascript
// Stakeholder removal trigger
eventBus.on('stakeholder.removed', async (event) => {
  const { stakeholderId, entityType, entityId, userId, teamId } = event;
  
  const pendingTransfers = await EntityTransfer.findAll({
    where: {
      entity_type: entityType,
      entity_id: entityId,
      status: 'pending',
      [Op.or]: [
        { from_user_id: userId },
        { from_team_id: teamId },
        { to_user_id: userId },
        { to_team_id: teamId }
      ]
    }
  });

    await transfer.update({
      status: 'cancelled',
      cancel_reason: 'Stakeholder removed from entity',
      cancelled_at: new Date(),
      cancelled_by_system: true
    });
    
    await notifyTransferCancellation(transfer, 'stakeholder_removed');
  }
});

// Commission reduction trigger
eventBus.on('stakeholder.commission_changed', async (event) => {
  const { stakeholderId, oldCommission, newCommission } = event;
  
  const stakeholder = await EntityStakeholder.findById(stakeholderId);
  const pendingTransfers = await EntityTransfer.findAll({
    where: {
      entity_type: stakeholder.entity_type,
      entity_id: stakeholder.entity_id,
      from_user_id: stakeholder.user_id,
      status: 'pending',
      from_commission_total: { [Op.gt]: newCommission }
    }
  });
  
  for (const transfer of pendingTransfers) {
    await transfer.update({
      status: 'cancelled',
      cancel_reason: `Commission reduced below transfer requirement (${newCommission}% < ${transfer.from_commission_total}%)`,
      cancelled_at: new Date(),
      cancelled_by_system: true
    });
  }
});
```

## Integration with stakeholder system

The transfer system integrates seamlessly with the broader stakeholder management system:

<Tabs>
<Tab title="Stakeholder integration">
**Unified stakeholder model:**

The transfer system leverages the existing `entity_stakeholder` table and related infrastructure:

```javascript
// Transfer approval updates stakeholder records directly
const approveTransfer = async (transferId, approverId) => {
  const transfer = await EntityTransfer.findById(transferId, { 
    include: ['fromStakeholder', 'toStakeholder'] 
  });
  
  await db.transaction(async (t) => {
    // Update transferor stakeholder
    await transfer.fromStakeholder.update({
      commission_percentage: transfer.from_commission_total * (transfer.from_keeps_percentage / 100),
      is_primary: transfer.from_keeps_percentage === 0 ? false : transfer.fromStakeholder.is_primary
    }, { transaction: t });
    
    // Create or update recipient stakeholder
    const [toStakeholder, created] = await EntityStakeholder.findOrCreate({
      where: {
        entity_type: transfer.entity_type,
        entity_id: transfer.entity_id, 
        user_id: transfer.to_user_id
      },
      defaults: {
        commission_percentage: transfer.from_commission_total * ((100 - transfer.from_keeps_percentage) / 100),
        is_primary: transfer.from_keeps_percentage === 0,
        role: 'owner',
        status: 'active',
        assigned_by_id: approverId
      },
      transaction: t
    });
    
    if (!created) {
      // Update existing stakeholder
      await toStakeholder.update({
        commission_percentage: toStakeholder.commission_percentage + 
          (transfer.from_commission_total * ((100 - transfer.from_keeps_percentage) / 100)),
        is_primary: transfer.from_keeps_percentage === 0 ? true : toStakeholder.is_primary
      }, { transaction: t });
    }
    
    // Update transfer status
    await transfer.update({
      status: 'approved',
      approved_by: approverId,
      approved_at: new Date()
    }, { transaction: t });
  });
};
```

## Business rules and constraints

Comprehensive business rules ensure transfer system integrity and compliance:

<AccordionGroup>
<Accordion title="Transfer eligibility rules">
**Entity-level constraints:**
- Only active leads and deals can be transferred (not archived, deleted, or closed)
- Lead transfers automatically cancelled upon conversion to contact/deal
- Deal transfers blocked once deal reaches 'won', 'lost', or 'cancelled' status
- Entity must belong to same organization as all transfer participants

**Stakeholder constraints:**
- Transferor must be current stakeholder with commission > 0%
- Cannot transfer more commission than currently held (protected by snapshot)
- Transferor must have been stakeholder for minimum 24 hours (prevents rapid churning)
- Total stakeholder commission after transfer cannot exceed 100%

**User/team constraints:**
- All participants must be active users/teams in same organization
- Cannot transfer to/from deactivated or suspended accounts
- Team transfers require all team members to be eligible for entity type
- Cross-organizational transfers prohibited
</Accordion>

<Accordion title="Commission and payment rules">
**Commission calculation rules:**
- Retention percentage must be between 0% and 100%
- 0% retention transfers primary stakeholder status to recipient
- Commission amounts rounded to 2 decimal places for consistency
- Snapshot protection prevents commission calculation drift during pending period

**Payment system integration:**
- Only pending payments can be recalculated after transfer approval
- Completed payments remain unchanged (financial integrity)
- New payment schedules follow recipient's preferences
- Multi-currency deals maintain exchange rate consistency

**High-value deal rules:**
- Deals > $100K require additional approval authority
- Commission payments > $50K require finance department review
- International transfers subject to tax jurisdiction validation
- Regulatory compliance checks for licensed agent requirements
</Accordion>

<Accordion title="Approval and workflow rules">
**Authorization matrix:**
```javascript
const getRequiredApprovalLevel = (transfer) => {
  const { entityType, entityValue, fromKeepsPercentage, organizationalLevel } = transfer;
  
  // High-value deals require senior approval
  if (entityType === 'deal' && entityValue > 100000) {
    return 'senior_manager';
  }
  
  // Complete transfers (0% retention) require manager approval
  if (fromKeepsPercentage === 0) {
    return 'manager';
  }
  
  // Cross-team transfers require team lead approval
  if (organizationalLevel === 'cross_team') {
    return 'team_lead';
  }
  
  // Standard transfers require supervisor approval
  return 'supervisor';
};
```

**Workflow constraints:**
- Maximum 72-hour pending period before auto-expiration
- Approvers cannot approve their own transfer requests
- Rejection requires detailed reason and alternative recommendations
- Emergency transfers (system failures) require dual approval
</Accordion>

<Accordion title="Data integrity and audit rules">
**Audit trail requirements:**
- All transfer operations logged with full context
- Stakeholder history maintained for compliance reporting
- Commission payment changes tracked with transfer attribution
- Failed transfers logged with detailed error information

**Data consistency checks:**
- Periodic validation of total commission percentages
- Orphaned transfer detection and cleanup
- Stakeholder relationship integrity verification  
- Cross-system synchronization status monitoring

**Rollback and recovery:**
- Transfer approval operations are atomic (complete or fail entirely)
- Failed transfers can be retried with same terms within 24 hours
- System errors during approval trigger automatic investigation workflow
- Data corruption detection triggers immediate transfer suspension
</Accordion>
</AccordionGroup>
