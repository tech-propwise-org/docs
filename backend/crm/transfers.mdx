## Transfer types

| From | To   | Description                      |
| ---- | ---- | -------------------------------- |
| User | User | Standard agent-to-agent transfer |
| User | Team | Agent transfers to a team pool   |
| Team | User | Team pool claim by a user        |
| Team | Team | Team-to-team reallocation        |

## Transfer workflow

<Steps>
<Step title="Transfer request">
Agent A requests transfer of Lead #1 to Agent B. An `entity_transfer` record is created with `from_commission_total` (snapshot) and `from_keeps_percentage`.
</Step>
<Step title="Pending review">
Agent B (or manager) reviews the request. They can approve, reject, or cancel.

<Warning>
**Auto-cancel:** If while a transfer is pending, any stakeholder on the entity is removed or has their commission reduced, the transfer is automatically cancelled because the commission split is now invalid.
</Warning>
</Step>
<Step title="Transfer approved">
On approval:
- Agent A's stakeholder is updated: `30%`, `isPrimary=false`
- Agent B's stakeholder is created: `isPrimary=true`, `70%`
- History records created for both agents
- Transfer status set to `approved`
</Step>
<Step title="Downstream effects">
When a new deal is created from the lead later, all lead stakeholders are auto-copied to the deal with their commission splits preserved.
</Step>
</Steps>

## Business rules

| Rule                 | Description                                                                                            |
| -------------------- | ------------------------------------------------------------------------------------------------------ |
| Approval required    | Transfer requires target user/team approval                                                            |
| Commission preserved | `from_keeps_percentage` sets what transferor keeps                                                     |
| Commission snapshot  | `from_commission_total` snapshots sender's commission at request time                                  |
| 0% transfer          | Allowed only when target is already a stakeholder (transfers primary status without moving commission) |
| Auto-cancel          | Pending transfers auto-cancelled when a stakeholder is removed or has commission reduced               |
| Team transfers       | Supports user‚Üíuser, user‚Üíteam, team‚Üíuser, team‚Üíteam                                                    |
| One pending          | Only one pending transfer per entity (enforced by unique partial index)                                |
| History recorded     | All changes logged to `entity_stakeholder_history`                                                     |
