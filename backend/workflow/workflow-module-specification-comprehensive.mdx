## Overview

The workflow module provides a comprehensive automation engine for PropWise CRM, enabling organizations to create, manage, and execute automated workflows triggered by various events. The module supports complex multi-step processes including messaging, data manipulation, external integrations, conditional logic, and AI agent integration.

<Info>
The workflow system operates on a graph-based execution model where workflows are defined as directed graphs of **nodes** with per-step **connections** and an `edges` array for editor consistency.
</Info>

### Canonical Data Structure

<CodeGroup>
```typescript Workflow Entity
Workflow.graph = { 
  nodes: WorkflowStepDefinition[], 
  edges: WorkflowEdge[] 
}

// Scoping
AutomationScope: "ORGANIZATION" | "PERSONAL"

// Status Management  
WorkflowStatus: "DRAFT" | "PUBLISHED" | "ARCHIVED"
```
</CodeGroup>

<Warning>
Legacy documentation that references `steps[]` + `connections[]` pairings, **`GLOBAL` scope**, `isActive`, or **activate/deactivate** APIs is obsolete.
</Warning>

### Validation System

<Warning>
Server-side publish validation enforces:
- Single entry point requirement
- No broken node references
- No unreachable nodes
- Cycle detection (unless explicitly modeled with `JUMP_TO`)
- No outgoing connections from `END` nodes
</Warning>

## Trigger System

### Event-Driven Activation

<CodeGroup>
```typescript Trigger Configuration
WorkflowTrigger {
  eventType: 'MESSAGE_RECEIVED' | 'CONTACT_CREATED' | 'WEBHOOK'
  conditions: TriggerCondition[]
  scope: 'ORGANIZATION' | 'PERSONAL'
  isActive: boolean
}
```

```typescript Scope-Aware Routing
// Organization-level workflows
organizationTriggers.filter(trigger => 
  trigger.scope === 'ORGANIZATION' && 
  trigger.matchesEvent(event)
)

// Personal workflows  
personalTriggers.filter(trigger =>
  trigger.scope === 'PERSONAL' &&
  trigger.userId === event.userId &&
  trigger.matchesEvent(event)
)
```
</CodeGroup>

## Scope Management

### Organization vs Personal Workflows

<CardGroup cols={2}>
<Card title="Organization Workflows" icon="building">
- Shared across all organization members
- Require organization-level permissions
- Triggered by organization-wide events
- Managed by administrators
</Card>

<Card title="Personal Workflows" icon="user">
- Private to individual users
- User-specific triggers and actions
- Personal productivity automation
- Self-managed by users
</Card>
</CardGroup>

### Permission Enforcement

<Warning>
All workflow operations enforce proper scope validation:
- Organization workflows require `WORKFLOW_MANAGE` permission
- Personal workflows are restricted to workflow owners
- Cross-scope operations are explicitly blocked
- Template access respects scope boundaries
</Warning>
