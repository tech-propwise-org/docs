## Overview

The workflow module provides a comprehensive automation engine for PropWise CRM, enabling organizations to create, manage, and execute automated workflows triggered by various events. The module supports complex multi-step processes including messaging, data manipulation, external integrations, conditional logic, and AI agent integration.

<Info>
  The workflow system operates on a graph-based execution model where workflows are defined as directed graphs of **nodes** with per-step **connections** (and an `edges` array for editor consistency). Workflows can be triggered by messaging/CRM events or webhooks.
</Info>

### Canonical Data Model

The persisted workflow shape follows this structure:

```typescript
Workflow.graph = { 
  nodes: WorkflowStepDefinition[], 
  edges: WorkflowEdge[] 
}
```

<Tabs>
  <Tab title="Automation Scope">
    - `ORGANIZATION`: Organization-wide workflows
    - `PERSONAL`: User-specific workflows
  </Tab>
  <Tab title="Workflow Status">
    - `DRAFT`: Editable workflow state
    - `PUBLISHED`: Active and executable
    - `ARCHIVED`: Inactive but preserved
  </Tab>
</Tabs>

<Warning>
  Legacy documentation referencing `steps[]` + `connections[]` pairings, `GLOBAL` scope, `isActive`, or activate/deactivate APIs is obsolete.
</Warning>

#### Validation Rules

<AccordionGroup>
  <Accordion title="Entry Point Validation">
    - Requires a single entry point
    - Rejects workflows without clear starting nodes
  </Accordion>
  <Accordion title="Reference Validation">
    - Rejects broken node references
    - Ensures all connections point to valid nodes
  </Accordion>
  <Accordion title="Reachability Check">
    - Rejects unreachable nodes
    - Ensures all nodes can be reached from entry point
  </Accordion>
  <Accordion title="Cycle Detection">
    - Rejects accidental cycles
    - Allows explicit loops via `JUMP_TO` steps
  </Accordion>
  <Accordion title="Terminal Validation">
    - Blocks outgoing connections from `END` nodes
    - Ensures proper workflow termination
  </Accordion>
</AccordionGroup>
