## Overview

The workflow module provides a comprehensive automation engine for PropWise CRM, enabling organizations to create, manage, and execute automated workflows triggered by various events. The module supports complex multi-step processes including messaging, data manipulation, external integrations, conditional logic, and AI agent integration.

<Info>
The workflow system operates on a graph-based execution model where workflows are defined as directed graphs of **nodes** with per-step **connections** and an `edges` array for editor parity.
</Info>

### Implementation Status

<Check>
The workflow module is **fully operational and integrated** into the main application with complete implementation across all components.
</Check>

## Core Architecture

### Workflow Structure

The canonical persisted shape for workflows follows this structure:

```typescript
Workflow.graph = { 
  nodes: WorkflowStepDefinition[], 
  edges: WorkflowEdge[] 
}
```

<Note>
**Scope Options**: `ORGANIZATION | PERSONAL`  
**Status Options**: `DRAFT | PUBLISHED | ARCHIVED`

Legacy documentation referencing `steps[]` + `connections[]` pairings, `GLOBAL` scope, `isActive`, or activate/deactivate APIs is obsolete.
</Note>

## Validation System

### Server-side Publish Validation

<Warning>
The workflow validation system enforces strict rules before publication:
</Warning>

- **Single Entry Point**: Requires exactly one entry node
- **Reference Validation**: Rejects broken node references
- **Reachability Check**: Rejects unreachable nodes
- **Cycle Detection**: Rejects accidental cycles (except explicit JUMP_TO loops)
- **End Node Protection**: Blocks outgoing connections from END nodes

### Graph Consistency

The system maintains consistency between:
- **Node Connections**: Per-step connection definitions
- **Edge Array**: Visual editor compatibility
- **Reference Integrity**: All connections reference valid nodes
