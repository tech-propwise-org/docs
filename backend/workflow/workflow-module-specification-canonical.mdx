## Architecture Overview

The workflow system operates on a graph-based execution model where workflows are defined as directed graphs of **nodes** with per-step **connections** and an `edges` array for editor parity.

<Info>
**Canonical Shape (2025+):** `Workflow.graph = { nodes: WorkflowStepDefinition[], edges: WorkflowEdge[] }` with `AutomationScope` (ORGANIZATION | PERSONAL) and `WorkflowStatus` (DRAFT | PUBLISHED | ARCHIVED)
</Info>

<Warning>
Legacy documentation referencing `steps[]` + `connections[]` pairings, `GLOBAL` scope, `isActive`, or activate/deactivate APIs is obsolete.
</Warning>

### Validation Rules

Server-side publish validation enforces:
- Single entry point requirement
- Rejection of broken references
- Rejection of unreachable nodes
- Cycle prevention (unless explicit `JUMP_TO` loops)
- Blocking outgoing connections from `END` nodes
