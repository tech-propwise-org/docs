## Overview

The workflow module provides a comprehensive automation engine for PropWise CRM, enabling organizations to create, manage, and execute automated workflows triggered by various events. The module supports complex multi-step processes including messaging, data manipulation, external integrations, conditional logic, and AI agent integration.

<Note>
The workflow system operates on a graph-based execution model where workflows are defined as directed graphs of **nodes** with per-step **connections** and an `edges` array for editor consistency.
</Note>

### Canonical Data Structure

The persisted workflow structure (2025+) uses:

```typescript
Workflow.graph = { 
  nodes: WorkflowStepDefinition[], 
  edges: WorkflowEdge[] 
}
```

With the following key properties:
- **AutomationScope**: `ORGANIZATION | PERSONAL`
- **WorkflowStatus**: `DRAFT | PUBLISHED | ARCHIVED`

<Warning>
Legacy documentation referencing `steps[]` + `connections[]` pairings, `GLOBAL` scope, `isActive`, or activate/deactivate APIs is obsolete.
</Warning>

## Execution Model

### Graph-Based Processing

Workflows execute as directed graphs with server-side validation that:
- Requires a single entry point
- Rejects broken references
- Rejects unreachable nodes
- Blocks accidental cycles (unless explicitly modeled with `JUMP_TO`)
- Prevents outgoing connections from `END` nodes

### Queue Integration

<CodeGroup>
```typescript Workflow Execution
// Queue-based execution with pg-boss
await this.workflowEngineService.executeWorkflow(
  workflowId,
  context,
  { retry: true }
);
```

```typescript Step Processing
// Individual step execution with tracking
const stepResult = await this.stepHandlerRegistry
  .getHandler(step.type)
  .execute(step, context);
```
</CodeGroup>

### Scope Management

Workflows support both:
- **Organization Scope**: Available to all organization members
- **Personal Scope**: Private to individual users with proper permission enforcement
