## Overview

The workflow module provides a comprehensive automation engine for PropWise CRM, enabling organizations to create, manage, and execute automated workflows triggered by various events. The module supports complex multi-step processes including messaging, data manipulation, external integrations, conditional logic, and AI agent integration.

<Note>
The workflow system operates on a graph-based execution model where workflows are defined as directed graphs of **nodes** with per-step **connections** and an `edges` array for editor consistency.
</Note>

### Canonical Data Structure

The persisted workflow shape follows this structure:

```typescript
Workflow.graph = { 
  nodes: WorkflowStepDefinition[], 
  edges: WorkflowEdge[] 
}
```

**Key Enums:**
- `AutomationScope`: `ORGANIZATION | PERSONAL`
- `WorkflowStatus`: `DRAFT | PUBLISHED | ARCHIVED`

<Warning>
Legacy documentation referencing `steps[]` + `connections[]` pairings, `GLOBAL` scope, `isActive`, or activate/deactivate APIs is obsolete.
</Warning>

## Step Handlers

The workflow engine supports a comprehensive set of step types:

### Communication Steps
- **SEND_MESSAGE**: Send messages via various channels
- **ASK_QUESTION**: Interactive elements with validation and timeout handling

### Logic and Control Flow
- **BRANCH**: Advanced branching with structured condition system
- **JUMP_TO**: Explicit loop modeling and flow control
- **TRIGGER_ANOTHER_WORKFLOW**: Workflow chaining with scope validation

### CRM Operations
- **CREATE_TASK**: Task creation and assignment
- **UPDATE_CONTACT**: Contact field updates and data manipulation
- **ASSIGN_TO**: Lead and contact assignment workflows

### Advanced Features
- **AI_AGENT**: AI agent activation and integration
- **DATE_TIME**: Time-based workflow operations
- **END**: Workflow termination nodes

## Execution Engine

### Workflow Processing

<Steps>
  <Step title="Trigger Detection">
    WorkflowTriggerService identifies applicable workflows based on events and scope
  </Step>
  <Step title="Queue Processing">
    Workflows are queued for asynchronous execution via pg-boss workers
  </Step>
  <Step title="Step Execution">
    WorkflowEngineService processes each step with comprehensive tracking
  </Step>
  <Step title="Error Handling">
    Retry logic and error recovery mechanisms ensure reliable execution
  </Step>
</Steps>

### Validation Rules

<Warning>
Server-side publish validation enforces strict rules:
- Requires single entry point
- Rejects broken references
- Rejects unreachable nodes
- Rejects accidental cycles (unless explicit JUMP_TO modeling)
- Blocks outgoing connections from END nodes
</Warning>
