## Overview

The Workflow Module delivers a powerful automation engine that enables organizations to create, manage, and execute sophisticated automated workflows. Built on a graph-based execution model, workflows are defined as directed graphs with nodes and connections, triggered by messaging events, CRM activities, or webhooks.

<Note>
**Current Implementation Status:** Fully operational with complete integration into the main application including template seeding, queue-based execution, and AI module integration.
</Note>

### Core Architecture

The workflow system operates using these fundamental concepts:

- **Graph-Based Design**: Workflows are directed graphs of nodes with step-by-step connections
- **Event-Driven Triggers**: Automatic initiation based on messaging events, CRM activities, or webhooks
- **Scope Management**: Personal or organization-level workflow automation with proper permission enforcement
- **Queue Processing**: Reliable asynchronous execution with retry logic and error handling

<Warning>
Legacy documentation referencing `steps[]` + `connections[]` pairings, `GLOBAL` scope, or `activate/deactivate` APIs is obsolete. The current canonical shape uses `Workflow.graph = { nodes: WorkflowStepDefinition[], edges: WorkflowEdge[] }`.
</Warning>

## Implementation Details

### Module Structure

The WorkflowModule registers all essential components:

- **Controllers**: API endpoints for workflow management
- **Services**: Core business logic and execution engine
- **Entities**: Database models and relationships
- **Queue Workers**: Asynchronous processing handlers
- **Validators**: Input validation and workflow integrity checks

### Execution Flow

<Steps>
  <Step title="Trigger Detection">
    WorkflowTriggerService identifies applicable workflows based on events
  </Step>
  
  <Step title="Queue Processing">
    Workflows are queued for asynchronous execution via pg-boss workers
  </Step>
  
  <Step title="Step Execution">
    WorkflowEngineService processes each step with proper error handling
  </Step>
  
  <Step title="Progress Tracking">
    Complete execution monitoring with step-level tracking
  </Step>
</Steps>

### Advanced Features

<Warning>
Workflow validation enforces strict rules: single entry points, no broken references, no unreachable nodes, cycle prevention (except explicit JUMP_TO loops), and no outgoing connections from END nodes.
</Warning>

- **Interactive Elements**: ASK_QUESTION steps with field validation and timeout handling
- **Advanced Branching**: Category-based field resolution with type-aware operators
- **Workflow Chaining**: TRIGGER_ANOTHER_WORKFLOW with scope validation and depth limits
- **Personal vs Organization**: Proper scoping with permission enforcement
- **Queue Integration**: Reliable processing with retry logic and error recovery
