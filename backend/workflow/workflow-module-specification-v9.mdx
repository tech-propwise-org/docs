## Overview

The workflow module provides a comprehensive automation engine for PropWise CRM, enabling organizations to create, manage, and execute automated workflows triggered by various events. The module supports complex multi-step processes including messaging, data manipulation, external integrations, conditional logic, and AI agent integration.

The workflow system operates on a **graph-based execution model** where workflows are defined as directed graphs of nodes with per-step connections. Workflows can be triggered by messaging/CRM events or webhooks. Server-side publish validation ensures workflow integrity by:

- Requiring a single entry point
- Rejecting broken references
- Rejecting unreachable nodes
- Rejecting accidental cycles (unless explicitly modeled with `JUMP_TO`)
- Blocking outgoing connections from `END` nodes

<Info>
**Canonical persisted shape (2025+):**  
`Workflow.graph = { nodes: WorkflowStepDefinition[], edges: WorkflowEdge[] }`  
with `AutomationScope` **`ORGANIZATION | PERSONAL`** and `WorkflowStatus` **`DRAFT | PUBLISHED | ARCHIVED`**
</Info>

<Warning>
Legacy documentation referencing `steps[]` + `connections[]` pairings, **`GLOBAL` scope**, `isActive`, or **activate/deactivate** APIs is obsolete.
</Warning>

  <Step title="Slug-Based Upsert">
    Templates are identified by stable slugs for idempotent seeding:
    
    ```typescript
    async seedTemplates(em: EntityManager): Promise<void> {
      for (const templateDef of TEMPLATE_CATALOG) {
        const existing = await em.findOne(WorkflowTemplate, {
          where: { slug: templateDef.slug },
        });

        if (existing) {
          // Update existing template
          Object.assign(existing, templateDef);
          await em.save(existing);
        } else {
          // Create new template
          const template = em.create(WorkflowTemplate, templateDef);
          await em.save(template);
        }
      }

      // Prune deleted templates
      await this.pruneDeletedTemplates(em);
    }
    ```
  </Step>

  <Step title="Template Pruning">
    Templates whose slugs are no longer in the catalog are soft-deleted:
    
    ```typescript
    private async pruneDeletedTemplates(em: EntityManager): Promise<void> {
      const catalogSlugs = TEMPLATE_CATALOG.map(t => t.slug);
      
      await em
        .createQueryBuilder()
        .update(WorkflowTemplate)
        .set({ deletedAt: new Date() })
        .where('slug NOT IN (:...slugs)', { slugs: catalogSlugs })
        .andWhere('deletedAt IS NULL')
        .execute();
    }
    ```
  </Step>
</Steps>

### Execution Engine

The **WorkflowEngineService** provides reliable workflow execution with comprehensive tracking:

<Steps>
  <Step title="Execution Initialization">
    Create a new workflow execution with initial context:
    
    ```typescript
    async startExecution(
      workflowId: string,
      context: Record<string, any>,
      organizationId: string,
      parentExecutionId?: string
    ): Promise<WorkflowExecution> {
      const workflow = await this.findWorkflow(workflowId);
      
      const execution = this.executionRepo.create({
        workflowId,
        organizationId,
        status: WorkflowExecutionStatus.PENDING,
        context,
        parentExecutionId,
        startedAt: new Date(),
      });
      
      await this.executionRepo.save(execution);
      
      // Queue first step
      await this.queueNextStep(execution, workflow);
      
      return execution;
    }
    ```
  </Step>

  <Step title="Step Execution">
    Execute individual workflow steps with handler delegation:
    
    ```typescript
    async executeStep(
      executionId: string,
      stepId: string
    ): Promise<void> {
      const execution = await this.findExecution(executionId);
      const workflow = await this.findWorkflow(execution.workflowId);
      const step = this.findStep(workflow, stepId);
      
      // Update execution status
      execution.currentStepId = stepId;
      execution.status = WorkflowExecutionStatus.RUNNING;
      await this.executionRepo.save(execution);
      
      try {
        // Get step handler
        const handler = this.getStepHandler(step.type);
        
        // Execute step
        const result = await handler.execute(step, execution);
        
        // Store result
        execution.stepResults[stepId] = result;
        await this.executionRepo.save(execution);
        
        // Queue next step(s)
        await this.queueNextSteps(execution, workflow, step, result);
      } catch (error) {
        await this.handleStepError(execution, step, error);
      }
    }
    ```
  </Step>

  <Step title="Next Step Resolution">
    Determine next step(s) based on current step and result:
    
    ```typescript
    private async queueNextSteps(
      execution: WorkflowExecution,
      workflow: Workflow,
      currentStep: WorkflowStepDefinition,
      result: any
    ): Promise<void> {
      // Handle END step
      if (currentStep.type === WorkflowStepType.END) {
        await this.completeExecution(execution);
        return;
      }
      
      // Handle BRANCH step
      if (currentStep.type === WorkflowStepType.BRANCH) {
        const branchPath = result.branchPath; // 'true' or 'false'
        const nextStepId = currentStep.connections[branchPath];
        
        if (nextStepId) {
          await this.queueStep(execution.id, nextStepId);
        }
        return;
      }
      
      // Handle standard connections
      const nextStepIds = currentStep.connections.default || [];
      
      for (const nextStepId of nextStepIds) {
        await this.queueStep(execution.id, nextStepId);
      }
    }
    ```
  </Step>

  <Step title="Error Handling">
    Handle step execution errors with retry logic:
    
    ```typescript
    private async handleStepError(
      execution: WorkflowExecution,
      step: WorkflowStepDefinition,
      error: Error
    ): Promise<void> {
      const retryCount = execution.stepResults[step.id]?.retryCount || 0;
      
      if (retryCount < MAX_RETRIES) {
        // Update retry count
        execution.stepResults[step.id] = {
          ...execution.stepResults[step.id],
          retryCount: retryCount + 1,
          lastError: error.message,
        };
        await this.executionRepo.save(execution);
        
        // Requeue step with backoff
        await this.queueStepWithBackoff(
          execution.id,
          step.id,
          retryCount + 1
        );
      } else {
        // Max retries exceeded, fail execution
        execution.status = WorkflowExecutionStatus.FAILED;
        execution.error = error.message;
        execution.completedAt = new Date();
        await this.executionRepo.save(execution);
      }
    }
    ```
  </Step>
</Steps>

### Queue Integration

Workflow execution leverages **pg-boss** for reliable asynchronous processing:

<CodeGroup>

```typescript Queue Worker
@Injectable()
export class WorkflowExecuteWorker {
  constructor(
    private readonly workflowEngine: WorkflowEngineService,
    @InjectPgBoss() private readonly pgBoss: PgBoss
  ) {}

  async onModuleInit() {
    await this.pgBoss.work(
      'workflow.execute-step',
      { teamSize: 5, teamConcurrency: 1 },
      this.handleExecuteStep.bind(this)
    );
  }

  private async handleExecuteStep(
    job: PgBoss.Job<{ executionId: string; stepId: string }>
  ): Promise<void> {
    const { executionId, stepId } = job.data;
    
    await this.workflowEngine.executeStep(executionId, stepId);
  }
}
```

```typescript Queue Job Submission
async queueStep(
  executionId: string,
  stepId: string,
  delaySeconds = 0
): Promise<void> {
  await this.pgBoss.send(
    'workflow.execute-step',
    { executionId, stepId },
    {
      retryLimit: 3,
      retryDelay: 60,
      retryBackoff: true,
      startAfter: delaySeconds,
    }
  );
}
```

</CodeGroup>

- **Scalability**: Multiple workers can process jobs concurrently
- **Fault tolerance**: Failed jobs are automatically retried with backoff

### Trigger Service

The **WorkflowTriggerService** handles event-driven workflow initiation with scope-aware routing:

<Steps>
  <Step title="Event Registration">
    Register workflow triggers with the event system:
    
    ```typescript
    async registerTriggers(workflow: Workflow): Promise<void> {
      if (!workflow.trigger) return;
      
      const { event, conditions } = workflow.trigger;
      
      // Register event listener
      await this.eventBus.subscribe(
        event,
        async (payload: any) => {
          await this.handleTriggerEvent(workflow, payload, conditions);
        }
      );
    }
    ```
  </Step>

  <Step title="Event Handling">
    Process trigger events and evaluate conditions:
    
    ```typescript
    private async handleTriggerEvent(
      workflow: Workflow,
      payload: any,
      conditions?: WorkflowCondition[]
    ): Promise<void> {
      // Check workflow status
      if (workflow.status !== WorkflowStatus.PUBLISHED) {
        return;
      }
      
      // Evaluate conditions
      if (conditions && !this.evaluateConditions(conditions, payload)) {
        return;
      }
      
      // Check scope and permissions
      if (!(await this.canExecute(workflow, payload))) {
        return;
      }
      
      // Start execution
      await this.workflowEngine.startExecution(
        workflow.id,
        payload,
        workflow.organizationId
      );
    }
    ```
  </Step>

  <Step title="Scope Validation">
    Validate workflow execution based on scope:
    
    ```typescript
    private async canExecute(
      workflow: Workflow,
      payload: any
    ): Promise<boolean> {
      // Organization workflows can be executed by anyone in org
      if (workflow.scope === AutomationScope.ORGANIZATION) {
        return payload.organizationId === workflow.organizationId;
      }
      
      // Personal workflows can only be executed by creator
      if (workflow.scope === AutomationScope.PERSONAL) {
        return (
          payload.organizationId === workflow.organizationId &&
          payload.userId === workflow.createdBy
        );
      }
      
      return false;
    }
    ```
  </Step>
</Steps>

      async execute(
        step: WorkflowStepDefinition,

    ```typescript
    @Injectable()
    export class BranchHandler implements WorkflowStepHandler {
      async execute(
        step: WorkflowStepDefinition,
        execution: WorkflowExecution
      ): Promise<WorkflowStepResult> {
        const { conditions, conditionLogic } = step.config;
        
        // Evaluate conditions
        const conditionResults = conditions.map(condition =>
          this.evaluateCondition(condition, execution.context)
        );
        
        // Apply logic
        const branchResult = conditionLogic === 'OR'
          ? conditionResults.some(r => r)
          : conditionResults.every(r => r);
        
        return {
          success: true,
          branchPath: branchResult ? 'true' : 'false',
        };
      }
      
      private evaluateCondition(
        condition: WorkflowCondition,
        context: Record<string, any>
      ): boolean {
        const value = this.resolveField(condition.category, condition.field, context);
        
        switch (condition.operator) {
          case 'equals':
            return value === condition.value;
          case 'contains':
            return String(value).includes(condition.value);
          case 'greater_than':
            return Number(value) > Number(condition.value);
          case 'less_than':
            return Number(value) < Number(condition.value);
          // ... other operators
        }
      }
    }
    ```

## Workflow Validation

### Publish Validation

The **WorkflowValidationService** enforces strict validation rules before workflow publication:

<Steps>
  <Step title="Entry Point Validation">
    Ensure workflow has exactly one entry point:
    
    ```typescript
    validateEntryPoint(graph: WorkflowGraph): ValidationResult {
      const entryNodes = graph.nodes.filter(node => 
        !graph.edges.some(edge => edge.target === node.id)
      );
      
      if (entryNodes.length === 0) {
        return { valid: false, error: 'No entry point found' };
      }
      
      if (entryNodes.length > 1) {
        return { valid: false, error: 'Multiple entry points found' };
      }
      
      return { valid: true };
    }
    ```
  </Step>

  <Step title="Reference Validation">
    Verify all node references are valid:
    
    ```typescript
    validateReferences(graph: WorkflowGraph): ValidationResult {
      const nodeIds = new Set(graph.nodes.map(n => n.id));
      
      for (const edge of graph.edges) {
        if (!nodeIds.has(edge.source)) {
          return { 
            valid: false, 
            error: `Invalid source node: ${edge.source}` 
          };
        }
        
        if (!nodeIds.has(edge.target)) {
          return { 
            valid: false, 
            error: `Invalid target node: ${edge.target}` 
          };
        }
      }
      
      return { valid: true };
    }
    ```
  </Step>

  <Step title="Reachability Validation">
    Ensure all nodes are reachable from entry point:
    
    ```typescript
    validateReachability(graph: WorkflowGraph): ValidationResult {
      const entryNode = this.findEntryNode(graph);
      const reachable = new Set<string>();
      
      this.traverseFrom(entryNode.id, graph, reachable);
      
      const unreachable = graph.nodes.filter(
        node => !reachable.has(node.id)
      );
      
      if (unreachable.length > 0) {
        return { 
          valid: false, 
          error: `Unreachable nodes: ${unreachable.map(n => n.id).join(', ')}` 
        };
      }
      
      return { valid: true };
    }
    ```
  </Step>

  <Step title="Cycle Detection">
    Detect accidental cycles (except explicit JUMP_TO loops):
    
    ```typescript
    validateCycles(graph: WorkflowGraph): ValidationResult {
      const visited = new Set<string>();
      const recursionStack = new Set<string>();
      
      for (const node of graph.nodes) {
        if (this.hasCycle(node.id, graph, visited, recursionStack)) {
          // Check if cycle is intentional (JUMP_TO)
          const cycle = this.extractCycle(node.id, graph);
          const hasJumpTo = cycle.some(n => n.type === 'JUMP_TO');
          
          if (!hasJumpTo) {
            return { 
              valid: false, 
              error: `Unintentional cycle detected at ${node.id}` 
            };
          }
        }
      }
      
      return { valid: true };
    }
    ```
  </Step>

  <Step title="END Node Validation">
    Block outgoing connections from END nodes:
    
    ```typescript
    validateEndNodes(graph: WorkflowGraph): ValidationResult {
      const endNodes = graph.nodes.filter(n => n.type === 'END');
      
      for (const endNode of endNodes) {
        const outgoing = graph.edges.filter(e => e.source === endNode.id);
        
        if (outgoing.length > 0) {
          return { 
            valid: false, 
            error: `END node ${endNode.id} has outgoing connections` 
          };
        }
      }
      
      return { valid: true };
    }
    ```
  </Step>

  <Step title="Configuration Validation">
    Validate step-specific configuration:
    
    ```typescript
    validateStepConfig(step: WorkflowStepDefinition): ValidationResult {
      const validator = this.getStepValidator(step.type);
      
      if (!validator) {
        return { 
          valid: false, 
          error: `Unknown step type: ${step.type}` 
        };
      }
      
      return validator.validate(step.config);
    }
    ```
  </Step>
</Steps>

<Warning>
Workflows cannot be published if any validation rule fails. Draft workflows can be saved with validation warnings, but must pass all checks before publication.
</Warning>
