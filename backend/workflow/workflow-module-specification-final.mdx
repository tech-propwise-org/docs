## Overview

The workflow module provides a comprehensive automation engine for PropWise CRM, enabling organizations to create, manage, and execute automated workflows triggered by various events. The module supports complex multi-step processes including messaging, data manipulation, external integrations, conditional logic, and AI agent integration.

<Note>
The workflow system operates on a graph-based execution model where workflows are defined as directed graphs of **nodes** with per-step **connections** and an `edges` array for editor parity.
</Note>

### Canonical Data Structure

The modern workflow structure follows this canonical format:

```typescript
Workflow.graph = { 
  nodes: WorkflowStepDefinition[], 
  edges: WorkflowEdge[] 
}
```

With scoping and status management:
- **AutomationScope**: `ORGANIZATION | PERSONAL`
- **WorkflowStatus**: `DRAFT | PUBLISHED | ARCHIVED`

<Warning>
Legacy documentation referencing `steps[]` + `connections[]` pairings, `GLOBAL` scope, `isActive`, or activate/deactivate APIs is obsolete.
</Warning>

### Advanced Features

- **Queue Integration**: Asynchronous execution via pg-boss workers with retry logic and error handling
- **Trigger System**: Event-driven workflow initiation with WorkflowTriggerService and scope-aware routing
- **Step Handlers**: Complete set of workflow steps including Send Message, Ask Question, Branch, Date/Time, Create Task, Update Contact, Assign To, AI Agent, and more
- **Interactive Elements**: ASK_QUESTION steps with validation, contact field updates, and timeout handling
- **Advanced Branching**: Structured condition system with category-based field resolution and type-aware operators
- **Workflow Chaining**: TRIGGER_ANOTHER_WORKFLOW steps with scope validation and depth limits
- **Personal vs Organization Scope**: Workflows scoped to organization or personal use with proper permission enforcement

### Template Seeding Process

The template seeding process runs automatically:

```typescript
// Runs on every server boot via AppInitializerService
WorkflowTemplateSeederService.seedTemplates(em)
```

<Info>
Template seeding executes inside a `TenantContext.executeWithBypass()` transaction and is also invoked by the thin `WorkflowTemplateSeeder` CLI seeder for development environments.
</Info>
