## Overview

The workflow module provides a comprehensive automation engine for PropWise CRM, enabling organizations to create, manage, and execute automated workflows triggered by various events. The module supports complex multi-step processes including messaging, data manipulation, external integrations, conditional logic, and AI agent integration.

The workflow system operates on a graph-based execution model where workflows are defined as directed graphs of **nodes** with per-step **connections**. Workflows can be triggered by messaging/CRM events or webhooks with server-side validation ensuring single entry points, rejecting broken references, unreachable nodes, and accidental cycles.

<Warning>
**Canonical persisted shape (2025+):** `Workflow.graph = { nodes: WorkflowStepDefinition[], edges: WorkflowEdge[] }` with `AutomationScope` **`ORGANIZATION | PERSONAL`** and `WorkflowStatus` **`DRAFT | PUBLISHED | ARCHIVED`**. Legacy documentation referencing `steps[]` + `connections[]` pairings, **`GLOBAL` scope**, `isActive`, or **activate/deactivate** APIs is obsolete.
</Warning>

### Scope Management

<Info>
Workflows can be scoped to organization or personal use with proper permission enforcement.
</Info>

**Scope Types:**
- `ORGANIZATION`: Available to all organization members
- `PERSONAL`: Available only to the creator

**Status Options:**
- `DRAFT`: Work in progress
- `PUBLISHED`: Active and executable
- `ARCHIVED`: Inactive but preserved
