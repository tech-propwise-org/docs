## Overview

The workflow module provides a comprehensive automation engine for PropWise CRM, enabling organizations to create, manage, and execute automated workflows triggered by various events. The module supports complex multi-step processes including messaging, data manipulation, external integrations, conditional logic, and AI agent integration.

<Warning>
The workflow system operates on a graph-based execution model where workflows are defined as directed graphs of **nodes** with per-step **connections** and an `edges` array that should remain consistent with those connections for editor parity.
</Warning>

### Workflow Structure

<CodeGroup>
```typescript Canonical Persisted Shape (2025+)
Workflow.graph = { 
  nodes: WorkflowStepDefinition[], 
  edges: WorkflowEdge[] 
}

// Automation Scope
enum AutomationScope {
  ORGANIZATION = 'ORGANIZATION',
  PERSONAL = 'PERSONAL'
}

// Workflow Status
enum WorkflowStatus {
  DRAFT = 'DRAFT',
  PUBLISHED = 'PUBLISHED',
  ARCHIVED = 'ARCHIVED'
}
```
</CodeGroup>

<Note>
Legacy documentation that references `steps[]` + `connections[]` pairings, **GLOBAL** scope, `isActive`, or **activate/deactivate** APIs is obsolete.
</Note>

### Server-side Validation

Server-side publish validation enforces:

<Steps>
  <Step title="Single Entry Point">
    Requires a single entry point for workflow execution
  </Step>
  <Step title="Reference Validation">
    Rejects broken references between workflow nodes
  </Step>
  <Step title="Reachability Check">
    Rejects unreachable nodes in the workflow graph
  </Step>
  <Step title="Cycle Prevention">
    Rejects accidental cycles unless the loop is explicitly modeled with `JUMP_TO`
  </Step>
  <Step title="End Node Protection">
    Blocks outgoing connections from `END` nodes
  </Step>
</Steps>

## Best Practices

<Tip>
Follow these best practices when designing and implementing workflows:
</Tip>

1. **Start Simple**: Begin with basic workflows and gradually add complexity
2. **Test Thoroughly**: Validate all workflow paths and edge cases
3. **Monitor Performance**: Track execution times and success rates
4. **Use Templates**: Leverage existing templates as starting points
5. **Document Logic**: Clearly document complex branching and conditions
6. **Error Handling**: Plan for failure scenarios and implement appropriate retry logic
7. **Scope Appropriately**: Choose between organization and personal scope based on use case
