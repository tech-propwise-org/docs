## Overview

The Subscription Module implements a **freemium SaaS billing system** for PropWise CRM. Every organization has a subscription tied to one of **three plan tiers** (Free / Pro / Business — Starter was removed; see §18). The module handles:

- **Plan-based feature gating** — binary feature flags per tier
- **Resource limits** — **source-aware** caps on leads, contacts, deals, companies (imports never count — §18.4), and storage
- **Unified AI-credit wallet** — one credit balance for Propilot, AI auto-reply, and unit valuation, with a per-action cost map, per-user ceilings, and personal credits (§18.5)
- **Single per-agent seat model** — one seat SKU per tier; Pro 5–10 seats (11th → upgrade to Business), Business 10+ with volume pricing (§18.3)
- **Stripe integration** — checkout, subscription management, mid-cycle plan changes, webhooks, billing portal, AED pricing, +25 GB storage packs, credit top-up packs
- **Evergreen 90-day trial** — Pro & Business signups get a card-upfront trial (§18.2)
- **Free organization ownership cap** — one user may own at most 2 active Free-plan organizations
- **Proration** — mid-cycle **tier** changes and seat changes are prorated to the day; **billing cycle** switches (Monthly ↔ Annual) are deferred to period end via Stripe Subscription Schedules
- **Suspension flow** — 2-day grace period on payment failure, then org goes read-only

<Warning>
  **§18 (Subscription Packaging Rollout)** is the authoritative description of the current Free/Pro/Business AED model, the single-seat collapse, the unified credit wallet, source-aware caps, the evergreen trial, and connection caps. Where earlier sections (written for the legacy 4-tier / dual-seat / dual-credit model) conflict with §18, **§18 wins**.
</Warning>

### Design principles

<AccordionGroup>
  <Accordion title="Core billing principles">
    | Principle | Decision |
    |-----------|----------|
    | Freemium model | Free plan with limited features; paid tiers unlock progressively |
    | Per-org billing | Billing is per organization; developer portal is free |
    | Dual seat types | Manager seats (Owner, Admin) and agent seats (Basic, custom roles); every user consumes a seat |
    | Seat type derived from role | No explicit seat assignment — seat type is automatically determined by the user's RBAC role |
    | Feature flags over tier checks | Gating uses `@RequiresFeature('flag')` on plan JSONB — changing what a tier includes requires only a seeder update, not code changes |
  </Accordion>

  <Accordion title="Enforcement and integration principles">
    | Principle | Decision |
    |-----------|----------|
    | Service-layer limit enforcement | Resource limits and credit consumption are checked in service methods, not guards, because they need entity counts |
    | Free-org creation protection | `POST /v1/organizations` locks the owner row, counts owned Free-plan orgs (missing subscription rows count as Free), and rejects the third active free workspace |
    | Stripe as source of truth for payments | Webhook-driven lifecycle: the app reacts to Stripe events rather than polling |
    | Billing cycle vs tier changes | **Tier changes** (Free → Pro, Pro → Business) are immediate and prorated. **Billing cycle switches** (Monthly ↔ Annual on same tier) are deferred to period end via Stripe Subscription Schedules with no proration. **Combined changes** (tier + cycle) are immediate and prorated with all line items re-priced atomically |
    | Checkout vs. change-plan separation | `POST /checkout` is for first-time subscription (Free → Paid); `POST /change-plan` is for switching between paid tiers |
    | Idempotent webhooks | Every Stripe event is logged in `BillingEvent` with a unique `stripeEventId` to prevent duplicate processing |
    | Graceful degradation | If `app.stripe.secretKey` (`STRIPE_SECRET_KEY`) is not set, billing features are unavailable but the app still starts |
  </Accordion>
</AccordionGroup>

### Data flow

<Tabs>
  <Tab title="First-time checkout">
    **First-time checkout flow (Free → Paid):**

    <Steps>
      <Step title="User initiates upgrade">
        Frontend "Upgrade" button triggers `POST /v1/subscriptions/checkout`
      </Step>

      <Step title="Create checkout session">
        - Rejects if org already has a Stripe subscription (use change-plan instead)
        - `SubscriptionService.createCheckoutSession()`
        - `StripeService.createCheckoutSession()` returns Stripe Checkout URL
      </Step>

      <Step title="User completes payment">
        User pays on Stripe's hosted page, then Stripe redirects to success URL with `session_id={CHECKOUT_SESSION_ID}`
      </Step>

      <Step title="Confirm checkout">
        Frontend calls `POST /v1/subscriptions/checkout/confirm { sessionId }`
        - `SubscriptionService.fulfillCheckoutSession()` (idempotent with webhook)
        - Subscription entity updated to ACTIVE (plan tier from session metadata)
      </Step>

      <Step title="Webhook confirmation">
        (Async) Stripe fires `checkout.session.completed` webhook
        - `StripeWebhookController` → `activateSubscription()` (same activation path)
      </Step>
    </Steps>
  </Tab>

  <Tab title="Plan change">
    **Mid-cycle plan change flow (Paid → different Paid tier):**

    <Steps>
      <Step title="User requests change">
        Frontend "Change Plan" button triggers `POST /v1/subscriptions/change-plan`
      </Step>

      <Step title="Validate and update">
        `SubscriptionService.changePlan()`:
        - Validates seat overflow (blocks if current users exceed new plan capacity)
        - `StripeService.swapSubscriptionPrice()` — prorated
        - Reconciles seat line items (old tier price → new tier price)
        - Updates local Subscription entity
        - Returns updated subscription immediately
      </Step>
    </Steps>
  </Tab>

  <Tab title="Renewal / failure">
    **Renewal / payment failure flow:**

    ```
    Stripe charges renewal invoice
      ├─ invoice.paid → handleInvoicePaid() 
      │                 → status stays ACTIVE, period updated
      │
      └─ invoice.payment_failed → handleInvoicePaymentFailed() 
                                  → status → PAST_DUE
           └─ Stripe retries for 2 days
                ├─ Payment succeeds → invoice.paid → back to ACTIVE
                │
                └─ All retries fail 
                   → customer.subscription.updated (status: unpaid)
                      → handleSubscriptionUpdated() 
                         → status → SUSPENDED
                            → Org is read-only 
                              (SubscriptionActiveGuard blocks writes)
    ```
  </Tab>
</Tabs>

  <Step title="Service enforces cap">
    ```typescript
    async checkResourceLimit(
      orgId: number, 
      resource: ResourceType, 
      source: 'user' | 'import'
    ) {
      const sub = await this.getOrgSubscription(orgId);
      const cap = sub.plan[`${resource}Cap`];
      
      if (cap === -1) return; // unlimited
      
      const usage = await this.getResourceUsage(orgId, resource, source);
      if (usage >= cap) {
        throw new ForbiddenException(
          `${resource} limit reached (${cap}). Upgrade to add more.`
        );
      }
    }
    ```
  </Step>

## Seat management

### Seat types

<Info>
  Seats are **role-derived**, not explicitly assigned. When a user is added to an organization, their RBAC role determines their seat type.
</Info>

<Tabs>
  <Tab title="Manager seats">
    **Manager seats** are consumed by:
    - Owner role
    - Admin role
    
    Managers have full CRM access plus administrative capabilities (billing, user management, settings).
  </Tab>

  <Tab title="Agent seats">
    **Agent seats** are consumed by:
    - Basic role (sales agent)
    - Custom roles (if created with agent-level permissions)
    
    Agents have CRM access but limited administrative capabilities.
  </Tab>
</Tabs>

### Seat counting

## Credit system

### Unified credit wallet (§18.5)

<Info>
  All AI features draw from a single credit balance: Propilot chat, AI auto-reply, and unit valuation. Credits are allocated per organization (org pool) and per user (personal credits).
</Info>

<Tabs>
  <Tab title="Credit sources">
    1. **Org pool credits** — monthly allowance from plan tier
    2. **Per-user credits** — monthly allowance per active user
    3. **Purchased credit packs** — one-time top-ups
    4. **Personal credits** — special allocations, never expire
  </Tab>

  <Tab title="Cost per action">
    ```typescript
    const CREDIT_COSTS = {
      propilot_message: 2,
      ai_auto_reply: 3,
      unit_valuation: 5
    };
    ```
  </Tab>

  <Tab title="Monthly refresh">
    Credits reset on the subscription anniversary date:
    - Org pool → `plan.aiCreditsPerMonth`
    - Per-user → `plan.aiCreditsPerUser × activeUserCount`
    - Unused credits do **not** roll over
    - Purchased packs persist until expiration
  </Tab>
</Tabs>

### Credit consumption (FIFO)

<Steps>
  <Step title="User triggers AI action">
    ```typescript
    async generateAIReply(userId: number, orgId: number) {
      const cost = CREDIT_COSTS.ai_auto_reply; // 3 credits
      await this.creditService.consumeCredits(
        orgId, 
        userId, 
        cost, 
        'ai_auto_reply'
      );
      // proceed with AI call
    }
    ```
  </Step>

  <Step title="FIFO deduction order">
    ```typescript
    async consumeCredits(
      orgId: number, 
      userId: number, 
      amount: number, 
      action: string
    ) {
      // 1. Deduct from org pool
      let remaining = amount;
      const orgPool = await this.getOrgCreditBalance(orgId);
      const fromOrg = Math.min(remaining, orgPool);
      remaining -= fromOrg;

      // 2. Deduct from user's personal credits
      if (remaining > 0) {
        const userPersonal = await this.getUserPersonalCredits(userId);
        const fromPersonal = Math.min(remaining, userPersonal);
        remaining -= fromPersonal;
      }

      // 3. Deduct from purchased packs (oldest first)
      if (remaining > 0) {
        await this.deductFromPurchasedPacks(orgId, remaining);
      }

      if (remaining > 0) {
        throw new ForbiddenException(
          'Insufficient AI credits. Purchase a credit pack to continue.'
        );
      }

      // Log consumption
      await this.logCreditUsage(orgId, userId, amount, action);
    }
    ```
  </Step>

  <Step title="Balance check">
    ```typescript
    async getCreditBalance(orgId: number): Promise<CreditBalance> {
      const sub = await this.getOrgSubscription(orgId);
      const usage = await this.subscriptionUsageRepo.findOne({ 
        subscription: sub 
      });

      const orgPool = sub.plan.aiCreditsPerMonth - usage.aiCreditsUsed;
      const purchasedPacks = await this.creditPurchaseRepo.find({
        organization: orgId,
        expiresAt: { $gt: new Date() },
        creditsRemaining: { $gt: 0 }
      });
      const purchasedTotal = purchasedPacks.reduce(
        (sum, pack) => sum + pack.creditsRemaining, 
        0
      );

      return {
        orgPool,
        purchased: purchasedTotal,
        total: orgPool + purchasedTotal
      };
    }
    ```
  </Step>
</Steps>

      organization: orgId,

    });

    await this.em.persistAndFlush(purchase);
  }

    async createCheckoutSession(
      orgId: number, 
      dto: CreateCheckoutDto
    ): Promise<{ url: string }> {
      const org = await this.orgRepo.findOneOrFail(orgId);
      
      // Reject if already has paid subscription
      if (org.subscription?.stripeSubscriptionId) {
        throw new BadRequestException(
          'Organization already has a subscription. Use change-plan instead.'
        );
      }

      const plan = await this.planRepo.findOneOrFail(dto.planId);
      
      // Build line items
      const lineItems: Stripe.Checkout.SessionCreateParams.LineItem[] = [
        {
          price: dto.billingCycle === 'annual' 
            ? plan.stripePriceIdAnnual 
            : plan.stripePriceIdMonthly,
          quantity: 1
        }
      ];

      // Add extra manager seats
      if (dto.seats?.manager > 0) {
        lineItems.push({
          price: dto.billingCycle === 'annual'
            ? plan.stripeManagerSeatPriceIdAnnual
            : plan.stripeManagerSeatPriceIdMonthly,
          quantity: dto.seats.manager
        });
      }

      // Add extra agent seats
      if (dto.seats?.agent > 0) {
        lineItems.push({
          price: dto.billingCycle === 'annual'
            ? plan.stripeAgentSeatPriceIdAnnual
            : plan.stripeAgentSeatPriceIdMonthly,
          quantity: dto.seats.agent
        });
      }

      // Create or retrieve Stripe customer
      if (!org.stripeCustomerId) {
        const customer = await this.stripe.customers.create({
          email: org.ownerEmail,
          name: org.name,
          metadata: { orgId: org.id }
        });
        org.stripeCustomerId = customer.id;
        await this.em.flush();
      }

      const session = await this.stripe.checkout.sessions.create({
        customer: org.stripeCustomerId,
        mode: 'subscription',
        line_items: lineItems,
        subscription_data: {
          trial_period_days: 90,  // Evergreen trial (§18.2)
          metadata: {
            orgId: org.id,
            planId: plan.id,
            billingCycle: dto.billingCycle,
            managerSeatsExtra: dto.seats?.manager || 0,
            agentSeatsExtra: dto.seats?.agent || 0
          }
        },
        success_url: `${this.config.frontendUrl}/billing/success?session_id={CHECKOUT_SESSION_ID}`,
        cancel_url: `${this.config.frontendUrl}/billing`
      });

      return { url: session.url };
    }

      const session = await this.stripe.checkout.sessions.retrieve(sessionId, {
        expand: ['subscription']
      });

      if (session.payment_status !== 'paid') {
        throw new BadRequestException('Payment not completed');
      }

      const { orgId, planId, billingCycle, managerSeatsExtra, agentSeatsExtra } 
        = session.metadata;

      const org = await this.orgRepo.findOneOrFail(orgId);
      const plan = await this.planRepo.findOneOrFail(planId);

      // Create or update subscription
      let sub = await this.subscriptionRepo.findOne({ organization: org });

      sub.agentSeatsPurchased = parseInt(agentSeatsExtra) || 0;

      await this.em.persistAndFlush(sub);

      return sub;
    }

    async changePlan(
      orgId: number, 
      dto: ChangePlanDto
    ): Promise<Subscription> {
      const sub = await this.getOrgSubscription(orgId);
      const newPlan = await this.planRepo.findOneOrFail(dto.newPlanId);

      // Validate seat capacity
      const usage = await this.getSeatUsage(orgId);
      if (usage.managerSeats > newPlan.managerSeatsIncluded + sub.managerSeatsPurchased) {
        throw new BadRequestException(
          `New plan supports ${newPlan.managerSeatsIncluded} manager seats. ` +
          `You currently have ${usage.managerSeats}. Remove users first.`
        );

      // Swap subscription price in Stripe
      const stripeSubItems = await this.stripe.subscriptionItems.list({
        subscription: sub.stripeSubscriptionId
      });

      const basePriceItem = stripeSubItems.data[0]; // First item is base plan

      await this.stripe.subscriptionItems.update(basePriceItem.id, {
        price: sub.billingCycle === 'annual'
          ? newPlan.stripePriceIdAnnual
          : newPlan.stripePriceIdMonthly,
        proration_behavior: 'create_prorations'

      // Update seat line items
      await this.reconcileSeatLineItems(
        sub.stripeSubscriptionId,
        sub.plan,
        newPlan,
        sub.billingCycle,
        sub.managerSeatsPurchased,
        sub.agentSeatsPurchased
      );

      // Update local record
      sub.plan = newPlan;
      await this.em.flush();

      return sub;
    }

    async switchBillingCycle(
      orgId: number, 
      newCycle: BillingCycle
    ): Promise<Subscription> {
      const sub = await this.getOrgSubscription(orgId);

      if (sub.billingCycle === newCycle) {
        throw new BadRequestException('Already on this billing cycle');
      }

      // Create subscription schedule for period end
      const schedule = await this.stripe.subscriptionSchedules.create({
        from_subscription: sub.stripeSubscriptionId
      });

      const newPriceId = newCycle === 'annual'
        ? sub.plan.stripePriceIdAnnual
        : sub.plan.stripePriceIdMonthly;

      await this.stripe.subscriptionSchedules.update(schedule.id, {
        phases: [
          {
            // Current phase continues until period end
            items: [{ price: sub.billingCycle === 'annual' 
              ? sub.plan.stripePriceIdAnnual 
              : sub.plan.stripePriceIdMonthly, quantity: 1 }],
            start_date: schedule.phases[0].start_date,
            end_date: schedule.phases[0].end_date
          },
          {
            // New phase starts at period end
            items: [{ price: newPriceId, quantity: 1 }],
            start_date: schedule.phases[0].end_date
          }
        ],
        proration_behavior: 'none'  // No proration for cycle switch
      });

      // Local update will happen via webhook at period end
      this.logger.log(
        `Scheduled billing cycle switch for org ${orgId} to ${newCycle} at period end`
      );

      return sub;
    }

    async processWebhook(event: Stripe.Event) {
      // Check idempotency (see BillingEvent entity)
      let billingEvent = await this.billingEventRepo.findOne({
        stripeEventId: event.id
      });

      if (billingEvent?.processed) {
        return; // Already handled
      }

      if (!billingEvent) {
        billingEvent = this.billingEventRepo.create({
          stripeEventId: event.id,
          eventType: event.type,
          payload: event.data.object
        });
        await this.em.persistAndFlush(billingEvent);
      }

      try {
        switch (event.type) {
          case 'checkout.session.completed':
            await this.handleCheckoutCompleted(event);
            break;
          case 'customer.subscription.created':
            await this.handleSubscriptionCreated(event);
            break;
          case 'customer.subscription.updated':
            await this.handleSubscriptionUpdated(event);
            break;
          case 'customer.subscription.deleted':
            await this.handleSubscriptionDeleted(event);
            break;
          case 'invoice.paid':
            await this.handleInvoicePaid(event);
            break;
          case 'invoice.payment_failed':
            await this.handleInvoicePaymentFailed(event);
            break;
          default:
            this.logger.warn(`Unhandled event type: ${event.type}`);
        }

        billingEvent.processed = true;
        billingEvent.processedAt = new Date();
      } catch (error) {
        billingEvent.errorMessage = error.message;
        throw error;
      } finally {
        await this.em.flush();
      }
    }

## Subscription lifecycle

### State diagram

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Subscription Status Flow                          │
└─────────────────────────────────────────────────────────────────────┘

    [NEW ORG]
        │
        ├─► FREE (default)
        │
        └─► Checkout initiated
                │
                ├─► TRIALING (90 days, card on file)
                │       │
                │       ├─► ACTIVE (trial converts)
                │       │
                │       └─► CANCELED (user cancels during trial)
                │
                └─► ACTIVE (immediate activation)
                        │
                        ├─► PAST_DUE (payment failure)
                        │       │
                        │       ├─► ACTIVE (payment succeeds)
                        │       │
                        │       └─► SUSPENDED (2-day grace period expired)
                        │               │
                        │               └─► ACTIVE (payment resumed)
                        │
                        ├─► CANCELED (user cancels, ends at period end)
                        │
                        └─► Plan change → ACTIVE (new tier)
```

### Status definitions

<AccordionGroup>
  <Accordion title="TRIALING">
    **Duration:** 90 days (evergreen trial, §18.2)
    
    **Behavior:**
    - Full access to paid plan features
    - Card required upfront (Stripe trial)
    - Auto-converts to ACTIVE when trial ends
    - Can cancel anytime during trial (no charge)
    
    **Enforcement:**
    - Feature gates use trial plan's feature flags
    - Resource limits apply from trial plan
    - Credits allocated from trial plan
  </Accordion>

  <Accordion title="ACTIVE">
    **Conditions:**
    - Trial converted OR immediate paid subscription
    - Last payment succeeded
    
    **Behavior:**
    - Full access to plan features
    - Charges occur at period end (monthly/annual)
    - Can upgrade/downgrade anytime
    - Can cancel (access continues until period end)
    
    **Enforcement:**
    - All features enabled per plan
    - Resource limits enforced
    - Credits replenished each period
  </Accordion>

  <Accordion title="PAST_DUE">
    **Trigger:** Payment failure on renewal
    
    **Behavior:**
    - **Grace period:** 2 days (Stripe retry logic)
    - Org remains fully functional during grace
    - Stripe auto-retries payment
    - Owner receives email notifications
    
    **Resolution:**
    - Payment succeeds → back to ACTIVE
    - All retries fail → SUSPENDED
  </Accordion>

  <Accordion title="SUSPENDED">
    **Trigger:** Payment failure after grace period
    
    **Behavior:**
    - **Read-only mode:** Users can view data but not create/edit
    - `SubscriptionActiveGuard` blocks write operations
    - Billing portal remains accessible
    - Data retained (not deleted)
    
    **Resolution:**
    - User updates payment method → back to ACTIVE
    - No action for 30 days → auto-cancel to FREE
    
    **Enforcement:**
    ```typescript
    @UseGuards(JwtAuthGuard, SubscriptionActiveGuard)
    @Post('leads')
    async createLead() {
      // Blocked if subscription.status === SUSPENDED
    }
    ```
  </Accordion>

  <Accordion title="CANCELED">
    **Trigger:** User cancels subscription
    
    **Behavior:**
    - Access continues until period end
    - No further charges
    - At period end → downgrade to FREE
    - Data retained on FREE plan (subject to Free limits)
    
    **Cancellation flow:**
    ```typescript
    async cancelSubscription(orgId: number, immediate: boolean = false) {
      const sub = await this.getOrgSubscription(orgId);
      
      await this.stripe.subscriptions.update(sub.stripeSubscriptionId, {
        cancel_at_period_end: !immediate
      });

      if (immediate) {
        await this.stripe.subscriptions.cancel(sub.stripeSubscriptionId);
        sub.status = SubscriptionStatus.CANCELED;
        sub.canceledAt = new Date();
      } else {
        sub.cancelAtPeriodEnd = true;
      }

      await this.em.flush();
    }
    ```
  </Accordion>

  <Accordion title="FREE">
    **Conditions:**
    - Default for new organizations
    - Downgrade destination from canceled/suspended paid plans
    
    **Behavior:**
    - Limited feature set (see feature flags)
    - Hard resource caps (50 leads, 100 contacts, etc.)
    - No AI credits (org pool = 0, per-user = 0)
    - No Stripe subscription (stripeSubscriptionId = null)
    
    **Enforcement:**
    - Free plan is always id=1 in SubscriptionPlan table
    - Missing subscription record is treated as Free
  </Accordion>
</AccordionGroup>
