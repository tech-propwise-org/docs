This document outlines the comprehensive rollout plan for the three-tier subscription system (Free/Pro/Business) with AED pricing. The implementation follows a backend-first approach across 11 phases.

## Overview

The rollout implements the subscription packaging strategy with:
- **Free/Pro/Business tiers** with AED pricing
- **90-day evergreen trial** for Pro & Business
- **Unified credit wallet** with shared org pool and per-user ceilings
- **Entity caps** for Free tier (10 companies, 25 inventory units)
- **Single-seat model** (collapsed from manager/agent dual-SKU)

<Note>
This plan addresses 12 identified gaps from the strategy document and has been code-verified against the current implementation.
</Note>

## Phase 3: Credit Metering Service

Create unified credit wallet system with enforcement flags.

<CodeGroup>
```typescript
// credit-metering.service.ts
@Injectable()
export class CreditMeteringService {
  async getBalance(orgId: string): Promise<CreditBalance> {
    if (!this.configService.isSubscriptionEnforcementEnabled()) {
      return { available: Number.MAX_SAFE_INTEGER, total: Number.MAX_SAFE_INTEGER };
    }
    
    const usage = await this.getUsage(orgId);
    const allowance = await this.getAllowance(orgId);
    
    return {
      available: Math.max(0, allowance.total - usage.consumed),
      total: allowance.total,
      consumed: usage.consumed
    };
  }

  async canSpend(orgId: string, amount: number): Promise<boolean> {
    if (!this.configService.isSubscriptionEnforcementEnabled()) {
      return true;
    }
    
    const balance = await this.getBalance(orgId);
    return balance.available >= amount;
  }
}
```

```typescript
// Integration with existing credit system
async addCredits(orgId: string, amount: number, context: CreditContext) {
  // Consolidate into single credit store
  await this.creditService.addCredits(orgId, CreditType.AI, amount);
  
  // Record purchase if applicable
  if (context.source === 'PURCHASE') {
    await this.creditPurchaseService.record(orgId, amount, context);
  }
}
```
</CodeGroup>

## Phase 7: Storage Limits

<Steps>
<Step title="Implement storage enforcement">
```typescript
// storage-limit.service.ts
async enforceStorageLimit(orgId: string, uploadSize: number): Promise<void> {
  if (!this.configService.isSubscriptionEnforcementEnabled()) {
    return; // Honor enforcement flag
  }

  const plan = await this.subscriptionService.getOrgPlan(orgId);
  const currentUsage = await this.getStorageUsage(orgId);
  
  if (currentUsage + uploadSize > plan.storageLimit) {
    throw new StorageLimitExceededError('Storage limit exceeded');
  }
}
```
</Step>

<Step title="Update upload handlers">
```typescript
// Hard block user uploads, soft degrade messaging media
async handleUserUpload(file: File, orgId: string) {
  await this.storageLimitService.enforceStorageLimit(orgId, file.size);
  // Proceed with upload
}

async handleMessagingMedia(media: Media, orgId: string) {
  try {
    await this.storageLimitService.enforceStorageLimit(orgId, media.size);
    // Process normally
  } catch (StorageLimitExceededError) {
    // Emit over-limit signal but don't drop inbound demand
    this.eventBus.emit('storage.limit.exceeded', { orgId, mediaId: media.id });
  }
}
```
</Step>

## Risk Mitigation

<AccordionGroup>
<Accordion title="Budget replacement risks">
Phase 5 replaces existing budget gates with behavior-equivalent credit gates. Verify that `AiUsageGuardService.assertCanCallModel` and agent-side guards maintain identical blocking behavior.
</Accordion>

<Accordion title="Seat model transition">
The single-seat collapse requires careful backfill of existing dual-seat data and Stripe subscription line items to avoid billing disruption.
</Accordion>

<Accordion title="Trial activation paths">
Both webhook and direct checkout paths must derive trial status from shared `buildActivationParamsFromCheckoutMetadata` to prevent divergence.
</Accordion>

<Accordion title="Entity cap enforcement">  
Hard-block user-initiated creates but soft-degrade inbound WhatsApp auto-create to avoid dropping live demand.
</Accordion>
