This implementation plan executes the subscription packaging strategy with a backend-first approach, followed by frontend integration. All phases are backed by code-level verification passes confirming current-state facts and dependencies.

## Overview

This rollout implements three subscription tiers (Free, Professional, Business) with:

- **90-day evergreen trial** on Pro & Business (card-upfront)
- **Unified credit wallet** (consolidated AI + messaging consumption)
- **Single per-agent seat model** (manager/agent SKUs collapsed)
- **Entity caps for Free tier** (10 companies, 25 units, 1GB storage)
- **Open inventory** to all tiers (25-unit cap on Free)

## Key Design Decisions

<AccordionGroup>
  <Accordion title="Tier Naming & Enum Strategy">
    **Internal tier ID remains `professional`** but displays as "Pro" throughout UI. This avoids renaming the enum across:
    - Database rows
    - Stripe `metadata.tier`
    - `OrganizationDto.subscriptionPlanTier`
    - Frontend types
    
    **Starter tier is hard-removed** via data migration (repoint existing Starter subscriptions → Professional) before enum deletion.
  </Accordion>

  <Accordion title="Unified Credit Wallet Architecture">
    - **Single org-level credit pool** replaces dual AI budget planes (`AiUsageAccount`, `AiCreditUsage`)
    - **Per-user ceilings** cap individual consumption from shared pool (not reserved carve-outs)
    - **Consumption order**: org pool first → personal credits fallback
    - **Managed by** new `CreditMeteringService` facade (wraps `CreditService` + consolidated allocation table)
    - **OpenAI-only** for billable traffic (per-org project key); Anthropic stays for eval judge only
  </Accordion>

  <Accordion title="Seat Model Consolidation">
    Collapse manager/agent two-SKU model into **single per-agent seat SKU**:
    - Repurpose existing `agentSeats*` columns as single seat counter
    - Zero out `managerSeats*` (no DDL required)
    - Backfill `agentSeatsTotal = manager + agent` for existing rows
    - Replace dual Stripe line items on cutover
  </Accordion>

  <Accordion title="Entity Cap Enforcement">
    **Count predicate**: non-imported AND `isDeleted = false` (archived rows still count)
    
    **Write enforcement split**:
    - **Hard-block**: user-initiated creates (manual, create-from-listing, API) → 403
    - **Soft-degrade**: inbound WhatsApp/Meta auto-create (create + emit over-limit signal)
    - **Skip**: bulk-import path (doesn't count against cap)
    
    **Source-of-record**:
    - Leads: reuse `Lead.leadSource === LeadSource.IMPORTED`
    - Contact/Deal/Company: new `isImported` boolean (migration required)
    - Auto-created companies during contact import MUST be stamped `isImported = true`
  </Accordion>

  <Accordion title="Budget Model & Allocation">
    **Shared pool with per-user ceilings**:
    - Effective user draw = `min(userLimit - userSpent, orgPoolRemaining)`
    - Ceilings MAY oversubscribe (no Σ caps ≤ allowance validation)
    - Remove `allowance - Σ budgets` math from agent budget allocation
    
    **Personal credit fallback**:
    - Applies to ANY human actor (owner/admin included)
    - Org-level-channel auto-reply has no human actor → no personal layer
    
    **Routing**:
    - Owner/Admin + org-level-channel AI → org pool
    - Regular agent assistant/Propilot + personal-channel AI → user allocation → personal credits
  </Accordion>

  <Accordion title="Access Control Split">
    **Organization buying & subscription management**: `system.owner` only
    - Route: `/settings/billing`
    - Actions: view balance/usage, buy credits/storage, manage subscription
    
    **Per-user ceiling distribution**: admin-level (`SYSTEM_ADMIN`)
    - Managed within org settings (no standalone `/settings/ai-credits` page)
  </Accordion>
</AccordionGroup>

    export class StorageGuardService {
      async assertCanUpload(orgId: string, fileSizeMB: number): Promise<void> {
        if (!this.configService.isSubscriptionEnforcementEnabled()) {
          return; // enforcement OFF → allow
        }
        
        const usage = await this.getUsage(orgId);

          { organizationId: orgId },
          'storageUsedGB',
          fileSizeMB / 1024
        );
      }
    }

    New service in `subscription/services/credit-metering.service.ts` (wraps `CreditService` + consolidated allocation table):
    
    ```typescript
    @Injectable()
    export class CreditMeteringService {
      constructor(
        private creditService: CreditService,
        private subscriptionUsageRepo: Repository<SubscriptionUsage>,
        private userCreditAllocationRepo: Repository<UserCreditAllocation>,
      ) {}
      
      async getOrgBalance(orgId: string): Promise<CreditBalance> {
        if (!this.configService.isSubscriptionEnforcementEnabled()) {
          return { available: Infinity, used: 0 }; // enforcement OFF
        }
        
        const usage = await this.subscriptionUsageRepo.findOne({ organizationId: orgId });
        const allowance = this.calculateMonthlyAllowance(usage); // Phase 5.4
        const purchased = await this.creditService.getPurchasedCredits(orgId);
        const spent = usage.aiCreditsUsed; // numeric(12,6), currently 0
        
        return {
          available: allowance + purchased - spent,
          used: spent,
          allowance,
          purchased,
        };
      }
      
      async getUserBalance(userId: string, orgId: string): Promise<CreditBalance> {
        const orgBalance = await this.getOrgBalance(orgId);
        const allocation = await this.userCreditAllocationRepo.findOne({ userId, orgId });
        const personalPurchased = await this.creditService.getUserPurchasedCredits(userId);
        
        const userOrgSpent = allocation?.spent || 0;
        const userLimit = allocation?.limit || Infinity;
        const effectiveOrgDraw = Math.min(userLimit - userOrgSpent, orgBalance.available);
        
        return {
          available: effectiveOrgDraw + personalPurchased,
          orgPoolAvailable: effectiveOrgDraw,
          personalCredits: personalPurchased,
          used: userOrgSpent,
          limit: userLimit,
        };
      }
    }
    ```
