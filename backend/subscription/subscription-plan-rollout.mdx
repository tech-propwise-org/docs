This document outlines the phased implementation plan for rolling out the Free/Pro/Business subscription tiers based on the subscription packaging strategy. The rollout is organized into 11 phases that prioritize backend changes first, then frontend implementation.

<Note>
This implementation plan addresses 12 identified gaps from the subscription packaging strategy, grouping them into sequential phases with backend-first approach.
</Note>

## Current State Verification

A comprehensive code-level verification confirmed the following current state facts:

- **Assistant budget gate is already LIVE** - `AiUsageGuardService.assertCanCallModel` throws `AiBudgetExceededError` to block calls today
- **USD tracking exists in `AiCreditUsage`/`AiUsageAccount`** - not in `aiCreditsUsed` field
- **One-time credit purchases need new webhook routing** - current `routeEvent` only handles 5 subscription events
- **Trial UX is partially implemented** - `trialEndsAt` DTO field and static "Trial" badge exist
- **Free tier currently has 20 `aiCreditsPerMonth`** vs proposed 2000 (intentional 100× rebase)

Create entity count guards:

```typescript
async enforceEntityLimit(orgId: string, entityType: 'contact' | 'company' | 'deal') {
  const plan = await this.getOrganizationPlan(orgId);
  const limit = plan.limits[`max${entityType}s`];
  
  const count = await this.getEntityCount(orgId, entityType);
  if (count >= limit) {
    throw new PlanLimitExceededError(`${entityType} limit reached`);
  }
}

private async getEntityCount(orgId: string, entityType: string) {
  // Count non-imported AND isDeleted = false
  // Archived rows still count (isArchived filter disabled)
  return this.repository.count({
    organizationId: orgId,
    isDeleted: false,
    isImported: false
  });
}
```
</Step>

<Step title="Apply Enforcement Strategy">
- **Hard block**: User-initiated creates (manual, API)
- **Soft degrade**: Inbound WhatsApp/Meta auto-creates
- **Skip**: Bulk import operations
</Step>
</Steps>

<Steps>

```typescript
interface UserAllocation {
  userId: string;
  orgId: string;

  orgSpent: number; // User's consumption from org pool

}
```
</Step>

<Step title="Create Consumption Logic">
Implement org-first, then personal consumption:

```typescript
async consumeCredits(context: SpendContext, amount: number) {
  const allocation = await this.getUserAllocation(context.userId, context.orgId);
  
  // Calculate available org credits for this user
  const userOrgAvailable = Math.min(
    allocation.personalLimit - allocation.orgSpent,
    await this.getOrgPoolRemaining(context.orgId)
  );
  
  if (amount <= userOrgAvailable) {
    // Consume from org pool
    await this.chargeOrgPool(context, amount);
  } else {
    // Consume org available + personal credits
    await this.chargeHybrid(context, amount, userOrgAvailable);
  }
}
```
</Step>

## Execution Sequencing

<Tip>
**Recommended execution order:**

1. **Phases 0-3**: Core infrastructure (tiers, webhooks, credit system)
2. **Phase 9**: Seat count consolidation (required for Phase 5.4)
3. **Phases 4-8**: Enforcement systems (entities, budgets, storage, inventory)

This order minimizes dependencies and allows for incremental testing.
</Tip>
