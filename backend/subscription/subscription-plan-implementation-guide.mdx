## Overview

This document implements the subscription packaging strategy with a phased approach, addressing 12 identified gaps through backend-first development followed by frontend integration. The rollout introduces three tiers:

- **Free**: Limited features with caps
- **Pro**: Enhanced capabilities for growing teams  
- **Business**: Full feature set for larger organizations

### Phase 3: Credit Metering Service

<Info>
The unified wallet consolidates all credit types into a single balance while maintaining enforcement flag compliance.
</Info>

<Steps>
<Step title="Service Architecture">
Create centralized credit management facade:

```typescript
// credit-metering.service.ts
@Injectable()
export class CreditMeteringService {
  constructor(
    private creditService: CreditService,
    private configService: ConfigService,
    private subscriptionService: SubscriptionService
  ) {}

  async getBalance(orgId: string, userId?: string): Promise<CreditBalance> {
    if (!this.isEnforcementEnabled()) {
      return { org: Infinity, personal: Infinity };
    }

    const orgBalance = await this.getOrgBalance(orgId);
    const personalBalance = userId ? 
      await this.getPersonalBalance(userId) : 0;
    
    return { org: orgBalance, personal: personalBalance };
  }

  private isEnforcementEnabled(): boolean {
    return this.configService.get<boolean>('SUBSCRIPTION_ENFORCEMENT_ENABLED');
  }
}
```
</Step>

<Step title="Consumption Model">
Implement org-first, personal-fallback consumption:

```typescript
async consumeCredits(context: SpendContext, amount: number): Promise<void> {
  const balance = await this.getBalance(context.orgId, context.userId);
  
  if (balance.org >= amount) {
    await this.deductOrgCredits(context.orgId, amount);
  } else if (context.userId && balance.personal >= amount) {
    await this.deductPersonalCredits(context.userId, amount);
  } else {
    throw new InsufficientCreditsError();
  }
}
```
</Step>
</Steps>

<Step title="Enforcement Guards">
Implement cap enforcement with soft/hard blocking:

```typescript
async enforceEntityCap(
  orgId: string, 
  entityType: EntityType,
  isUserInitiated: boolean = true
): Promise<void> {
  const plan = await this.subscriptionService.getPlan(orgId);
  const limit = plan.limits[entityType];
  const current = await this.getEntityCount(orgId, entityType);
  
  if (current >= limit) {
    if (isUserInitiated) {
      throw new EntityCapExceededError(entityType, limit);
    } else {
      // Soft degrade for auto-created entities
      await this.emitOverLimitSignal(orgId, entityType);
    }
  }
}
```

### Phase 6: Storage Management

<Steps>
<Step title="Storage Guards">
Implement storage quota enforcement:

```typescript
// storage-guard.service.ts
async checkStorageQuota(orgId: string, fileSize: number): Promise<void> {
  if (!this.isEnforcementEnabled()) return;
  
  const plan = await this.subscriptionService.getPlan(orgId);
  const currentUsage = await this.getStorageUsage(orgId);
  
  if (currentUsage + fileSize > plan.storageLimit) {
    throw new StorageQuotaExceededError();
  }
}

private isEnforcementEnabled(): boolean {
  return this.configService.get<boolean>('SUBSCRIPTION_ENFORCEMENT_ENABLED');
```
