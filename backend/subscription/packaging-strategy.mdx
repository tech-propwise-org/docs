## Summary of Decisions

The PropWise CRM subscription model has been designed specifically for the UAE real estate market with the following key decisions:

| Decision | Choice |

| **Currency** | **AED** native (charged in AED on Stripe) |
| **Tiers** | **Free â†’ Pro â†’ Business** (Starter dropped) |
| **Growth wedge** | **3-month free trial on Pro & Business**, card upfront, charged only after day 90, cancel anytime before with no charge |
| **Seat model** | **Flat per-agent pricing** (one seat SKU per tier; dual manager/agent model dropped). **Pro 5â€“10 seats** (11+ must upgrade to Business); **Business 10+ seats with a volume discount** |
| **Off-plan directory** | **Public to everyone** (incl. Free) |
| **Secondary directory** | **Public to everyone** (incl. Free) |

## The 3-Month Free Trial (Growth Wedge)

The centerpiece of our go-to-market strategy: anyone can subscribe to **Pro or Business** by adding a card, get the **full plan experience for 90 days at AED 0**, and cancel anytime before day 90 with no charge.

### Trial Mechanics

<Steps>
  <Step title="Card Capture at Checkout">
    Card captured at checkout; **AED 0 charged** until day 90
  </Step>
  
  <Step title="Full Access During Trial">
    During trial: subscription status is `TRIALING` and grants **full access to the chosen plan**
  </Step>
  
  <Step title="Cancellation Before Day 90">
    Cancel before day 90 â†’ no charge â†’ downgrade to Free
  </Step>
  
  <Step title="Auto-Conversion">
    No cancel â†’ auto-converts to the paid plan on day 90
  </Step>
  
  <Step title="Reminder Notification">
    Reminder sent ~3 days before trial ends
  </Step>
</Steps>

<Info>
**Business trial** lets you onboard a **large agency** risk-free. 50+-seat agencies still come through contact-sales, where the trial terms are confirmed as part of the deal.
</Info>

### Margin Protection

<Warning>
The trial waives only the **subscription fee**. The unified AI-credit wallet still applies at the plan's levels during the trial, and overage still requires buying top-up packs (a real charge). This prevents 90 days of free AI cost-of-goods â€” which matters most on Business, where the included credit pool is large.
</Warning>

<Tip>
Most of the plumbing already exists: Stripe checkout accepts `trialPeriodDays`, and the subscription entity has `trialEndsAt` + a `TRIALING` status. Remaining work is enforcement wiring + the `trial_will_end` reminder webhook.
</Tip>

### How Entity Caps Are Counted

<Info>
The lead / contact / deal / company caps apply **only to records created inside the app**. **Imported records never count** against the cap and are always unlimited on every tier.
</Info>

| Source of Record | Counts Toward Cap? |
|------------------|-------------------|
| CSV / Excel / bulk import | **No** â€” always unlimited |
| Manual create (Add Lead / Add Contact / â€¦) | Yes |
| Auto-created from a channel (WhatsApp / Meta inbound) | Yes |
| Created from a listing / property interest | Yes |
| Created by distribution / automation / API | Yes |

<Tip>
**Why this design:** Brokerages can migrate their entire existing database in for free (removes the #1 onboarding blocker), while the cap still meters ongoing *organic* growth created through the product.
</Tip>

<Note>
**Trade-off to be aware of:** Because imports are uncapped, record counts are a soft lever, not a hard paywall â€” a team could bulk-import everything and rarely hit the cap. That's intentional here: the real upgrade drivers become **features** (distribution, analytics, teams), **seats**, and the **unified AI-credit wallet** (Propilot, AI messaging, valuations), not raw record volume.
</Note>

<Check>
**Implementation note:** Enforcing this requires the counter to be **source-aware** â€” increment only on in-app creation paths, and skip the import path entirely.
</Check>

## What Is Public vs Gated

### Public to Everyone (Including Free)

The following features are **never gated** and available to all users:

- Core CRM: leads, contacts, deals, companies, pipeline, tasks
- Off-plan directory
- Secondary listings directory
- Developer portal
- **Teams & hierarchy** (team management + team-scoped roles)
- **Advanced analytics** â€” the `/analytics` reporting module (agent & manager performance, CSV export); the base `/dashboard` dashboard is always available too
- **Lead / deal transfer system** (assignment handoff with approval)

### The Paid Real-Estate Lever

<Warning>
**Unit valuation** â€” metered from the **unified AI-credit wallet**. Anyone can browse off-plan / secondary for free; *valuing* a unit costs **300 credits**, charged every time (cache or fresh). It is the most credit-expensive action, so heavy valuation users burn the wallet fastest and convert to top-ups.
</Warning>

### Gated by Tier (The SaaS Engine)

Features that require paid plans:

- Distribution engine, escalation engine
- Commission tracking
- Custom roles (Pro & Business)
- Audit log retention, channel/integration counts, resource caps

### Removed from Scope

The following features are **not offered** in any tier:

- **Marketplace** â€” unlinked from modules
- **White-label branding** â€” not offered
- **Public API access** â€” not offered

## Credit System (One Unified Wallet)

All metered AI usage draws from **one balance** â€” there are no separate AI / messaging / valuation buckets. The org gets a monthly pool of **AI credits**, pooled across all seats, and spends it on whatever it wants (chat, auto-reply, or valuations).

### Credit Value Model

<Info>
Credits are denominated at **20 credits = AED 1 of value** (the Ã—20 factor keeps in-app numbers large and decoupled from cash, so customers reason in credits, not dirhams).
</Info>

### Per-Action Credit Cost

<Note>
Prices were recalibrated 2025-07-15 against measured per-action COGS (gpt-5.6 tier models) to a **uniform ~10% COGS-to-face-value ratio** (`credits â‰ˆ 200 Ã— COGS_AED`), which makes the full-allowance burn cost mix-invariant (â‰ˆ AED 20â€“25 on a 5,000-credit Pro seat regardless of action mix). Valuation stays value-priced above formula; optimize/KB sit at the 5â€“20 floor.
</Note>

| Action | Credits | Notes |
|--------|---------|-------|
| **AI message reply (auto-reply)** | 5 | Only **AI-generated** replies burn credits; tiny-context action (~6K tokens) â€” 5 credits â‰ˆ 1,000 replies/month on a Pro seat |
| **Propilot prompt â€” lite model** | 20 | Cheapest model tier |
| **Propilot prompt â€” standard model** | 45 | Default tier (gpt-5.6-luna) |
| **Propilot prompt â€” max model** | 100 | Flagship model (gpt-5.6-terra) |
| **AI agent prompt optimize** | 5 | Near-zero COGS; floor-priced so users are not discouraged from improving agent instructions |
| **Deep web research surcharge** | +30 | Added **on top** of the per-tier prompt cost when the Market & Regulations persona runs a `deep` web-research turn that actually scrapes the web (Fast research adds nothing) |
| **Knowledge-base ingestion** | 20 + 5/extra chunk | **Dynamic** â€” base 20 (â‰¤1 chunk) + 5 per additional chunk, capped at 1,000, so a large document/site costs proportionally more than a one-line FAQ |
| **Unit valuation** | 300 | Charged **every time** (cache or fresh); value-priced premium anchor |

### Monthly Allowance (Pooled at Org Level)

| Tier | Allowance | Min-Seat Org Total |
|------|-----------|-------------------|
| **Free** | 2,000 / org | 2,000 |
| **Pro** | 5,000 / seat | 25,000 (5 seats) |
| **Business** | 9,000 / seat | 90,000 (10 seats) |

<Info>
**Pooled** means a heavy agent draws from the shared org balance â€” there is no hard per-seat cap. Top-up packs refill the same wallet, sold as **non-round bundles** with the per-credit cash rate never advertised.
</Info>

### Economics

<Tabs>
  <Tab title="Value Anchor">
    Internal value anchor: **20 credits = AED 1** (1 credit â‰ˆ AED 0.05). Retail credits sell at ~**90% gross margin** over model cost-of-goods (uniform ~10% COGS ratio, measured 2025-07-15) â€” because we price the **outcome**, not the tokens.
  </Tab>
  
  <Tab title="Cache Pricing">
    **Always charge on cache:** A valuation costs 300 credits even when served from the 7-day cache (cache hits cost ~AED 0 â†’ ~100% margin on those).
  </Tab>
  
  <Tab title="External Fees">
    **Meta fees are external:** WhatsApp/Meta conversation & template charges are paid by the customer directly to Meta (their own WABA). They are **not** in the wallet â€” only the AI model cost of an auto-reply is.
  </Tab>
</Tabs>

### Out of Credits Behavior

When the wallet hits zero, the AI **hard-stops with a warning** ("Propilot is out of credits â€” buy a top-up or upgrade").

<Warning>
To avoid dropping live leads:
- Owner/admins are alerted at **80%** and **100%** consumption (in-app + email)
- On the **AI auto-reply** path, the conversation is set to **needs human** (and/or an AI draft is saved) instead of going silent, so an agent can step in
</Warning>

```typescript Wallet Initialization
interface CreditWallet {
  orgId: string;
  balance: number;

    at80: boolean;
    at100: boolean;
  };

function calculateMonthlyAllowance(
  tier: 'FREE' | 'PRO' | 'BUSINESS',
  seatCount: number
): number {
  switch (tier) {
    case 'FREE':
      return 2000;
    case 'PRO':
      return 5000 * seatCount;
    case 'BUSINESS':
      return 9000 * seatCount;
  }
}
```

```typescript Credit Deduction
async function deductCredits(
  orgId: string,
  action: CreditAction,
  amount: number
): Promise<DeductionResult> {
  const wallet = await getOrgWallet(orgId);
  
  if (wallet.balance < amount) {
    await sendOutOfCreditsAlert(orgId);
    return {
      success: false,
      reason: 'INSUFFICIENT_CREDITS',
      remaining: wallet.balance
    };
  }
  
  const newBalance = wallet.balance - amount;
  await updateWalletBalance(orgId, newBalance);
  
  // Check alert thresholds
  const usagePercent = 
    (wallet.monthlyAllowance - newBalance) / wallet.monthlyAllowance;
  
  if (usagePercent >= 0.8 && !wallet.alertsSent.at80) {
    await send80PercentAlert(orgId);
  }
  
  if (usagePercent >= 1.0 && !wallet.alertsSent.at100) {
    await send100PercentAlert(orgId);
  }
  
  return {
    success: true,
    remaining: newBalance,
    action,
    amount
  };
}
```

```typescript Import Counter Bypass
async function createLead(
  data: LeadData,
  source: 'MANUAL' | 'IMPORT' | 'CHANNEL'
): Promise<Lead> {
  const lead = await Lead.create(data);
  
  // Only count toward cap if NOT from import

  }
  
  return lead;
}
```

### AI Auto-Reply Fallback

<CodeGroup>
```typescript Auto-Reply Handler
async function handleAutoReply(
  conversationId: string,
  inboundMessage: Message
): Promise<void> {
  const org = await getConversationOrg(conversationId);
  const wallet = await getOrgWallet(org.id);
  
  if (wallet.balance < 5) {
    // Out of credits - fallback to human
    await setConversationStatus(
      conversationId,
      'NEEDS_HUMAN'
    );
    
    // Still generate draft for agent (free)
    const draft = await generateReplyDraft(inboundMessage);
    await saveDraftReply(conversationId, draft);
    
    await notifyAssignedAgent(conversationId, {
      reason: 'OUT_OF_CREDITS',
      hasDraft: true
    });
    
    return;
  }
  
  // Generate and send AI reply
  const reply = await generateAIReply(inboundMessage);
  await deductCredits(org.id, 'AI_MESSAGE_REPLY', 5);
  await sendMessage(conversationId, reply);
}
```
