# API Authentication Guide

Learn how to authenticate with the PropWise CRM API using API keys and Bearer tokens. All API endpoints require proper authentication to ensure secure access to your data.

## Overview

All API endpoints require a Bearer token for authentication. You obtain a short-lived token by exchanging your **API Key** and **API Secret** via the token endpoint.

**Base URL:** `https://<host>/api`

<Note>
Keep your API Secret secure and never expose it in client-side code. Treat it like a password.
</Note>

## Authentication Process

<Steps>
<Step title="Get Your Credentials">
You will receive two values from the PropWise platform:

- **API Key** — A public identifier for your client (e.g., `pk_live_abc123...`)
- **API Secret** — A private secret that must be kept secure

<Warning>
Never expose your API Secret in client-side code or public repositories. Store it securely in your backend environment.
</Warning>
</Step>

<Step title="Exchange Credentials for a Token">
Make a POST request to the token endpoint with your credentials:

```bash
POST /api/auth/token
Content-Type: application/json

{
  "apiKey": "your-api-key",
  "apiSecret": "your-api-secret"
}
```

**Successful Response (200):**

```json
{
  "access_token": "eyJhbGciOi...",
  "token_type": "Bearer",
  "expires_in": 1800
}
```

### Response Fields

| Field          | Description                              |
|----------------|------------------------------------------|
| `access_token` | JWT token to use in subsequent requests  |
| `token_type`   | Always `Bearer`                          |
| `expires_in`   | Token lifetime in seconds (default: 30m) |

### Error Responses

| Status | Reason                          |
|--------|---------------------------------|
| `401`  | Invalid API key or secret       |
| `401`  | API client is disabled          |
</Step>

<Step title="Use the Token in API Requests">
Pass the token as a `Bearer` token in the `Authorization` header on every API request:

```bash
GET /api/developers/search?q=emaar
Authorization: Bearer eyJhbGciOi...
```

<Info>
When the token expires, request a new one via `POST /api/auth/token` again. Tokens are valid for 30 minutes by default.
</Info>
</Step>
</Steps>

## Access Levels

Your API client has an assigned access level that controls which endpoints you can use:

<CardGroup cols={2}>
<Card title="Basic Access" icon="shield">
Standard access to public-facing endpoints
</Card>
<Card title="Super Access" icon="shield-check">
Extended access to restricted endpoints
</Card>
</CardGroup>

<Warning>
If you call an endpoint that requires a higher access level, you will receive a `403 Forbidden` response.
</Warning>

## Rate Limiting

API requests are rate-limited per client to ensure fair usage and system stability.

<Note>
The default limit is **60 requests per window**. If you exceed this limit, you will receive a `429 Too Many Requests` response. Back off and retry after the window resets.
</Note>

## Quick Reference

| What                | Value                          |
|---------------------|--------------------------------|
| Token endpoint      | `POST /api/auth/token`         |
| Auth header format  | `Authorization: Bearer <token>`|
| Default token TTL   | 30 minutes                     |
| Default rate limit  | 60 req/window                  |

## Complete Example

<CodeGroup>
```bash cURL
# 1. Get a token
curl -X POST https://<host>/api/auth/token \
  -H "Content-Type: application/json" \
  -d '{"apiKey":"your-api-key","apiSecret":"your-api-secret"}'

# 2. Use the token
curl https://<host>/api/developers/search?q=emaar \
  -H "Authorization: Bearer <access_token>"
```

```javascript Node.js
// 1. Get a token
const tokenResponse = await fetch('https://<host>/api/auth/token', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    apiKey: 'your-api-key',
    apiSecret: 'your-api-secret'
  })
});

const { access_token } = await tokenResponse.json();

// 2. Use the token
const apiResponse = await fetch('https://<host>/api/developers/search?q=emaar', {
  headers: {
    'Authorization': `Bearer ${access_token}`
  }
});
```

```python Python
import requests

# 1. Get a token
token_response = requests.post('https://<host>/api/auth/token', json={
    'apiKey': 'your-api-key',
    'apiSecret': 'your-api-secret'
})

access_token = token_response.json()['access_token']

# 2. Use the token
api_response = requests.get('https://<host>/api/developers/search?q=emaar', 
    headers={'Authorization': f'Bearer {access_token}'}
)
```
</CodeGroup>

<Tip>
Consider implementing token refresh logic in your application to automatically obtain a new token when the current one expires.
</Tip>
