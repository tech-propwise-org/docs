## Overview

All API endpoints require a Bearer token. You obtain a short-lived token by exchanging your **API Key** and **API Secret** via the token endpoint.

**Base URL:** `https://<host>/api`

<Note>
Your API credentials should be treated as sensitive information. Never expose your API Secret in client-side code or public repositories.
</Note>

## Authentication flow

<Steps>
<Step title="Get your credentials">
You will receive two values from the platform:

- **API Key** — a public identifier for your client (e.g., `pk_live_abc123...`)
- **API Secret** — a private secret (treat like a password, do not expose in client-side code)
</Step>

<Step title="Exchange credentials for a token">
Make a POST request to the token endpoint with your credentials:

<CodeGroup>
```bash cURL
curl -X POST https://<host>/api/auth/token \
  -H "Content-Type: application/json" \
  -d '{"apiKey":"your-api-key","apiSecret":"your-api-secret"}'
```

```javascript JavaScript
const response = await fetch('https://<host>/api/auth/token', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    apiKey: 'your-api-key',
    apiSecret: 'your-api-secret'
  })
});

const data = await response.json();
```

```python Python
import requests

response = requests.post('https://<host>/api/auth/token', 
  json={
    'apiKey': 'your-api-key',
    'apiSecret': 'your-api-secret'
  }
)

data = response.json()
```
</CodeGroup>

**Response `200`:**

```json
{
  "access_token": "eyJhbGciOi...",
  "token_type": "Bearer",
  "expires_in": 1800
}
```

| Field          | Description                              |
|----------------|------------------------------------------|
| `access_token` | JWT token to use in subsequent requests  |
| `token_type`   | Always `Bearer`                          |
| `expires_in`   | Token lifetime in seconds (default: 30m) |

<AccordionGroup>
<Accordion title="Common authentication errors">
| Status | Reason                          |
|--------|---------------------------------|
| `401`  | Invalid API key or secret       |
| `401`  | API client is disabled          |
</Accordion>
</AccordionGroup>
</Step>

<Step title="Use the token">
Pass the token as a `Bearer` token in the `Authorization` header on every API request:

<CodeGroup>
```bash cURL
curl https://<host>/api/developers/search?q=emaar \
  -H "Authorization: Bearer <access_token>"
```

```javascript JavaScript
const response = await fetch('https://<host>/api/developers/search?q=emaar', {
  headers: {
    'Authorization': `Bearer ${access_token}`
  }
});
```

```python Python
import requests

headers = {
    'Authorization': f'Bearer {access_token}'
}

response = requests.get('https://<host>/api/developers/search?q=emaar', headers=headers)
```
</CodeGroup>

<Warning>
When the token expires, request a new one via `POST /api/auth/token` again. Tokens have a default lifetime of 30 minutes.
</Warning>
</Step>
</Steps>

## Access levels

Your API client has an assigned access level that controls which endpoints you can use:

| Level   | Description                              |
|---------|------------------------------------------|
| `basic` | Standard access to public-facing endpoints |
| `super` | Extended access to restricted endpoints    |

<Info>
If you call an endpoint that requires a higher access level, you will receive a `403 Forbidden` response.
</Info>

## Rate limiting

API requests are rate-limited per client. The default limit is **60 requests per window**. If you exceed it, you will receive a `429 Too Many Requests` response.

<Tip>
When you hit rate limits, back off and retry after the window resets to avoid continued throttling.
</Tip>

## Quick reference

| What                | Value                          |
|---------------------|--------------------------------|
| Token endpoint      | `POST /api/auth/token`         |
| Auth header format  | `Authorization: Bearer <token>`|
| Default token TTL   | 30 minutes                     |
| Default rate limit  | 60 req/window                  |

## Complete example

Here's a complete example showing the full authentication flow:

<CodeGroup>
```bash cURL
# 1. Get a token
curl -X POST https://<host>/api/auth/token \
  -H "Content-Type: application/json" \
  -d '{"apiKey":"your-api-key","apiSecret":"your-api-secret"}'

# 2. Use the token
curl https://<host>/api/developers/search?q=emaar \
  -H "Authorization: Bearer <access_token>"
```

```javascript JavaScript
// 1. Get a token
const authResponse = await fetch('https://<host>/api/auth/token', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    apiKey: 'your-api-key',
    apiSecret: 'your-api-secret'
  })
});

const { access_token } = await authResponse.json();

// 2. Use the token
const apiResponse = await fetch('https://<host>/api/developers/search?q=emaar', {
  headers: {
    'Authorization': `Bearer ${access_token}`
  }
});

const data = await apiResponse.json();
```

```python Python
import requests

# 1. Get a token
auth_response = requests.post('https://<host>/api/auth/token', 
  json={
    'apiKey': 'your-api-key',
    'apiSecret': 'your-api-secret'
  }
)

access_token = auth_response.json()['access_token']

# 2. Use the token
headers = {'Authorization': f'Bearer {access_token}'}
api_response = requests.get('https://<host>/api/developers/search?q=emaar', 
                          headers=headers)

data = api_response.json()
```
</CodeGroup>
