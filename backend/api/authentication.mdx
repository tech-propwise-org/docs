## Overview

All API endpoints require a Bearer token. You obtain a short-lived token by exchanging your **API Key** and **API Secret** via the token endpoint.

**Base URL:** `https://<host>/api`

<Warning>
Your API Secret is private and should be treated like a password. Do not expose it in client-side code.
</Warning>

## Authentication flow

<Steps>
<Step title="Get your credentials">
You will receive two values from the platform:

- **API Key** — a public identifier for your client (e.g., `pk_live_abc123...`)
- **API Secret** — a private secret (treat like a password, do not expose in client-side code)
</Step>

<Step title="Exchange credentials for a token">
Make a POST request to the token endpoint with your credentials:

```json
POST /api/auth/token
Content-Type: application/json

{
  "apiKey": "your-api-key",
  "apiSecret": "your-api-secret"
}
```

**Response `200`:**

```json
{
  "access_token": "eyJhbGciOi...",
  "token_type": "Bearer",
  "expires_in": 1800
}
```

| Field          | Description                              |
|----------------|------------------------------------------|
| `access_token` | JWT token to use in subsequent requests  |
| `token_type`   | Always `Bearer`                          |
| `expires_in`   | Token lifetime in seconds (default: 30m) |

**Common errors:**

| Status | Reason                          |
|--------|---------------------------------|
| `401`  | Invalid API key or secret       |
| `401`  | API client is disabled          |
</Step>

<Step title="Use the token">
Pass the token as a `Bearer` token in the `Authorization` header on every API request:

```http
GET /api/developers/search?q=emaar
Authorization: Bearer eyJhbGciOi...
```

<Note>
When the token expires, request a new one via `POST /api/auth/token` again.
</Note>
</Step>
</Steps>

## Access levels

Your API client has an assigned access level that controls which endpoints you can use:

<AccordionGroup>
<Accordion title="Basic access">
Standard access to public-facing endpoints. This is the default level for most API clients.
</Accordion>

<Accordion title="Super access">
Extended access to restricted endpoints with additional functionality and data access.
</Accordion>
</AccordionGroup>

<Info>
If you call an endpoint that requires a higher access level, you will receive a `403 Forbidden` response.
</Info>

## Rate limiting

API requests are rate-limited per client. The default limit is **60 requests per window**. 

<Warning>
If you exceed the rate limit, you will receive a `429 Too Many Requests` response. Back off and retry after the window resets.
</Warning>

## Code examples

<CodeGroup>
```bash cURL
# 1. Get a token
curl -X POST https://<host>/api/auth/token \
  -H "Content-Type: application/json" \
  -d '{"apiKey":"your-api-key","apiSecret":"your-api-secret"}'

# 2. Use the token
curl https://<host>/api/developers/search?q=emaar \
  -H "Authorization: Bearer <access_token>"
```

```javascript JavaScript
// 1. Get a token
const response = await fetch('https://<host>/api/auth/token', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    apiKey: 'your-api-key',
    apiSecret: 'your-api-secret'
  })
});

const { access_token } = await response.json();

// 2. Use the token
const apiResponse = await fetch('https://<host>/api/developers/search?q=emaar', {
  headers: {
    'Authorization': `Bearer ${access_token}`
  }
});
```

```python Python
import requests

# 1. Get a token
token_response = requests.post('https://<host>/api/auth/token', json={
    'apiKey': 'your-api-key',
    'apiSecret': 'your-api-secret'
})

access_token = token_response.json()['access_token']

# 2. Use the token
api_response = requests.get(
    'https://<host>/api/developers/search?q=emaar',
    headers={'Authorization': f'Bearer {access_token}'}
)
```
</CodeGroup>
