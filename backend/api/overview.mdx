# API Overview

This document provides a comprehensive overview of the PropWise CRM backend API, including authentication requirements, available endpoints, and entity operations.

<Note>
All endpoints require proper authentication using either API keys or Bearer tokens as specified in each section.
</Note>

## Authentication

The API uses two authentication methods depending on the endpoint:

- **API Key Authentication** (`ApiAuthGuard`): For developer search endpoints
- **Bearer Token Authentication**: For all entity CRUD operations

<CodeGroup>
```bash API Key
curl -H "X-API-Key: your-api-key" \
  "https://api.propwise.com/developers/search?q=emaar"
```

```bash Bearer Token
curl -H "Authorization: Bearer your-token" \
  "https://api.propwise.com/api/developers"
```
</CodeGroup>

## Developer Search

<Info>
The developer search uses PostgreSQL's `pg_trgm` extension for fuzzy text matching across both English and Arabic names.
</Info>

### Search Developers

`GET /developers/search`

**Authentication:** API Key

#### Query Parameters

| Parameter | Type   | Required | Default | Description |
|-----------|--------|----------|---------|-------------|
| `q`       | string | Yes      | —       | Search query (min length: 1) |
| `limit`   | int    | No       | `10`    | Results limit (1-50) |

<Steps>
<Step title="Process Search Query">
The system trims the query string and runs similarity matching against both `name_en` and `name_ar` columns.
</Step>

<Step title="Filter Results">
Only returns rows where either similarity score is greater than 0.15.
</Step>

<Step title="Sort and Limit">
Orders results by highest similarity score (descending) and applies the specified limit.
</Step>
</Steps>

#### Response

<CodeGroup>
```json Response Format
[
  {
    "id": 42,
    "nameEn": "Emaar Properties",
    "nameAr": "إعمار العقارية",
    "developerNumber": "DEV-001",
    "logo": "https://example.com/logo.png",
    "logoDark": "https://example.com/logo-dark.png",
    "licenseUrl": "https://example.com/license.pdf",
    "similarity": 0.85
  }
]
```
</CodeGroup>

<Warning>
All fields except `id` and `similarity` are nullable and may not be present in the response.
</Warning>

## Entity GET Endpoints

All entity endpoints require Bearer token authentication and return paginated responses.

### Pagination Envelope

<CodeGroup>
```json Standard Response
{
  "data": [...],
  "total": 1234,
  "page": 1,
  "limit": 20
}
```
</CodeGroup>

### Configurable Includes

<Tip>
Use the `include` parameter to optimize API responses by controlling which relations and computed fields are returned.
</Tip>

| Include Value | Behavior |
|---------------|----------|
| _(omitted)_   | Default relations (backward-compatible) |
| `none`        | No relations or stats - only scalar fields |
| `all`         | All allowed relations and stats |
| `field1,field2` | Only specified relations/stats |

<Warning>
Invalid include values return a `400` error with a list of allowed options.
</Warning>

## Developers

### List Developers

`GET /api/developers`

#### Query Parameters

<AccordionGroup>
<Accordion title="Pagination & Sorting">
| Parameter   | Type   | Default  | Description |
|-------------|--------|----------|-------------|
| `page`      | int    | `1`      | Page number (≥ 1) |
| `limit`     | int    | `20`     | Results per page (1-100) |
| `sortBy`    | string | `nameEn` | Sort field: `id`, `nameEn`, `nameAr`, `developerNumber`, `createdAt` |
| `sortOrder` | string | `asc`    | Sort direction: `asc`, `desc` |
</Accordion>

<Accordion title="Filtering">
| Parameter | Type   | Description |
|-----------|--------|-------------|
| `nameEn`  | string | Substring filter for English name |
| `nameAr`  | string | Substring filter for Arabic name |
| `search`  | string | Searches both `nameEn` and `nameAr` |
</Accordion>

<Accordion title="Includes">
| Parameter | Type   | Allowed Values |
|-----------|--------|----------------|
| `include` | string | `stats` |
</Accordion>
</AccordionGroup>

#### Response Fields

| Field             | Type   | Always Present |
|-------------------|--------|----------------|
| `id`              | number | Yes            |
| `nameEn`          | string | No             |
| `nameAr`          | string | No             |
| `developerNumber` | string | No             |
| `logo`            | string | No             |
| `logoDark`        | string | No             |

### Get Developer Details

`GET /api/developers/:id`

**Default includes:** `stats`

#### Additional Fields (Detail View)

| Field               | Type   | Description |
|---------------------|--------|-------------|
| `sourceDeveloperId` | string | External source identifier |
| `stats`             | object | `{ projectsCount, masterProjectsCount }` |

## Cities

### List Cities

`GET /api/cities`

#### Query Parameters

<Tabs>
<Tab title="Basic Parameters">
| Parameter   | Type   | Default  | Constraints |
|-------------|--------|----------|-------------|
| `page`      | int    | `1`      | ≥ 1 |
| `limit`     | int    | `20`     | 1-100 |
| `sortBy`    | string | `nameEn` | `id`, `nameEn`, `nameAr`, `state`, `createdAt` |
| `sortOrder` | string | `asc`    | `asc`, `desc` |
</Tab>

<Tab title="Filters">
| Parameter | Type   | Description |
|-----------|--------|-------------|
| `nameEn`  | string | Substring filter |
| `nameAr`  | string | Substring filter |
| `search`  | string | Searches `nameEn`, `nameAr`, `state.nameEn` |
| `stateId` | int    | Filter by state |
</Tab>

<Tab title="Includes">
| Parameter | Type   | Allowed Values |
|-----------|--------|----------------|
| `include` | string | `state`, `stats` |
</Tab>
</Tabs>

**Default includes:** `state`

#### Response Fields

| Field    | Type   | Description |
|----------|--------|-------------|
| `id`     | number | City identifier |
| `nameEn` | string | English name |
| `nameAr` | string | Arabic name |
| `state`  | object | `{ id, nameEn, code }` if included |

### Get City Details

`GET /api/cities/:id`

**Default includes:** `state`, `stats`

#### Stats Object

<CodeGroup>
```json City Stats
{
  "areasCount": 15,
  "projectsCount": 128,
  "buildingsCount": 456,
  "unitsCount": 2340,
  "transactionsCount": 1890,
  "rentContractsCount": 567
}
```
</CodeGroup>

## Areas

### List Areas

`GET /api/areas`

#### Query Parameters

| Parameter           | Type   | Default  | Description |
|---------------------|--------|----------|-------------|
| `page`              | int    | `1`      | Page number |
| `limit`             | int    | `20`     | Results per page |
| `sortBy`            | string | `nameEn` | Sort field |
| `sortOrder`         | string | `asc`    | Sort direction |
| `nameEn`            | string | —        | Name filter |
| `nameAr`            | string | —        | Arabic name filter |
| `search`            | string | —        | Search across names and city |
| `cityId`            | int    | —        | Filter by city |
| `include`           | string | —        | `city`, `city.state`, `stats` |

**Default includes:** `city`

#### Response Fields

| Field                | Type   | Description |
|----------------------|--------|-------------|
| `id`                 | number | Area identifier |
| `nameEn`             | string | English name |
| `nameAr`             | string | Arabic name |
| `municipalityNumber` | string | Municipality reference |
| `imageUrl`           | string | Area image URL |
| `city`               | object | `{ id, nameEn }` if included |

### Get Area Details

`GET /api/areas/:id`

**Default includes:** `city`, `stats`

<Check>
Area details include comprehensive statistics for projects, buildings, units, transactions, and rent contracts within the area.
</Check>
