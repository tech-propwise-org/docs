# Lookup Endpoints

Reference data endpoints that provide lookup tables and dropdown options for various entity relationships. These endpoints are optimized for form controls, autocomplete functionality, and maintaining referential integrity across the system.

## Authentication

All lookup endpoints require an API key passed via the `ApiAuthGuard`:

```
X-API-Key: your_api_key_here
```

## Response Format

All lookup endpoints return arrays of objects with consistent structure:

```json
[
  {
    "id": 1,
    "nameEn": "English Name",
    "nameAr": "الاسم العربي",
    "code": "CODE_VALUE"
  }
]
```

<Note>
Some lookup tables may include additional fields specific to their domain. The `id`, `nameEn`, and `nameAr` fields are standard across all endpoints.
</Note>

## Best Practices

<Tips>
- Cache lookup data on the client side as it changes infrequently
- Use hierarchical filtering (state → city → area) for better user experience
- Always validate lookup IDs exist before submitting to main entity endpoints
</Tips>

### Caching Strategy

<Steps>
  <Step title="Load on Application Start">
    Fetch all lookup tables when the application initializes to minimize API calls.
  </Step>
  
  <Step title="Implement Local Storage">
    Store lookup data in browser localStorage with timestamps for cache invalidation.
  </Step>
  
  <Step title="Refresh Periodically">
    Set up a background refresh mechanism to update lookup data every 24 hours.
  </Step>
</Steps>

<Warning>
Always validate that referenced lookup IDs exist in your local cache before submitting forms to prevent referential integrity errors.
</Warning>

### Error Handling

All lookup endpoints return `200 OK` with an empty array if no results are found. Standard error responses:

| Status Code | Description |
|------------|-------------|
| `401` | Invalid or missing API key |
| `400` | Invalid query parameters |
| `500` | Server error |

<AccordionGroup>
  <Accordion title="Example Error Response">
    ```json
    {
      "error": "Unauthorized",
      "message": "Invalid API key provided",
      "statusCode": 401
    }
    ```
  </Accordion>
  
  <Accordion title="Handling Missing Data">
    ```javascript
    const handleLookupError = (response) => {
      if (!response.ok) {
        console.error('Lookup failed:', response.status);
        return [];
      }
      return response.json();
    };
    ```
  </Accordion>
</AccordionGroup>
