The Lookup Endpoints provide access to reference data used throughout the system, including states, legal statuses, license types, and other standardized lookup values. These endpoints support the core entities and ensure data consistency across the platform.

<Note>
All lookup endpoints require Bearer token authentication (`Authorization: Bearer <token>`).
</Note>

## States

States represent the top-level geographic divisions in the system.

### `GET /api/states`

Returns a paginated list of all states.

#### Query Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `page` | integer | no | `1` | Page number (≥ 1) |
| `limit` | integer | no | `20` | Items per page (1-100) |
| `sortBy` | string | no | `nameEn` | Sort field: `id`, `nameEn`, `nameAr`, `code`, `createdAt` |
| `sortOrder` | string | no | `asc` | Sort direction: `asc`, `desc` |
| `nameEn` | string | no | — | Filter by English name (substring) |
| `nameAr` | string | no | — | Filter by Arabic name (substring) |
| `search` | string | no | — | Search across `nameEn`, `nameAr` |
| `include` | string | no | — | Allowed: `stats` |

#### Response

```json
{
  "data": [
    {
      "id": 1,
      "nameEn": "Dubai",
      "nameAr": "دبي",
      "code": "DXB"
    }
  ],
  "total": 7,
  "page": 1,
  "limit": 20
}
```

### `GET /api/states/:id`

Returns detailed information for a specific state.

#### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `id` | integer | yes | State ID |

#### Query Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `include` | string | no | Allowed: `stats` |

**Default includes:** `stats`

#### Response

```json
{
  "id": 1,
  "nameEn": "Dubai",
  "nameAr": "دبي", 
  "code": "DXB",
  "stats": {
    "citiesCount": 15,
    "areasCount": 120,
    "projectsCount": 450,
    "transactionsCount": 12500
  }
}
```

## Response Patterns

<Info>
All lookup endpoints follow consistent response patterns with paginated envelopes for list endpoints and detailed objects for single-item endpoints.
</Info>

### Pagination Envelope

All list endpoints return data in this format:

```json
{
  "data": [...],
  "total": 1234,
  "page": 1,
  "limit": 20
}
```

### Include Parameter

<Tip>
Use the `include` parameter to control which related data is returned. This helps optimize API performance by only fetching needed data.
</Tip>

Available include options:
- **Default**: Backward-compatible relations
- **`none`**: Only scalar fields, no relations
- **`all`**: All available relations and statistics
- **`field1,field2`**: Specific relations only

### Error Responses

Invalid `include` values return a `400` status with available options:

```json
{
  "error": "Invalid include parameter",
  "allowedIncludes": ["stats", "cities", "areas"]
}
```
