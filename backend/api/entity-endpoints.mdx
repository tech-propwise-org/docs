<Info>
All entity endpoints require Bearer token authentication (`Authorization: Bearer <token>`) and return paginated results in a standardized envelope format.
</Info>

## Response Format

All list endpoints return data in a paginated envelope:

```json
{
  "data": [...],
  "total": 1234,
  "page": 1,
  "limit": 20
}
```

## Configurable Includes

Every entity supports an optional `include` query parameter to control which relations and computed fields are returned:

<CardGroup cols={2}>
  <Card title="Default Behavior" icon="gear">
    Omit parameter for backward-compatible default relations
  </Card>
  <Card title="Minimal Response" icon="minimize">
    Use `none` for scalar fields only
  </Card>
  <Card title="Complete Data" icon="expand">
    Use `all` for all allowed relations and stats
  </Card>
  <Card title="Custom Selection" icon="list-check">
    Use `field1,field2` for specific relations only
  </Card>
</CardGroup>

<Warning>
Invalid include values return `400` with the list of allowed options. Virtual includes like `stats` and `buildingAreas` control expensive aggregate queries.
</Warning>

## Developers

### List Developers

<CodeGroup>
```http GET /api/developers
GET /api/developers?page=1&limit=20&sortBy=nameEn&sortOrder=asc
Authorization: Bearer <token>
```

```json Response
{
  "data": [
    {
      "id": 42,
      "nameEn": "Emaar Properties",
      "nameAr": "إعمار العقارية", 
      "developerNumber": "DEV-001",
      "logo": "https://example.com/logo.png",
      "logoDark": "https://example.com/logo-dark.png"
    }
  ],
  "total": 150,
  "page": 1,
  "limit": 20
}
```
</CodeGroup>

#### Query Parameters

| Parameter | Type | Default | Constraints | Description |
|-----------|------|---------|-------------|-------------|
| `page` | int | `1` | ≥ 1 | Page number |
| `limit` | int | `20` | 1–100 | Items per page |
| `sortBy` | string | `nameEn` | `id`, `nameEn`, `nameAr`, `developerNumber`, `createdAt` | Sort field |
| `sortOrder` | string | `asc` | `asc`, `desc` | Sort direction |
| `nameEn` | string | — | — | Filter by English name (substring) |
| `nameAr` | string | — | — | Filter by Arabic name (substring) |
| `search` | string | — | — | Search across `nameEn` and `nameAr` |
| `include` | string | — | `stats` | Include additional data |

<Note>
**Default includes (list):** None
</Note>

### Get Developer Details

<CodeGroup>
```http GET /api/developers/:id
GET /api/developers/42?include=stats
Authorization: Bearer <token>
```

```json Response
{
  "id": 42,
  "nameEn": "Emaar Properties",
  "nameAr": "إعمار العقارية",
  "developerNumber": "DEV-001",
  "sourceDeveloperId": "ext-dev-001",
  "logo": "https://example.com/logo.png",
  "logoDark": "https://example.com/logo-dark.png",
  "stats": {
    "projectsCount": 25,
    "masterProjectsCount": 8
  }
}
```
</CodeGroup>

<Note>
**Default includes (detail):** `stats`  
**Allowed includes:** `stats`
</Note>

## Cities

### List Cities

<CodeGroup>
```http GET /api/cities
GET /api/cities?stateId=1&include=state,stats
Authorization: Bearer <token>
```

```json Response
{
  "data": [
    {
      "id": 1,
      "nameEn": "Dubai",
      "nameAr": "دبي",
      "state": {
        "id": 1,
        "nameEn": "Dubai",
        "code": "DU"
      }
    }
  ],
  "total": 7,
  "page": 1,
  "limit": 20
}
```
</CodeGroup>

#### Query Parameters

| Parameter | Type | Default | Constraints | Description |
|-----------|------|---------|-------------|-------------|
| `page` | int | `1` | ≥ 1 | Page number |
| `limit` | int | `20` | 1–100 | Items per page |
| `sortBy` | string | `nameEn` | `id`, `nameEn`, `nameAr`, `state`, `createdAt` | Sort field |
| `sortOrder` | string | `asc` | `asc`, `desc` | Sort direction |
| `nameEn` | string | — | — | Filter by English name |
| `nameAr` | string | — | — | Filter by Arabic name |
| `search` | string | — | — | Search across `nameEn`, `nameAr`, `state.nameEn` |
| `stateId` | int | — | — | Filter by state ID |
| `include` | string | — | `state`, `stats` | Include additional data |

<Note>
**Default includes (list):** `state`
</Note>

| Parameter | Type | Default | Constraints | Description |
|-----------|------|---------|-------------|-------------|

<Tip>

</Tip>

## Common Patterns

<Steps>
  <Step title="Authentication">
    Include Bearer token in Authorization header for all requests
  </Step>
  <Step title="Pagination">
    Use `page` and `limit` parameters to control result sets
  </Step>
  <Step title="Filtering">
    Apply entity-specific filters like `cityId`, `stateId`, or text search
  </Step>
  <Step title="Includes">
    Use `include` parameter to optimize response data and performance
  </Step>
</Steps>

<Warning>
Always validate include parameters - invalid values return `400` status with allowed options listed.
</Warning>
