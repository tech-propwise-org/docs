## Overview

All entity GET endpoints require Bearer token authentication (`Authorization: Bearer <token>`) and return paginated data with consistent filtering, sorting, and inclusion options.

<Note>
All list endpoints return data in a paginated envelope format:
```json
{ "data": [...], "total": 1234, "page": 1, "limit": 20 }
```
</Note>

## Configurable Includes

Every entity supports an optional `include` query parameter that controls which relations and computed fields are returned:

| Value | Behavior |
|-------|----------|
| _(omitted)_ | Default relations populated (backward-compatible) |
| `none` | No relations or stats — only scalar fields |
| `all` | All allowed relations and stats |
| `field1,field2` | Only the specified relations/stats |

<Warning>
Invalid include values return `400` status with a list of allowed options for that endpoint.
</Warning>

## Developers

### List Developers

<CodeGroup>
```http GET /api/developers
GET /api/developers?page=1&limit=20&sortBy=nameEn&sortOrder=asc
Authorization: Bearer <token>
```
</CodeGroup>

#### Query Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `page` | integer | No | `1` | Page number (≥ 1) |
| `limit` | integer | No | `20` | Items per page (1-100) |
| `sortBy` | string | No | `nameEn` | Sort field: `id`, `nameEn`, `nameAr`, `developerNumber`, `createdAt` |
| `sortOrder` | string | No | `asc` | Sort order: `asc`, `desc` |
| `nameEn` | string | No | — | Filter by English name (substring) |
| `nameAr` | string | No | — | Filter by Arabic name (substring) |
| `search` | string | No | — | Search across `nameEn` and `nameAr` |
| `include` | string | No | — | Allowed: `stats` |

**Default includes (list):** _(none)_

#### Response

<CodeGroup>
```json Response
{
  "data": [
    {
      "id": 42,
      "nameEn": "Emaar Properties",
      "nameAr": "إعمار العقارية",
      "developerNumber": "DEV-001",
      "logo": "https://example.com/logo.png",
      "logoDark": "https://example.com/logo-dark.png"
    }
  ],
  "total": 1,
  "page": 1,
  "limit": 20
}
```
</CodeGroup>

### Get Developer by ID

<CodeGroup>
```http GET /api/developers/:id
GET /api/developers/42?include=stats
Authorization: Bearer <token>
```
</CodeGroup>

**Default includes (detail):** `stats`  
**Allowed includes:** `stats`

#### Response

<CodeGroup>

```json Response
{
  "id": 42,
  "nameEn": "Emaar Properties",
  "nameAr": "إعمار العقارية", 
  "developerNumber": "DEV-001",
  "sourceDeveloperId": "emaar-001",

  "logoDark": "https://example.com/logo-dark.png",
  "stats": {
    "projectsCount": 25,
    "masterProjectsCount": 8
  }
}
```
</CodeGroup>

## Cities

### List Cities

<CodeGroup>
```http GET /api/cities
GET /api/cities?include=state,stats&stateId=1
Authorization: Bearer <token>
```
</CodeGroup>

#### Query Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `page` | integer | No | `1` | Page number (≥ 1) |
| `limit` | integer | No | `20` | Items per page (1-100) |
| `sortBy` | string | No | `nameEn` | Sort field: `id`, `nameEn`, `nameAr`, `state`, `createdAt` |
| `sortOrder` | string | No | `asc` | Sort order: `asc`, `desc` |
| `nameEn` | string | No | — | Filter by English name (substring) |
| `nameAr` | string | No | — | Filter by Arabic name (substring) |
| `search` | string | No | — | Search across `nameEn`, `nameAr`, `state.nameEn` |
| `stateId` | integer | No | — | Filter by state ID |
| `include` | string | No | — | Allowed: `state`, `stats` |

**Default includes (list):** `state`

#### Response

<CodeGroup>
```json Response
{
  "data": [
    {
      "id": 1,
      "nameEn": "Dubai",
      "nameAr": "دبي",
      "state": {
        "id": 1,
        "nameEn": "Dubai",
        "code": "DU"
      }
    }
  ],
  "total": 1,
  "page": 1,
  "limit": 20
}
```
</CodeGroup>

### Get City by ID

**Default includes (detail):** `state`, `stats`

<Info>
The stats object includes: `areasCount`, `projectsCount`, `buildingsCount`, `unitsCount`, `transactionsCount`, `rentContractsCount`
</Info>

## Areas

### List Areas

<CodeGroup>
```http GET /api/areas
GET /api/areas?cityId=1&include=city.state
Authorization: Bearer <token>
```
</CodeGroup>

#### Query Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `page` | integer | No | `1` | Page number (≥ 1) |
| `limit` | integer | No | `20` | Items per page (1-100) |
| `sortBy` | string | No | `nameEn` | Sort field: `id`, `nameEn`, `nameAr`, `city`, `municipalityNumber`, `createdAt` |
| `sortOrder` | string | No | `asc` | Sort order: `asc`, `desc` |
| `nameEn` | string | No | — | Filter by English name (substring) |
| `nameAr` | string | No | — | Filter by Arabic name (substring) |
| `search` | string | No | — | Search across `nameEn`, `nameAr`, `city.nameEn` |
| `cityId` | integer | No | — | Filter by city ID |
| `include` | string | No | — | Allowed: `city`, `city.state`, `stats` |

**Default includes (list):** `city`

#### Response

<CodeGroup>
```json Response
{
  "data": [
    {
      "id": 1,
      "nameEn": "Downtown Dubai",
      "nameAr": "وسط مدينة دبي",
      "municipalityNumber": "MUN-001",
      "imageUrl": "https://example.com/area.jpg",
      "city": {
        "id": 1,
        "nameEn": "Dubai"
      }
    }
  ],
  "total": 1,
  "page": 1,
  "limit": 20
}
```
</CodeGroup>

### Get Area by ID

**Default includes (detail):** `city`, `stats`  
**Allowed includes:** `city`, `city.state`, `stats`

<Tip>
Use nested includes like `city.state` to get related state information along with city data.
</Tip>
