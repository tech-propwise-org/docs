---
title: "Entity GET Endpoints"
description: "Paginated endpoints for retrieving developers, cities, and other entities with configurable includes"
---

All entity endpoints require Bearer token authentication (`Authorization: Bearer <token>`) and return paginated results in a standard envelope format.

<Note>
All list endpoints return data in the following paginated structure:
```json
{ "data": [...], "total": 1234, "page": 1, "limit": 20 }
```
</Note>

## Configurable Includes

Every entity supports an optional `include` query parameter that controls which relations and computed fields are returned:

<AccordionGroup>
  <Accordion title="Include Parameter Options">
    | Value           | Behavior                                                          |
    | --------------- | ----------------------------------------------------------------- |
    | _(omitted)_     | Default relations populated (backward-compatible, same as before) |
    | `none`          | No relations or stats — only scalar fields                        |
    | `all`           | All allowed relations and stats                                   |
    | `field1,field2` | Only the specified relations/stats                                |
  </Accordion>
</AccordionGroup>

<Warning>
Invalid include values return `400` with the list of allowed options. Virtual includes like `stats` and `buildingAreas` control expensive aggregate queries rather than ORM relations.
</Warning>

## Developers

### List developers

<CodeGroup>
```http GET /api/developers
GET /api/developers?page=1&limit=20&sortBy=nameEn&sortOrder=asc
Authorization: Bearer <token>
```
</CodeGroup>

#### Query Parameters

| Param       | Type   | Required | Default  | Constraints                                              |
| ----------- | ------ | -------- | -------- | -------------------------------------------------------- |
| `page`      | int    | no       | `1`      | ≥ 1                                                      |
| `limit`     | int    | no       | `20`     | 1–100                                                    |
| `sortBy`    | string | no       | `nameEn` | `id`, `nameEn`, `nameAr`, `developerNumber`, `createdAt` |
| `sortOrder` | string | no       | `asc`    | `asc`, `desc`                                            |
| `nameEn`    | string | no       | —        | substring filter                                         |
| `nameAr`    | string | no       | —        | substring filter                                         |
| `search`    | string | no       | —        | searches `nameEn`, `nameAr`                              |
| `include`   | string | no       | —        | allowed: `stats`                                         |

<Info>
**Default includes (list):** _(none)_
</Info>

#### Response

```json
{
  "data": [
    {
      "id": 42,
      "nameEn": "Emaar Properties",
      "nameAr": "إعمار العقارية",
      "developerNumber": "DEV-001",
      "logo": "https://example.com/logo.png",
      "logoDark": "https://example.com/logo-dark.png"
    }
  ],
  "total": 1234,
  "page": 1,
  "limit": 20
}
```

### Get developer by ID

<CodeGroup>
```http GET /api/developers/:id
GET /api/developers/42?include=stats
Authorization: Bearer <token>
```
</CodeGroup>

#### Path Parameters

| Param | Type | Required | Description |
| ----- | ---- | -------- | ----------- |
| `id`  | int  | yes      | Developer ID |

#### Query Parameters

| Param     | Type   | Required | Allowed values |
| --------- | ------ | -------- | -------------- |
| `include` | string | no       | `stats`        |

<Info>
**Default includes (detail):** `stats`
</Info>

#### Response

```json
{
  "id": 42,
  "nameEn": "Emaar Properties",
  "nameAr": "إعمار العقارية",
  "developerNumber": "DEV-001",
  "logo": "https://example.com/logo.png",
  "logoDark": "https://example.com/logo-dark.png",
  "sourceDeveloperId": "EXT-DEV-001",
  "stats": {
    "projectsCount": 15,
    "masterProjectsCount": 3
  }
}
```

## Cities

### List cities

<CodeGroup>
```http GET /api/cities
GET /api/cities?page=1&limit=20&include=state,stats
Authorization: Bearer <token>
```
</CodeGroup>

#### Query Parameters

| Param       | Type   | Required | Default  | Constraints                                    |
| ----------- | ------ | -------- | -------- | ---------------------------------------------- |
| `page`      | int    | no       | `1`      | ≥ 1                                            |
| `limit`     | int    | no       | `20`     | 1–100                                          |
| `sortBy`    | string | no       | `nameEn` | `id`, `nameEn`, `nameAr`, `state`, `createdAt` |
| `sortOrder` | string | no       | `asc`    | `asc`, `desc`                                  |
| `nameEn`    | string | no       | —        | substring filter                               |
| `nameAr`    | string | no       | —        | substring filter                               |
| `search`    | string | no       | —        | searches `nameEn`, `nameAr`, `state.nameEn`    |
| `stateId`   | int    | no       | —        | filter by state                                |
| `include`   | string | no       | —        | allowed: `state`, `stats`                      |

<Info>
**Default includes (list):** `state`
</Info>

#### Response

```json
{
  "data": [
    {
      "id": 1,
      "nameEn": "Dubai",
      "nameAr": "دبي",
      "state": {
        "id": 1,
        "nameEn": "Dubai",
        "code": "DU"
      }
    }
  ],
  "total": 50,
  "page": 1,
  "limit": 20
}
```

### Get city by ID

<CodeGroup>
```http GET /api/cities/:id
GET /api/cities/1?include=state,stats
Authorization: Bearer <token>
```
</CodeGroup>

<Info>

</Info>

#### Response

```json
{
  "id": 1,
  "nameEn": "Dubai",
  "nameAr": "دبي",
  "state": {
    "id": 1,
    "nameEn": "Dubai",
    "code": "DU"
  },
  "stats": {
    "areasCount": 25,
    "projectsCount": 150,
    "buildingsCount": 500,
    "unitsCount": 2500,
    "transactionsCount": 1200,
    "rentContractsCount": 800
  }
}
```

## Areas

### List areas

<CodeGroup>
```http GET /api/areas
GET /api/areas?cityId=1&include=city,stats
Authorization: Bearer <token>
```
</CodeGroup>

#### Query Parameters

| Param       | Type   | Required | Default  | Constraints                                                         |
| ----------- | ------ | -------- | -------- | ------------------------------------------------------------------- |
| `page`      | int    | no       | `1`      | ≥ 1                                                                 |
| `limit`     | int    | no       | `20`     | 1–100                                                               |
| `sortBy`    | string | no       | `nameEn` | `id`, `nameEn`, `nameAr`, `city`, `municipalityNumber`, `createdAt` |
| `sortOrder` | string | no       | `asc`    | `asc`, `desc`                                                       |
| `nameEn`    | string | no       | —        | substring filter                                                    |
| `nameAr`    | string | no       | —        | substring filter                                                    |
| `search`    | string | no       | —        | searches `nameEn`, `nameAr`, `city.nameEn`                          |
| `cityId`    | int    | no       | —        | filter by city                                                      |
| `include`   | string | no       | —        | allowed: `city`, `city.state`, `stats`                              |

<Info>
**Default includes (list):** `city`
</Info>

#### Response

```json
{
  "data": [
    {
      "id": 1,
      "nameEn": "Downtown Dubai",
      "nameAr": "وسط مدينة دبي",
      "municipalityNumber": "MUN-001",
      "imageUrl": "https://example.com/area-image.jpg",
      "city": {
        "id": 1,
        "nameEn": "Dubai"
      }
    }
  ],
  "total": 100,
  "page": 1,
  "limit": 20
}
```

```json
    "projectsCount": 15,
    "buildingsCount": 50,
    "unitsCount": 250,
    "transactionsCount": 120,
    "rentContractsCount": 80
  }
}
```

## Communities

### List communities

<CodeGroup>
```http GET /api/communities
GET /api/communities?areaId=1&include=area,stats
Authorization: Bearer <token>
```
</CodeGroup>

#### Query Parameters

| Param       | Type   | Required | Default  | Constraints                                   |
| ----------- | ------ | -------- | -------- | --------------------------------------------- |
| `page`      | int    | no       | `1`      | ≥ 1                                           |
| `limit`     | int    | no       | `20`     | 1–100                                         |
| `sortBy`    | string | no       | `nameEn` | `id`, `nameEn`, `nameAr`, `area`, `createdAt` |
| `sortOrder` | string | no       | `asc`    | `asc`, `desc`                                 |
| `nameEn`    | string | no       | —        | substring filter                              |
| `nameAr`    | string | no       | —        | substring filter                              |
| `search`    | string | no       | —        | searches `nameEn`, `nameAr`, `area.nameEn`    |
| `areaId`    | int    | no       | —        | filter by area                                |
| `include`   | string | no       | —        | allowed: `area`, `area.city`, `stats`         |

<Tip>
Communities represent sub-areas within larger areas, providing more granular location data for properties and projects.
</Tip>
