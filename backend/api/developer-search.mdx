# Developer Search API

The Developer Search API provides fuzzy search capabilities for finding developers by name using PostgreSQL's `pg_trgm` similarity extension. This endpoint is designed for fast, intelligent name matching across both English and Arabic developer names.

## Search Endpoint

### `GET /developers/search`

Search for developers using fuzzy name matching with PostgreSQL trigram similarity.

<Note>
This endpoint requires API key authentication using the `ApiAuthGuard`.
</Note>

#### Query Parameters

| Parameter | Type   | Required | Default | Description                    |
|-----------|--------|----------|---------|--------------------------------|
| `q`       | string | Yes      | —       | Search query (min length: 1)  |
| `limit`   | int    | No       | `10`    | Results limit (range: 1-50)   |

#### How the Search Algorithm Works

<Steps>
<Step title="Query preprocessing">
The search query `q` parameter is trimmed of whitespace.
</Step>

<Step title="Similarity calculation">
PostgreSQL's `similarity()` function is run against both `name_en` and `name_ar` columns using trigram matching.
</Step>

<Step title="Filtering">
Only rows where either English or Arabic name similarity score is greater than 0.15 are included.
</Step>

<Step title="Ranking">
Results are ordered by the highest similarity score (either English or Arabic) in descending order.
</Step>

<Step title="Limiting">
Returns up to the specified `limit` number of results.
</Step>
</Steps>

<Warning>
This endpoint requires the `pg_trgm` PostgreSQL extension to be installed and enabled.
</Warning>

#### Response Format

**Success Response: `200 OK`**

```json
[
  {
    "id": 42,
    "nameEn": "Emaar Properties",
    "nameAr": "إعمار العقارية",
    "developerNumber": "DEV-001",
    "logo": "https://example.com/logo.png",
    "logoDark": "https://example.com/logo-dark.png",
    "licenseUrl": "https://example.com/license.pdf",
    "similarity": 0.85
  }
]
```

<Info>
All fields except `id` and `similarity` are nullable. The `similarity` field indicates the matching confidence score.
</Info>

## Usage Examples

<CodeGroup>
```bash cURL
curl -X GET "https://api.example.com/developers/search?q=emaar&limit=5" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript
const searchDevelopers = async (query, limit = 10) => {
  const response = await fetch(
    `/developers/search?q=${encodeURIComponent(query)}&limit=${limit}`,
    {
      headers: {
        'Authorization': 'Bearer YOUR_API_KEY'
      }
    }
  );
  return response.json();
};

// Search for developers matching "emaar"
const results = await searchDevelopers('emaar', 5);
```

```python Python
import requests

def search_developers(query, limit=10):
    url = f"/developers/search"
    params = {"q": query, "limit": limit}
    headers = {"Authorization": "Bearer YOUR_API_KEY"}
    
    response = requests.get(url, params=params, headers=headers)
    return response.json()

# Search for developers
results = search_developers("emaar", 5)
```
</CodeGroup>

## Performance Considerations

<Tip>
For optimal performance:
- Keep search queries focused (shorter queries generally perform better)
- Use appropriate limits to balance between completeness and speed
- The trigram similarity threshold of 0.15 provides a good balance between recall and precision
</Tip>

The search endpoint is optimized for fast response times through:
- Indexed `name_en` and `name_ar` columns
- Efficient trigram similarity calculations
- Limited result sets with configurable bounds
