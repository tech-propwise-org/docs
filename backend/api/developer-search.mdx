# Developer Search

The Developer Search API provides fuzzy search capabilities for developers by name using PostgreSQL's `pg_trgm` similarity algorithm. This allows for intelligent matching even when search terms contain typos or partial matches.

## Authentication

<Note>
All developer search requests require API key authentication using `ApiAuthGuard`.
</Note>

## Search Endpoint

### `GET /developers/search`

Performs fuzzy search across developer names in both English and Arabic.

#### Query Parameters

| Parameter | Type   | Required | Default | Constraints    | Description |
|-----------|--------|----------|---------|----------------|-------------|
| `q`       | string | Yes      | —       | min length 1   | Search query |
| `limit`   | int    | No       | `10`    | 1–50          | Maximum results to return |

#### How the Search Works

<Steps>
<Step title="Query Processing">
The search query `q` is trimmed and then processed using PostgreSQL's `similarity()` function against both `name_en` and `name_ar` columns.
</Step>

<Step title="Similarity Filtering">
Results are filtered to include only rows where either the English or Arabic name has a similarity score greater than 0.15.
</Step>

<Step title="Result Ordering">
Results are ordered by the highest similarity score between the two language columns in descending order.
</Step>

<Step title="Limit Application">
Up to `limit` results are returned (default 10, maximum 50).
</Step>
</Steps>

<Info>
This endpoint requires the `pg_trgm` PostgreSQL extension to be installed and enabled.
</Info>

#### Response Format

**Success Response: `200 OK`**

```json
[
  {
    "id": 42,
    "nameEn": "Emaar Properties",
    "nameAr": "إعمار العقارية",
    "developerNumber": "DEV-001",
    "logo": "https://example.com/logo.png",
    "logoDark": "https://example.com/logo-dark.png",
    "licenseUrl": "https://example.com/license.pdf",
    "similarity": 0.85
  }
]
```

<Note>
All fields except `id` and `similarity` are nullable and may not be present in the response.
</Note>

#### Response Fields

| Field             | Type   | Description |
|-------------------|--------|-------------|
| `id`              | number | Unique developer identifier |
| `nameEn`          | string | Developer name in English (nullable) |
| `nameAr`          | string | Developer name in Arabic (nullable) |
| `developerNumber` | string | Developer's registration number (nullable) |
| `logo`            | string | URL to developer's logo image (nullable) |
| `logoDark`        | string | URL to developer's dark theme logo (nullable) |
| `licenseUrl`      | string | URL to developer's license document (nullable) |
| `similarity`      | number | Similarity score (0.0 to 1.0) |

### Entity Relationships

The developer entity has the following relationships:

- **Projects**: One-to-many relationship via `developer` foreign key
- **Master Projects**: One-to-many relationship via `masterDeveloper` foreign key

## Example Usage

<CodeGroup>

```bash cURL
curl -X GET "https://api.example.com/developers/search?q=emaar&limit=5" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript
const response = await fetch('/developers/search?q=emaar&limit=5', {
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY'
  }
});
```

```python Python

response = requests.get(

    headers={'Authorization': 'Bearer YOUR_API_KEY'}

developers = response.json()
```

</CodeGroup>

## Search Tips

<Tip>
The fuzzy search algorithm is tolerant of:
- Typos and misspellings
- Partial matches
- Different word orders
- Mixed language queries (English and Arabic)
</Tip>

<Warning>
Search queries must have a minimum length of 1 character. Empty or whitespace-only queries will result in a validation error.
</Warning>

## Performance Considerations

- The search uses PostgreSQL's trigram similarity algorithm, which is optimized for fuzzy text matching
- Both `name_en` and `name_ar` columns are indexed for optimal performance
- The similarity threshold of 0.15 balances result relevance with search recall
- Results are limited to prevent excessive resource usage
