This API provides comprehensive access to real estate data including developers, properties, transactions, and administrative functions. All endpoints require proper authentication and follow RESTful conventions.

<Note>
All API endpoints use Bearer token authentication unless otherwise specified. Include the token in the Authorization header: `Authorization: Bearer <token>`
</Note>

## Developer Search

Fuzzy search for developers by name using PostgreSQL `pg_trgm` similarity for intelligent matching across English and Arabic names.

### Search Developers

<CodeGroup>
```http GET /developers/search
GET /developers/search?q=emaar&limit=10
Authorization: x-api-key YOUR_API_KEY
```

```javascript
const response = await fetch('/developers/search?q=emaar&limit=10', {
  headers: {
    'x-api-key': 'YOUR_API_KEY'
  }
});
const developers = await response.json();
```
</CodeGroup>

**Authentication:** API key (`x-api-key` header)

#### Query Parameters

| Parameter | Type    | Required | Default | Constraints    | Description |
|-----------|---------|----------|---------|----------------|-------------|
| `q`       | string  | Yes      | —       | min length 1   | Search query |
| `limit`   | integer | No       | `10`    | 1–50           | Results limit |

#### Search Algorithm

<Steps>
<Step title="Query Processing">
Trims the search query and runs similarity matching against both `name_en` and `name_ar` columns
</Step>
<Step title="Similarity Filtering">
Filters rows where either similarity score > 0.15 using PostgreSQL's trigram similarity
</Step>
<Step title="Result Ordering">
Orders by the highest similarity score (descending) from either language
</Step>
<Step title="Limit Application">
Returns up to the specified limit of results
</Step>
</Steps>

<Info>
Requires the `pg_trgm` PostgreSQL extension for trigram similarity matching
</Info>

#### Response

```json 200 OK
[
  {
    "id": 42,
    "nameEn": "Emaar Properties",
    "nameAr": "إعمار العقارية",
    "developerNumber": "DEV-001",
    "logo": "https://example.com/logo.png",
    "logoDark": "https://example.com/logo-dark.png",
    "licenseUrl": "https://example.com/license.pdf",
    "similarity": 0.85
  }
]
```

<Note>
All fields except `id` and `similarity` are nullable. The `similarity` score ranges from 0 to 1.
</Note>

## Entity GET Endpoints

All list endpoints return paginated results with the following envelope structure:

```json
{
  "data": [...],
  "total": 1234,
  "page": 1,
  "limit": 20
}
```

### Configurable Includes

Every entity supports an optional `include` query parameter for controlling which relations and computed fields are returned:

<Tabs>
<Tab title="Include Options">
| Value | Behavior |
|-------|----------|
| _(omitted)_ | Default relations populated (backward-compatible) |
| `none` | No relations or stats — only scalar fields |
| `all` | All allowed relations and stats |
| `field1,field2` | Only the specified relations/stats |
</Tab>
<Tab title="Virtual Includes">
- `stats` - Controls expensive aggregate queries
- `buildingAreas` - Building area calculations
- Relations like `city.state` for nested includes
</Tab>
</Tabs>

<Warning>
Invalid include values return `400` with a list of allowed options for that endpoint
</Warning>

## Developers

### List Developers

<CodeGroup>
```http GET /api/developers
GET /api/developers?page=1&limit=20&sortBy=nameEn&sortOrder=asc
Authorization: Bearer YOUR_TOKEN
```

```javascript
const response = await fetch('/api/developers?include=stats', {
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN'
  }
});
const { data, total, page, limit } = await response.json();
```
</CodeGroup>

#### Query Parameters

| Parameter   | Type    | Required | Default    | Constraints | Description |
|-------------|---------|----------|------------|-------------|-------------|
| `page`      | integer | No       | `1`        | ≥ 1         | Page number |
| `limit`     | integer | No       | `20`       | 1–100       | Results per page |
| `sortBy`    | string  | No       | `nameEn`   | See allowed values | Sort field |
| `sortOrder` | string  | No       | `asc`      | `asc`, `desc` | Sort direction |
| `nameEn`    | string  | No       | —          | —           | Filter by English name |
| `nameAr`    | string  | No       | —          | —           | Filter by Arabic name |
| `search`    | string  | No       | —          | —           | Search across names |
| `include`   | string  | No       | —          | `stats`     | Include relations |

<Accordion title="Allowed sortBy values">
- `id`
- `nameEn` 
- `nameAr`
- `developerNumber`
- `createdAt`
</Accordion>

**Default includes (list):** None

#### Response

```json 200 OK
{
  "data": [
    {
      "id": 42,
      "nameEn": "Emaar Properties",
      "nameAr": "إعمار العقارية", 
      "developerNumber": "DEV-001",
      "logo": "https://example.com/logo.png",
      "logoDark": "https://example.com/logo-dark.png"
    }
  ],
  "total": 1,
  "page": 1,
  "limit": 20
}
```

### Get Developer

<CodeGroup>
```http GET /api/developers/:id
GET /api/developers/42?include=stats
Authorization: Bearer YOUR_TOKEN
```

```javascript
const response = await fetch('/api/developers/42?include=stats', {
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN'
  }
});
const developer = await response.json();
```
</CodeGroup>

**Default includes (detail):** `stats`  
**Allowed includes:** `stats`

#### Response

```json 200 OK
{
  "id": 42,
  "nameEn": "Emaar Properties",
  "nameAr": "إعمار العقارية",
  "developerNumber": "DEV-001",
  "logo": "https://example.com/logo.png",
  "logoDark": "https://example.com/logo-dark.png",
  "sourceDeveloperId": "EXT-DEV-001",
  "stats": {
    "projectsCount": 150,
    "masterProjectsCount": 25
  }
}
```

## Cities

### List Cities

<CodeGroup>
```http GET /api/cities
GET /api/cities?stateId=1&include=state,stats
Authorization: Bearer YOUR_TOKEN
```

```javascript
const response = await fetch('/api/cities?include=state', {
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN'
  }
});
const { data } = await response.json();
```
</CodeGroup>

#### Query Parameters

| Parameter   | Type    | Required | Default    | Constraints | Description |
|-------------|---------|----------|------------|-------------|-------------|
| `page`      | integer | No       | `1`        | ≥ 1         | Page number |
| `limit`     | integer | No       | `20`       | 1–100       | Results per page |
| `sortBy`    | string  | No       | `nameEn`   | See allowed values | Sort field |
| `sortOrder` | string  | No       | `asc`      | `asc`, `desc` | Sort direction |
| `nameEn`    | string  | No       | —          | —           | Filter by English name |
| `nameAr`    | string  | No       | —          | —           | Filter by Arabic name |
| `search`    | string  | No       | —          | —           | Search names and state |
| `stateId`   | integer | No       | —          | —           | Filter by state |
| `include`   | string  | No       | —          | `state`, `stats` | Include relations |

<Accordion title="Allowed sortBy values">
- `id`
- `nameEn`
- `nameAr` 
- `state`
- `createdAt`
</Accordion>

**Default includes (list):** `state`

#### Response

```json 200 OK
{
  "data": [
    {
      "id": 1,
      "nameEn": "Dubai",
      "nameAr": "دبي",

  "total": 1,
  "page": 1, 
  "limit": 20
}
```

## Areas

### List Areas

Similar structure to cities with the following specific parameters:

#### Query Parameters

| Parameter   | Description |
|-------------|-------------|
| `cityId`    | Filter by city |
| `include`   | Allowed: `city`, `city.state`, `stats` |

**Default includes (list):** `city`

#### Response Item

```json
{
  "id": 1,
  "nameEn": "Downtown Dubai",
  "nameAr": "وسط مدينة دبي",
  "municipalityNumber": "MUN-001",
  "imageUrl": "https://example.com/area.jpg",
  "city": {
    "id": 1,
    "nameEn": "Dubai"
  }
}
```

### Get Area

**Default includes (detail):** `city`, `stats`

#### Extended Response

```json 200 OK
{
  "sourceAreaId": "EXT-AREA-001",
  "stats": {
    "projectsCount": 125,
    "buildingsCount": 890, 
    "unitsCount": 12500,
    "transactionsCount": 2500,
    "rentContractsCount": 1500
  }
}
```

## Communities

Community endpoints follow the same pattern as areas with area-based filtering:

#### Query Parameters

| Parameter | Description |
|-----------|-------------|
| `areaId`  | Filter by area |
| `include` | Allowed: `area`, `area.city`, `area.city.state`, `stats` |

**Default includes (list):** `area`
