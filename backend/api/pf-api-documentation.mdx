## Authentication

When working with the API, you need to pass an access token in the `Authorization` header of all your requests. This will be in the form of a Bearer token.

### Obtaining OAuth 2.0 credentials

<Steps>
  <Step title="Access PF Expert">
    Open the **PF Expert** application and log in with your credentials.
  </Step>
  
  <Step title="Navigate to developer resources">
    Go to the **API Credentials** section under the **Developer Resources** tab in the left sidebar.
  </Step>
  
  <Step title="Generate credentials">
    Create a new API key and API secret. Be sure to use the type as **API Integration**.
    
    - **API Key**: Your unique client identifier on PF Expert
    - **API Secret**: Your client secret, used for authentication when requesting an access token
  </Step>
</Steps>

### Requesting an access token

Exchange your OAuth 2.0 credentials for an access token by calling the **Issue JWT Token Endpoint**.

<CodeGroup>
```bash cURL
curl --location 'https://atlas.propertyfinder.com/v1/auth/token' \
--header 'Content-Type: application/json' \
--header 'Accept: application/json' \
--data '{
  "apiKey":"<API_KEY>",
  "apiSecret":"<API_SECRET>"
}'
```

```javascript JavaScript
const response = await fetch('https://atlas.propertyfinder.com/v1/auth/token', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Accept': 'application/json'
  },
  body: JSON.stringify({
    apiKey: '<API_KEY>',
    apiSecret: '<API_SECRET>'
  })
});

const data = await response.json();
```

```python Python
import requests

response = requests.post(
    'https://atlas.propertyfinder.com/v1/auth/token',
    headers={
        'Content-Type': 'application/json',
        'Accept': 'application/json'
    },
    json={
        'apiKey': '<API_KEY>',
        'apiSecret': '<API_SECRET>'
    }
)

data = response.json()
```
</CodeGroup>

#### Response

```json
{
  "accessToken": "<ACCESS_TOKEN>",
  "expiresIn": 1800,
  "tokenType": "Bearer"
}
```

<Info>
Your access token will be valid for **30 minutes** (1800 seconds). When it expires, you'll need to request a new one. No refresh token flow is supported in this integration.
</Info>

### Making authenticated requests

Use the Bearer token in the `Authorization` header of your API requests:

```bash
curl --location 'https://atlas.propertyfinder.com/v1/users?page=1&perPage=15' \
--header 'Accept: application/json' \
--header 'Authorization: Bearer <ACCESS_TOKEN>'
```

### Unauthorized requests

If you attempt to access a protected resource without valid authorization, you will receive a `401 Unauthorized` or `403 Forbidden` HTTP error.

```json
{
  "detail": "access denied (decision id: 5593a6df-2c67-4891-bd0a-83f3edfe9b14)",
  "title": "Unauthorized",
  "type": "AUTHENTICATION"
}
```

## Authorization - Enterprise API keys

Enterprise API keys are generated in **PF Expert > Developer Resources** and provide authenticated access to the PF API. API keys support scope-based authorization and enforced expiration to control access and manage key lifecycle.

### Key characteristics

<CardGroup cols={2}>
  <Card title="Scope-based access" icon="list-check">
    Each API key is created with explicitly assigned scopes that define which endpoints and operations are authorized
  </Card>
  <Card title="Immutable configuration" icon="lock">
    Once API keys are created, their scopes and configuration cannot be modified
  </Card>
  <Card title="Automatic expiration" icon="clock">
    Maximum validity period is 365 days, after which keys are automatically invalidated
  </Card>
  <Card title="Endpoint protection" icon="shield">
    Requests to endpoints outside assigned scopes are rejected
  </Card>
</CardGroup>

### Default scopes

The following scopes are enabled by default and **cannot be disabled**:

| Category | Scope |
| --- | --- |
| Compliance | `compliances:read` |
| Listing Verification | `listing_verification:full_access` |
| Location | `locations:read` |
| Project | `projects:read` |
| Webhooks | `webhooks:full_access` |

### Optional scopes

The following scopes can be enabled or disabled based on integration requirements:

| Category | Scopes |
| --- | --- |
| Users | `users:read`, `users:full_access` |
| Listings | `listings:read`, `listings:full_access` |
| Leads | `leads:read` |
| Credits | `credits:read` |
| Statistics | `statistics:read` |
| Roles | `roles:read` |

### Webhook event-based permissions

When subscribing to webhook events, additional scope requirements apply based on the event type:

| Event Category | Event Types | Required Scope |
| --- | --- | --- |
| Listings events | `listing.published`, `listing.unpublished`, `listing.action` | `listings:read` |
| Leads events | `lead.created`, `lead.updated`, `lead.assigned` | `leads:read` |
| Users events | `user.created`, `user.updated`, `user.deleted`, `user.activated`, `user.deactivated` | `users:read` |
| Public Profile events | `publicProfile.verification.approved`, `publicProfile.verification.rejected` | `users:read` |

### Unauthorized scope access

If you attempt to access an endpoint or event type without the required scope, you will receive a `403 Forbidden` HTTP error:

```json
{
  "detail": "action not allowed (decision id: 212c6af7-a651-462e-8250-2139c1c83ee3)",
  "title": "Forbidden",
  "type": "AUTHORIZATION"
}
```

### Best practices

<Tip>
**Principle of least privilege**: Only request scopes that your integration actually needs to minimize security exposure and maintain clean access patterns.
</Tip>

<Check>
**Key rotation**: Plan for key expiration by implementing a key rotation strategy before the 365-day limit to ensure uninterrupted service.
</Check>

<Info>
**Scope documentation**: When onboarding clients, clearly communicate which scopes your integration requires to streamline the setup process.
</Info>

## Rate limiting

To ensure fair use and maintain high availability of our APIs, we enforce rate limiting on all requests made to our endpoints.

### Default rate limits

Unless otherwise specified, all clients are subject to the following default limits:

| Request Type | Limit | Example Operations |
| --- | --- | --- |
| Issue JWT Token endpoint | 60 requests per minute | Issue JWT token |
| All other endpoints | 650 requests per minute | Create user, Search Listings, etc. |

<Note>
The rate limits are applied per IP address and per client. Other factors may also be considered.
</Note>

### Rate limit exceeded

If you exceed your allotted rate limit, you will receive a `429 Too Many Requests` HTTP response. The request can then be safely retried after some time.

```http
HTTP/1.1 429 Too Many Requests
```

### Best practices

<Tip>
To avoid frequently hitting the rate limit, implement the following strategies:

- **Retry with backoff**: Use an incremental backoff with jitter when retrying failed requests
- **Response caching**: Cache responses for frequently accessed data to reduce duplicate API calls
- **Request batching**: Where possible, batch multiple operations into single requests
- **Monitoring**: Track your API usage patterns to identify optimization opportunities
</Tip>

## Image requirements

When uploading images for listings or other resources, please ensure your images meet the following technical requirements to ensure optimal display quality and prevent processing issues.

### Accepted file types

The following image formats are supported:

- JPEG/JPG (`image/jpeg`)
- PNG (`image/png`)
- WebP (`image/webp`)

### File size

- **Minimum**: 5 KB per image
- **Maximum**: 15 MB per image

<Warning>
Images outside this range may fail to upload or process correctly.
</Warning>

### Color space requirements

**Supported color spaces:**

- ✅ sRGB (recommended)
- ✅ Adobe RGB
- ✅ RGB

**Not supported:**

- ❌ CMYK

<Warning>
CMYK color space is **NOT** supported. Images using CMYK may appear oversaturated or display incorrect colors when shown on the Property Finder website. Please convert CMYK images to sRGB before uploading to ensure accurate color representation.
</Warning>

### Aspect ratio and resolution

**Recommended aspect ratio:**

- Landscape orientation (e.g., 16:9, 4:3)

**Maximum resolution:**

- 1920 x 1080 pixels

<Info>
Images with higher resolutions will be automatically resized to fit within these dimensions while maintaining aspect ratio.
</Info>
