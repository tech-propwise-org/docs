## Overview

The Meta Platform module provides integration capabilities with Meta's Graph API ecosystem, including Facebook and Instagram messaging, advertising, and webhook handling. This module serves as a foundational layer for interacting with Meta's various APIs while handling authentication, error management, and webhook verification.

<Info>
The module abstracts common Meta API operations, implements OAuth flows, and provides secure webhook validation to ensure reliable communication with Meta's services.
</Info>

**Last Updated:** 2026-04-14  
**Status:** Active

## Services

### MetaGraphApiService

Core service for making authenticated requests to Meta's Graph API.

#### Key Features

- Automatic retry logic with exponential backoff
- Comprehensive error handling with Meta-specific error codes
- Type-safe request/response handling
- Configurable base URL and retry attempts
- Per-account circuit breaker isolation

#### Methods

<AccordionGroup>
  <Accordion title="get<T>">
    **Signature:** `(path, params, token, version?, circuitKey?, baseUrl?, timeoutMs?)`
    
    GET request with query parameters. All optional parameters provide flexibility for different API endpoints and circuit breaker isolation.
  </Accordion>
  
  <Accordion title="post<T>">
    **Signature:** `(path, body, token, version?, circuitKey?, baseUrl?)`
    
    POST request with JSON body for standard API operations.
  </Accordion>
  
  <Accordion title="postFormData<T>">
    **Signature:** `(path, formData, token, version?, circuitKey?, baseUrl?)`
    
    POST request with multipart form data, typically used for media uploads.
  </Accordion>
  
  <Accordion title="delete<T>">
    **Signature:** `(path, token, version?, circuitKey?, baseUrl?)`
    
    DELETE request with no body or query params. Append query params to `path` if needed. Used by `ChannelAccountService` for Meta webhook unsubscription on channel account deletion.
  </Accordion>
</AccordionGroup>

<Tip>
All methods accept optional `version`, `circuitKey`, and `baseUrl` parameters. The `circuitKey` enables per-account circuit breaker isolation, while `baseUrl` allows overriding the default `graph.facebook.com` for services like Instagram Login API.
</Tip>

#### Configuration Dependencies

- `app.meta.graphApiBaseUrl`: Base URL for Graph API (defaults to `https://graph.facebook.com`)
- `app.meta.whatsappApiVersion`: Default API version when callers don't pass an explicit version

### MetaOAuthService

Handles OAuth token operations for Meta platform authentication.

#### Configuration Dependencies

- `app.meta.graphApiBaseUrl`: Base URL for OAuth endpoints
- `app.meta.appId`: Meta application ID
- `app.meta.appSecret`: Meta application secret
- `app.meta.oauthApiVersion`: API version for OAuth dialog and token exchange URLs (default `v22.0`)

#### Methods

<Tabs>
  <Tab title="Authorization">
    **`getAuthorizationUrl(scopes, redirectUri, state)`**
    
    Builds the Facebook OAuth dialog URL with the configured `oauthApiVersion`.
    
    ```typescript
    const authUrl = await metaOAuthService.getAuthorizationUrl(
      ['pages_messaging', 'instagram_basic'],
      'https://app.example.com/callback',
      'random-state-string'
    );
    ```
  </Tab>
  
  <Tab title="Token Exchange">
    **`exchangeCodeForToken(code, redirectUri)`**
    
    Exchanges an authorization code for a short-lived access token.
    
    **`exchangeEmbeddedSignupCode(code)`**
    
    Exchanges a WhatsApp Embedded Signup code for a non-expiring BISUAT. Uses `axios` directly (no retries) because the code is single-use with a 30s TTL.
  </Tab>
  
  <Tab title="Token Management">
    **`exchangeForLongLivedToken(shortLivedToken)`**
    
    Exchanges a short-lived user token for a long-lived token (~60 days).
    
    **`refreshToken(token)`**
    
    Refreshes a token (delegates to `exchangeForLongLivedToken`).
  </Tab>
</Tabs>

#### Token Response Interface

```typescript
interface TokenResponse {
  access_token: string;
  token_type: string;
  expires_in?: number;
}
```

<Warning>
WhatsApp Embedded Signup codes have a 30-second TTL and are single-use. Retries would fail with Meta error 36007.
</Warning>

### MetaWebhookGuard

NestJS guard that validates webhook requests from Meta using HMAC-SHA256 signature verification.

#### Security Features

- HMAC-SHA256 signature validation
- Timing-safe comparison to prevent timing attacks
- Automatic request rejection for invalid signatures
- Route-level throttling protection

<Note>
Route-level throttling remains enabled on webhook endpoints to protect CPU, database writes, and pg-boss queue capacity. Signature validation proves authenticity, but does not prevent signed flood/retry storms.
</Note>

#### Configuration Dependencies

- `app.meta.webhookVerifyToken`: Secret token for webhook verification

## Error Handling

### MetaApiError

Specialized error class for Meta API operations with enhanced context:

```typescript
class MetaApiError extends Error {
  constructor(
    message: string,
    statusCode: number,
    metaErrorCode?: number,
    metaErrorSubcode?: number,
    isRetryable: boolean = false,
  );
}
```

#### Properties

<Steps>
  <Step title="statusCode">
    HTTP status code from the API response
  </Step>
  <Step title="metaErrorCode">
    Meta-specific error code (if available)
  </Step>
  <Step title="metaErrorSubcode">
    Meta-specific error subcode (if available)
  </Step>
  <Step title="isRetryable">
    Indicates if the operation can be safely retried
  </Step>
</Steps>

<Info>
Both `MetaApiError` and `CircuitOpenError` call `Object.setPrototypeOf(this, <Class>.prototype)` in their constructors to ensure `instanceof` checks work correctly with TypeScript class inheritance from `Error`.
</Info>

### Configurable API Versions

Meta deprecates Graph API versions on a rolling basis. Each channel's API version is configurable via environment variables so upgrades require only a config change, not code changes:

| Environment Variable | Config Key | Default | Used By |
|---------------------|------------|---------|---------|
| `META_WHATSAPP_API_VERSION` | `app.meta.whatsappApiVersion` | `v22.0` | WhatsAppProviderService, TemplateService, ChannelAccountService |
| `META_INSTAGRAM_API_VERSION` | `app.meta.instagramApiVersion` | `v23.0` | InstagramProviderService, ChannelAccountService |
| `META_MESSENGER_API_VERSION` | `app.meta.messengerApiVersion` | `v23.0` | MessengerProviderService, ChannelAccountService |
| `META_OAUTH_API_VERSION` | `app.meta.oauthApiVersion` | `v22.0` | MetaOAuthService |

<Tip>
**Upgrade procedure:** Update the environment variable, deploy, and test. No code changes needed.
</Tip>

## Circuit Breaker Integration

`MetaGraphApiService` supports per-account circuit breaker isolation. All Meta Graph API call sites (13 across 4 files in the Messaging module) pass the `channelAccountId` as the `circuitKey` parameter.

<Check>
This ensures one degraded account does not trip the circuit breaker for others, maintaining system resilience.
</Check>

## Business Rules

<Steps>
  <Step title="Retry Logic">
    Failed requests are retried up to 3 times with exponential backoff
  </Step>
  <Step title="Signature Validation">
    All webhook requests must include valid `x-hub-signature-256` header
  </Step>
  <Step title="Webhook Capacity Guard">
    Meta webhook POST endpoints use explicit high throttles (1000/min) and verification GET endpoints use low throttles (20/min). Do not use `@SkipThrottle()` on Meta webhook controllers.
  </Step>
  <Step title="Token Management">
    OAuth tokens should be refreshed before expiration and stored securely with encryption
  </Step>
</Steps>

<Warning>
Meta's APIs have specific rate limits and error codes. Always implement proper error handling and respect the documented limits to avoid service disruptions.
</Warning>
