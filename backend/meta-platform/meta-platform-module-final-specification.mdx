## Overview

The Meta Platform module provides integration capabilities with Meta's Graph API ecosystem, including Facebook and Instagram messaging, advertising, and webhook handling. This module serves as a foundational layer for interacting with Meta's various APIs while handling authentication, error management, and webhook verification.

<Info>
The module abstracts common Meta API operations, implements OAuth flows, and provides secure webhook validation to ensure reliable communication with Meta's services.
</Info>

## Architecture

The module follows a service-oriented architecture with clear separation of concerns:

- **MetaGraphApiService**: Core HTTP client for Graph API interactions
- **MetaOAuthService**: Handles OAuth token management and exchange
- **MetaWebhookGuard**: Validates incoming webhook requests from Meta
- **MetaApiError**: Specialized error class for Meta API responses

<Note>
The module is designed to be stateless and relies on dependency injection for configuration management.
</Note>

## Services

### MetaGraphApiService

Core service for making authenticated requests to Meta's Graph API.

**Key Features:**

- Automatic retry logic with exponential backoff
- Comprehensive error handling with Meta-specific error codes
- Type-safe request/response handling
- Configurable base URL and retry attempts
- Per-account circuit breaker isolation

**Methods:**

| Method | Signature | Description |
| --- | --- | --- |
| `get<T>` | `(path, params, token, version?, circuitKey?, baseUrl?, timeoutMs?)` | GET request with query parameters |
| `post<T>` | `(path, body, token, version?, circuitKey?, baseUrl?)` | POST request with JSON body |
| `postFormData<T>` | `(path, formData, token, version?, circuitKey?, baseUrl?)` | POST request with multipart form data |
| `delete<T>` | `(path, token, version?, circuitKey?, baseUrl?)` | DELETE request (no body or query params — append query params to `path` if needed) |

<Tip>
All methods accept optional parameters for `version`, `circuitKey` for per-account circuit breaker isolation, and `baseUrl` to override the default `graph.facebook.com`.
</Tip>

**Configuration Dependencies:**

- `app.meta.graphApiBaseUrl`: Base URL for Graph API (defaults to `https://graph.facebook.com`)
- `app.meta.whatsappApiVersion`: Default API version when callers don't pass an explicit version

### MetaOAuthService

Handles OAuth token operations for Meta platform authentication.

**Configuration Dependencies:**

- `app.meta.graphApiBaseUrl`: Base URL for OAuth endpoints
- `app.meta.appId`: Meta application ID
- `app.meta.appSecret`: Meta application secret
- `app.meta.oauthApiVersion`: API version for OAuth dialog and token exchange URLs (default `v22.0`)

**Methods:**

| Method | Description |
| --- | --- |
| `getAuthorizationUrl(scopes, redirectUri, state)` | Builds the Facebook OAuth dialog URL with the configured `oauthApiVersion` |
| `exchangeCodeForToken(code, redirectUri)` | Exchanges an authorization code for a short-lived access token |
| `exchangeEmbeddedSignupCode(code)` | Exchanges a WhatsApp Embedded Signup code for a non-expiring BISUAT |
| `exchangeForLongLivedToken(shortLivedToken)` | Exchanges a short-lived user token for a long-lived token (~60 days) |
| `refreshToken(token)` | Refreshes a token (delegates to `exchangeForLongLivedToken`) |

<Warning>
The `exchangeEmbeddedSignupCode` method uses `axios` directly (no retries) because the code is single-use with a 30s TTL — retries would fail with Meta error 36007.
</Warning>

**Token Response Interface:**

```typescript
interface TokenResponse {
  access_token: string;
  token_type: string;
  expires_in?: number;
}
```

### MetaWebhookGuard

NestJS guard that validates webhook requests from Meta using HMAC-SHA256 signature verification.

**Security Features:**

- HMAC-SHA256 signature validation
- Timing-safe comparison to prevent timing attacks
- Automatic request rejection for invalid signatures

**Configuration Dependencies:**

- `app.meta.webhookVerifyToken`: Secret token for webhook verification

## Error Handling

### MetaApiError

Specialized error class for Meta API operations with enhanced context:

```typescript
class MetaApiError extends Error {
  constructor(
    message: string,
    statusCode: number,
    metaErrorCode?: number,
    metaErrorSubcode?: number,
    isRetryable: boolean = false,
  );
}
```

**Properties:**

<AccordionGroup>
<Accordion title="Error Properties">
- `statusCode`: HTTP status code from the API response
- `metaErrorCode`: Meta-specific error code (if available)
- `metaErrorSubcode`: Meta-specific error subcode (if available)
- `isRetryable`: Indicates if the operation can be safely retried
</Accordion>
</AccordionGroup>

<Note>
Both `MetaApiError` and `CircuitOpenError` call `Object.setPrototypeOf(this, <Class>.prototype)` in their constructors to ensure `instanceof` checks work correctly with TypeScript class inheritance from `Error`.
</Note>

## Configuration

The module requires the following configuration parameters:

```typescript
meta: {
  graphApiBaseUrl: string; // Meta Graph API base URL
  appId: string; // Meta application ID
  appSecret: string; // Meta application secret
  webhookVerifyToken: string; // Webhook verification token
  tokenEncryptionKey: string; // AES-256-GCM key for token encryption at rest
  whatsappApiVersion: string; // WhatsApp Cloud API version (e.g. 'v22.0')
  instagramApiVersion: string; // Instagram Messaging API version (e.g. 'v23.0')
  messengerApiVersion: string; // Messenger Platform API version (e.g. 'v23.0')
  oauthApiVersion: string; // OAuth dialog/token exchange version (e.g. 'v22.0')
}
```

### Configurable API Versions

<Info>
Meta deprecates Graph API versions on a rolling basis. Each channel's API version is configurable via environment variables so upgrades require only a config change, not code changes.
</Info>

| Env Variable | Config Key | Default | Used By |
| --- | --- | --- | --- |
| `META_WHATSAPP_API_VERSION` | `app.meta.whatsappApiVersion` | `v22.0` | WhatsAppProviderService, TemplateService, ChannelAccountService |
| `META_INSTAGRAM_API_VERSION` | `app.meta.instagramApiVersion` | `v23.0` | InstagramProviderService, ChannelAccountService |
| `META_MESSENGER_API_VERSION` | `app.meta.messengerApiVersion` | `v23.0` | MessengerProviderService, ChannelAccountService |
| `META_OAUTH_API_VERSION` | `app.meta.oauthApiVersion` | `v22.0` | MetaOAuthService |

## Entities

### WebhookEventLog

Raw webhook payload storage used for idempotency and debugging. Persisted before async processing so duplicate webhooks from Meta are detected and skipped.

| Field | Type | Description |
| --- | --- | --- |
| id | UUID | Primary key |
| platform | WebhookPlatform | `META` or `GOOGLE` |
| eventType | WebhookEventType | `MESSAGING`, `STATUS_UPDATE`, or `LEAD_FORM` |
| externalEventId | string | Deterministic ID for idempotency (unique constraint) |
| payload | JSONB | Raw webhook payload |
| status | WebhookEventStatus | `RECEIVED` → `PROCESSING` → `PROCESSED` / `FAILED` |
| organization | Organization? | Resolved after processing (nullable) |
| processedAt | Date? | When processing completed |
| error | string? | Error message if processing failed |
| createdAt | Date | Receive timestamp |

<Note>
- No RLS — webhook arrives before organization context is known
- No soft delete — rows are hard-deleted by retention cron after 30 days
</Note>

## Circuit Breaker Integration

`MetaGraphApiService` supports per-account circuit breaker isolation. All Meta Graph API call sites pass the `channelAccountId` as the `circuitKey` parameter, ensuring one degraded account does not trip the circuit breaker for others.

<Tip>
This isolation pattern prevents cascading failures when individual Meta accounts experience rate limiting or other issues.
</Tip>

### MessagingModule Dependencies

The `MetaPlatformModule` provides the following to `MessagingModule`:

- `MetaGraphApiService` for outbound API calls
- `MetaOAuthService` for personal account OAuth flows
- `InstagramOAuthService` for Instagram Login OAuth flows
- `MetaWebhookGuard` and `InstagramWebhookGuard` for webhook validation
- `WebhookEventLog` entity for webhook idempotency

## Business Rules

<CardGroup cols={2}>
<Card title="Retry Logic" icon="refresh">
Failed requests are retried up to 3 times with exponential backoff
</Card>
<Card title="Signature Validation" icon="shield">
All webhook requests must include valid `x-hub-signature-256` header
</Card>
<Card title="Token Management" icon="key">
OAuth tokens should be refreshed before expiration
</Card>
<Card title="Error Classification" icon="warning">
API errors are classified as retryable or non-retryable based on status codes
</Card>
</CardGroup>

## Security Considerations

<Warning>
All webhook requests are validated using HMAC-SHA256 signatures with timing-safe comparison to prevent timing attack vectors.
</Warning>

- Application secrets are loaded from environment variables
- Raw request bodies are preserved for signature verification
- Per-account circuit breaker isolation prevents cross-contamination

## Usage Patterns

### Webhook Validation

```typescript
@UseGuards(MetaWebhookGuard)
@Post('/webhooks/meta')
async handleWebhook(@Req() request: Request) {
  // Webhook signature is automatically validated
  const payload = request.body;
  // Process webhook payload
}
```

### Graph API Calls

```typescript
// GET request with circuit breaker isolation
const response = await this.metaGraphApiService.get<UserProfile>(
  '/me',
  { fields: 'id,name,email' },
  accessToken,
  'v22.0',
  `account_${channelAccountId}` // Circuit breaker key
);

// POST request with form data
const mediaResponse = await this.metaGraphApiService.postFormData<MediaUpload>(
  '/media',
  formData,
  accessToken,
  'v22.0',
  `account_${channelAccountId}`
);
```

### OAuth Flow

```typescript
// Generate authorization URL
const authUrl = await this.metaOAuthService.getAuthorizationUrl(
  ['pages_messaging', 'instagram_basic'],
  redirectUri,
  state
);

// Exchange code for token
const tokenResponse = await this.metaOAuthService.exchangeCodeForToken(
  authorizationCode,
  redirectUri
);

// Get long-lived token
const longLivedToken = await this.metaOAuthService.exchangeForLongLivedToken(
  tokenResponse.access_token
);
```
