The Meta Platform module provides integration capabilities with Meta's Graph API ecosystem, including Facebook and Instagram messaging, advertising, and webhook handling. This module serves as a foundational layer for interacting with Meta's various APIs while handling authentication, error management, and webhook verification.

<Note>
Last Updated: 2026-06-01 | Status: Active
</Note>

## Overview

The module abstracts common Meta API operations, implements OAuth flows, and provides secure webhook validation to ensure reliable communication with Meta's services.

### Key Features

- **Graph API Integration**: Core HTTP client for Meta's Graph API
- **OAuth Management**: Complete OAuth token lifecycle handling
- **Webhook Security**: HMAC-SHA256 signature verification
- **Error Handling**: Specialized error classes for Meta API responses
- **Circuit Breaker**: Per-account isolation for reliability

## Architecture

The module follows a service-oriented architecture with clear separation of concerns:

## Services

### MetaGraphApiService

Core service for making authenticated requests to Meta's Graph API.

#### Key Features

- Automatic retry logic with exponential backoff
- Comprehensive error handling with Meta-specific error codes
- Type-safe request/response handling
- Configurable base URL with a fixed 3-total-attempt retry cap
- Per-account circuit breaker isolation

#### Methods

<AccordionGroup>
  <Accordion title="get<T>(path, params, token, version?, circuitKey?, baseUrl?, timeoutMs?)">
    GET request with query parameters
    
    **Parameters:**
    - `path`: API endpoint path
    - `params`: Query parameters object
    - `token`: Access token for authentication
    - `version`: API version (optional, defaults to `app.meta.whatsappApiVersion`)
    - `circuitKey`: Circuit breaker key for per-account isolation (optional)
    - `baseUrl`: Override default base URL (optional)
    - `timeoutMs`: Request timeout in milliseconds (optional)
  </Accordion>

  <Accordion title="post<T>(path, body, token, version?, circuitKey?, baseUrl?)">
    POST request with JSON body
    
    **Parameters:**
    - `path`: API endpoint path
    - `body`: Request body object
    - `token`: Access token for authentication
    - `version`: API version (optional)
    - `circuitKey`: Circuit breaker key (optional)
    - `baseUrl`: Override default base URL (optional)
  </Accordion>

  <Accordion title="postFormData<T>(path, formData, token, version?, circuitKey?, baseUrl?)">
    POST request with multipart form data
    
    **Parameters:**
    - `path`: API endpoint path
    - `formData`: FormData object
    - `token`: Access token for authentication
    - `version`: API version (optional)
    - `circuitKey`: Circuit breaker key (optional)
    - `baseUrl`: Override default base URL (optional)
  </Accordion>

  <Accordion title="uploadResumableAsset({ fileName, fileType, fileBuffer, token, version?, circuitKey? })">
    Creates a Meta Resumable Upload session under the configured app id and uploads raw bytes
    
    **Returns:** Asset handle (`h`) used for WhatsApp template media header examples
    
    **Parameters:**
    - `fileName`: Name of the file
    - `fileType`: MIME type of the file
    - `fileBuffer`: File content as buffer
    - `token`: Access token for authentication
    - `version`: API version (optional)
    - `circuitKey`: Circuit breaker key (optional)
  </Accordion>

  <Accordion title="delete<T>(path, token, version?, circuitKey?, baseUrl?)">
    DELETE request (no body or query params — append query params to `path` if needed)
    
    Used by messaging account cleanup for webhook unsubscription and app-permission revocation.
    
    **Parameters:**
    - `path`: API endpoint path with any query parameters
    - `token`: Access token for authentication
    - `version`: API version (optional)
    - `circuitKey`: Circuit breaker key (optional)
    - `baseUrl`: Override default base URL (optional)
  </Accordion>
</AccordionGroup>

#### Configuration Dependencies

- `app.meta.graphApiBaseUrl`: Base URL for Graph API (defaults to `https://graph.facebook.com`)
- `app.meta.whatsappApiVersion`: Default API version when callers don't pass an explicit version

### MetaOAuthService

Handles OAuth token operations for Meta platform authentication.

#### Configuration Dependencies

- `app.meta.graphApiBaseUrl`: Base URL for OAuth endpoints
- `app.meta.appId`: Meta application ID
- `app.meta.appSecret`: Meta application secret
- `app.meta.oauthApiVersion` (`META_OAUTH_API_VERSION` env var): API version for OAuth dialog and token exchange URLs (default `v22.0`)

#### Methods

<Tabs>
  <Tab title="Authorization">
    **`getAuthorizationUrl(scopes, redirectUri, state)`**
    
    Builds the Facebook OAuth dialog URL with the configured `oauthApiVersion`
    
    ```typescript
    const authUrl = await metaOAuthService.getAuthorizationUrl(
      ['pages_messaging', 'whatsapp_business_messaging'],
      'https://app.example.com/callback',
      'random-state-string'
    );
    ```
  </Tab>
  
  <Tab title="Token Exchange">
    **`exchangeCodeForToken(code, redirectUri)`**
    
    Exchanges an authorization code for a short-lived access token
    
    ```typescript
    const tokenResponse = await metaOAuthService.exchangeCodeForToken(
      'authorization-code',
      'https://app.example.com/callback'
    );
    ```
  </Tab>
  
  <Tab title="Embedded Signup">
    **`exchangeEmbeddedSignupCode(code)`**
    
    Exchanges a WhatsApp Embedded Signup code for a non-expiring BISUAT.
    
    <Warning>
    Uses `axios` directly (no retries) because the code is single-use with a 30s TTL — retries would fail with Meta error 36007.
    </Warning>
    
    ```typescript
    const tokenResponse = await metaOAuthService.exchangeEmbeddedSignupCode(
      'embedded-signup-code'
    );
    ```
  </Tab>
  
  <Tab title="Long-Lived Tokens">
    **`exchangeForLongLivedToken(shortLivedToken)`**
    
    Exchanges a short-lived user token for a long-lived token (~60 days)
    
    **`refreshToken(token)`**
    
    Refreshes a token (delegates to `exchangeForLongLivedToken`)
    
    ```typescript
    const longLivedToken = await metaOAuthService.exchangeForLongLivedToken(
      'short-lived-token'
    );
    ```
  </Tab>
</Tabs>

#### Token Response Interface

```typescript
interface TokenResponse {
  access_token: string;
  token_type: string;
  expires_in?: number;
}
```

### MetaWebhookGuard

NestJS guard that validates webhook requests from Meta using HMAC-SHA256 signature verification.

#### Security Features

- HMAC-SHA256 signature validation
- Timing-safe comparison to prevent timing attacks
- Automatic request rejection for invalid signatures
- Route-level throttling remains enabled on webhook endpoints to protect CPU, database writes, and pg-boss queue capacity

<Note>
Signature validation proves authenticity, but it does not prevent signed flood/retry storms.
</Note>

#### Configuration Dependencies

- `app.meta.webhookVerifyToken`: Secret token for webhook verification (single value — correct for one-Meta-App architecture)

## Error Handling

### MetaApiError

Specialized error class for Meta API operations with enhanced context:

```typescript
class MetaApiError extends Error {
  constructor(
    message: string,
    statusCode: number,
    metaErrorCode?: number,
    metaErrorSubcode?: number,
    isRetryable: boolean = false,
  );
}
```

#### Properties

<AccordionGroup>
  <Accordion title="statusCode">
    HTTP status code from the API response
  </Accordion>
  
  <Accordion title="metaErrorCode">
    Meta-specific error code (if available)
  </Accordion>
  
  <Accordion title="metaErrorSubcode">
    Meta-specific error subcode (if available)
  </Accordion>
  
  <Accordion title="isRetryable">
    Indicates if the operation can be safely retried
  </Accordion>
</AccordionGroup>

<Tip>
Both `MetaApiError` and `CircuitOpenError` call `Object.setPrototypeOf(this, <Class>.prototype)` in their constructors to ensure `instanceof` checks work correctly with TypeScript class inheritance from `Error`.
</Tip>

### WhatsApp Billing Errors

Meta WhatsApp send/status failures that indicate WABA billing setup problems are handled in the Messaging module, not by changing Propwise subscription billing.

<Warning>
The canonical billing/payment error code is `131042` ("Business eligibility payment issue"). Messaging classifies this and related payment-method/billing/credit-line text as `MessageFailureCategory.META_BILLING`.
</Warning>

The system preserves the raw `failedReason` and includes the category on message DTOs and `message-status-updated` socket events so the frontend can guide the user to Meta Business Suite billing settings.

### Configurable API Versions

Meta deprecates Graph API versions on a rolling basis. Each channel's API version is configurable via environment variables so upgrades require only a config change, not code changes:

| Env Variable                 | Config Key                     | Default | Used By                                                                |
| ---------------------------- | ------------------------------ | ------- | ---------------------------------------------------------------------- |
| `META_WHATSAPP_API_VERSION`  | `app.meta.whatsappApiVersion`  | `v22.0` | WhatsAppProviderService, TemplateService, ChannelAccountService        |
| `META_INSTAGRAM_API_VERSION` | `app.meta.instagramApiVersion` | `v23.0` | InstagramProviderService, ChannelAccountService, InstagramMediaService |
| `META_MESSENGER_API_VERSION` | `app.meta.messengerApiVersion` | `v23.0` | MessengerProviderService, ChannelAccountService                        |
| `META_OAUTH_API_VERSION`     | `app.meta.oauthApiVersion`     | `v22.0` | MetaOAuthService                                                       |

## Entities

### WebhookEventLog

Raw webhook payload storage used for idempotency and debugging. Persisted before async processing so duplicate webhooks from Meta are detected and skipped.

| Field           | Type               | Description                                                        |
| --------------- | ------------------ | ------------------------------------------------------------------ |
| id              | UUID               | Primary key                                                        |
| platform        | WebhookPlatform    | `META` or `GOOGLE`                                                 |
| eventType       | WebhookEventType   | `MESSAGING`, `STATUS_UPDATE`, or `LEAD_FORM`                       |
| externalEventId | string             | Deterministic ID for idempotency (unique constraint)               |
| payload         | JSONB              | Raw webhook payload                                                |
| status          | WebhookEventStatus | `RECEIVED` → `PROCESSING` → `PROCESSED` / `FAILED`                 |
| organization    | Organization?      | Resolved after processing (nullable — org unknown at receive time) |
| processedAt     | Date?              | When processing completed                                          |
| error           | string?            | Error message if processing failed                                 |

<Info>
The `externalEventId` field ensures webhook idempotency by creating a unique constraint that prevents duplicate processing of the same webhook event.
</Info>
