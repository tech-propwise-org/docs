## Overview

The Meta Platform module provides integration capabilities with Meta's Graph API ecosystem, including Facebook and Instagram messaging, advertising, and webhook handling. This module serves as a foundational layer for interacting with Meta's various APIs while handling authentication, error management, and webhook verification.

<Info>
The module abstracts common Meta API operations, implements OAuth flows, and provides secure webhook validation to ensure reliable communication with Meta's services.
</Info>

**Last Updated:** 2026-04-14  
**Status:** Active

## Architecture

The module follows a service-oriented architecture with clear separation of concerns:

- **MetaGraphApiService**: Core HTTP client for Graph API interactions
- **MetaOAuthService**: Handles OAuth token management and exchange
- **MetaWebhookGuard**: Validates incoming webhook requests from Meta
- **MetaApiError**: Specialized error class for Meta API responses

<Note>
The module is designed to be stateless and relies on dependency injection for configuration management.
</Note>

#### Available Methods

<AccordionGroup>
  <Accordion title="GET Request">
    **Signature:** `get<T>(path, params, token, version?, circuitKey?, baseUrl?, timeoutMs?)`
    
    GET request with query parameters and optional timeout configuration.
  </Accordion>
  
  <Accordion title="POST Request">
    **Signature:** `post<T>(path, body, token, version?, circuitKey?, baseUrl?)`
    
    POST request with JSON body payload.
  </Accordion>
  
  <Accordion title="POST Form Data">
    **Signature:** `postFormData<T>(path, formData, token, version?, circuitKey?, baseUrl?)`
    
    POST request with multipart form data for file uploads.
  </Accordion>
  
  <Accordion title="DELETE Request">
    **Signature:** `delete<T>(path, token, version?, circuitKey?, baseUrl?)`
    
    DELETE request with no body. Query parameters should be appended to the path if needed. Used by `ChannelAccountService` for Meta webhook unsubscription on channel account deletion.
  </Accordion>
</AccordionGroup>

<Tip>
All methods accept an optional `version` (defaults to `app.meta.whatsappApiVersion`), `circuitKey` for per-account circuit breaker isolation, and `baseUrl` to override the default `graph.facebook.com`.
</Tip>

#### Configuration Dependencies

```yaml
app.meta.graphApiBaseUrl: Base URL for Graph API (defaults to https://graph.facebook.com)
app.meta.whatsappApiVersion: Default API version when callers don't pass an explicit version
```

### MetaOAuthService

Handles OAuth token operations for Meta platform authentication.

#### Configuration Dependencies

```yaml
app.meta.graphApiBaseUrl: Base URL for OAuth endpoints
app.meta.appId: Meta application ID
app.meta.appSecret: Meta application secret
app.meta.oauthApiVersion: API version for OAuth dialog and token exchange URLs (default v22.0)
```

#### OAuth Methods

<Tabs>
  <Tab title="Authorization">
    **`getAuthorizationUrl(scopes, redirectUri, state)`**
    
    Builds the Facebook OAuth dialog URL with the configured `oauthApiVersion`.
  </Tab>
  
  <Tab title="Token Exchange">
    **`exchangeCodeForToken(code, redirectUri)`**
    
    Exchanges an authorization code for a short-lived access token.
  </Tab>
  
  <Tab title="Embedded Signup">
    **`exchangeEmbeddedSignupCode(code)`**
    
    Exchanges a WhatsApp Embedded Signup code for a non-expiring BISUAT. Uses `axios` directly (no retries) because the code is single-use with a 30s TTL.
  </Tab>
  
  <Tab title="Long-Lived Token">
    **`exchangeForLongLivedToken(shortLivedToken)`**
    
    Exchanges a short-lived user token for a long-lived token (~60 days).
  </Tab>
</Tabs>

#### Token Response Interface

```typescript
interface TokenResponse {
  access_token: string;
  token_type: string;
  expires_in?: number;
}
```

### MetaWebhookGuard

NestJS guard that validates webhook requests from Meta using HMAC-SHA256 signature verification.

<Warning>
Route-level throttling remains enabled on webhook endpoints to protect CPU, database writes, and pg-boss queue capacity. Signature validation proves authenticity, but it does not prevent signed flood/retry storms.
</Warning>

#### Security Features

- HMAC-SHA256 signature validation
- Timing-safe comparison to prevent timing attacks
- Automatic request rejection for invalid signatures
- Protection against replay attacks

#### Configuration

```yaml
app.meta.webhookVerifyToken: Secret token for webhook verification
```

<Note>
Single verify token value is correct for one-Meta-App architecture. See MESSAGING_MODULE_SPECIFICATION.md § "Webhook Verify Token" for rationale.
</Note>

## Error Handling

### MetaApiError

Specialized error class for Meta API operations with enhanced context:

```typescript
class MetaApiError extends Error {
  constructor(
    message: string,
    statusCode: number,
    metaErrorCode?: number,
    metaErrorSubcode?: number,
    isRetryable: boolean = false,
  );
}
```

#### Error Properties

| Property | Type | Description |
| --- | --- | --- |
| `statusCode` | number | HTTP status code from the API response |
| `metaErrorCode` | number? | Meta-specific error code (if available) |
| `metaErrorSubcode` | number? | Meta-specific error subcode (if available) |
| `isRetryable` | boolean | Indicates if the operation can be safely retried |

<Check>
Both `MetaApiError` and `CircuitOpenError` call `Object.setPrototypeOf(this, <Class>.prototype)` in their constructors to ensure `instanceof` checks work correctly with TypeScript class inheritance from `Error`.
</Check>

## Data Models

### WebhookEventLog Entity

Raw webhook payload storage used for idempotency and debugging. Persisted before async processing so duplicate webhooks from Meta are detected and skipped.

| Field | Type | Description |
| --- | --- | --- |
| `id` | UUID | Primary key |
| `platform` | WebhookPlatform | `META` or `GOOGLE` |
| `eventType` | WebhookEventType | `MESSAGING`, `STATUS_UPDATE`, or `LEAD_FORM` |
| `externalEventId` | string | Deterministic ID for idempotency (unique constraint) |
| `payload` | JSONB | Raw webhook payload |
| `status` | WebhookEventStatus | `RECEIVED` → `PROCESSING` → `PROCESSED` / `FAILED` |
| `organization` | Organization? | Resolved after processing (nullable) |
| `processedAt` | Date? | When processing completed |
| `error` | string? | Error message if processing failed |
| `createdAt` | Date | Receive timestamp |

<Warning>
No RLS (Row Level Security) — webhook arrives before organization context is known. No soft delete — rows are hard-deleted by retention cron after 30 days.
</Warning>

## Circuit Breaker Integration

<Tip>
`MetaGraphApiService` supports per-account circuit breaker isolation. All Meta Graph API call sites pass the `channelAccountId` as the `circuitKey` parameter, ensuring one degraded account does not trip the circuit breaker for others.
</Tip>

The circuit breaker pattern prevents cascading failures by:
- Monitoring failure rates per account
- Opening circuits when thresholds are exceeded
- Allowing gradual recovery with half-open states

## Business Rules

<AccordionGroup>
  <Accordion title="Retry Logic">
    Failed requests are retried up to 3 times with exponential backoff to handle transient network issues and temporary API unavailability.
  </Accordion>
  
  <Accordion title="Signature Validation">
    All webhook requests must include valid `x-hub-signature-256` header for HMAC-SHA256 verification to ensure authenticity.
  </Accordion>
  
  <Accordion title="Webhook Capacity Guard">
    Meta webhook POST endpoints use explicit high throttles (1000/min) and verification GET endpoints use low throttles (20/min). Do not use `@SkipThrottle()` on Meta webhook controllers.
  </Accordion>
  
  <Accordion title="Token Management">
    OAuth tokens should be refreshed before expiration to maintain uninterrupted API access. Long-lived tokens last approximately 60 days.
  </Accordion>
  
  <Accordion title="Error Classification">
    API errors are classified as retryable or non-retryable based on Meta error codes to optimize recovery strategies and prevent unnecessary retry storms.
  </Accordion>
</AccordionGroup>
