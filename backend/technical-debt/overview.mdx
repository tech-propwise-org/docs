## Active debt items

### Remove AI Assistant knowledge-base glue

<Info>
**Status:** Open — interim structural alignment DONE (2026-05-30, on `ai-assistant`); final deletion still pending the PCD-64 merge
</Info>

**Owner:** TBD  
**Estimated effort:** ~2 hours + verification  
**Risk:** Low — pure refactor, no schema/contract change  
**Created:** 2026-05-28

#### Background

When the `dev` branch cherry-picked PCD-64's shared knowledge-base infrastructure, it could not adopt the full AI Agent runtime. To maintain the internal AI assistant's `searchKnowledge` tool functionality, a temporary glue layer was implemented:

- `src/modules/ai/services/ai-knowledge.service.ts` — slim search-only service
- `src/modules/ai/ai-knowledge-tools.provider.ts` — wraps the service into AI tools
- Registration lines in `src/modules/ai/ai.module.ts`

<Warning>
This glue layer duplicates functionality that will be available in `AiAgentKnowledgeBaseService` when PCD-64 merges, creating maintenance overhead and potential drift.
</Warning>

#### Cleanup tasks

<Steps>
<Step title="Migrate tool provider">
Update `AiKnowledgeToolsProvider` to inject and call `AiAgentKnowledgeBaseService.searchAgentKnowledge()` instead of the temporary `AIKnowledgeService`
</Step>

<Step title="Delete temporary files">
Remove `src/modules/ai/services/ai-knowledge.service.ts` entirely
</Step>

<Step title="Clean up module registration">
Remove `AIKnowledgeService` from `ai.module.ts` providers and exports while preserving the tool registration
</Step>

<Step title="Update documentation">
Remove references to "dev-only glue" from `AI_MODULE_SPECIFICATION.md`
</Step>
</Steps>

## Debt categorization

### Risk levels

<CardGroup cols={2}>
<Card title="Low risk" icon="check-circle" color="#green">
Pure refactors with no schema changes or external contract modifications
</Card>

<Card title="Medium risk" icon="exclamation-triangle" color="#yellow">
Changes affecting internal APIs or requiring coordinated deployment
</Card>

<Card title="High risk" icon="times-circle" color="#red">
Schema migrations, breaking changes, or complex cross-service modifications
</Card>
</CardGroup>

### Effort estimation

- **< 2 hours:** Simple deletions, configuration updates
- **2-8 hours:** Service refactoring, interface changes
- **1-2 days:** Module restructuring, database migrations
- **> 2 days:** Architecture changes, major feature deprecation
