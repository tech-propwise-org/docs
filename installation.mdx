---
title: "Installation"
description: "Set up the PropWise CRM backend and frontend for local development."
---

This guide walks you through cloning, configuring, and running both the backend and frontend locally.

## Prerequisites

- **Node.js** v20+ (backend) / v22+ (frontend)
- **npm**
- **Git**
- **Cursor IDE**
- A **Neon API key** (generated from the [Neon console](https://console.neon.tech/app/settings/api-keys))

<Warning>
  Do **not** install the Neon MCP server in Cursor. It is prohibited in this project. Database branch management is handled entirely through CLI scripts and the Neon API key in `.env.branches`.
</Warning>

## Backend setup

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/propwise/propwise-crm-backend.git
    cd propwise-crm-backend
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    npm install
    ```
  </Step>

  <Step title="Configure environment variables">
    Copy the example environment file:

    ```bash
    cp .env.example .env.local
    ```

    Get the real values from your team (Slack / 1Password) and update `.env.local`. The following variables are required:

    <AccordionGroup>
      <Accordion title="Database">
        | Variable | Description |
        |----------|-------------|
        | `DATABASE_URL` | Neon PostgreSQL connection string (superuser for migrations) |
        | `RUNTIME_DATABASE_URL` | Runtime database URL (RLS user for app queries) |
        | `DATABASE_SSL` | Enable SSL for database connections (`true`) |
        | `REFERENCE_DB_URL` | Reference database connection (read-only, separate Neon project) |
      </Accordion>

      <Accordion title="Application">
        | Variable | Description |
        |----------|-------------|
        | `NODE_ENV` | Environment (`dev`, `staging`, `prod`) |
        | `PORT` | Server port (default: `3002`) |
        | `FRONTEND_URL` | Frontend URL for CORS (default: `http://localhost:3003`) |
      </Accordion>

      <Accordion title="Secrets & JWT">
        | Variable | Description |
        |----------|-------------|
        | `JWT_ACCESS_SECRET` | Access token signing secret |
        | `JWT_REFRESH_SECRET` | Refresh token signing secret |
        | `JWT_TENANT_SECRET` | Tenant token signing secret |
        | `JWT_SECRET` | General JWT secret |
        | `JWT_VERIFICATION_TOKEN_SECRET` | Email verification token secret |
        | `JWT_DEVELOPER_SECRET` | Developer portal JWT secret |
        | `COOKIE_SECRET` | Cookie signing secret |
      </Accordion>

      <Accordion title="R2 storage (Cloudflare)">
        | Variable | Description |
        |----------|-------------|
        | `R2_ACCOUNT_ID` | Cloudflare account ID |
        | `R2_ACCESS_KEY_ID` | R2 access key |
        | `R2_SECRET_ACCESS_KEY` | R2 secret key |
        | `R2_BUCKET_NAME` | R2 bucket name |
        | `R2_PUBLIC_BASE_URL` | Public URL for uploaded files |
        | `CLOUDFLARE_TOKEN` | Cloudflare API token |
        | `CLOUDFLARE_URL` | Full Cloudflare R2 storage URL |
      </Accordion>

      <Accordion title="WhatsApp Cloud API">
        | Variable | Description |
        |----------|-------------|
        | `WHATSAPP_API_URL` | Graph API base URL |
        | `WHATSAPP_ACCESS_TOKEN` | WhatsApp access token |
        | `WHATSAPP_PHONE_NUMBER_ID` | Phone number ID |
        | `WHATSAPP_BUSINESS_ACCOUNT_ID` | Business account ID |
        | `WHATSAPP_VERIFY_TOKEN` | Webhook verification token |
      </Accordion>

      <Accordion title="OAuth (email integrations)">
        | Variable | Description |
        |----------|-------------|
        | `OUTLOOK_CLIENT_ID` | Microsoft OAuth client ID |
        | `OUTLOOK_CLIENT_SECRET` | Microsoft OAuth client secret |
        | `OUTLOOK_REDIRECT_URI` | Outlook callback URL (`http://localhost:3002/auth/outlook/callback`) |
        | `GMAIL_CLIENT_ID` | Google OAuth client ID |
        | `GMAIL_CLIENT_SECRET` | Google OAuth client secret |
        | `GMAIL_REDIRECT_URI` | Gmail callback URL (`http://localhost:3002/auth/gmail/callback`) |
      </Accordion>

      <Accordion title="AI & integration services">
        | Variable | Description |
        |----------|-------------|
        | `TIPTAP_AI_SECRET` | Tiptap AI secret key |
        | `TIPTAP_AI_APP_ID` | Tiptap AI application ID |
        | `GOOGLE_GEMINI_API_KEY` | Google Gemini API key |
        | `FIRECRAWL_API_KEY` | Firecrawl API key |
        | `FIRECRAWL_MAX_PAGES` | Max pages for Firecrawl (default: `20`) |
        | `RESEND_API_KEY` | Resend email service API key |
        | `N8N_API_KEY` | N8N automation API key |
        | `DGIS_API_KEY` | 2GIS map API key |
      </Accordion>
    </AccordionGroup>

    <Warning>
      Never commit `.env.local` or `.env.branches` to version control. They contain database credentials and API keys.
    </Warning>
  </Step>

  <Step title="Set up Neon branch database URLs">
    Copy the branches example file:

    ```bash
    cp .env.branches.example .env.branches
    ```

    Generate an API key from the [Neon console API keys page](https://console.neon.tech/app/settings/api-keys) and add it to `.env.branches` along with your branch-specific database URLs. The CLI scripts use this key for branch management.
  </Step>

  <Step title="Install Git hooks">
    ```bash
    npm run db:setup-hooks
    ```

    This installs a `post-checkout` Git hook that automatically syncs your Neon database branch when you switch Git branches. Disable auto-sync by setting `NEON_AUTO_SYNC=false` in your environment.
  </Step>

  <Step title="Run database migrations">
    ```bash
    npm run migrate:up
    ```
  </Step>

  <Step title="Start the development server">
    ```bash
    npm run start:dev
    ```

    The API starts at `http://localhost:3002`. Swagger UI is available at `http://localhost:3002/api`.
  </Step>
</Steps>

### Database branch management

The backend uses Neon's branch-based development. Each Git branch can have its own isolated database branch.

| Command | Description |
|---------|-------------|
| `npm run db:sync` | Create a Neon branch matching the current Git branch |
| `npm run db:sync -- --delete` | Delete the Neon branch for the current Git branch |
| `npm run db:switch <branch>` | Manually switch to a specific Neon branch |
| `npm run db:current` | Show the current Neon branch |

<Tip>
  With Git hooks installed via `db:setup-hooks`, branch syncing happens automatically on `git checkout`. You rarely need to run these commands manually.
</Tip>

### Other useful commands

| Command | Description |
|---------|-------------|
| `npm run migrate` | Create a new migration |
| `npm run migrate:down` | Rollback the last migration |
| `npm run migrate:pending` | Check for pending migrations |
| `npm run seed` | Run database seeders |
| `npm run cli:system-admin` | Create a system admin user |
| `npm run test` | Run unit tests |
| `npm run lint` | Lint with auto-fix |

## Frontend setup

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/propwise/propwise-crm-frontend.git
    cd propwise-crm-frontend
    ```
  </Step>

  <Step title="Configure the Tiptap Pro registry">
    Create an `.npmrc` file in the project root with your Tiptap Pro token:

    ```ini .npmrc
    @tiptap-pro:registry=https://registry.tiptap.dev/
    //registry.tiptap.dev/:_authToken=YOUR_TIPTAP_PRO_TOKEN
    ```

    Get the token from your team. This is required to install Tiptap Pro packages used by the rich text editor.

    <Warning>
      Never commit `.npmrc` to version control. It contains a private registry token.
    </Warning>
  </Step>

  <Step title="Install dependencies">
    ```bash
    npm install
    ```

    <Note>
      `npm install` will fail if `.npmrc` is not configured â€” the Tiptap Pro packages require authentication to download.
    </Note>
  </Step>

  <Step title="Configure environment variables">
    Copy the example environment file:

    ```bash
    cp .env.example .env
    ```

    Update the following variables:

    | Variable | Description |
    |----------|-------------|
    | `NEXT_PUBLIC_ENV_NODE` | Environment (`dev`) |
    | `NEXT_PUBLIC_BACKEND_URL` | Backend API URL (`http://localhost:3002/`) |
    | `NEXT_PUBLIC_TIPTAP_AI_APP_ID` | Tiptap AI application ID |
    | `NEXT_PUBLIC_TIPTAP_AI_TOKEN` | Tiptap AI token |
    | `TIPTAP_PRO_TOKEN` | Tiptap Pro token (same as in `.npmrc`) |
  </Step>

  <Step title="Start the development server">
    ```bash
    npm run dev
    ```

    The frontend starts at `http://localhost:3003` using Next.js with Turbopack.
  </Step>
</Steps>

## Verify the setup

2. Check the backend Swagger UI at `http://localhost:3002/api`

<Check>
  If both servers respond successfully, your local environment is ready.
</Check>
