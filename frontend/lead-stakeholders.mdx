# Lead Stakeholders: Holders, Observers & Contributed Teams

## Requirement

> Display a list of **holders** (anyone with stakeholder of the entity with commission > 0), a list of **observers** (anyone with commission = 0), and a list of **contributed teams** (all teams that exist in any stakeholder of that lead).

## Phase 2: Frontend — Data Layer

### 2.1 — Create a TanStack Query hook for lead stakeholders

**File:** `src/hooks/use-lead-stakeholders.ts` (new file, or add to `src/hooks/use-query-hooks.ts`)

```typescript
import { useQuery } from '@tanstack/react-query';
import { queryKeys } from '@/lib/query-keys';
import { LeadApi } from '@/services/api/lead.api';

export function useLeadStakeholders(leadId: string) {
  return useQuery({
    queryKey: queryKeys.leads.stakeholders(leadId),
    queryFn: () => LeadApi.getStakeholders(leadId, { limit: 100 }),
    enabled: !!leadId,
  });
}
```

### 2.2 — Add client-side derivation helpers

Either in the hook or as utility functions in the consuming component:

```typescript
import { useMemo } from 'react';

// Inside the component or hook:
const holders = useMemo(
  () => stakeholders.filter((s) => s.commissionPercentage > 0),
  [stakeholders],
);

const observers = useMemo(
  () => stakeholders.filter((s) => s.commissionPercentage === 0),
  [stakeholders],
);

const contributedTeams = useMemo(
  () =>
    Array.from(
      new Map(stakeholders.filter((s) => s.team).map((s) => [s.team!.id, s.team!])).values(),
    ),
  [stakeholders],
);
```

## Phase 3: Frontend — UI Component

### 3.1 — Create `LeadStakeholdersCard` component

**New file:** `src/components/pages/leads/lead-stakeholders-card.tsx`

**Props:**

```typescript
interface LeadStakeholdersCardProps {
  leadId: string;
}
```

**Structure — three sections inside a Card (or three separate Cards):**

1. **Holders section** — stakeholders with `commissionPercentage > 0`
2. **Observers section** — stakeholders with `commissionPercentage === 0`
3. **Contributed Teams section** — deduplicated teams from all stakeholders

**Skeleton/Loading:** Follow the pattern from `src/components/pages/deals/deal-commission-breakdown.tsx` (lines 102–118).

**Empty state:** Follow the same file's pattern (lines 120–137).

### 3.2 — Holder row design

Each holder row should display:

| Element       | Source Field                | Notes                                         |
| ------------- | --------------------------- | --------------------------------------------- |
| Avatar        | `user.avatarUrl`            | Fallback initials from `firstName`/`lastName` |
| Name (linked) | `user.firstName + lastName` | Click → `/home/staff/:userId`                 |
| Team badge    | `team.name`                 | Only if `team` exists                         |
| Commission %  | `commissionPercentage`      | e.g. "25%" in a colored badge                 |
| Locked icon   | `isCommissionLocked`        | Show lock icon if `true`                      |
| Primary badge | `isPrimary`                 | "Primary" badge if `true`                     |
| Access level  | `accessLevel`               | Subtle badge: FULL / WRITE / READ             |

### 3.3 — Observer row design

Same as holder but **without** commission/locked columns. Use a lighter/muted style to visually differentiate from holders.

### 3.4 — Contributed Teams design

Horizontal list of team name badges, or a simple list with team name and optional description. Deduplicated from all stakeholders' `team` field.

### 3.5 — Empty states

| Scenario                  | Message                            |
| ------------------------- | ---------------------------------- |
| No holders beyond primary | "No additional commission holders" |
| No observers              | "No observers on this lead"        |
| No contributed teams      | "No teams involved"                |

### 3.6 — (Optional) Commission Summary bar

At the top of the card, show data from `CommissionSummaryDto`:

- Total allocated: `totalCommission`%
- Available: `availableToAllocate`%
- Visual progress bar showing allocation
- Valid indicator from `isValid`
