# Component Extraction & Page Structure Guide

**PropWise CRM Frontend**

This guide defines the standard for organizing page files and extracting reusable components across the frontend codebase.

---

## Core Principle

> **Page files should ONLY contain the page function. All other components must be extracted to the components folder.**

---

## File Structure Standard

### Rule: One Page Function Per Page File

```
src/app/{page-name}/
└── page.tsx              # Contains ONLY the default export page function
```

### Rule: Page Components Live in Components Folder

```
src/components/pages/{page-name}/
├── component-one.tsx     # Extracted component
├── component-two.tsx     # Extracted component
├── component-modal.tsx   # Modal components
└── index.ts              # Barrel export
```

## What Goes Where

### Page File (`src/app/{page}/page.tsx`)

**ONLY include:**

- Default export page function
- Page-level state (`useState` for UI state like selected items, modal open state)
- Page-level hooks (`useRouter`, `useAuth`, `useQuery` for page data)
- Page layout structure (wrapping components)
- Event handlers that coordinate between components

**DO NOT include:**

- Child component definitions
- Modal component definitions
- Card/list item component definitions
- Complex UI sub-sections

### Components Folder (`src/components/pages/{page}/`)

**Extract these types of components:**

| Component Type | Example | Naming Convention |
|----------------|---------|-------------------|
| Cards | Organization card, Lead card | `{entity}-card.tsx` |
| Modals | Create modal, Edit modal | `create-{entity}-modal.tsx` |
| List items | Table rows, Grid items | `{entity}-list-item.tsx` |
| Forms | Inline forms, Filter forms | `{entity}-form.tsx` |
| Sections | Overview, Timeline | `{entity}-{section}.tsx` |
| Dialogs | Confirmation, Selection | `{action}-dialog.tsx` |

## Implementation Example

### Before (Anti-pattern)

```tsx
// src/app/organization-selection/page.tsx
// ❌ BAD: 500+ lines with inline components

'use client';

import { useState } from 'react';
// ... many imports

// ❌ Component defined inside page file
function CreateOrganizationModal({ open, setOpen, onSuccess }) {
  // 100+ lines of modal code
}

// ❌ Another component defined inside page file
function OrganizationCard({ organization, onSelect, isSelected }) {
  // 80+ lines of card code
}

// Page function buried below components
export default function OrganizationSelectionPage() {
  // Page logic
}
```

### After (Correct Pattern)

**Page file:**

```tsx
// src/app/organization-selection/page.tsx
// ✅ GOOD: Clean, focused page file (~100-200 lines)

'use client';

import { useState, useEffect } from 'react';
import { useRouter } from 'next/navigation';
import { useMyOrganizations } from '@/hooks/use-query-hooks';
import {
  OrganizationCard,
  CreateOrganizationModal,
} from '@/components/pages/organization-selection';

export default function OrganizationSelectionPage() {
  const [selectedOrgId, setSelectedOrgId] = useState<string | null>(null);
  const [createModalOpen, setCreateModalOpen] = useState(false);
  const { data: organizations = [], isLoading } = useMyOrganizations();

  // Page-level handlers
  const handleSelectOrganization = (org) => setSelectedOrgId(org.id);
  const handleCreateSuccess = (newOrg) => setSelectedOrgId(newOrg.id);

  return (
    <div>
      {organizations.map((org) => (
        <OrganizationCard
          key={org.id}
          organization={org}
          onSelect={handleSelectOrganization}
          isSelected={selectedOrgId === org.id}
        />
      ))}
      <CreateOrganizationModal
        open={createModalOpen}
        setOpen={setCreateModalOpen}
        onSuccess={handleCreateSuccess}
      />
    </div>
  );
}
```

**Components folder:**

```tsx
// src/components/pages/organization-selection/organization-card.tsx
// ✅ Extracted, reusable component

'use client';

import { Card, CardHeader, CardContent } from '@/components/ui/card';

interface OrganizationCardProps {
  organization: OrganizationDto;
  onSelect: (organization: OrganizationDto) => void;
  isSelected?: boolean;
}

export function OrganizationCard({ organization, onSelect, isSelected }: OrganizationCardProps) {
  return (
    <Card onClick={() => onSelect(organization)}>
      {/* Card content */}
    </Card>
  );
}
```

```tsx
// src/components/pages/organization-selection/create-organization-modal.tsx
// ✅ Extracted modal with its own logic

'use client';

import { useMutation, useQueryClient } from '@tanstack/react-query';
import { Dialog, DialogContent } from '@/components/ui/dialog';

interface CreateOrganizationModalProps {
  open: boolean;
  setOpen: (open: boolean) => void;
  onSuccess: (organization: OrganizationDto) => void;
}

export function CreateOrganizationModal({ open, setOpen, onSuccess }: CreateOrganizationModalProps) {
  const queryClient = useQueryClient();
  
  const createMutation = useMutation({
    mutationFn: (data) => OrganizationApi.create(data),
    onSuccess: (newOrg) => {
      queryClient.invalidateQueries({ queryKey: queryKeys.organizations.myOrganizations });
      onSuccess(newOrg);
      setOpen(false);
    },
  });

  return (
    <Dialog open={open} onOpenChange={setOpen}>
      <DialogContent>
        {/* Modal content */}
      </DialogContent>
    </Dialog>
  );
}
```

```tsx
// src/components/pages/organization-selection/index.ts
// ✅ Barrel export for clean imports

export { OrganizationCard } from './organization-card';
export { CreateOrganizationModal } from './create-organization-modal';
```

## Barrel Export Pattern

Every page components folder should have an `index.ts` file for clean imports:

```tsx
// src/components/pages/{page-name}/index.ts

export { ComponentOne } from './component-one';
export { ComponentTwo } from './component-two';
export { SomeModal } from './some-modal';
```

This enables clean imports in the page file:

```tsx
// ✅ Clean import from barrel export
import {
  ComponentOne,
  ComponentTwo,
  SomeModal,
} from '@/components/pages/{page-name}';

// ❌ Avoid multiple import statements
import { ComponentOne } from '@/components/pages/{page-name}/component-one';
import { ComponentTwo } from '@/components/pages/{page-name}/component-two';
```

## When to Extract Components

### Always Extract

| Scenario | Reason |
|----------|--------|
| Component > 50 lines | Keeps page file readable |
| Modal/Dialog components | Self-contained with own state |
| Card/List item components | Often reusable, testable |
| Form sections | Complex validation logic |
| Repeated patterns | DRY principle |

### Keep Inline (Exceptions)

| Scenario | Reason |
|----------|--------|
| Simple loading state | `{isLoading && <Spinner />}` |
| Simple empty state | `{items.length === 0 && <Empty />}` |
| Layout wrappers | `<div className="...">` |

## Component Props Pattern

### Standard Props Interface

```tsx
// For display components
interface {Entity}CardProps {
  {entity}: {Entity}Dto;
  onSelect?: (item: {Entity}Dto) => void;
  isSelected?: boolean;
}

// For modal components
interface Create{Entity}ModalProps {
  open: boolean;
  setOpen: (open: boolean) => void;
  onSuccess: (item: {Entity}Dto) => void;
}

// For edit modals
interface Edit{Entity}ModalProps {
  open: boolean;
  setOpen: (open: boolean) => void;
  {entity}: {Entity}Dto;
  onSuccess: (item: {Entity}Dto) => void;
}
```

## Benefits of This Pattern

| Benefit | Description |
|---------|-------------|
| **Readability** | Page files stay focused and scannable |
| **Maintainability** | Changes to components don't bloat page files |
| **Testability** | Extracted components can be unit tested |
| **Reusability** | Components can be imported in other pages |
| **Code Review** | Easier to review smaller, focused files |
| **Performance** | Better code splitting opportunities |

---

## Checklist for New Pages

When creating a new page:

```
[ ] Create page file at src/app/{page}/page.tsx
[ ] Create components folder at src/components/pages/{page}/
[ ] Extract all child components to components folder
[ ] Create index.ts barrel export
[ ] Page file contains ONLY the page function
[ ] Page file is under 200 lines
[ ] All modals/cards/sections are extracted
[ ] Components have proper TypeScript interfaces
```
